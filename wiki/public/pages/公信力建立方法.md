# 公信力建立方法

新制度推行前，如何以最小成本建立最大信任？基于《史记·商君列传》"立木为信"案例的提炼与延伸。

**核心价值**：用最小成本建立最大信任

**来源**：《史记·商君列传》（卷068）

---

## 核心问题：信任缺失

**背景**：
> "令既具，未布，恐民之不信"

含义：
- 法令已制定完备
- 但尚未颁布
- 担心民众不相信政府会执行

**信任缺失的原因**：
- 历史信任赤字（政府以前曾食言）
- 信息不对称（民众不了解新制度）
- 变革阻力（既得利益者的抵制）
- 变革成本不明确（民众不知道是否值得配合）

---

## 立木为信：经典操作

**商鞅的具体做法**：

> 乃立三丈之木于国都市南门，募民有能徙置北门者予十金。民怪之，莫敢徙。复曰"能徙者予五十金"。有一人徙之，辄予五十金，以明不欺。卒下令。

**操作分解**：

| 步骤 | 商鞅的行动 | 背后逻辑 |
|------|-----------|---------|
| 1. 选择简单可见的承诺 | 立木于市场 | 所有人都能看到，无信息不对称 |
| 2. 承诺要超出预期 | 10金→50金（涨价5倍）| 超预期才能克服"不信任"惯性 |
| 3. 100%兑现承诺 | 有人徙木，立即给50金 | 即时兑现，无延迟无折扣 |
| 4. 公开执行 | "以明不欺" | 让所有人都知道政府守信 |
| 5. 随即颁布新法令 | "卒下令" | 信任建立后，立即行动 |

**成本分析**：
- 50金代价：小（相对于全国推行新法的收益）
- 信任收益：大（民众相信政府会兑现承诺）

## 信任建立的核心原理

### 原理1：信任是可以用小代价"购买"的

**逻辑**：
- 民众对新政策的信任=0（历史赤字）
- 通过一次小规模、可见的"超额兑现"，信任迅速跳跃

**数学模型**：
```
信任度 × 政策接受率 = 政策效果

商鞅改革前：信任度≈0 → 政策效果≈0
立木为信后：信任度显著提升 → 政策效果大幅提升
```

### 原理2：承诺必须"超出预期"才能打破不信任惯性

**为什么从10金涨到50金？**
- 10金时没人移动：预期 > 实际承诺（人们认为"这是骗局"）
- 50金时有人行动：超预期承诺打破心理防线（"即使是骗局，风险也值得承担"）

**计算框架**：
```
行动阈值 = 不信任度 × 历史失信次数 × 风险厌恶系数

当"承诺价值 > 行动阈值"时，才有人愿意行动
```

### 原理3：可见性是信任的倍增器

**为什么要立在"国都市南门"（人流最多的地方）？**
- 所有人都能看到承诺被兑现
- 口口相传，信任指数级扩散
- 无法抵赖或悄悄取消承诺

## 现代应用框架

### 框架1：新领导者上任

**场景**：新任CEO/市长/校长，需要快速建立公信力。

**立木为信操作**：
1. 选择一个"小但可见"的承诺（如：每周开放员工直接沟通渠道）
2. 超额兑现（第一次会议亲自主持+推动实施反馈）
3. 公开展示结果（发布员工建议采纳报告）
4. 随即推行更大的改革

**案例**：某互联网公司新CEO第一周：
- 承诺：所有Bug报告72小时内回复
- 兑现：平均响应时间24小时
- 效果：工程师团队信任度显著提升，后续组织变革阻力小

### 框架2：新产品/服务上市

**场景**：电商平台推出"7天无理由退货"

**立木为信操作**：
1. 承诺超出行业标准（行业普遍3天，我们承诺7天）
2. 100%兑现（早期即使有损失也坚持退货）
3. 公开展示数据（退货率公开，证明承诺执行）
4. 建立信任后推出更多服务

### 框架3：政策/制度改革

**场景**：企业推行新的绩效考核制度

**立木为信操作**：
1. 选择一个具体、可见的承诺（如：本季度KPI达标者100%按时发放奖金）
2. 超额兑现（不仅按时，还提前3天）
3. 全员公告（让所有人知道奖金已发）
4. 随即推行新的考核标准

## 操作清单

### 第一步：诊断信任缺失程度

- [ ] 历史信任赤字评估（过去有多少次食言？）
- [ ] 目标群体信任调查（问卷/访谈）
- [ ] 竞争对手信任对比（我vs行业平均）

**信任缺失程度分级**：
- 轻度（单次失信）：一次小规模信任行动即可
- 中度（多次失信）：需要多次持续的信任行动
- 重度（长期失信）：需要大规模、高成本的信任重建

### 第二步：设计"立木"行动

- [ ] 选择"承诺"：简单、可见、可量化
- [ ] 设置"超预期"幅度：超出预期20%-50%
- [ ] 确保"100%兑现能力"：资源是否充足？
- [ ] 计划"公开展示"方式：如何让所有人都知道？

**好承诺的标准（SMART+V）**：
- Specific（具体）：不能模糊（"我会改善"❌，"48小时内回复"✅）
- Measurable（可量化）：有明确衡量标准
- Achievable（可实现）：自己有能力100%兑现
- Relevant（相关）：与目标群体关心的问题直接相关
- Time-bound（有时限）：有明确时间节点
- Visible（可见）：让所有人都能看到承诺和兑现

### 第三步：执行与放大

- [ ] 100%兑现承诺（无例外、无延迟、无折扣）
- [ ] 公开展示兑现结果
- [ ] 收集反馈（信任度是否提升？）
- [ ] 随即推行更大的改革

## 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 承诺太大，无法兑现 | 信任二次崩溃，比之前更差 | 只承诺有把握兑现的 |
| 承诺太小，无法打动 | 没人相信，信任未建立 | 超出预期20%-50% |
| 兑现不公开 | 信任传播有限 | 主动公开展示兑现结果 |
| 兑现有例外 | "99%兑现"≠信任建立 | 100%无例外兑现 |
| 承诺太复杂 | 民众无法判断是否兑现 | 简单、一句话说清楚 |

## 信任的可持续性

**立木为信只是起点**：
- 一次超额兑现建立初始信任
- 后续每次兑现巩固信任
- 一次食言摧毁信任（需要10次超额兑现才能恢复）

**信任资产管理**：
```
信任资产 = 历次兑现总量 - 历次失信总量 × 失信放大系数

失信放大系数 ≈ 5-10（食一次言的负面效果是兑现一次的5-10倍）
```
