# 反思 2026-04-22 (v7 · Bug 复盘)

> 触发: 用户指出 `鲁桓公` 页面有错误.
> 类型: 非周期性反思, bug 驱动.

---

## 一、Bug 现象

`鲁桓公.md` 的"生平大事"区包含 6 条事件, 其中 **4 条是其他时代的**:

- 前 681 柯地之盟 — 鲁庄公时期
- 前 660 季友立釐公诛庆父 — 鲁庄公末/釐公初
- 前 562 三桓分军 — 鲁襄公时期
- 前 468 哀公欲伐三桓 — 鲁哀公时期

鲁桓公实际**在位前 712-694**, 被齐襄公所杀后这些事件都与他无关.

## 二、根因 (技术)

`enrich_timeline.py extract_events` 的过滤逻辑是:

```python
for pn in person_names:
    if pn in people:  # 子串匹配!
        events.append(...)
```

鲁桓公的 aliases = `[桓, 桓公]`:
- `桓` 是**单字**别名
- KG 事件的"主要人物"列里出现:
  - `三桓` (鲁公族三支) → 含"桓"
  - `齐桓公` → 含"桓公"
  - `楚桓公` 等 → 同理

子串 `桓` 匹到**一切含"桓"字的人名**, 误伤大量不相关事件.

## 三、根因 (方法论)

三个系统性问题:

1. **裸串匹配忽略了 KG 结构**. KG 的人物列是 `〖@<name>〗` 标注形式, butler 却丢失这个结构去做子串.

2. **无抽样验证**. 批量 enrich 146 页时我说"✓ 添加 N events", 但没回验单条数据。结果整批数据污染.

3. **单字别名的风险评估缺失**. "桓/文/武/宣/季" 这类常见单字做别名, 理应触发警告.

## 四、修复

### 技术修复

改 `extract_events` 为**严格正则**匹配:

```python
# 只匹 〖@<canonical>〗 / 〖@<alias>|<canonical>〗 / 〖@<长别名>〗
patterns = []
patterns.append(r'〖@' + re.escape(canonical) + r'(?:\|[^〗]+)?〗')
patterns.append(r'〖@[^〗]+\|' + re.escape(canonical) + r'〗')
for a in aliases:
    if len(a) >= 2:  # 单字别名一律不用于匹配
        patterns.append(r'〖@' + re.escape(a) + r'(?:\|[^〗]+)?〗')
```

全量重写 195 页 timeline, 重新 enrich 171 页.

鲁桓公现在只剩 2 条**本人**事件 (即位 + 被弑). 正确.

### 方法论修复

**规则**: butler 涉及 "抽取-过滤-写回" 类动作时必须:

1. **尊重数据结构**. KG 标注 `〖@@〗` 不是装饰, 是消歧信号. 丢掉它等于放弃一半的准确性.
2. **单字别名禁用于匹配**. 不管什么动作, aliases 里 `len(a) == 1` 的一律跳过.
3. **批量动作必须先抽样**. 先对 3-5 页人工验证输出, 再上全量. 修进 W2 atomic 行动前置条件.

## 五、沉淀入 Skill (提案 20-22)

### 提案 20 · W2 前置条件加 "抽样验证"

每个 batch-style 动作（enrich-timeline / add-tag / cite-doc-report 等）执行前:
1. 对 3 随机 target 先跑一次
2. 人工/脚本 sanity check (如"是否指向本人"/"是否合理")
3. 通过才批量

当前 butler 在 `sys.stderr` 输出进度, 不回验; 加一个 `--dry-run` 和 `--sample <N>` 模式.

### 提案 21 · 抽象出"标注感知"助手

所有 butler 脚本需要从 `〖...〗` 标注文本中抽信息, 应共用一个 `kg_text_utils.py`:
- `extract_people(text)` — 只返回 `〖@〗` 完整包裹的名字列表
- `strip_annotations(text)` — 去标注, 保 value
- `match_canonical(text, canonical)` — 严格匹配

避免每个脚本各自写（易错）.

### 提案 22 · butler 自检脚本

新增 `wiki/scripts/butler/selfcheck.py`:
- 对每个 timeline 随机抽 3 条事件, 回 kg 事件索引交叉验证
- 对每个 tag 检查是否合理 (如 生卒年 与 era tag 一致)
- 对 cite-doc-report 链接检查目标文件存在

每周跑一次, 结果入 reflection.
