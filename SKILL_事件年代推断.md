# SKILL: 事件年代推断

## 任务定义

将古籍（史记）事件索引中缺少公元纪年标注的事件，推断其大致年代，并标注推断依据。

## 标记规范

| 标记 | 含义 | 示例 |
|------|------|------|
| `（公元前XXX年）` | 精确纪年（原文有明确纪年或已知事件） | `（公元前206年）` |
| `[约公元前XXX年]` | 推断纪年（由推理规则得出） | `[约公元前209年]` |

推断纪年在详细记录中附加推断依据行：
```
- **年代推断**: {推断方法和依据}
```

## 推断策略（优先级从高到低）

### 策略0: 已知事件精确定年

部分事件是有公认年份的重大历史事件，直接赋予精确纪年。

| 已知事件 | 公元年 | 备注 |
|---------|--------|------|
| 陈胜起义/大泽乡起义 | 前209年 | 秦二世元年七月 |
| 项梁渡淮 | 前209年 | |
| 投奔项梁/从项梁 | 前208年 | |
| 巨鹿之战/破釜沉舟 | 前207年 | |
| 鸿门宴 | 前206年 | 汉元年 |
| 还定三秦/入汉中 | 前206年 | |
| 彭城之战 | 前205年 | 汉二年 |
| 背水一战/韩信破赵 | 前204年 | |
| 垓下之战/乌江自刎 | 前202年 | |
| 白登之围/平城之围 | 前200年 | 汉七年 |
| 诛吕（诛诸吕立文帝） | 前180年 | 高后八年 |

**注意**：名称匹配优先于描述匹配，避免因事件描述中提及历史事件而误赋年份。

### 策略1: 章节内插值（最常用）

史记各篇按时间顺序叙事，利用同一章节中已知纪年事件的前后锚点进行推断。

**规则**：
- 找到该事件在章节中前方最近的已知纪年事件（前锚）和后方最近的（后锚）
- 取前锚年份作为推断值（因为事件发生在前锚之后、后锚之前）
- 仅在前后锚跨度 < 200年时使用（跨度太大说明章节内事件时间跨度大，插值不可靠）

**示例**：
- 章节007 事件007-005（无纪年），前锚007-004=前209年，后锚007-008=前207年 → 推断约前209年

### 策略2: 人物生卒年修正

当插值结果与事件主要人物的生卒年严重矛盾时（偏离20年以上），改用人物生卒年交集中点。

**规则**：
- 收集事件所有人物的生卒年
- 计算生存期交集 [最晚出生, 最早去世]
- 如果插值年份不在交集±20年内，改用交集中点

### 策略3: 人物生卒年中点（无锚点时）

当章节内无已知纪年事件或跨度 > 200年时，用人物生卒年交集中点作为推断值。

### 策略4: 大跨度插值

锚点跨度 > 200年时，仍取最近锚点年份（标记为大跨度插值，精度较低）。

### 策略5: 章节时代兜底

当以上策略均不可用时，使用预设的章节默认时代（如第001章=约前2500年，第007章=约前207年）。

## 常见错误模式

### 1. 君主混淆错误

同一篇章内可能涉及多个君主的纪年。例如：
- 009_吕太后本纪中，"七年"可能指孝惠七年（前188年），也可能指高后七年（前181年）
- 自动标注脚本可能默认使用篇章主角（高后）来计算所有纪年，导致错误

**检查方法**：逐事件确认纪年所指的君主，而非默认使用全篇统一君主。

### 2. 描述引用误匹配

事件描述中提及其他历史事件（如"陈胜起义后..."），不能据此将该事件定年为被引用事件的年份。

**防范**：
- 事件名称匹配优先于描述匹配
- 描述匹配限定章节范围（如"还定三秦"仅在章节008/051/054/057/095/098中匹配）
- 使用 `^` 锚定描述开头，避免内容中间的引用

### 3. 跨章节不一致

同一事件在不同章节可能有不同年份标注。

**检查方法**：跨章节验证，同名事件的纪年应一致。若不一致，以本纪/表的纪年为准。

### 4. 人物活跃期过宽

用人物生卒年中点推断时，如人物在世时间很长（如吕不韦 前292-前235），中点可能偏离事件实际时间较远。

**改进**：优先使用章节插值，人物生卒年仅作为合理性检查和兜底。

## 跨章节验证流程

1. 提取所有事件的名称和纪年
2. 按名称分组，检查同名事件纪年是否一致
3. 不一致的，以精确纪年（本纪/表）优先
4. 输出冲突报告供人工审核
