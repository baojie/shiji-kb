## 一、标题格式一致性问题

### 1.1 发现的问题

当前存在**三种不同的标题格式**：

#### 格式A：`META: 标题 — 副标题`（6个文档）
- 00-META-01_反思.md: `META: 反思 — Agent驱动的质量审查通用方法论`
- 00-META-02_迭代工作流.md: `META: 迭代工作流 — 从冷启动到收敛的执行策略`
- 00-META-03_冷启动.md: `META: 冷启动 — 从零开始构建AI驱动工序的方法论`
- 00-META-08_标注体系设计.md: `META: 标注体系设计 — 语义标记符号的工程化设计原则`
- 00-META-09_Agent提示词工程.md: `META: Agent提示词工程 — 方法论驱动的AI任务设计`
- 00-META-10_质量控制.md: `META: 质量控制 — 自动化质检体系与工具链设计`

#### 格式B：`元技能:标题`或`元技能：标题`（7个文档）
- 00-META-04_柳叶刀方法.md: `元技能：柳叶刀方法（精细分解与混合解决方案）`
- 00-META-05_知识作为上下文压缩.md: `元技能:知识作为上下文压缩`
- 00-META-06_SKILL优化与演化.md: `元技能:SKILL优化与演化`
- 00-META-07_可读性.md: `元技能:可读性`
- 00-META-11_数据体感培养.md: `元技能:数据体感培养`
- 00-META-12_数据融合.md: `元技能:数据融合`
- 00-META-13_技能迁移.md: `元技能:技能迁移`

#### 格式C：`元元技能`（1个文档）
- 00-META-14_好东西都是总结出来的.md: `元元技能：好东西都是总结出来的`

### 1.2 细节问题

1. **冒号不一致**：有的用全角`：`，有的用半角`:`
2. **分隔符不一致**：格式A用` — `（em dash），格式B用括号或无副标题
3. **"元技能"术语**：中英文混用（META vs 元技能）

### 1.3 推荐统一格式

建议采用**格式A**（META: 标题 — 副标题），理由：
- 更规范（META是项目术语的英文标准形式）
- 副标题提供更多上下文
- 与SKILL文档格式风格一致（如"SKILL_03a: 实体标注"）

**统一后的格式规范**：
```
# META: 标题 — 副标题（可选）

# 或简化版：
# META-编号: 标题 — 副标题
```

## 六、目录缩进一致性问题 ⚠️

### 6.1 问题描述

**所有元技能文档**的自动生成目录都存在缩进层级混乱问题。

**示例（00-META-02_迭代工作流.md第6-24行）**：

```markdown
- [一、迭代工作流的四个阶段](#一迭代工作流的四个阶段)
  - [阶段模型](#阶段模型)
  - [1.1 Phase 0: 冷启动](#11-phase-0-冷启动)
- [冷启动清单](#冷启动清单)           ❌ 应该缩进（属于1.1的子项）
- [质量预期（Phase 0）](#质量预期phase-0)  ❌ 应该缩进
  - [1.2 Phase 1: 模式积累](#12-phase-1-模式积累)
```

**正确的缩进应该是**：

```markdown
- [一、迭代工作流的四个阶段](#一迭代工作流的四个阶段)
  - [阶段模型](#阶段模型)
  - [1.1 Phase 0: 冷启动](#11-phase-0-冷启动)
    - [冷启动清单](#冷启动清单)         ✅ 三级缩进（4空格）
    - [质量预期（Phase 0）](#质量预期phase-0)  ✅ 三级缩进
  - [1.2 Phase 1: 模式积累](#12-phase-1-模式积累)
    - [Phase 1执行清单](#phase-1执行清单)   ✅ 三级缩进
```

### 6.2 影响范围

检查结果：**14个元技能文档全部受影响**

### 6.3 根本原因

自动生成目录的工具无法正确识别文档的层级结构，特别是：
- 列表项（如"清单"、"标准"）被误识别为一级标题
- 嵌套的小节标题缩进层级错误

### 6.4 修复建议

建议删除所有自动生成的目录，理由：
1. 目前所有目录都有缩进错误
2. 元技能文档篇幅适中（200-600行），无需目录
3. 现代编辑器（VSCode/Claude Code）都有大纲导航功能
4. 维护目录成本高（每次修改标题都要更新）

### 6.5 紧急程度

⚠️ **高优先级** — 错误的目录比没有目录更糟糕
