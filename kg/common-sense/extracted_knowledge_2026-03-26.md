## 一、反思迭代方法应用

### 1.1 提取流程

```
【第1轮：案例收集】
├─ 输入: Factual SKU_030(五刑) + SKU_080(周爵位) + SKU_081(汉爵位)
├─ 提问: "这些SKU揭示了什么普适规律?"
└─ 输出: 候选常识条目列表

【第2轮：普适性检验】
├─ 输入: 候选常识条目
├─ 提问: "是否适用于多个案例? 是否有时代边界? 是否可量化?"
├─ 反思: ✅ 若适用3+案例且有时代标注 → 保留
└─ 输出: 精炼后的常识条目

【第3轮：来源标注】
├─ 输入: 精炼常识
├─ 行动: 标注数据源(SKU/案例/史料) + 置信度(高/中/低)
└─ 输出: 规范化常识条目

【第4轮：规则化编码】
├─ 输入: 规范化常识
├─ 行动: 转化为可检测规则
└─ 输出: IF-THEN规则

【第5轮：实战验证】
├─ 输入: 检测规则
├─ 行动: 在已有案例中测试假阳性率
└─ 输出: 验证后的常识

【第6轮：录入常识库】
├─ 输入: 验证后的常识
├─ 行动: 整合到对应常识类别
└─ 输出: 更新后的常识库
```

## 六、方法论优化

### 6.1 反思问题清单增强

在原有9个问题基础上,新增:

**Q10: 标准化检查**
- 数字是否有明确单位?(里/日/人/率)
- 分类是否有量化标准?(大中小/轻重/高低)

**Q11: 边界条件检查**
- 是否有例外情况?(如"特殊贡献可突破")
- 例外比例是否<20%?

**Q12: 可组合性检查**
- 这条常识是否可与其他常识组合?(如人口+兵力→承受力)
- 组合后是否产生新洞见?

### 6.2 假阳性降低策略

**策略1: 明确量化标准**
- 将"大城/中城/小城"量化为人口数或城墙长度
- 将"大国/中国/小国"量化为人口或疆域面积

**策略2: 增加条件判断**
- 从"IF 条件 THEN 异常"
- 改为"IF 条件 AND 无例外理由 THEN 异常"

**策略3: 置信度分级**
- 高置信度常识(假阳性<5%): 直接标记异常
- 中置信度常识(假阳性5-20%): 标记"疑似异常"
- 低置信度常识(假阳性>20%): 仅提示,不标记
