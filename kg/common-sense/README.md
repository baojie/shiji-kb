## 5. 社会等级常识

### 5.1 身份等级

| 等级 | 身份 | 权力范围 | 时代 | 数据源 |
|------|------|---------|------|--------|
| **天子/皇帝** | 最高统治者 | 天下 | 天子(周至战国)，皇帝(秦始皇后) | 政治常识 |
| **诸侯王** | 封国君主 | 一国/一郡 | 分封制/郡国制 | 封建体系 |
| **列侯** | 封县君主 | 一县或部分县 | 秦汉 | 封爵制度 |
| **卿大夫** | 高级贵族 | 官职 | 周至战国 | 贵族等级 |

- 庶民→士 : 通过军功或察举

- 列侯→王 : 通常不可，除非皇族

| **军队编制** | 伍-什-百-千-万 | 先秦至汉 | 五人为伍，五伍为什 | 军制 |

## 附录A: 常识应用于反常检测

### A.1 检测流程

```
结构化数据（实体+事件+关系）
    ↓ 加载常识库规则
常识规则库 (common_sense.md)
    ↓ 规则匹配
候选反常列表（违反数字/制度/逻辑常识）
    ↓ LLM评估
有价值的反常 (anomaly_report.json)
```

### A.2 规则引用格式

在 SKILL_07c 反常推理中引用本常识库时，使用以下格式：

```markdown
**常识依据**: [1.1 人口与兵力规模] - "单支军队常规规模: 5-20万"
**违反情况**: 长平之战赵军四十余万降卒 (超出常规规模上限)
**异常等级**: 高
```

### A.3 常识库更新机制

1. **新增常识**: 从新发现的案例中提炼普适性规则
2. **修正常识**: 根据学界新研究成果调整数值范围
3. **版本控制**: 每次更新记录版本号和变更日期
4. **引用溯源**: 每条常识标注数据来源
