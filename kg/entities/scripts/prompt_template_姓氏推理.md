# 提示词模板：姓氏推理

本模板用于生成《史记》各章节或人物群体的姓氏推理提示词。

## 迭代机制

本模板与 SKILL 文件构成滚动改进循环：

1. **执行 Round N** → 按模板生成提示词 → agent 推理并写入 `person_xingshi.json`
2. **agent 发现新规则或新例外** → 更新 `SKILL_07b_姓氏推理.md`（补充推理规则或特殊案例）
3. **执行 Round N+1** → 提示词第一步读取已更新的 SKILL 和已有数据 → 自动获得前序轮次的积累
4. **如需调整模板本身** → 手动编辑本文件

**核心原则**：`person_xingshi.json` 是"数据积累层"（每轮增量写入），SKILL 是"规则积累层"（发现新规则时更新），本模板是"工作流程层"。

---

## 模板正文

```
你的任务是对《史记》人物进行姓氏推理，将结果写入 `kg/entities/data/person_xingshi.json`。

## 工作流程

本次执行推理轮次：{round}
本次处理范围：{scope}（章节 / 人物群体 / 邦国）

### Step 1：读取已有知识

1. 读取推理规则：`skills/SKILL_07b_姓氏推理.md`
2. 读取背景规范：`doc/spec/姓氏制度.md`
3. 读取已有数据：`kg/entities/data/person_xingshi.json`（已确认的姓氏记录）
4. 读取姓族索引：`kg/entities/data/xing_index.json`
5. 读取本次处理的章节或人物列表：{input_files}

### Step 2：按轮次执行推理

**Round 1（仅在首次运行时执行）：直接提取**
- 在文本中搜索明确记载姓/氏的句式：
  - "X者，Y姓也" / "X，姓Y氏Z" / "以Z为氏" / "同姓" 后接家世说明
- 记录：xing, shi, ming, zi（如有），evidence（原文引用），confidence=high/exact

**Round 2：邦国推理**
- 对文本中每个〖@人名〗，找出其所属〖'邦国〗
- 查 xing_index.json 获取该邦国的姓
- 若该人物在 person_xingshi.json 中无姓记录 → 推理补填
- confidence = medium，标注推理路径：`"via": "邦国:{邦国名}"`

**Round 3：氏族推理**
- 对文本中每个〖@人名〗，找出其所属〖&氏族〗
- 若该氏族已在 person_xingshi.json 中有人物记录 → 提取氏
- 若该人物无氏记录 → 推理补填
- confidence = high（氏族关联明确），标注推理路径：`"via": "氏族:{氏族名}"`

**Round 4：父系推理**
- 从实体关系或文本中识别父子关系（"某之子" / "某父" / 世系列表）
- 对父已知姓氏、子未知的情况：
  - 子继承父的姓（规则 R4）
  - 若无创氏事件，子继承父的氏（规则 R5）
  - 若有创氏事件（受封 / 官职 / 祖名），记录新氏，不继承（规则 R6）
- confidence = 父的置信度降一级，最高 medium

### Step 3：冲突检测

对新推理出的条目，检查：
1. 同一人物是否出现两个不同的姓 → 输出冲突报告，不写入
2. 同一人物是否出现两个不同的氏 → 检查是否为改氏，记录变化时间点
3. "同姓"记载是否与推理结果一致 → 不一致则输出警告

### Step 4：输出

对每个推理出的新条目，按以下格式追加到 `person_xingshi.json`：

```json
"人物名": {
  "xing": "...",
  "shi": "...",（如无则省略）
  "ming": "...",
  "zi": "...",（如无则省略）
  "period": "pre-qin",
  "xing_heritage": "...",（可选，说明血缘来源）
  "shi_type": "封地|官职|祖名|其他",（如有氏）
  "shi_origin": "...",（如有氏）
  "common_confusion": "...",（如有典型误解）
  "confidence": "exact|high|medium|low",
  "via": "...",（推理路径，如"邦国:楚"或"父系:屈瑕"）
  "evidence": ["原文引用1", "原文引用2"]
}
```

最后输出本轮推理摘要：
- 新增条目数
- 各置信度级别的分布
- 发现的冲突或特殊案例（如有）
- 建议更新 SKILL_07b 的新规则或例外（如有）

## 本次任务说明

{task_notes}
```

## 调用示例

### Round 1：楚国人物直接提取

```
轮次：Round 1（直接提取）
范围：040_楚世家、084_屈原贾生列传
输入文件：
  - chapter_md/040_楚世家.md
  - chapter_md/084_屈原贾生列传.md
任务说明：
  重点处理楚国贵族。芈姓邦国，主要氏族为屈/景/昭。
  注意"楚之同姓"的语义——指芈姓血缘同源，不代表近亲。
```

### Round 2：姬姓诸侯国全扫

```
轮次：Round 2（邦国推理）
范围：004_周本纪 及 031-044 世家中的姬姓封国
输入文件：
  - chapter_md/004_周本纪.md
  - chapter_md/031_吴太伯世家.md 至 chapter_md/039_晋世家.md
任务说明：
  周王室分封的诸侯皆为姬姓，以封国名为氏（如鲁氏、晋氏）。
  注意田氏代齐后齐国人物改为妫姓。
```

### Round 4：世家父系链传播

```
轮次：Round 4（父系推理）
范围：某一世家的完整世系列表
输入文件：
  - chapter_md/{章节}.md
  - kg/entities/data/person_xingshi.json（已有种子数据）
任务说明：
  从世家的世系列表中提取父子链，从已知祖先向下传播姓氏。
  标注每个新推理条目的 via 字段为"父系:{父名}"。
  若某代有明确的受封记录，在该代起记录新氏并停止向下传旧氏。
```
