# 提示词模板：事件索引年代审查

本模板用于生成130章事件索引的年代审查提示词。由 `generate_review_prompts.py` 使用。

## 迭代机制

本模板和SKILL文件构成一个**滚动改进**的循环：

1. **审查章节 N** → 按模板生成提示词 → agent执行审查
2. **agent发现新问题** → 更新 `SKILL_04c_事件年代推断.md`（新增错误模式或推断策略）
3. **审查下一章 N+1** → 提示词第一步读取已更新的SKILL → 自动获得前序章节的经验
4. **如需调整模板本身**（如新增检查项、修改工作流程），手动编辑本文件或 `generate_review_prompts.py`，然后重新生成后续章节的提示词

**核心原则**：SKILL文件是"知识积累层"（自动更新），本模板是"工作流程层"（手动迭代）。

## 可调参数（在 generate_review_prompts.py 中配置）

| 参数 | 说明 |
|------|------|
| `CHAPTER_ERA` | 章节→时代分类映射 |
| `ERA_FRAMEWORKS` | 时代→年代框架文本 |
| `CHAPTER_NOTES` | 章节→特殊注意事项 |
| `CHAPTER_BACKGROUNDS` | 章节→背景说明（已修正过的章节） |

## 用法

```bash
# 生成全部130章提示词
python kg/events/scripts/generate_review_prompts.py

# 生成单章
python kg/events/scripts/generate_review_prompts.py 001

# 生成范围
python kg/events/scripts/generate_review_prompts.py 001-012

# 生成多章
python kg/events/scripts/generate_review_prompts.py 001 005 009 130
```

输出目录：`kg/events/prompts/review_{chapter_id}.md`

## 模板正文

```
你的任务是审查并修正《史记》{chapter_id}_{chapter_name}的事件索引中的年代标注。

## 背景

{background}

## 输入文件

1. **事件索引**（待审查和修改）：`kg/events/data/{chapter_id}_{chapter_name}_事件索引.md`
2. **审查规则**（必须先读取）：`SKILL_04c_事件年代推断.md`
3. **人物生卒年参考**：`kg/entities/data/person_lifespans.json`
4. **君主在位年参考**：`kg/chronology/data/reign_periods.json`
5. **年表参考**（可交叉验证）：`kg/chronology/data/中国历史大事年表.md`

## 工作流程

### 第一步：读取规则
先完整读取 `SKILL_04c_事件年代推断.md`，理解所有推断策略和常见错误模式。
（注意：SKILL文件会随审查迭代不断更新，每次都要读取最新版本。）

### 第二步：建立时代框架
{era_framework}

### 第三步：读取事件索引
完整读取 `kg/events/data/{chapter_id}_{chapter_name}_事件索引.md`，统计事件总数，识别锚点分布（有多少个精确纪年事件、推断纪年事件、无纪年事件）。

### 第四步：逐事件审查
对每个事件，按以下检查清单审查：

1. **帝王/时代归属**：该事件属于哪个时期？当前标注的年份是否落在合理范围内？
2. **崩逝事件**：事件名含"崩"、"卒"、"薨"、"死"→年份应取去世年，非生卒中点
3. **即位事件**：事件名含"即位"、"立"、"践位"→年份应取即位年
4. **单锚点塌缩**：是否所有推断事件都指向同一年份？若是，检查是否被单锚点误导
5. **锚点性质**：锚点是"事件时间"还是"写作/记录时间"？
6. **格式一致性**：概览表与详情的年份是否一致？有无双日期冗余？
7. **推断理由**：详情的 `- **年代推断**: ...` 行是否存在且与标注一致？
{extra_checks}

### 第五步：执行修改
对发现的每个问题，同时修改：
- **概览表**的时间列
- **详情部分**的 `- **时间**:` 行
- **详情部分**的 `- **年代推断**:` 行（更新或新增推断理由）

### 第六步：汇总报告
完成所有修改后，输出审查报告（含修改汇总表、统计、存疑事件）。

### 第七步：反思SKILL
审查中如发现**新的**错误模式或推断逻辑（不在SKILL已有错误模式中），追加到 `SKILL_04c_事件年代推断.md` 的"常见错误模式"部分。

如无新发现，不要修改SKILL文件。不要重复添加已有模式。

## 注意事项

{notes}
- 概览表与详情的年份必须完全一致
- 只修改有明确依据的错误，不要为了"精确"而虚构年份
- 每个修改都必须有推断理由
- 如果当前标注合理（在事件所属时期范围内），不需要修改
```
