## 技术说明

### 提取方法

**a2o-lite（Anything2Ontology Lite）核心原则**：

1. **MECE原则**（Mutually Exclusive, Collectively Exhaustive）
   - Facts相互独立，避免重复
   - Skills完全穷尽，覆盖核心方法
   - 保持合理粒度，质量重于数量

2. **结构化数据保留**
   - 世系、年表等结构化数据使用JSON格式
   - 保持表格完整性，不拆分单元格
   - 叙事性内容使用Markdown格式

3. **事实与技能分离**
   - Facts：回答"是什么"（人物、事件、制度）
   - Skills：回答"怎么做"（方法、流程、框架）
   - Eureka：回答"为什么"（深层原理、跨域启示）

4. **质量控制**
   - 每个SKU包含清晰的name、description
   - 每个SKU映射到具体使用场景
   - Eureka仅包含真正跨领域的洞察

### 文件格式

**Facts格式**：
```markdown
---
name: short-identifier-name
description: 一句话描述
---

# 标题

## 内容结构
- 分段清晰
- 数据完整
- 引文准确
```

**Skills格式**：
```markdown
---
name: skill-name-in-hyphen-case
description: 何时使用此技能
---

## 概述
## 步骤
## 决策点
## 预期结果
```

**JSON格式**（结构化数据）：
```json
{
  "name": "identifier-name",
  "description": "一句话描述",
  "data": {
    // 结构化数据
  }
}
```

### 目录结构

每个章节的标准结构：
```
chapter_NNN/
├── skus/
│   ├── facts/          # 事实性知识单元
│   │   ├── fact_001.md/.json
│   │   └── ...
│   └── skills/         # 程序性知识单元
│       ├── skill_001.md
│       └── ...
├── mapping.md          # SKU使用场景映射
├── eureka.md           # 跨领域洞察
└── README.md           # 章节概览与说明
```
