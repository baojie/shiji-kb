# 专家型领导 vs 协调型领导

## 概述

吴起与田文的对话揭示了一个重要问题：**个人能力强不等于适合最高位置**。不同时期、不同职位需要不同类型的领导者。

## 核心区别

### 吴起型：专家型领导

**擅长领域**：
1. **军事指挥**："将三军，使士卒乐死，敌国不敢谋"
2. **内政管理**："治百官，亲万民，实府库"
3. **战略防御**："守西河而秦兵不敢东乡，韩赵宾从"

**特点**：
- 垂直领导能力强（对下属）
- 专业能力突出
- 执行力强
- 亲力亲为
- 以身作则

**局限**：
- 难以协调各方利益
- 容易树敌（"废公族"→贵族反对）
- 个人风格强烈（"刻暴少恩"）
- 不善政治平衡

### 田文型：协调型领导

**核心能力**：
- 不是单项能力最强
- 而是"让大臣附，让百姓信"

**特点**：
- 水平协调能力强（各部门、各阶层）
- 大局观
- 平衡各方利益
- 稳定性
- 凝聚共识

**适用场景**：
"主少国疑，大臣未附，百姓不信"

## 识别场景

### 场景1：平时vs危机

**平时（稳定期）**：
- 需要：协调型领导
- 原因：平衡各方，维持稳定
- 案例：田文任相

**危机（动荡期）**：
- 需要：看具体危机类型
- 如果是军事危机→专家型（吴起守西河）
- 如果是政治危机→协调型（田文稳定大局）

### 场景2：创业vs守成

**创业期**：
- 需要：专家型领导
- 原因：突破、执行力、专业能力
- 特点：强势推进，不怕树敌

**守成期**：
- 需要：协调型领导
- 原因：稳定、平衡、可持续
- 特点：协调各方，长治久安

### 场景3：改革vs稳定

**改革期**：
- 需要：取决于改革策略
- 激进改革→专家型（如吴起在楚）
- 渐进改革→协调型（如田文）

**稳定期**：
- 需要：协调型领导
- 原因：维持现状，平衡利益

## 判断标准

### 标准1：关键挑战是什么？

**田文的问题**：
"主少国疑，大臣未附，百姓不信，方是之时，属之于子乎？属之于我乎？"

**分解**：
1. **主少**：君主年幼，权威不足
2. **国疑**：国家动荡，人心不稳
3. **大臣未附**：重臣各怀心思
4. **百姓不信**：民众缺乏信心

**核心挑战**：凝聚共识，稳定大局

**需要**：协调型领导（田文）

**如果是军事危机**：
- 敌国入侵，需要速战速决
- 核心挑战：军事指挥
- 需要：专家型领导（吴起）

### 标准2：成功关键是什么？

**协调型场景**：
- 成功关键：让各方接受、合作
- 例子：政治过渡、联盟组建、组织整合

**专家型场景**：
- 成功关键：专业突破、快速执行
- 例子：技术攻关、市场开拓、军事作战

### 标准3：失败后果是什么？

**如果选错（吴起在楚国）**：
- 吴起能力强但用协调型任务
- 以专家型方式（"刻暴少恩"）推进改革
- 触犯所有贵族
- 结果：被杀，改革失败

**如果选对（田文稳定魏国）**：
- 田文能力不如吴起
- 但在政治过渡期用协调型方式
- "让大臣附，让百姓信"
- 结果：魏国稳定

## 操作步骤

### 步骤1：诊断当前情况

**问题清单**：
1. 组织处于什么阶段？（创业/成长/成熟/衰退）
2. 面临什么挑战？（技术/市场/管理/政治）
3. 关键任务是什么？（突破/稳定/转型/整合）
4. 最大风险是什么？（外部竞争/内部分裂/资源短缺/时机错过）

### 步骤2：评估领导者类型

**专家型特征**：
- 在某领域能力突出
- 亲力亲为
- 决策果断
- 执行力强
- 个人风格强烈

**协调型特征**：
- 综合能力平衡
- 善于授权
- 决策谨慎
- 平衡能力强
- 善于妥协

### 步骤3：匹配场景与领导者

**匹配矩阵**：

| 场景 | 需要类型 | 原因 |
|------|----------|------|
| 主少国疑 | 协调型 | 凝聚共识 |
| 外敌入侵 | 专家型 | 军事指挥 |
| 技术攻关 | 专家型 | 专业突破 |
| 组织整合 | 协调型 | 平衡利益 |
| 市场开拓 | 专家型 | 快速执行 |
| 联盟组建 | 协调型 | 多方协调 |

### 步骤4：调整或更换

**选项A：调整领导风格**
- 专家型学习协调（难度大）
- 协调型学习专业（难度大）
- 吴起的教训：知道但做不到

**选项B：分工合作**
- 专家型负责执行（吴起守西河）
- 协调型负责协调（田文任相）
- 各司其职

**选项C：换人**
- 田文死后→吴起离开魏国
- 认识到自己不适合在魏国发展

## 决策点

### 决策点1：我是哪种类型？

**自我评估**：
1. 我的强项是什么？（专业vs协调）
2. 我的性格是什么？（强势vs温和）
3. 我的成功经验是什么？（突破vs稳定）

**吴起的自知之明**：
- 三项能力都强于田文
- 但承认"主少国疑"时应"属之田文"
- 这是难得的自知之明

### 决策点2：组织需要什么？

**田文的判断**：
- 不是比较单项能力
- 而是判断组织需求
- "主少国疑"需要协调型

**如果判断错误**：
- 吴起在楚国：以为需要专家型改革
- 实际：也需要协调型稳定（废公族树敌太多）
- 结果：失败

### 决策点3：如何过渡？

**场景变化时**：
- 从创业到守成：专家型→协调型
- 从稳定到危机：协调型→专家型
- 关键：及时识别场景变化

**魏国案例**：
- 魏文侯时：需要吴起守西河（专家型）
- 魏武侯初期：需要田文稳定（协调型）
- 分阶段使用不同类型领导

## 现代应用

### 1. 企业管理

**创业公司CEO**：
- 早期：需要专家型（产品、市场突破）
- 成长期：需要协调型（组织建设、文化塑造）
- 很多创业者失败在转型期

**解决方案**：
- 自我转型（难）
- 引入职业经理人（专家型创始人+协调型CEO）
- 分工（创始人负责产品，CEO负责管理）

### 2. 技术vs管理路线

**技术专家**（吴起型）：
- 擅长：技术攻关、项目执行
- 适合：CTO、技术总监
- 不一定适合：CEO、总经理

**管理者**（田文型）：
- 擅长：团队建设、资源协调
- 适合：CEO、总经理
- 不一定适合：CTO（如果需要技术深度）

**吴起的困惑**：
- 为何我技术、管理、战略都强
- 但田文当CEO？
- 因为：CEO需要的是协调，不是单项最强

### 3. 项目经理选择

**技术项目**：
- 需要：技术专家型PM
- 原因：技术决策是关键

**跨部门项目**：
- 需要：协调型PM
- 原因：协调各部门是关键

**客户项目**：
- 需要：取决于客户类型
- 技术型客户→专家型PM
- 决策型客户→协调型PM

### 4. 组织变革

**激进改革**（吴起在楚）：
- 需要：强势专家型领导
- 风险：阻力大，可能失败
- 适合：危机时刻，破釜沉舟

**渐进改革**（田文式）：
- 需要：协调型领导
- 优势：阻力小，可持续
- 适合：正常时期，稳步推进

## 吴起的悲剧

### 知道vs做到

**吴起知道**：
- 在德不在险（对魏武侯）
- 田文的价值（主少国疑需要协调）
- 自己不如田文（某些方面）

**吴起做到**：
- 在楚国"刻暴少恩"（违反"在德"）
- 触犯所有贵族（没学会田文的协调）
- 死于反扑（悲剧结局）

### 根源分析

**可能原因**：

1. **个性使然**：
   - 猜忍人也（性格刻薄）
   - 难以改变天性

2. **环境压力**：
   - 楚国积弊深重
   - 需要快速见效
   - 没时间慢慢协调

3. **角色错位**：
   - 吴起是执行者思维
   - 相位需要协调者思维
   - 能力与职位不匹配

4. **认知盲区**：
   - 理论上知道
   - 实践中不会
   - 知行不一

## 核心洞察

### 1. 最高职位≠最强能力

**田文不如吴起（单项能力）**
**但田文适合相位（综合能力）**

**原因**：
- 相位需要协调，不是单项最强
- CEO需要平衡，不是专业最深
- 最高职位需要的能力和中层不同

### 2. 识时务者为俊杰

**吴起的自知之明**：
- 承认田文适合当时的魏国
- "属之子矣"

**吴起的盲点**：
- 在楚国没有同样的判断
- 以为自己适合推进改革
- 忘记了需要协调

### 3. 能力陷阱

**陷阱**：
- 因为某方面强→被提拔
- 但新职位需要其他能力→失败

**吴起案例**：
- 因为军事、内政强→被任为相
- 但相位需要协调→失败

**预防**：
- 识别新职位需要什么
- 评估自己是否具备
- 必要时拒绝（孙膑：刑余之人不可）
