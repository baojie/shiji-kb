## 版本说明

这是应用**Reflection-Driven Iteration方法论**对史记知识图谱进行重构的第一轮迭代。

### 与ontology-v1的差异

| 维度 | ontology-v1 | ontology-v2/v1 |
|------|-------------|----------------|
| **组织策略** | 章节级分散提取（675个SKU） | 主题级深度提炼（目标10-30个高价值SKU） |
| **SKU类型** | Hard Facts主导（64.3%） | Soft Facts + Meta Skills主导（目标50%+） |

| **方法论** | 传统知识抽取 | Gap分析+迭代优化 |

### v1的定位

根据Reflection-Driven Iteration的5步法，v1是**"框架建立"阶段**：
- **目标**: 快速建立知识框架（而非完整覆盖）
- **覆盖率目标**: 30-40%
- **时间分配**: 20%总时间
- **核心任务**: 识别ontology-v1的最大Gap，创建最核心的SKU

## Gap分析摘要

完整Gap分析见：[GAP_ANALYSIS.md](GAP_ANALYSIS.md)

### 发现的最大Gap

应用8种Gap发现方法后，识别出ontology-v1的三大缺口：

#### 1. **Soft Facts严重不足**（占比7.4%，目标20-30%）
- 缺少：情感分析、象征意义、哲学洞察
- 示例：司马迁的历史哲学、史记的悲剧精神、天人之际的思考

#### 2. **Meta Facts完全缺失**（占比0%，目标5-10%）
- 缺少：如何读史记、史学方法论、纪传体vs编年体
- 示例：阅读指南、史学思想、知识组织框架

#### 3. **跨章节综合主题缺失**
- 缺少：春秋五霸、战国七雄、史记十大经典场景
- 示例：需要跨越多个世家/列传的综合性叙事

## 核心创新点

### 1. 方法论驱动

**ontology-v1的问题**:
- 缺少"如何使用这个知识库"的指导
- 缺少"为什么史记重要"的价值论证

**v1的解决方案**:
- skill_001：提供系统性阅读方法
- fact_001：提供哲学框架与价值论证

### 2. 深度优先于广度

**ontology-v1的策略**:
- 675个SKU，平均每个SKU 200-400行
- 追求"全面覆盖"

**v1的策略**:
- 2个SKU，每个SKU 5,000-6,000字
- 追求"深度理解"

**哲学差异**:
- v1：知道"什么"（事实） + "为什么"（思想） + "怎么做"（方法）
- v2：主要回答"什么"（事实） + "怎么做"（技能）

### 3. 用户导向

**典型用户问题**（v1重点解决）:
1. ❓ "我想读史记，从哪里开始？" → ✅ skill_001
2. ❓ "史记为什么重要？" → ✅ fact_001
3. ❓ "司马迁是怎么写史记的？" → ✅ fact_001
4. ❓ "如何理解史记的结构？" → ✅ skill_001
5. ❓ "读史记需要什么背景知识？" → ✅ skill_001

## 使用指南

### 如何使用v1

**场景1：史记入门**
1. 先读 `skill_001`（如何阅读史记）→ 获得阅读方法
2. 按照skill_001的建议，开始阅读史记原文
3. 读完30篇后，回到 `fact_001`（司马迁的史学思想）→ 理解更深

**场景2：理解史记的价值**
1. 先读 `fact_001`（司马迁的史学思想）→ 理解"为什么史记重要"
2. 对照ontology-v1的具体人物/事件SKU → 看具体案例如何体现史学思想

**场景3：教学/研究**
- 使用 `fact_001` 作为"史记导论"课的理论框架
- 使用 `skill_001` 作为"史记阅读指南"讲义

### 与ontology-v1的协同使用

**v1与v2互补**:
- **v1（ontology-v1）**：回答"XXX是谁""XXX发生了什么"（事实查询）
- **v2/v1（本版本）**：回答"为什么""怎么做"（深度理解）

**推荐工作流**:
1. 从v2/v1开始（建立框架）
2. 使用v1（查询具体事实）
3. 回到v2/v1（深化理解）

## 方法论总结

### Reflection-Driven Iteration的适用性

**在史记知识图谱构建中的适用性**: ⭐⭐⭐⭐⭐

**为什么适用**:
1. **复杂度高**：史记52万字，130篇，不可能一次性完美提取
2. **价值层次多**：事实层（人物/事件）→ 规律层（历史演变）→ 智慧层（哲学思考）
3. **用户需求多样**：初学者需要指南，研究者需要深度分析，普通读者需要精彩故事

**8种Gap发现方法的收获**:

| 方法 | 在史记项目中的应用 | 推荐度 |
|------|-------------------|--------|
| 方法1: 系统性重读 | 重新审视史记的五体结构 | ⭐⭐⭐⭐⭐ |
| 方法2: 维度检查 | 发现叙事维度、方法论维度缺失 | ⭐⭐⭐⭐⭐ |
| 方法3: 对比法 | 发现典故gap | ⭐⭐⭐⭐ |
| 方法4: 引文检查 | 发现经典名句gap | ⭐⭐⭐⭐⭐ |
| 方法5: 用户视角 | 发现"如何读史记"gap | ⭐⭐⭐⭐⭐ |
| 方法6: 类型覆盖 | 发现Soft Facts、Meta Facts缺失 | ⭐⭐⭐⭐⭐ |
| 方法7: 叙事弧线 | 发现跨章节综合主题gap | ⭐⭐⭐⭐ |
| 方法8: 段落追踪 | 不适用（130篇太大） | ⭐⭐ |

### 下一轮迭代的改进点

**v1的不足**:
- SKU数量太少（只有2个），未达到"5-10个"的目标
- Hard Facts完全缺失
- 跨章节综合主题（春秋五霸、战国七雄）未覆盖

**v2改进方向**:
1. 新增5-8个SKU，达到总数7-10个
2. 平衡SKU类型（补充Hard Facts）
3. 新增至少3个跨域洞察（eureka.md）
