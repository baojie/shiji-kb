# 史记知识图谱 Gap 分析报告（基于 ontology-v1）

**分析日期**: 2026-04-05
**分析对象**: kg/ontology/ontology-v1（675个SKU）
**分析方法**: 应用 Reflection-Driven Iteration 的8种Gap发现方法

---

## 执行摘要

Ontology-v1 已经是一个相当完整的知识库，包含：
- **Factual SKUs**: 434项（事实性知识）
- **Procedural SKUs**: 241项（技能知识）
- **Relational knowledge**: 1,336条关系三元组
- **覆盖范围**: 130篇史记章节的核心人物、事件、制度

**核心Gap识别**：ontology-v1 采用的是"章节级分散提取"策略，每个SKU对应一个具体的人物/事件/制度。这种方式适合细粒度查询，但缺少：
1. **跨章节综合性主题**（如"战国七雄的兴衰"需要跨越多个世家章节）
2. **历史分期视角**（如"春秋霸主时代"vs"战国变法时代"的结构性差异）
3. **叙事弧线**（史记作为一部"纪传体史书"的叙事结构如何组织这些知识）
4. **方法论知识**（如何阅读史记、如何理解司马迁的编史思想）

- 统一的实体标注体系（entities.json）
- 完整的索引系统（facts_index.md, skills_index.md, relational_index.md）

**3. 工程化实现**
- 可机器读取的JSON格式（skus_index.json, ontology_manifest.json）
- 与实体索引系统集成（kg/entities/）
- 可扩展的SKU框架

### B.2 ontology-v1的局限

**1. 粒度过细**

| 维度 | 现状 | 问题 | 示例 |
|------|------|------|------|
| 人物 | 每个人物一个SKU | 缺少人物类型学的综合分析 | 有"荆轲"SKU，但缺"刺客群体的共性"分析 |
| 事件 | 每个事件一个SKU | 缺少事件链的综合叙事 | 有"长平之战"SKU，但缺"战国战争演变"综合分析 |
| 制度 | 每个制度一个SKU | 缺少制度演变的长时段视角 | 有"郡县制"SKU，但缺"从封建到郡县"的转型分析 |

**案例对比**：

**ontology-v1的方式**（分散）:
```
sku_086: 荆轲传记
sku_087: 专诸传记
sku_088: 聂政传记
sku_089: 豫让传记
sku_090: 要离传记
```

**ontology-v2的方式**（综合）:
```
fact_XXX: 刺客群体的共性分析
- 社会背景：战国乱世，君主需要"非常手段"
- 心理动机："士为知己者死"的义士精神
- 成功率：大多失败（荆轲、聂政），少数成功（专诸）
- 历史意义：个人vs权力的象征
- 司马迁的评价：同情+赞美（《刺客列传》序）
```

**2. 缺少"为什么"层面的分析**

| SKU类型 | 回答的问题 | 未回答的问题 |
|---------|-----------|-------------|
| Factual SKU | "XXX是什么""XXX发生了什么" | "为什么XXX重要""XXX的深层意义" |
| Procedural SKU | "如何做XXX"（具体技能） | "为什么这样做""方法论原理" |

**案例**：sku_433（太史公自序）

**现有内容**（事实层）:
- 司马迁的家世
- 《史记》的写作过程
- 李陵事件
- 宫刑的屈辱

**缺失内容**（思想层）:
- **为什么**司马迁要写"究天人之际，通古今之变，成一家之言"？
- **如何理解**"发愤著书"的史学传统？
- **为什么**纪传体是一种创新？
- **如何评价**司马迁的历史哲学观？

→ 这正是ontology-v2/v1的fact_001要补充的内容

**3. 缺少跨章节的综合视角**

**问题**：ontology-v1是"章节级提取"，导致跨章节主题分散

**案例1：春秋五霸**

在ontology-v1中，五霸分散在不同SKU：
- sku_XXX: 齐桓公（《齐世家》）
- sku_YYY: 晋文公（《晋世家》）
- sku_ZZZ: 秦穆公（《秦本纪》）
- sku_AAA: 楚庄王（《楚世家》）
- sku_BBB: 吴王阖闾（《吴世家》）

**缺失**：
- "五霸"作为一个历史阶段的整体叙事
- 霸权为何在五国之间转移？
- 五霸时代vs战国时代的差异是什么？

**案例2：战国七雄**

在ontology-v1中，七雄分散在30篇世家中

**缺失**：
- 七雄如何从春秋诸侯演变而来？
- 为何最终秦统一？（不仅是"秦强"，还有结构性原因）
- 七雄的政治制度、经济模式、军事策略的对比

**4. 缺少方法论指导**

**问题**：ontology-v1是一个"知识仓库"，但缺少"使用说明书"

**用户困境**：
- 675个SKU，从哪里开始？
- 如何找到自己需要的SKU？（虽有mapping.md，但仍需要"如何使用mapping"）
- 如何理解史记的整体结构？
- 如何用这个知识库学习史记？

**ontology-v2/v1的补充**：
- skill_001：如何阅读史记（入门指南）
- fact_001：司马迁的史学思想（理解框架）
- skill_002（v2待补充）：如何使用史记知识库

### B.3 两种策略的适用场景对比

| 场景 | ontology-v1 | ontology-v2 |
|------|-------------|-------------|
| **快速查询** | ✅ 优秀 | ⚠️ 不适用 |
| 示例："项羽是谁" | 直接查sku_XXX（项羽本纪） | 需先理解框架，效率低 |
| **深度学习** | ⚠️ 分散，需自行整合 | ✅ 优秀 |
| 示例："如何理解史记" | 需读多个SKU自行综合 | fact_001+skill_001完整覆盖 |
| **主题研究** | ⚠️ 需跨多个SKU | ✅ 优秀 |
| 示例："春秋五霸兴衰" | 需读5个世家SKU | fact_XXX（v2待补充）一篇完整 |
| **教学使用** | ⚠️ 需教师自行组织 | ✅ 优秀 |
| 示例："史记导论课" | 需从675个SKU中选取 | fact_001作为理论框架 |

**结论**：两者互补，而非替代
- ontology-v1：细粒度查询，全面覆盖
- ontology-v2：深度分析，主题综合

   - 不可能一次性完美提取
   - 需要多轮迭代优化

   - 8种方法中，7种高度适用（⭐⭐⭐⭐⭐）
   - 发现了v1未发现的系统性缺口（Soft Facts、Meta Facts）

   - 三维模型（重要性+独特性+叙事价值）符合史记的特性
   - 量化评分避免了主观随意

4. **迭代节奏的科学性**：
   - v1聚焦框架（35%覆盖率）
   - v2扩展广度（预计65%）
   - v3深化深度（预计85%）
   - v4-v5收尾（预计95-98%）

### E.2 方法论的局限与改进

**局限1：段落追踪法不适用**

- 方法8（段落级覆盖追踪）在busy-beaver中有效（单一文本，2185行）

- 将"段落级"改为"章节级"
- 创建 CHAPTER_COVERAGE_TRACKING.md（130行，而非52万行）

**示例格式**：
```markdown
| 章节 | 篇名 | v1覆盖 | v2覆盖 | v3覆盖 | 覆盖的SKU | 覆盖率 |

| 008 | 高祖本纪 | ✗ | ○ | ✓ | fact_XXX (待补充) | 70% |
...
```

**局限2：v1 SKU数量未达目标**

**问题**：
- 原定目标：5-10个SKU
- 实际完成：2个SKU（fact_001, skill_001）

**改进建议**：
