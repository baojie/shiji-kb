## 技术细节

### 目录结构创建

```python
facts_dir = chapter_dir / 'skus' / 'facts'
skills_dir = chapter_dir / 'skus' / 'skills'
facts_dir.mkdir(parents=True, exist_ok=True)
skills_dir.mkdir(parents=True, exist_ok=True)
```

使用`mkdir(parents=True, exist_ok=True)`确保：
- 自动创建多层父目录
- 目录已存在时不报错

### JSON模板生成

```python
fact_001_data = create_fact_json_template(chapter_num, info)
fact_001_path.write_text(
    json.dumps(fact_001_data, ensure_ascii=False, indent=2),
    encoding='utf-8'
)
```

关键参数：
- `ensure_ascii=False`：保留中文字符
- `indent=2`：格式化输出，便于阅读
- `encoding='utf-8'`：确保中文正确写入

### Markdown模板生成

使用f-string和`chr(10).join()`生成：
```python
{chr(10).join(f'- **{theme}**' for theme in info['core_themes'])}
```

等价于：
```python
'\n'.join(['- **主题1**', '- **主题2**', ...])
```

### 问题1：章节特色差异大

**挑战**：16个章节类型多样（诸侯世家、功臣世家、起义世家）

### 问题2：内容填充工作量大

**解决**：
- 先建框架后填内容（框架优先）
- 分组分优先级执行
- 重点章节深度提炼，次要章节保持框架
