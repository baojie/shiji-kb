# 每日工作日志

## ⏰ 重要说明

**工作日划分以早上7:00为界，而非午夜0:00**

- `YYYY-MM-DD.md` 包含：`(前一天 07:00)` 至 `(当天 07:00)` 的提交
- 例如：`2026-03-17.md` = `2026-03-16 07:00` ~ `2026-03-17 07:00`
- 详细说明见：[TIME_BOUNDARY.md](TIME_BOUNDARY.md) | [USAGE.md](USAGE.md#⏰-重要工作日划分规则)

## 工作流程

### 单个日期（日常使用）

1. **生成详细日志**：`python scripts/generate_daily_log.py YYYY-MM-DD`
2. **AI生成并更新微信通知**：
   - 请Claude根据 `SKILL_daily_wechat_summary.md` 总结
   - AI会自动将摘要添加到日志文件最前面
3. **发送到群**：打开日志文件，复制"微信群通知"部分的内容

### 批量生成所有历史日志（一次性）

```bash
# 为所有有git提交的日期生成日志和微信通知
cd logs/daily
python generate_all_daily_logs.py
```

这个脚本会：
- 扫描git历史，找出所有有提交的日期
- 为每个日期生成详细日志（如不存在）
- 自动生成并添加微信群通知（如不存在）
- 跳过已有微信通知的日志

💡 日志文件格式：
```markdown
# 工作日志 YYYY-MM-DD

## 微信群通知

```
【史记知识库 YYYY-MM-DD】

· 核心工作1
· 核心工作2
...
· 提交N次代码
```  ← 直接复制代码块内容

---

## 核心功能变化  ← 详细内容
...
```

## 日志模板

每个日志包含以下部分：

1. **核心功能变化** - 主要功能开发、重要bug修复、架构调整
2. **技术细节** - 具体实现细节、数据统计
3. **下一步计划** - 待办事项和规划

## 微信群通知格式建议

```
【史记知识库 YYYY-MM-DD】
✅ 核心进展：
- 第一条重点
- 第二条重点

📊 数据：关键指标

🔜 下一步：计划事项
```
