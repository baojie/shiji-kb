# 每日工作日志使用说明

## ⏰ 重要：工作日划分规则

**本系统以早上7:00为工作日分界线，而非传统的午夜0:00**

### 时间范围定义

- `YYYY-MM-DD.md` 包含的提交时间：`(YYYY-MM-DD-1) 07:00` 至 `(YYYY-MM-DD) 07:00`
- 例如：`2026-03-17.md` 包含 `2026-03-16 07:00` ~ `2026-03-17 07:00` 的所有提交

### 为什么这样设计？

1. **符合实际作息** - 凌晨0-7点的工作通常是前一天的延续
2. **方便统计** - 次日早上7点后可以完整统计前一天的工作
3. **避免切分** - 深夜加班不会被切分到两天

### 示例

| 提交时间 | 计入日志 | 说明 |
|---------|---------|------|
| 2026-03-16 23:30 | 2026-03-17.md | 前一天深夜算次日 |
| 2026-03-17 02:00 | 2026-03-17.md | 凌晨算当日 |
| 2026-03-17 06:59 | 2026-03-17.md | 7点前算当日 |
| 2026-03-17 07:00 | 2026-03-18.md | 7点起算次日 |
| 2026-03-17 08:00 | 2026-03-18.md | 白天算次日 |

## 快速开始

### 1. 自动生成昨天的日志

```bash
cd logs/daily
python generate_log.py
```

这会：
- 分析昨天的所有git提交
- 自动生成 `YYYY-MM-DD.md` 日志文件
- 输出微信群通知格式

### 2. 生成指定日期的日志

```bash
cd logs/daily
python generate_log.py 2026-03-17
```

### 3. 批量生成历史日志

```bash
cd logs/daily
# 生成最近7天的日志
for i in {0..6}; do
  date=$(date -d "-$i days" +%Y-%m-%d)
  python generate_log.py "$date"
done
```

## 每日工作流程

### 每天下班前（或次日早上7点后）

**步骤1：生成详细日志**

```bash
cd logs/daily
python generate_log.py
```

这会生成包含多级列表和详细分类的日志文件（方便查阅）。

**步骤2：AI生成并更新微信通知**

请Claude Code根据SKILL生成并更新微信通知：

```
根据 logs/daily/SKILL_daily_wechat_summary.md，为2026-03-17生成微信通知
```

或简单地说：
```
总结2026-03-17的工作日志
```

Claude会：
1. 读取 `SKILL_daily_wechat_summary.md` 了解规则
2. 读取详细日志文件 `2026-03-17.md`
3. 提取3-4条核心工作
4. 简化技术术语，生成平实文本
5. **自动将摘要追加到日志文件最前面**
6. 输出确认信息

**步骤3：发送通知**

打开日志文件 `2026-03-17.md`，在开头会看到：

```markdown
## 微信群通知

```
【史记知识库 2026-03-17】

· 跨章反思修正615处人名标注错误
· 完成001-010章单字人名消歧39处
· 完成011章第二次反思32处修正
...
· 提交13次代码
```
```

复制代码块中的内容（使用全角中点列表），粘贴到微信群即可。

### 日志文件结构

每个日志包含：
- **核心功能变化** - 按类别自动分类（新功能/实体反思/反思规则/Bug修复等）
- **技术细节** - 提交统计、涉及文件数等
- **下一步计划** - 需要手动填写

### 微信群通知格式

Claude AI会根据SKILL_daily_wechat_summary生成列表格式（示例）：

```
【史记知识库 2026-03-17】

· 跨章反思修正615处人名标注错误
· 完成001-010章单字人名消歧39处
· 完成011章第二次反思32处修正
· 完成012章反思59处修正
· 统一18类实体标注格式
· 更新实体统计文档（15190词条）
· 姓氏推理覆盖2053人物（56.6%）
· 重建HTML和实体索引
· 提交13次代码
```

**特点：**
- 使用全角中点（·）列表
- 每行一个工作项
- 5-9项工作内容
- 平实语言，突出关键数字
- 最后一项为"提交N次代码"

直接复制粘贴到微信群即可。

## 自定义

### 修改分类规则

编辑 `scripts/generate_daily_log.py` 中的 `categorize_changes()` 函数。

当前分类包括：
- 反思规则更新
- 实体反思
- 可视化
- 文档
- Bug修复
- 新功能
- 工具脚本
- 其他

### 修改微信通知格式

编辑 `generate_wechat_message()` 函数。

## 提示

1. **提交信息要规范** - 脚本依赖提交信息进行分类，写清楚提交内容很重要
2. **每天提交** - 没有提交记录就无法生成日志
3. **手动完善** - 自动生成后可以手动编辑 `.md` 文件，补充细节
4. **下一步计划** - 记得填写"下一步计划"部分
