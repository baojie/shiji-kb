# SKILL: 每日工作日志微信群总结

## 目标

根据详细的每日工作日志（markdown格式，包含多级列表和分类），生成适合微信群发送的简短平实文本，并将摘要追加到日志文件的最前面。

## 输入

用户会提供：
1. 日期（如 2026-03-17）
2. 或直接提供日志文件路径

## 工作流程

### 1. 读取日志文件

```bash
# 日志文件位置
logs/daily/YYYY-MM-DD.md
```

使用Read工具读取完整内容。

**重要**：日志文件由脚本自动生成，格式为：
```markdown
# 工作日志 YYYY-MM-DD

## 核心功能变化
...

## 技术细节
...
```

### 2. 分析日志内容

从日志中提取：
- 核心功能变化（按类别）
- 提交次数
- 关键数字（修正数、新增数等）

重点关注：
- **新功能** - 优先级最高
- **实体反思** - 核心工作
- **反思规则更新** - 方法论进展
- **Bug修复** - 重要修复
- 其他分类次之

### 3. 生成微信通知

**格式要求：**

```
【史记知识库 YYYY-MM-DD】

· 核心工作1（包含关键数字）
· 核心工作2（包含关键数字）
· 核心工作3
...
· 提交N次代码
```

**具体要求：**

1. **列表格式**
   - 使用全角中点（·）作为列表符号
   - 每行一个工作项
   - 最多10项（包括"提交N次"）
   - 一般6-8项为宜

2. **纯文本格式**
   - 无markdown语法
   - 无阿拉伯数字编号
   - 无多级结构
   - 不使用emoji（除标题的【】）

3. **语言风格**
   - 平实易懂的中文
   - 避免技术黑话
   - 简化版本号（如v2.8→直接说做了什么）
   - 将技术动作转化为成果描述

4. **内容选择**
   - 提取5-9条核心工作
   - 每条尽量包含关键数字
   - 最后一项固定为"提交N次代码"

5. **优先级**
   - 带数字的成果（如"修正615处"）优先
   - 完成性工作（如"完成XX"）优先
   - 新增功能优先于维护
   - 核心模块优先于辅助工具

### 4. 简化技术术语示例

| 原文 | 简化后 |
|------|--------|
| v2.8格式统一：18类实体全部迁移为〖TYPE X〗格式 | 统一18类实体标注格式 |
| 内联消歧语法 v2.7：扩展到所有13种实体类型 + lint修复 | 消歧语法扩展到13类实体 |
| 重建HTML/实体索引，更新反思文档和SKILL规则 | 更新实体索引和反思文档 |
| 人名实体跨章反思：615处修正（邦国/氏族/动词/边界损坏） | 跨章反思修正615处人名标注 |

## 核心功能变化

### 实体反思
- 人名实体跨章反思：615处修正（邦国/氏族/动词/边界损坏）

【史记知识库 2026-03-17】

## 实施步骤

当用户请求生成微信通知时：

1. **确认日期**
   ```
   用户：生成今天的微信通知
   或：总结2026-03-17的工作
   ```

2. **读取日志**
   ```python
   Read logs/daily/2026-03-17.md
   ```

3. **分析提取**
   - 列出所有核心变化
   - 标注优先级
   - 提取关键数字

4. **生成总结**
   - 按优先级选择3-4条
   - 简化技术术语
   - 用顿号连接
   - 加上提交次数

5. **更新日志文件**

   使用Edit工具，在日志文件的**最前面**插入微信摘要：

   ```markdown
   # 工作日志 YYYY-MM-DD

   ## 微信群通知

   ```
   【史记知识库 YYYY-MM-DD】

   · 核心工作1（包含数字）
   · 核心工作2（包含数字）
   · 核心工作3
   · 核心工作4
   · 提交N次代码
   ```

   ---

   ## 核心功能变化

   （原有内容保持不变）
   ...
   ```

   **重要**：
   - 在标题 `# 工作日志 YYYY-MM-DD` 之后立即插入
   - 使用 `## 微信群通知` 作为二级标题
   - 摘要用代码块包裹（方便复制）
   - 用 `---` 分隔线与后续内容分开
   - 不改动原有的详细内容

6. **输出确认**

   更新完成后，输出：
   ```
   ✅ 已将微信通知摘要添加到 logs/daily/YYYY-MM-DD.md

   可直接复制以下内容发送到微信群：

   【史记知识库 YYYY-MM-DD】

   核心内容1、核心内容2、核心内容3。提交N次。
   ```

## 质量检查

生成后检查：
- [ ] 是否纯文本（无markdown语法）
- [ ] 是否有编号或层级结构（应该没有）
- [ ] 是否超过150字（应该≤150）
- [ ] 是否包含关键数字
- [ ] 是否易于非技术人员理解
- [ ] 是否适合直接复制到微信群

## 使用方式

```bash
# 1. 先生成详细日志
cd logs/daily
python generate_log.py 2026-03-17

# 2. 然后请Claude根据此SKILL生成微信通知
用户：根据SKILL_daily_wechat_summary，为2026-03-17生成微信通知
```

## 注意事项

1. **不要**自动生成，必须根据实际日志内容总结
2. **不要**使用模板，每天的总结应该不同
3. **不要**遗漏重要数字
4. **要**确保语言平实，适合群发
5. **要**保持简洁，突出核心成果
