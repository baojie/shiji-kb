# logs/ 目录说明

本目录存放项目运行日志、工作记录、反思报告与分析结果。

## 目录结构

```
logs/
├── daily/           # 每日工作日志（YYYY-MM-DD.md）
├── reflection/      # Agent反思与审查报告
├── lint/            # 脚本运行日志与校验结果
├── curation/        # 文本整理记录
├── analysis/        # 数据分析报告
└── runs/            # 其他运行日志
```

---

## 各子目录说明

### `daily/` - 每日工作日志

**用途**：记录每日工作进展、commit统计、任务完成情况

**文件命名**：`YYYY-MM-DD.md`

**生成方式**：
```bash
python logs/daily/generate_log.py YYYY-MM-DD
```

**内容结构**：
- 微信群通知（简要总结）
- Git提交统计
- 详细工作内容（按commit分组）

**参考规范**：[`SKILL_10b_每日工作日志维护.md`](../skills/SKILL_10b_每日工作日志维护.md)

---

### `reflection/` - Agent反思与审查报告

**用途**：存放AI Agent的自动反思、批量审查、问题修复报告

**典型文件**：
- `entity_boundary_*.md/json` - 实体边界检测与修复
- `*_batch_review_*.md` - 批量审查总结
- `reflection_XXX_*.json` - 章节反思结果
- `time_entity_reflection_report.*` - 时间实体反思报告
- `issue_*.md` - 问题修复报告

**文件特点**：
- 通常由Agent自动生成
- 包含错误检测、修复建议、执行结果
- JSON格式文件用于结构化数据，MD格式用于可读报告

---

### `lint/` - 脚本运行日志与校验结果

**用途**：存放自动化校验脚本的运行日志

**典型文件**：
- `lint_text_integrity_*.txt` - 文本完整性校验日志（按章节编号命名）
- `lint_text_integrity_chapter_md/` - 按章节组织的校验结果

**文件命名规则**：
- 单章：`lint_text_integrity_XXX.txt`（XXX为章节编号）
- 多章：`lint_text_integrity_XXX_YYY_ZZZ.txt`
- 最新：`lint_text_integrity_latest.txt`

**生成方式**：
```bash
python scripts/lint_text_integrity.py
```

**说明**：此目录包含94个校验日志文件，记录了文本完整性检查的历史过程。

---

### `curation/` - 文本整理记录

**用途**：存放人工整理发现的文本问题与修正记录（包括校勘、勘误等）

**子目录结构**：
```
curation/
├── curation_XXX_问题描述.md    # 单个问题记录
└── reports/                     # 系统化校对报告
    └── NNN_篇名_校对报告.md
```

**单个问题记录示例**：
- `curation_016_项羽分封后表头.md` - 项羽本纪表头问题
- `curation_088_劳人是愆脱字.md` - 扶苏列传脱字问题
- `curation_089_信诸侯vs信都侯.md` - 张耳陈馀列传校勘

**校对报告示例**：
- `reports/001_五帝本纪_校对报告.md` - 完整章节校对记录

**内容结构**：
- 问题描述
- 底本对照
- 修正建议
- 参考文献

---

### `analysis/` - 数据分析报告

**用途**：存放对标注数据的统计分析、模式挖掘报告

**典型文件**：
- `officials_*.txt` - 官职标注分析报告

**说明**：这些分析有助于发现标注规律、优化标注策略。

---

### `runs/` - 其他运行日志

**用途**：存放不属于上述分类的脚本运行日志

**说明**：目前为空，预留用于未来的其他自动化任务日志。

---

## 文件管理原则

### 何时创建新文件

1. **每日工作日志**：每个工作日结束时生成
2. **反思报告**：Agent执行反思流程后自动生成
3. **校验日志**：运行校验脚本时自动生成
4. **校勘记录**：发现文本问题时手工创建

### 文件清理策略

- **保留**：
  - 所有每日工作日志（永久保留）
  - 重要的反思报告（如最终修复报告）
  - 最新的校验日志

- **可归档**：
  - 中间过程的校验日志（如多章联合校验的临时文件）
  - 已解决问题的反思报告（移至 `archive/`）

### 版本管理建议

- `daily/` - 必须纳入Git版本管理
- `reflection/` - 重要报告纳入版本管理
- `lint/` - 最新结果纳入版本管理，历史日志可选
- `curation/` - 必须纳入Git版本管理
- `analysis/` - 纳入版本管理
- `runs/` - 通常不纳入版本管理（可添加至 `.gitignore`）
