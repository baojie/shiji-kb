## 技术实现

### 1. 年表章节表格解析器

**适用范围**: 014_十二诸侯年表、015_六国年表

**核心原理**:
- 每行`[rX]`对应一个公元年（从year_state_map.json获取）
- 每个单元格包含年份文本（元年、二、三、十、二十...）
- 使用正则`(元年|[一二三四五六七八九十百]+)`提取年份
- 排除在`〖〗`标注内的文本
- 段落ID使用`rX`而非普通段落号（避免覆盖）

**数据流**:
```
year_state_map.json (635年)
    ↓
build_row_to_year_map() → {1: -841, 2: -840, ...}
    ↓
process_table_chapter() → 提取单元格年份
    ↓
year_ce_map.json (4654条table_row映射)
```

**关键代码**:
```python
def process_table_chapter(filepath, chapter_id, row_to_year, year_state_map):
    # Parse table header
    state_cols = extract_state_columns(header_line)

    # Process each row
    for line in lines:
        if match('[rX]'):
            ce_year = row_to_year[X]
            for cell_idx, cell in enumerate(data_cells):
                year_texts = extract_year_numbers(cell)
                ruler_key = year_state_map[ce_year][state_cols[cell_idx]][0]

                for year_text in year_texts:
                    year_map[f'r{X}'][year_text] = {
                        'ce_year': ce_year,
                        'ruler': ruler_key,
                        'method': 'table_row'
                    }
```

### Unresolved分析（99个）

经分析，Unresolved主要包含：
1. **上古时期**（约40%）：夏商周早期，无公元年数据
2. **特殊格式**（约30%）：年号年份（如"建元六年"），年号未在reign_periods中
3. **缺少上下文**（约20%）：年份周围无人物标注，无法确定君主
4. **孤立引用**（约10%）：引用其他文献的年份，缺乏本章上下文

## 经验总结

### 成功经验

1. **数据驱动**: 通过统计分析找到真正的问题（年表章节被跳过）
2. **结构化方法**: 表格解析器利用了固有结构（行=年）
3. **渐进优化**: 先解决大头（table_row 69.9%），再优化细节（+0.07%）
4. **质量优先**: 宁可Unresolved，不强制错误映射

### 技术亮点

1. **表格行解析**: 无需标注，直接从表格结构提取
2. **去重机制**: dict自动去重，避免重复计算
3. **多层策略**: 5层消歧策略（era→nearby→sequential→section→raw）
4. **容错设计**: ±5年容错范围，out-of-range自动修正

### 教训与改进

1. **统计口径**: 早期混淆"标注数"vs"处理数"vs"映射数"，导致分析偏差
2. **过滤逻辑**: 月份/季节/时长的过滤规则需要不断完善
3. **参数调优**: nearby_ruler搜索范围（60→120）是关键，但没有系统化调优过程
4. **文档先行**: 应先写SKILL文档再写代码，避免事后补文档
