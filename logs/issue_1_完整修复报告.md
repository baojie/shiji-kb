# Issue #1 时间实体类型错误完整修复报告

## 问题来源

GitHub Issue: https://github.com/baojie/shiji-kb/issues/1

**问题描述**：
- 发现有一些「地理范围」被归类到了「时间」里面
- 纯数字好像也被归类到了「时间」里面

## 修复过程

### 第一阶段：数量词误标为时间

#### 问题发现

通过系统性分析，发现 **44处** 数量词被错误标注为时间类型 `〖%X〗`，涉及 20 个章节。

#### 问题类型

1. **数量+度量单位**：
   - `〖%千石〗` → `〖$千石〗`（容量单位）
   - `〖%八尺馀〗` → `〖$八尺馀〗`（长度单位）
   - `〖%二百蹄〗` → `〖$二百蹄〗`（马匹数量）
   - `〖%千足〗` → `〖$千足〗`（动物数量）

2. **数量+物品**：
   - `〖%八篇〗` → `〖$八篇〗`（书篇数量）
   - `〖%三章〗` → `〖$三章〗`（法律章节数）
   - `〖%十驷〗` → `〖$十驷〗`（车马数量，四马为一驷）
   - `〖%千社〗` → `〖$千社〗`（土地社稷数量）

3. **数量+行政单位**：
   - `〖%十五郡〗` → `〖$十五郡〗`（郡县数量）
   - `〖%五六郡〗` → `〖$五六郡〗`（郡县数量）

#### 修复工具

创建 [`scripts/fix_time_quantity_confusion.py`](../scripts/fix_time_quantity_confusion.py)

**检测规则**：
- 含有数量单位字（蹄、足、石、章、畦、里、步、亩、丈、尺、寸、篇、卷、社、车、马、人、户、家、郡）的标注
- 不含时间特征字的标注
- 排除特殊词（战国、国中、国家、郡国）

## 技术方案总结

### 1. 自动化脚本

#### fix_time_quantity_confusion.py
- **功能**：批量修复数量词误标为时间
- **输入**：所有 `.tagged.md` 文件
- **输出**：修正后的标注文件 + 修复报告
- **特点**：支持预览模式（--dry-run）

#### reflect_time_entity_types.py
- **功能**：系统性审查所有时间标注
- **分析维度**：
  - 时间特征（年、月、日、季、干支、纪元、时辰）
  - 非时间特征（度量单位、物品、行政单位）
  - 可疑度评估
- **输出**：JSON + Markdown 双格式报告

### 2. 检测规则

#### 时间特征字
```
年、月、日、岁、世、元、载、
春、夏、秋、冬、
子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥、
甲、乙、丙、丁、戊、己、庚、辛、壬、癸、
朝、夕、夜、晨、暮、旦、昏、时、刻、
前、后、先、今、昔、往、来
```

#### 数量单位字
```
度量：蹄、足、石、章、畦、里、步、亩、顷、丈、尺、寸、斤、两、钱、贯
物品：匹、驷、双、辈、篇、卷、社、车、马
行政：人、户、家、郡
```

### 3. 排除规则

特殊词语虽含数量字，但不做修复：
- **战国**：历史时期（已移除标注）
- **国中、国家、郡国**：地名/政治概念
