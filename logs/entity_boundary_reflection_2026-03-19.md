## 一、反思方法论

### 1.1 边界错误的定义

实体边界错误是指标注的起止位置不正确，导致：
- **吞噬前后文字**：标注包含了不应该包含的字符
- **切分错误**：正确的实体被错误切分
- **遗漏部分**：实体的一部分未被标注

### 1.2 检测策略

采用启发式规则检测8类潜在边界错误：

1. **身份词吞噬前文**：数量词/范围词被标注吞噬
2. **动词被标注吞噬**：实体标注内包含动词
3. **官职与人名分裂**：人名+官职分裂为两个标注
4. **姓氏与名字分裂**：姓氏被标注，名字在外
5. **地名吞噬后缀**：地名标注包含结构助词
6. **实体内嵌套其他标注**：标注内部嵌套另一标注
7. **连续身份标注**：连续身份标注可能需要合并
8. **数量词被其他实体吞噬**：非数量实体标注内包含数量词

## 三、真实错误识别与修复

### 3.1 误报分析

检测到的15,332处"疑似错误"中，大多数是**误报**：

#### (1) 姓氏与名字分裂 (6,058处)

**误报原因**：检测规则过于简单，将"〖@X〗+助词"都判定为分裂

**示例**：
- `〖@黄帝〗者` - ✅ 正确（"者"是判断词）
- `〖@黄帝〗之时` - ✅ 正确（"之时"是时间状语）
- `〖@蚩尤〗作乱` - ✅ 正确（"作乱"是谓语动词）

**真实错误极少**，需要人工逐一审查。

#### (2) 数量词被其他实体吞噬 (7,390处)

**误报原因**：很多实体名本身包含数量词

**示例**：
- `〖@三公〗` - ✅ 正确（"三公"是固定身份词）
- `〖@九卿〗` - ✅ 正确（"九卿"是固定官职）
- `〖'六国〗` - ✅ 正确（"六国"是历史概念）

**白名单已过滤**：三公/九卿/五帝/六国等

### 3.2 真实边界错误

#### **类型1: 诸侯服边界错误**

**问题**：
```
诸〖^侯服〗〖'秦〗
```

**分析**：
- 误将"侯服"标注为〖^制度〗
- 实际应为：〖#诸侯〗服〖'秦〗
- "诸侯"是身份，"服"是动词（服从）

**修复**：
- 章节: 086_刺客列传, 087_李斯列传
- 数量: 2处
- 规则: `诸〖^侯服〗` → `〖#诸侯〗服`

**修复后验证**：
```
✅ 086: 〖#诸侯〗服〖'秦〗，莫敢〖^合从〗
✅ 087: 今〖#诸侯〗服〖'秦〗，譬若郡县
```

## 五、待进一步分析的模式

虽然本轮只修复了2处确认的边界错误，但检测报告揭示了以下值得进一步审查的模式：

### 5.1 实体内嵌套标注 (679处)

**模式**: `〖X...〖Y〗...〗`

**可能原因**：
- 标注损坏（格式错误）
- 复合实体处理不当

**建议**: 逐一检查，可能需要格式修复

### 5.2 连续身份标注 (286处)

**模式**: `〖_X〗〖_Y〗`

**示例**:
- `〖_弟〗〖_子〗` → 应合并为 `〖_弟子〗`
- `〖_士〗〖_大夫〗` → 应合并为 `〖_士大夫〗`

**建议**: 已有部分脚本处理（fix_identity_tags_global.py），需全面审查

### 5.3 地名吞噬后缀 (86处)

**模式**: `〖=X(之|者|人|氏)〗`

**示例**:
- `〖=秦之〗` → 应为 `〖=秦〗之`（如果"之"不是地名一部分）

**建议**: 需要语境判断，可能部分是误报

## 六、方法论反思

### 6.1 启发式规则的局限

1. **误报率高**：简单模式匹配导致大量误报
2. **语境缺失**：无法理解语义，难以判断边界
3. **白名单维护成本**：需要持续补充例外情况

### 6.2 改进建议

#### (1) 分阶段审查

- **Phase 1**: 高置信度规则（如"诸侯服"）→ 自动修复
- **Phase 2**: 中等置信度模式 → 生成TSV人工审查
- **Phase 3**: 低置信度模式 → 标记待查，不自动修复

#### (2) 基于语境的检测

需要引入：
- 词性判断（动词/名词/助词）
- 语法结构分析（主谓宾）
- 领域词表（固定搭配）

#### (3) 增量式白名单

建立分类白名单：
```
kg/entities/data/boundary_whitelist.json
{
  "compound_identities": ["三公", "九卿", "士大夫", ...],
  "compound_feudal": ["六国", "七国", "三晋", ...],
  "valid_suffixes": ["者", "之", "也", "矣", ...],
}
```
