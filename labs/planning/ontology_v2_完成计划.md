## 一、完成标准

每个章节达到"完成"需同时满足：

1. **facts 真实化**：`skus/facts/` 下所有文件无 `待补充` / `placeholder` / `TEMPLATE` 字样，且 ≥200 行或 ≥1200 字节（结构化 JSON 不受行数约束）
2. **skills 真实化**：`skus/skills/` 下同上
3. **三文件齐备**：`mapping.md`、`eureka.md`、`README.md` 存在且非骨架
4. **可追溯**：内容来自 `chapter_md/NNN_*.tagged.md` 原文，关键引文标注出处

**参考样本**（五个已完整章节）：`chapter_001`、`chapter_031`、`chapter_061`、`chapter_091`、`chapter_121`

## 三、单章执行循环

**串行执行，不开并行 Agent**。每章按下列步骤：

```
1. 读原文：chapter_md/NNN_*.tagged.md
2. 盘点：ls chapter_NNN/skus/facts/ 与 skills/
3. 判定每文件是否骨架（size ≤1200b 或 含"待补充/placeholder/TEMPLATE"）
4. 对每个骨架文件，从原文提炼真实 SKU
5. 补齐/更新 mapping.md、eureka.md、README.md
6. 汇报：产出 SKU 数、新增字数、剩余骨架数
```

**注**：遇到同一模板在多章重复（049-060 / 093-110），每章独立提炼，不复用模板文字。
