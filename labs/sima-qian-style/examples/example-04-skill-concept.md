# 示例04：论Skill之道（技术概念说明）

本示例演示如何将**技术概念的现代解释**（370字）改写为太史公风格的论说文体。

---

## 原始素材（现代白话，370字）

> 很多人对 Skill 的理解停留在"就是个 Markdown 文件嘛，写点提示词让 AI 照着做"。Anthropic 的人说，这是最常见的误解。Skill 其实是一个文件夹。里面可以放脚本、数据文件、参考代码、模板，甚至 SQLite 数据库。Claude 能自己去翻这个文件夹，找到它需要的东西，然后用起来。这就像你给一个新来的同事安排工作，如果你只给他一张纸条写着"去把这个功能做了"，他大概率会做得磕磕绊绊。但如果你给他一个完整的工具箱，里面有参考代码、常见问题清单、测试脚本、模板文件，他上手就快得多。Skill 就是这个工具箱。想想看，我们平时做任何工作，效率最高的时候往往是手边有一套趁手的工具和参考资料。Skill 的设计思路其实跟这个道理一模一样，把你的经验、流程、工具打包成一个可复用的单元，下次再遇到类似的事，直接调用就行。

## 第1步：白话文 → 文言文

🔗 **调用技能**：`SKILL-白话转文言.md`（第1-8类转换规则）

### 关键转换要点

**1. 虚词删除**：
- 很多人对技艺**的**理解 → 众人以**为**技艺
- 就**是**个文书 → 文书**也**
- 新来**的**同僚 → 新至之僚
- 删除所有"的""着""了"

**2. 动词转换**：
- 理解停留在 → 以为
- **是**一个匣 → **为**匣
- 能自己去翻 → 能自取
- 做**得**磕磕绊绊 → 必多碍
- 入手**就**快**得**多 → 入手易
- **就是** → 即

**3. 句式重组**：
- "很多人对技艺的理解停留在..." → "众人以为技艺者，文书也，书令使智能行之"
- "这就像你给一个新来的同事安排工作" → "譬诸授新至之僚以事"
- "想想看，我们平时做任何工作" → "凡为事者"

**4. 删除口语化表达**：
- "大概率" → 必
- "快得多" → 易
- "一模一样" → 同
- "就行" → 可

### 第1步转换结果（标准文言文，180字）

> 众人以为技艺者，文书也，书令使智能行之。Anthropic公司曰：此最常见之误也。技艺者，匣也，中可置程式、数簿、范式、范本，甚或数库。Claude能自取所需而用之。譬诸授新至之僚以事，若仅予一笺书曰"为此事"，必多碍；若予完备之器匣，内具范式、疑难录、验式、范本，则入手易矣。技艺即此器匣也。凡为事者，效最高时，莫不趁手之器与参典在侧。技艺之制，与此理同，聚验、序、器为可传之具，异日复遇此类事，取用即可。

## 第2步：文言文 → 太史公文风

🔗 **调用技能**：`SKILL-太史公曰.md`

本文属于**论赞体**（技术概念论述），应用太史公议论文特有技法。

### 应用技法详解

#### 1. 开篇定式（X者，Y也）

```
众人以为技艺者，文书也。
```

✅ 先述众人之误见，再破之（先抑后扬）

#### 2. 引述权威

```
Anthropic公司曰：此最常见之误也。
```

✅ 引述权威以破误解（类似"孔子曰""老子曰"）

#### 3. 正面定义

```
技艺者，匣也。
```

✅ 用判断句明确定义

#### 4. 列举内容

```
中可置程式、数簿、范式、范本，甚或数库。
```

✅ 排比列举，最后用"甚或"递进

#### 5. 类比论证（譬诸...）

```
譬诸授新至之僚以事：
若仅予一笺书曰"为此事"，必多碍；
若予完备之器匣...则入手易矣。
```

✅ 太史公常用"譬诸"引出类比，正反对比

**太史公原文参考**：
《史记·货殖列传》："**譬**若锥之处囊中，其末立见。"

#### 6. 先抑后扬

```
【抑】若仅予一笺书曰"为此事"，必多碍；
【扬】若予完备之器匣...则入手易矣。
```

#### 7. 归纳升华

```
凡为事者，效最高时，莫不趁手之器与参典在侧。
```

✅ 用"凡...者"引出普遍规律

#### 8. 类比收束

```
技艺之制，与此理同。
```

✅ 回扣主题，呼应前文

#### 9. 简短断言

```
聚验、序、器为可传之具，异日复遇此类事，取用即可。
```

✅ 最后以简洁语句收束

#### 10. 新增太史公曰

```
太史公曰：
工欲善其事，必先利其器。古之言也。
今之所谓技艺者，非一器，乃百器之匣也。
匣中有序、有验、有范，授人以匣，不若授人以鱼，授人以渔也。
智能虽巧，无匣则茫然；人虽能，无验则蹇滞。
故曰：技可授，匣不可缺也。
```

**结构分析**：
- **引古**：引"工欲善其事"古训
- **联今**：技艺即"百器之匣"
- **类比**：呼应"授人以渔"
- **评判**：工具与经验并重
- **升华**：技可授，匣不可缺

## 常见陷阱

### ❌ 陷阱1：技术术语不统一

**错误**：同时出现"脚本""程式""代码"等混用
```
❌ 里面可以放脚本、代码、程式...
```

**正确**：统一术语
```
✅ 中可置程式、数簿、范式、范本
```

### ❌ 陷阱2：保留现代口语

**错误**：`上手就快得多`、`一模一样`、`就行`

**正确**：改为文言
```
✅ 入手易矣
✅ 与此理同
✅ 取用即可
```

### ❌ 陷阱3：类比论证不完整

**错误**：只有类比，没有对比
```
❌ 这就像给同事安排工作，如果给他工具箱，他就能做好。
```

**正确**：正反对比，先抑后扬
```
✅ 譬诸授新至之僚以事：
   若仅予一笺...必多碍；
   若予完备之器匣...则入手易矣。
```

### ❌ 陷阱4：太史公曰流于空泛

**错误**：
```
❌ 太史公曰：技艺很重要，大家要好好用。
```

**正确**：引古联今，有具体论述
```
✅ 工欲善其事，必先利其器，古之言也。
   今之所谓技艺者，非一器，乃百器之匣也。
   ...
```

## 改写心得

### 技术概念文本的特殊性

**挑战**：
1. 现代科技术语多（30个），需逐一古化
2. 有具体类比（工具箱比喻），需保留类比结构
3. 论述性文本，需选用"论赞体"而非"列传体"

**解决方案**：
1. 第0步彻底古化所有术语（Markdown→文书、脚本→程式）
2. 保留类比结构，但用太史公"譬诸"引出
3. 应用太史公议论文技法：引述权威、类比论证、先抑后扬、归纳升华

### 本文亮点

1. **类比论证完整**：用"譬诸"引出，正反对比（若...必...；若...则...）
2. **引述权威**：保留"Anthropic公司曰"，类似《史记》引"孔子曰"
3. **太史公曰引经据典**：引"工欲善其事"古训，自然过渡到当代技术
4. **赞曰押韵工整**：器-书-内-成，四句总结全文

### 适用场景

本示例适合：
- 技术概念解释
- 产品设计理念阐述
- 方法论论述
- 任何需要类比论证的论说文
