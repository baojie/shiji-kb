# 太史公曰（Taishi Gong Yue - Sima Qian's Style）

将现代素材以司马迁《史记》的文笔风格重写。

---

## 技能分层

本技能采用**两层架构**：

1. **基础层**：调用 `SKILL-白话转文言` 完成现代汉语→文言文转换
2. **风格层**：在文言基础上，应用太史公特有句法和修辞

---

## 工作流（两步转换法）

### 第一步：现代汉语 → 文言文

**🔗 调用技能**：`SKILL-白话转文言.md`

将现代白话彻底转换为文言文。完整规则见该SKILL文档（8大类转换规则、5大句式、自检清单）。

**核心要点**：
- ✅ 删除所有"的、地、得、了、吗、呢、着"
- ✅ 替换所有现代动词（是→为，有→具，被→为...所...）
- ✅ 改写所有现代句式（被动句、疑问句、判断句）
- ✅ 数字汉字化：130 → 百三十，3,185 → 三千馀（概数化）
- ✅ 使用文言专用连词、副词、代词

**结果**：完全文言化的文本，无任何现代白话痕迹

---

### 第二步：文言文 → 太史公文风

**核心原则**：在文言基础上，应用太史公特有句法和修辞

#### 1. 解析与分流

识别素材类型，选择对应文体：

| 素材类型 | 文体 | 特征 |
|---------|------|------|
| 人物传记 | **列传体** | 以人物为中心，突出关键转折，惜墨如金 |
| 历史事件 | **本纪体** | 编年叙事，简洁记事，跳跃式剪辑 |
| 评论议论 | **论赞体** | 引经据典，排比对仗，寓褒贬于叙事 |

#### 2. 应用太史公句式

参考 `references/style-core.md`，应用以下核心句式：

##### 2.1 开篇定式

**模式**：`X者，Y也`

```
史记知识库者，开源古籍知识工程也。
西瓜者，名鲍捷，治智能知识工程三十载，好文史。
```

**变体**：
- 籍贯式：`X者，Y人也`（项籍者，下相人也）
- 职业式：`X者，为Y也`

##### 2.2 列举式

**模式**：`其所为有二：其一...其二...`

```
其所为有二：
其一，以智能之术，化《史记》百三十篇...
其二，沉淀其法为技能文档二十六篇...
```

**注意**：用"其一/其二"而非"一曰/二曰"（后者常用于引述对话）

##### 2.3 层递并列

**模式**：`A，B，C`（动词连用，省略主语，节奏渐强）

```
类十有八，标近十万；事三千馀，系七千馀，单元六百馀
（数据排比，节奏感强）

学书不成，去学剑，又不成。
（反复强调，凸显志向）

易一书，线不改，调参数，新库立成。
（四短句，层层递进）
```

##### 2.4 因果承转

**模式**：`A，故B` / `A，以是B` / `A，然B`

```
古之治文史者，困於人力。（A）
华夏典籍以亿言计，宝库深藏，未尽其用。（进一步说明A）

以其笃学，故绩甚佳。（以A，故B）

虽难，然必为之。（A，然B）
```

##### 2.5 简短断言

**特征**：单句陈述，不加修饰，直击要害，常作收束

```
粲然备矣。（华丽完备了）
靡不具也。（无不具备）
新库立成。（新库立即成功）
竟其功。（完成了任务）
```

##### 2.6 设问自答

**模式**：问题（常用连续两问，递进强化）+ 简短回答

```
尝自问：智能之术，能压皓首穷经之数十载为旬月乎？能令知识活用乎？
（连续两问，递进）

今以一人携群智能体，竟其功。
（简短回答：完成了）
```

#### 3. 应用太史公修辞

##### 3.1 反复强调

**技法**：重复关键词，递进加强

```
学书不成，去学剑，又不成。
（"不成"重复 → 凸显项羽志向之大）

能...乎？能...乎？
（连续设问 → 强化疑问）

命也夫，命也夫！
（"命也夫"重复 → 司马谈的悲愤）
```

##### 3.2 先抑后扬

**模式**：`A则B，然其C`

```
古之治文史者，困於人力。（抑：古人困境）
↓
今以一人携群智能体，竟其功。（扬：今人突破）

儒者博而寡要，劳而少功；然其序君臣父子之礼，不可易也。
（先抑：博而寡要 → 后扬：礼不可易）
```

##### 3.3 排比对仗

**技法**：句式整齐，音律工整，常用于总结性论述

```
类十有八，标近十万；
事三千馀，系七千馀，
单元六百馀，粲然备矣。
（数据排比，三段式，节奏分明）

太史公曰（对仗）：
古人穷经，皓首一生，不过数卷。
今藉智能，旬月可竟全史。
（古 vs 今，皓首一生 vs 旬月，数卷 vs 全史）
```

##### 3.4 白描手法

**原则**：删除所有装饰性形容词，只写动作和结果

```
❌ 繁冗：项籍看到秦始皇威严的仪仗队浩浩荡荡地过江，心中顿时涌起无限豪情...
✅ 太史公：秦始皇帝游会稽，渡浙江，梁与籍俱观。籍曰："彼可取而代也。"

❌ 装饰：非常重要的核心成果
✅ 白描：成果（直接删除"非常重要的核心"）
```

##### 3.5 跳跃式剪辑

**技法**：省略过渡，直接跳到下一场景，时间跨度可能数年

```
[场景1] 项梁杀人，与籍避仇於吴中。
[场景2] 吴中贤士大夫皆出项梁下。
[场景3] 秦始皇帝游会稽...

（省略中间数年经历，不交代如何在吴中建立声望）
```

#### 4. 人物刻画技法

##### 4.1 关键细节定性格

**原则**：一个细节胜过千言万语

```
项籍曰："书足以记名姓而已。剑一人敌，不足学，学万人敌。"
→ 性格：雄心勃勃、不拘小节（不直说，让对话说话）

西瓜者，名鲍捷，治智能知识工程三十载，好文史。
→ 能力+爱好（简洁勾勒人物）
```

##### 4.2 行为即性格

**原则**：不评价，让行为说话

```
❌ 说教：项羽是个骄傲自大的人
✅ 太史公：（后文）项王不肯竟学兵法、分封诸侯失当等行为
→ 读者自己得出结论
```

##### 4.3 他人之口借力

**原则**：用旁人评价暗示主角地位

```
吴中贤士大夫皆出项梁下。
（侧面写项梁声望，不直说"项梁很厉害"）

梁以此奇籍。
（借项梁之眼，写项羽不凡）
```

#### 5. 太史公曰（可选）

**结构**：引古（历史事例/经典语录）→ 联今 → 评判

```
太史公曰：
古人穷经，皓首一生，不过数卷。        （引古）
今藉智能，旬月可竟全史，奇哉！        （联今）
然工具虽利，志在人也。                （评判）
西瓜三十年磨一剑，方得此业。          （具体化）
技可授，志不可夺也。                  （升华哲理）
```

#### 6. 自检与润色

- [ ] 确保无现代白话痕迹（"的""了""吗""很""非常"等）
- [ ] 对话只截取关键一句
- [ ] 用行为代替心理描写
- [ ] 句式简短有力（平均<15字/句）
- [ ] 应用至少3种太史公句式（开篇定式、层递并列等）
- [ ] 应用至少2种太史公修辞（白描、先抑后扬等）
- [ ] **严禁**：直接复用《史记》原文语句

#### 7. 输出成文

- 仅输出最终文本（Markdown格式）
- 可选章节小标题（### 格式）
- 末尾可附"太史公曰"式评论段落（可选）

## 输出格式

**默认**: 仅输出太史公文笔正文，不显示转换过程。

**排版要求**：
- ✅ **每个句子之间空一行**（重要！）
- ✅ 段落分明，易于阅读
- ✅ 使用markdown引用块（`>`）或普通段落格式

**格式示例**：
```
史记知识库者，开源古籍学问事业也。

其所为有二：

其一，以智能之术，化《史记》百三十篇、五十七万言为条理之学。

类十有八，注近十万；事三千馀，系七千馀，学条六百馀。

其二，沉淀其法为技艺文二十六篇，涵九序，自校勘至推理，靡不具也。

易一书，法序不改，调参值，新学府立成。
```

**可选展示**（用户明确要求时）:
- 原文 vs. 改写对比
- 两步转换过程
- 使用的具体句式模板
- 风格自检清单结果

**禁止**:
- ❌ 添加作者署名（如"司马迁""太史公"）
- ❌ 在标题中使用"史记""列传"等字样（除非用户明确要求）
- ❌ 句子连写不空行（影响阅读体验）

内部完成以下检查（不输出，除非用户要求）：

## 注意事项

### 1. 语言时代

- 使用文言文（非现代白话）
- 避免网络用语、新词
- 年份可用公元纪年（如"公元1976年"）或传统纪年（如"癸丑年"）

### 2. 价值中立

- 不强加现代价值观
- 不美化或丑化历史人物
- 保持太史公的"实录精神"（客观记事，寓褒贬于细节）

### 3. 篇幅控制

- 根据素材量适度扩充或压缩
- 典型列传体: 300-800字
- 短评（太史公曰）: 50-150字

### 4. 信息密度

- 文言文信息密度高，字数通常压缩30-40%
- 240字现代白话 → 约160字文言文
