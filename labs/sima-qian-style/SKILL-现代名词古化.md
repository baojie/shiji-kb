# SKILL：现代名词古化

## 技能定位

将现代学术、科技、社会名词转化为古代汉语表达，消除"现代感"，使文本具有古籍韵味。

**调用者**：`SKILL-白话转文言.md`（第9类转换规则）

---

## 核心原则

### 1. 古化策略（按优先级）

| 优先级 | 策略 | 示例 |
|-------|------|------|
| **最优** | 找古代对应物 | 电脑 → 算筹<br>飞机 → 飞舆<br>电话 → 传音符 |
| **次优** | 用功能描述 | 知识图谱 → 知识经纬<br>数据库 → 典籍库 |
| **可选** | 音译+注释 | 计算机 → 计数之器<br>互联网 → 万机互联 |
| **避免** | 直接音译 | ❌ 康普优特（computer）<br>❌ 因特耐特（internet） |

### 2. 禁忌词表（必须替换）

| 现代词 | 禁忌原因 |
|--------|---------|
| 系统、体系、结构化、框架 | 现代学术用语 |
| 项目、工程、计划、方案 | 现代管理用语 |
| 数据、信息、资料、资源 | 现代信息用语 |
| 技术、科技、智能、算法 | 现代科技用语 |
| 平台、网络、系统、模块 | 现代IT用语 |
| 流程、机制、模式、范式 | 现代理论用语 |

## 组合词处理

### 规则1：双音节→单音节

| 现代词 | 古化 | 规则 |
|--------|------|------|
| 知识库 | 学府 | 知识→学，库→府 |
| 数据库 | 典库 | 数据→典，库→库（保留） |
| 项目组 | 事班 | 项目→事，组→班 |
| 工程师 | 工师 | 工程→工，师→师（保留） |
| 研究员 | 治学之士 | 研究→治学，员→士 |

### 规则2：三字词→二字或重组

| 现代词 | 古化 | 规则 |
|--------|------|------|
| 知识图谱 | 知识经纬 / 学问图志 | 图谱→经纬 |
| 知识体系 | 学统 | 知识→学，体系→统 |
| 数据分析 | 数析 / 考数 | 数据→数，分析→析 |
| 人工智能 | 智能之术 | 重组为"之"字结构 |
| 机器学习 | 机习之法 | 重组为"之"字结构 |

### 规则3：四字词→精简

| 现代词 | 古化 | 规则 |
|--------|------|------|
| 结构化知识 | 条理化之学 / 知识条理 | 结构化→条理化 |
| 知识工程 | 学问之事 / 治学事业 | 工程→事业 |
| 智能知识工程 | 智能治学之事 | 删除冗余词 |
| 自然语言处理 | 语文析理 | 大幅简化 |

## 特殊场景

### 场景1：现代专有名词（无古代对应物）

**策略**：用功能描述 + 删减修饰词

| 现代名词 | 不推荐 | 推荐 | 原因 |
|---------|-------|------|------|
| 知识图谱 | ❌ 知道地图 | ✅ 知识经纬 | "经纬"既有图谱含义，又古雅 |
| 数据库 | ❌ 数字仓库 | ✅ 典库 / 簿库 | "典"有"经典文献"意，"簿"有"账册"意 |
| 智能体 | ❌ 聪明机器 | ✅ 智能体 / 机灵 | "智能体"已较古雅 |
| 机器学习 | ❌ 机器读书 | ✅ 机习 / 自习之术 | 简洁为上 |

### 场景2：叠床架屋的现代词组

**策略**：拆解→古化→重组

**示例**：
```
现代：智能知识工程研究项目
拆解：智能 + 知识 + 工程 + 研究 + 项目
古化：智能之术 + 学问 + 事业 + 治 + 事
重组：治智能学问之事
精简：智能治学事
```

**示例2**：
```
现代：结构化知识图谱构建方法论
拆解：结构化 + 知识 + 图谱 + 构建 + 方法论
古化：条理 + 学问 + 经纬 + 建 + 法
重组：建学问经纬之法
精简：构学问经纬法
```

### 场景3：数量+名词

**策略**：数字后置 + 名词简化

| 现代 | 古化 | 规则 |
|------|------|------|
| 18类实体 | 类十有八 | 数字后置，实体→删除 |
| 10万次标注 | 标注十万 / 注十万 | 次→删除 |
| 3185个事件 | 事三千馀 | 概数化，个→删除 |
| 26个技能文档 | 技艺文二十六篇 | 技能→技艺，文档→文 |

**初步古化**：
> 此为开源古籍知识工程事。以智能体化《史记》百三十篇为结构化知识图谱。类十有八，标注近十万。

**深度古化（应用本SKILL）**：
| 现代词 | 古化 | 应用规则 |
|--------|------|---------|
| 项目 | 事 | 二、科技工程类 |
| 知识工程 | 治学事业 → 事 | 二、科技工程类 + 精简 |
| AI Agent | 智能体 | 二、科技工程类 |
| 结构化知识图谱 | 条理化之学问经纬 → 知识经纬 | 一、学术研究类 + 精简 |
| 实体类型 | 名物之类 → 类 | 一、学术研究类 + 精简 |
| 标注 | 注 | 一、学术研究类 |

**最终版本**：
> 此为开源古籍治学之事。以智能体化《史记》百三十篇为知识经纬。类十有八，注近十万。

**原文**：
> 西瓜（鲍捷）从事智能知识工程研究三十年。

**初步古化**：
> 西瓜（鲍捷）从事智能知识工程研究三十载。

## 使用注意

### ⚠️ 不要过度古化

**错误示范**：
```
原文：AI能否将皓首穷经数十载的工作压缩到几个月？
过度古化：机灵之术能否将白首竭经数十载之劳压为数朔乎？
（"机灵""白首""竭经""数朔"过于生僻，影响理解）

推荐版本：智能之术，能压皓首穷经之数十载为旬月乎？
（"智能之术"保留"智能"，"旬月"为常见文言词）
```

### ⚠️ 平衡古雅与可读性

**原则**：
- 通用词（如"是""有""很"）→ 必须古化
- 专有名词（如"知识图谱""AI"）→ 适度古化
- 人名地名 → 不古化
- 数字 → 汉字化但保留可读性

**示例**：
```
现代：AI Agent
过度古化：机器之灵（过于晦涩）
适度古化：智能体（保留"智能"，易懂）
```

## 词典使用流程

1. **查表**：在分类词典中查找现代名词对应的古化方案
2. **选择**：根据上下文选择最合适的古化词（优先选"古代对应物"）
3. **组合**：如果是组合词，拆解后逐个古化再重组
4. **精简**：删除冗余修饰词，力求简洁
5. **检查**：用自检清单确认是否消除现代感
