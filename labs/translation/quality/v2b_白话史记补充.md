# v2b 提示词迭代 · 白话史记补充

v2 分析（`v2_分析.md`）主要以 hunterhug 段译为对照，归纳出了 8 类规则，已落入 `SKILL_01h` v2。本轮专注从 **白话史记**（`corpus/shiji/白话史记.txt`，整本单文件、不含原文）抽取补充信号，看它是否能为 v2 提示词再添一批有价值的新规则。

## 1. 采样说明

### 数据源

- 对齐文件：`data/translation_alignment/{002,007,067,087,129}.json`，每 PN 含 `ours / hunterhug / baihua` 三列
- `baihua` 字段来源：整本白话史记按"章内比例切段"粗估，**飘移 1-2 PN**
- 章节覆盖：
  - 002 夏本纪（69 PN）
  - 007 项羽本纪（588 PN，抽样池大）
  - 067 仲尼弟子列传（152 PN）
  - 087 李斯列传（69 PN）
  - 129 货殖列传（43 PN）

### 采样方法

两阶段：

1. **粗筛 50 条**：每章随机抽 10 条（要求 `baihua` ≥15 字），写入 `/tmp/v2b/samples.txt`
2. **精筛 50 条**：用"`ours` vs `baihua` 字符集重合度 ≥0.35"过滤出 **167 条**真正描述"同一史实"的样本（避开飘移），再按每章取重合度最高的 10 条，写入 `/tmp/v2b/aligned.txt`

> **白话史记对齐的根本局限**：由于整本单文件+比例切段，单个 PN 对齐飘移较严重。167/287（58%）样本 overlap 能达到 0.35 以上；其中 19 条达到 0.7 以上——这 19 条才是真正可逐字比对的"金样本"。
>
> 本分析结论基于两类样本：**金样本**（19 条）做字面规则归纳，**中等样本**（~50 条）做风格印象汇总。

### 语料风格速描

- **文风简练**：同一场景比 hunterhug 短 30-50%，删繁就简
- **段落合并**：常把史记相邻多段合为一段，无内部分段标记
- **添译括号纪年**："秦二世元年(前209)"（与 hunterhug 相同的通病）
- **现代化程度高**：消失率统计（`reports/terminology_audit.md`）显示"元年→84%"、"群臣→90%"、"寡人→99%"，远超 hunterhug
- **不保留实体标注**：纯白话，没有 `〖@人名〗` 结构，因此不能用来学消歧

## 2. 模式分类

### 模式 α：地名后缀补全（中频，severity: **medium**，建议写入）

**描述**：原文用单字/二字地名（"蓟"、"博阳"、"栗"），白话史记常补"县/郡/地/城"后缀，让现代读者明白这是地理单位而非人名/通名。hunterhug 有时做（"栗县"），有时不做；白话史记做得更一致。

**金样本**：

| 章#PN | 原文 | 本库 | 白话史记 |
|-------|------|------|---------|
| 007#100.1 | `都蓟` | `建都蓟` | `建都蓟县` |
| 007#20.2 | `栗` | `栗` | `栗县`（hunterhug 同） |
| 007#101.2 | `都博阳` | `建都博阳` | `建都博阳` *（两家都没补，可不补）* |

**判断**：`都X` 这一句式中，若 X 是二字以上的古地名，白话史记常保留；若 X 是单字，补"县"或"地"能显著提升可读性。**本库在单字地名后宜补后缀**。

**建议规则**（新增小条款，附到 v2 规则 #14 "古今歧义地名"之下）：

> **单字地名**（`都蓟 / 至栗 / 都薛`）：在地名后补"县/城/邑/地"，使地理单位清楚。若原文已消歧 `〖= X|Y〗` 则遵从。

---

### 模式 β：文言量词/数词现代化（低频，severity: **low**，建议写入）

**描述**：文言 `数城 / 数十万 / 数百 / 数千万`，白话史记会现代化为 `几座城邑 / 几十万 / 几百 / 几千万`。本库保留文言"数X"，但在现代语境中"数X"有时像"具体数字 4-5"的歧义（"数十" vs "几十"）。

**金样本**：

| 章#PN | 原文 | 本库 | 白话史记 |
|-------|------|------|---------|
| 007#101.2 | `下济北数城` | `攻下了济北数城` | `攻下了济北几座城邑` |

**判断**：轻度改动，`数X` → "几X"在可读性上略有提升，但本库风格一贯保留文言数词（如 129#3 "千里往往山出釭置"），**算风格选择**；若要写入规则，建议仅限 `数X`（X 为名词）场景。

**建议规则**（可选条款）：

> **不定数词白话化**：文言 `数X`（X 为名词，如"数城/数人/数月"）可译为"几X"或"若干X"；**大数保留原形式**（如"数十万"保留为"数十万"或"几十万"均可，不强制改）。

---

### 模式 γ：复合动作展开（中频，severity: **medium**，建议写入）

**描述**：文言"起大泽中"、"至栗"这种"动词+地名"句式，白话史记常展开为"在大泽乡起义"、"到达栗县"（补行为对象/性质），hunterhug 也类似。本库的"陈涉等人在大泽中起事"较接近白话史记，但"大泽" → "大泽乡"这种**补三字地名**值得学习（大泽乡是秦制乡邑，补后更精准）。

**金样本**：

| 章#PN | 原文 | 本库 | 白话史记 |
|-------|------|------|---------|
| 007#7.1 | `陈涉等起大泽中` | `陈涉等人在大泽中起事` | `陈涉等人在大泽乡起义` |
| 007#20.2 | `章邯军至栗` | `章邯的军队进抵栗` | *（未直接命中）* hunterhug: `章邯的军队到达栗县` |

**判断**：白话史记在"历史地名是秦汉制度层面的乡/县/邑"时会补制度后缀。但这类补充**涉及历史知识判断**（需知"大泽"是大泽乡，"博阳"是济北博阳县），错补可能引入新的错误（如"山东" → "山东省"是灾难）。

**建议**：**不作为强规则**，但加一句柔性提示。

**建议规则**（柔性，附到 v2 规则 #14 之下）：

> **著名地名的制度后缀**：对于已被读者熟知为"X乡/X县"的地点（如"大泽乡"、"鸿门"、"垓下"），白话译文可保留原名；若原文单字地名易误读为通名，可按"模式 α"补"县/城/邑"。**不要把"山东"扩写为"山东省"类现代行政名**。

---

### 模式 δ：纪年括号注公元（高频现象，severity: **high**，**保持禁止**）

**描述**：白话史记和 hunterhug 都有此风格：`秦二世元年(前209)七月`、`汉五年(前202)`。**v2 规则 #3（简洁性）已明确禁止"公元前 X 年"类括注**。此处无新规则，只是**再次确认 v2 立场**。

**金样本**：

- 007#7.1 白话：`秦二世元年(前209)七月` — **本库不采纳此风格**
- 007#161.1 白话：`汉五年(前202)` — **同上**

**判断**：此点与 hunterhug 完全一致，白话史记没有给出新的反驳证据。v2 规则 #3 继续执行。

**建议**：**不改 SKILL_01h**。

---

### 模式 ε：专名规范化（中频，severity: **medium**，建议写入）

**描述**：白话史记在复述时倾向把"单字 → 全名"、"异体字 → 规范字"（如"襃氏"→"褒氏"、"荼"→"臧荼"、"籍"→"项籍"）。hunterhug 也这么做。v2 规则 #1 已强制"单字简称展开"，覆盖了人名部分；但**姓氏异体字**（襃/褒、靑/青、壄/野）未成文。

**金样本**：

| 章#PN | 原文 | 本库 | 白话史记 |
|-------|------|------|---------|
| 007#100.1 | `故立荼为燕王` | `所以立荼为燕王` | `所以立臧荼为燕王` |
| 002#28 | `襃氏` | `襃氏` | `褒氏` |
| 007#101.2 | `田安` 处 `立安为济北王` | `立安为济北王` | `立田安为济北王` |

**判断**：**单字名一律全展开**——v2 规则 #1 已覆盖，但"立荼为燕王"这种**分封句式**里的"立+单字名"，本库常漏（以为是专名不用展开）。应明确补充"`立X为…`、`封X为…`、`以X为…` 中的单字 X 必须展开为全名"。

**异体字方面**：白话史记会规范化（襃→褒），本库遵循原文保持"襃"是正确的（这是我们**不擅改原文字形**原则的体现），但在白话译文中展开时可否采用现代常用字形，是个风格问题。v2 没约束，暂**保持现状**（保留异体），由 surface 翻译脚本按需统一。

**建议规则**（并入 v2 规则 #1 第二项）：

> **分封句式展开**：`立X为Y / 封X为Y / 以X为Y / 拜X为Y` 中，若 X 是单字名或单字姓（如 `立荼为燕王`、`立安为济北王`、`立信为楚王`），白话译文**必须展开为全名**：`立臧荼为燕王`、`立田安为济北王`、`立韩信为楚王`。即使该单字在前文已出现，此类句式中仍要展开。

---

### 模式 ζ：语气词/情态词更自然（低频，severity: **low**，不写入）

**描述**：白话史记的情态表达较本库自然，例如：

- 007#105.1 `不使之国` → 本库 `不让他之国` （"之国"两字保留文言） vs 白话 `不许赴封国`（"赴封国"更现代）
- 067#16 `千室之邑` → 本库 `千户的城邑`（直译）vs 白话 `千户之家的城邑`（加"之家"并不更好）
- 007#8.4 `诺` → 本库 `好` vs 白话 `好`（同）

**判断**：白话史记的改写**有优有劣**，不稳定。本库的"千户的城邑"比白话"千户之家的城邑"更简洁准确。hunterhug 的"千户的城邑"和本库一致，可信度更高。

**建议**：**不写入规则**，本库"不使之国"→"不让他赴封国"可个案改进，但不作为强制规则（属于模式 B 的细分）。

---

### 模式 η：量词"之"字串替换（低频，severity: **low**，已被 v2 规则覆盖）

**描述**：本库大量保留 `X之Y`（如 `燕将之属 / 济北数城 / 兵之威`），白话史记几乎不保留。v2 规则 #2 已要求"议论段落句式白话化"且已覆盖此类虚词改写。

**判断**：**不新增规则**，但**建议 v2 规则 #2 在例子里增加 `X之Y → X的Y`**（当前清单中已有 `於是/乃` 等，但没有 `之`）。

**建议规则**（微调 v2 规则 #2 的动词清单，加一条）：

> 在 "议论段落句式白话化" 列表中追加：
> - `X之Y`（X、Y 为实词）→ "X的Y"（如"数城" 保留，但"燕将之属" → "燕将这一类人"、"千里之外" → "千里以外"）。**姓名/朝代名中的"之"勿动**（如"晏平仲"、"宰予之学"—— 前者不含之，后者保留原貌）。

---

### 模式 θ：对话补说话人（中频，severity: **medium**，建议写入）

**描述**：白话史记对连续对话（`A 曰"…" 曰"…" 曰"…"`，后续省略主语）会**主动补全主语**。v2 规则 #1"代词展开"虽已提到"补说话人主语"，但没给出明确的连续对话补全示例。白话史记给出了很好的模板。

**金样本**：

| 章#PN | 原文 | 本库 | 白话史记 |
|-------|------|------|---------|
| 007#58.3 | `曰："毋从俱死也。"`（省略主语"项伯"） | `项伯说："不要跟着一起送死。"`（已补，OK） | *（飘移未命中）* |
| 067#16 | `复问：` → 省略主语"季康子" | `又问：` *（未补主语）* | `季康子又问` |
| 007#8.3 | `梁复入，与守坐，曰：` | `梁再进去…说：` *（单字"梁"未展开，且无"项梁"）* | `项梁又进去，与郡守同坐，说` |

**判断**：白话史记的"**连续对话中每次说话前都明示主语**"是一条清晰规则，可上升为强制条款，与 v2 规则 #1 形成互补。

**建议规则**（强化 v2 规则 #1 第四项，或新增第 16 条）：

> **连续对话必补主语**：多人对话中，第二轮之后每次 `曰` 前都应明示说话人（如"季康子又问"、"孔子回答"、"李斯反驳说"），**不允许出现只用 `又问：` / `复曰：` 而无主语的情况**。同一说话人连续表达可用"接着说"/"又说"，但不同说话人交替必须标名。

**以原文文意裁决冲突**：

## 4. 对 SKILL_01h 的增补建议（草稿文本）

以下是可落入 `SKILL_01h` v2b 的**具体段落文本**（不是此轮直接修改，仅提供素材；由用户审阅后决定是否合并入正式 SKILL）。

### 4.1 强规则（建议并入 v2 规则 #1）

在现有 "**代词/单字简称必须展开**" 子条之后追加：

> - **分封句式必展开**（v2b 新）：`立X为Y / 封X为Y / 以X为Y / 拜X为Y` 中，若 X 为单字名或单字姓（如 `立荼为燕王 / 立安为济北王 / 立信为楚王 / 以胜为陈王`），**白话译文必须展开为全名**：`立臧荼为燕王 / 立田安为济北王 / 立韩信为楚王 / 以陈胜为陈王`。此规则优先级高于"前文已出现则省略"，分封句中必须展开。
> - **连续对话必补主语**（v2b 新）：多人对话中，从第二轮起每次 `曰` 前都应明示说话人（"季康子又问 / 李斯反驳说 / 孔子回答"），**不允许出现 `又问："…"` / `复曰："…"` 这种无主语形式**。同一说话人连续表达可用"接着说 / 又说"。

### 4.2 强规则（建议并入 v2 规则 #14）

在现有 "古今歧义地名" 之下追加：

> - **单字地名补后缀**（v2b 新）：原文中的单字地名作分封/建都的宾语时（`都蓟 / 至栗 / 都薛 / 封于鲁`），白话译文宜补"县/邑/城/地"等后缀（`建都蓟县 / 到达栗县 / 建都薛邑 / 封于鲁地`），以明确地理单位。**不得**补现代行政名（如"山东省 / 北京市"）。对已普遍知道是乡/县的地名（"大泽乡 / 鸿门 / 垓下"）按原形式处理即可。

### 4.3 柔性提示（建议并入 v2 规则 #2 虚词清单）

在现有清单中追加：

> - `X之Y`（实词+之+实词）→ "X的Y"（如"千里之外" → "千里以外"、"燕将之属" → "燕将这一类人"）。**姓/名/字中的"之"不动**（如"王翦之孙"保持为"王翦的孙子"），异体字型字形不改。
> - `数X`（X 为可数名词："数城 / 数人 / 数月"）→ "几X / 若干X"；**大数保留原形**（"数十万"可保留为"数十万"，不强制改"几十万"）。

### 4.4 不写入的项（记录备查）

- **纪年括号注公元**：白话史记 & hunterhug 都做，**v2 明确禁止**，立场保持不变。
- **扩译/合并段**：白话史记的常态特征，与本库"PN 独立、按段翻译"风格根本不兼容，无法作为提示词信号。
- **"礼 → 礼记" 误渲染**：v2 分析已彻底处理（见 `v2_分析.md` 模式 F），本轮无新信号。
- **异体字规范化**（襃→褒、靑→青）：属字形层面，**由原文为准**，不在 LLM 翻译阶段处理；若要统一，应在 `translate_surface.py` 或 `sync_translation_disambig.py` 层面做。

## 5. 核心发现（80 字内）

白话史记对 PN 级对齐飘移较大，可靠信号主要是 4 条：**分封句式单字名必展开**、**连续对话必补主语**、**单字地名补"县/邑"后缀**、**`之` 字串与 `数X` 做现代化**。前两条应升格强规则，后两条作柔性提示。
