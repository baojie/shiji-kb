## 项目目标

构建一套标准化的测试问题集，用于评估各种知识库系统对《史记》知识的覆盖度和理解深度。

### 核心指标

- **问题总数**：1000个（10组 × 100个）
- **章节覆盖**：130个章节（本纪12 + 表10 + 书8 + 世家30 + 列传70）
- **覆盖目标**：每组覆盖 > 100个章节（77%+）
- **互斥性要求**：10组问题完全无交集

## 项目创新点

### 1. 多维度均衡设计

不同于传统的按章节或人物设计问题，本项目采用10个维度的均衡覆盖：
- 人物维度（2组）：基本信息 + 关系网络
- 事件维度（1组）：重大历史事件
- 时空维度（2组）：时间 + 地理
- 制度维度（1组）：官职、礼仪、法律等
- 文化维度（2组）：思想文化 + 器物自然
- 传承维度（1组）：成语典故
- 推理维度（1组）：综合推理

### 2. 互斥性保证

通过以下机制确保10组问题完全无交集：
- **问题文本去重**：任意两组不能有相同问题
- **核心实体不重叠**：同一人物/事件/地点不在多组中作为主要查询对象
- **知识点不交叉**：每个知识点只被一组问题覆盖
- **自动化验证**：使用脚本检测重复和交叉

### 3. 可验证性设计

每个答案都包含：
- **原文引用**：可在史记原文中验证
- **章节出处**：明确的位置标注
- **置信度**：high/medium/low三级
- **相关实体**：使用标准化标注符号（@人名、=地名等）

### 4. 难度梯度设计

- 简单40%：单一事实查询，适合基础知识库
- 中等40%：需要理解上下文，适合中级知识库
- 困难20%：跨章节推理，测试知识图谱能力

## 使用场景

### 场景1：知识库系统评估

**用户**：知识库开发者

**用法**：
1. 使用10组问题测试知识库的查询能力
2. 计算答案准确率（对比标准答案）
3. 分析覆盖率（哪些维度表现好/差）
4. 识别知识盲点

### 场景2：RAG系统测试

**用户**：RAG（检索增强生成）开发者

**用法**：
1. 测试检索精度（能否找到正确章节）
2. 测试生成质量（答案与标准答案的相似度）
3. 难度分级测试（简单/中等/困难的准确率差异）

### 场景3：学习系统评估

**用户**：教育平台开发者

**用法**：
1. 作为学习效果测试题库
2. 按难度分级提供练习
3. 生成个性化学习路径

### 场景4：学术研究基准

**用户**：NLP研究者

**用法**：
1. 作为古文问答（QA）任务的benchmark
2. 测试命名实体识别（NER）
3. 测试关系抽取（RE）
4. 测试知识图谱构建（KG）
