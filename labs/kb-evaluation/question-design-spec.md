# 史记知识库测试问题设计规范

## 文档目的

本文档定义史记知识库测试问题集的详细设计规范，确保问题集的质量、一致性和可验证性。

## 问题设计原则

### 1. SMART原则

每个问题必须符合SMART原则：
- **Specific**：具体明确，避免模糊表述
- **Measurable**：答案可测量、可验证
- **Achievable**：答案可在史记原文中找到
- **Relevant**：与测试目标相关
- **Time-bound**：答案不随时间变化（基于史记原文）

### 2. 问题类型分类

#### 事实型问题 (Factual)
直接查询原文明确记载的事实
- 示例：孔子的字是什么？
- 答案特征：唯一确定答案

#### 关系型问题 (Relational)
查询实体之间的关系
- 示例：韩信与萧何是什么关系？
- 答案特征：关系类型 + 相关证据

#### 推理型问题 (Inferential)
需要基于多处信息进行推理
- 示例：根据史记记载，秦始皇统一六国的顺序是什么？
- 答案特征：需要整合多处信息

#### 比较型问题 (Comparative)
对比多个实体或事件
- 示例：项羽和刘邦的军事策略有何不同？
- 答案特征：多维度对比分析

#### 评价型问题 (Evaluative)
基于史记内容的评价判断
- 示例：太史公如何评价韩信？
- 答案特征：基于"太史公曰"等评论

### 3. 难度分级

#### 简单 (Easy) - 40%
- 直接查询单一事实
- 答案在单一章节明确记载
- 不需要推理

**示例**：
- Q: 项羽的籍贯是哪里？
- A: 下相 (出处: 007_项羽本纪)

#### 中等 (Medium) - 40%
- 需要理解上下文
- 答案可能分散在同一章节的不同位置
- 需要简单推理

**示例**：
- Q: 韩信被封为齐王后，又被改封为什么王？
- A: 楚王 (出处: 092_淮阴侯列传，需要理解事件时序)

#### 困难 (Hard) - 20%
- 需要跨章节信息整合
- 需要复杂推理
- 可能有多个合理答案

**示例**：
- Q: 刘邦重用的"汉初三杰"分别在哪些方面对建立汉朝做出了贡献？
- A: 需要整合092_淮阴侯列传、055_留侯世家、053_萧相国世家等多处信息

## 章节覆盖验证方法

### 覆盖率计算

```python
def calculate_coverage(questions):
    """
    计算问题集的章节覆盖率
    """
    total_chapters = 130
    covered_chapters = set()

    for q in questions:
        covered_chapters.update(q['target_chapters'])

    coverage_rate = len(covered_chapters) / total_chapters
    return coverage_rate, covered_chapters
```

### 分布均衡性检验

使用卡方检验检验章节分布是否均衡：

```python
import scipy.stats as stats

def test_balance(questions):
    """
    检验问题在章节上的分布是否均衡
    """
    chapter_counts = count_questions_per_chapter(questions)
    expected_count = len(questions) / 130

    chi_square, p_value = stats.chisquare(
        chapter_counts,
        [expected_count] * 130
    )

    return p_value > 0.05  # 不显著则认为分布均衡
```

## 互斥性验证方法

### 互斥性定义

10组问题之间完全互斥，定义为：
1. **问题文本不重复**：任意两组间不存在相同的问题文本
2. **核心实体不重叠**：同一核心实体不在多组中作为主要查询对象
3. **知识点不交叉**：同一知识点不被多组问题覆盖

### 互斥性检验算法

```python
def validate_uniqueness(all_sets):
    """
    验证10组问题的互斥性
    """
    # 1. 问题文本去重
    all_questions = []
    for s in all_sets:
        all_questions.extend([q['question'] for q in s['questions']])

    if len(all_questions) != len(set(all_questions)):
        return False, "存在重复问题文本"

    # 2. 核心实体不重叠检验
    entity_to_sets = {}
    for set_id, s in enumerate(all_sets):
        for q in s['questions']:
            core_entity = extract_core_entity(q)
            if core_entity:
                if core_entity in entity_to_sets:
                    entity_to_sets[core_entity].append(set_id)
                else:
                    entity_to_sets[core_entity] = [set_id]

    # 检查是否有实体出现在多个组
    overlaps = {e: sets for e, sets in entity_to_sets.items()
                if len(sets) > 1}

    if overlaps:
        return False, f"实体重叠: {overlaps}"

    # 3. 知识点交叉检验
    # (基于答案相似度)
    for i in range(len(all_sets)):
        for j in range(i+1, len(all_sets)):
            if has_knowledge_overlap(all_sets[i], all_sets[j]):
                return False, f"第{i+1}组与第{j+1}组存在知识点交叉"

    return True, "10组问题完全互斥"
```

## 答案质量标准

### 答案完整性

每个答案必须包含：
1. **核心答案**：简短明确的回答
2. **原文依据**：史记原文引用
3. **章节出处**：具体章节位置
4. **补充说明**：必要的背景解释

### 答案可验证性

- **单一出处**：答案可在单一章节验证（简单问题）
- **多处印证**：答案需要多个章节印证（中等问题）
- **推理依据**：列出推理所需的所有前提（困难问题）

### 答案置信度

- **高置信**：原文明确记载，无歧义
- **中置信**：需要合理推理，但推理路径清晰
- **低置信**：存在多种合理解释，需要标注
