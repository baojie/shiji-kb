# 修改意见建议

> 基于 quality_audit.csv 和 quality_audit_report.md 的数据，按优先级排列

---

## P0 — 紧急修复（数据安全 + 核心功能）

### 1. 清理 934 个孤儿文件

**问题**：磁盘上有 934 个 `.md` 文件，其文件名与 frontmatter ID 不匹配。924 个是旧版本的残存副本，与当前版本内容不同。

**方案**：编写 `scripts/cleanup_orphan_files.py`，扫描 `docs/wiki/pages/` 目录，对每个文件检查 `filename ≠ id.md` 且 registry 中无此条目时，先与正确文件名版本做 diff，确认内容已涵盖后删除旧文件。

**优先级**：高。冗余文件可能导致后续编辑时混淆。

### 2. 补「史记引文」章节（A1）

**问题**：7,812 页（38%）缺少「史记引文」节，其中 5,841 页有 sources 字段（具备生成条件）。

**方案**：
- 对已有 sources 的 5,841 页：批量运行 `/quote` 流程（引用 butler 的 W3 补引文 pipeline）
- 对无 sources 的页面（约 2,000 页）：先补 sources，再补引文
- **预期产出**：可新增约 6,000+ 页的引文内容

**优先级**：高。引文是史记知识库的核心差异化特性。

### 3. 修复 291 处断链（D1）

**问题**：142 页面含有指向不存在条目的链接。最突出的是 `避讳` 页面（91 处断链）和多个带反斜杠后缀的章节页链接。

**方案**：
- 反斜杠后缀类（如 `008_高祖本纪\`）：批量替换删除尾部 `\`
- `避讳` 页面：先创建被引用的页面（如 `汉元帝`、`景驹` 等），或改写链接
- WantedPages 类：对链接失效的实体，评估是否需要创建新页面

**优先级**：高。断链影响用户体验和知识图谱完整性。

## P1 — 批量改进（元数据 + 分类）

### 4. 批量补 tags（B3：9,585 页无 tag）

**问题**：近一半页面缺少 tags 标签，无法被分类检索。

**方案**：按页面类型自动赋值默认 tag：
- person 无 tag → 加 `[人物]`
- event 无 tag → 加 `[事件]`
- place 无 tag → 加 `[地名]`
- 再根据 sources 字段关联章节，推断二级 tag（如 `[秦本纪]`、`[本纪]`）

**优先级**：高。机械操作，可批量脚本完成。

### 5. 补 canonical_name（E2：4,489 个 person 页面缺）

**问题**：所有 person 页面均缺少 `canonical_name` 字段。

**方案**：
- 已有 label 的 person 页面：`canonical_name = label`（大多数情况成立）
- 需要消歧的页面（如同名不同人）：手动补充

**优先级**：中高。影响跨页面链接消歧。

### 6. 补 place/state 坐标（E1：1,463 页缺）

**问题**：1,463 个 place/state/侯国 页面缺坐标，无法生成历史地图叠加图。

**方案**：
- 通过谭其骧地图坐标反查，批量获取古地名坐标
- 手动补充知名地点（县、邑、都）
- 使用 `/map` 命令逐页补充

**优先级**：中。地图是视觉亮点但不是所有页面必需。

## P2 — 质量档提升（批量升级）

### 7. 事件页质量提升（E5：2,899 个 basic event）

**问题**：94% 的事件页面为 basic 档，内容仅 1-2 句话 + 来源链接。

**方案**：
- 批量补充事件结构：**起因 → 经过 → 结果 → 影响 → 相关人物**
- 用 `/enrich` pipeline 批量升级至 standard
- 先从 k_score 较高（>30）的 event 页开始，仅约 200 页

**收益**：将事件页从"链接站点"变成"微型叙事"。

### 8. 成语页扩展（E4：316 个 basic chengyu）

**问题**：99% 的成语页面只有"出自 X 章节"的信息。

**方案**：
- 批量补充：典故原文摘录 + 释义 + 用法示例
- 关联《史记》原文段落中该成语出现的上下文

### 9. 章节页升 premium（E3：130 个 standard chapter）

**问题**：130 个史记章节页全部为 standard，作为知识库骨架应有最高质量。

**方案**：
- 每章补充：地图位置、关键人物链接图、深度导读
- 加配图（已有画作/地图/文物图像）
- 补充「太史公曰」原文与译注

### 10. 其他类型批量提升

| 类型 | 页数 | basic 占比 | 建议 |
|------|------|-----------|------|
| official（官职） | 1,008 | 72% | 补职掌、品级、沿革 |
| artifact（器/物） | 934 | 79% | 补出土地点、现藏 |
| institution（制度） | 640 | 80% | 补设立背景、演变 |
| book（典籍） | 186 | 94% | 补作者、存佚、内容提要 |
| state（邦国） | 224 | 85% | 补疆域、都城、国君世系 |

## P3 — 内容美化（长期优化）

### 11. 图片补充（B5：18,402 页缺图）

**问题**：90% 的页面没有配图。person 页缺肖像、place 页缺地图、artifact 页缺文物照片。

**方案**：分三阶段：
1. **高价值页面**（premium/featured/standard）：优先配图（约 6,000 页）
2. **place/state 页**：生成谭其骧地图截图（通过 `/map`）
3. **person 页**：收集公共领域历史画像

**预估工作**：阶段 1 可复用已有资源，阶段 2 通过脚本批量完成，阶段 3 需外部素材。

### 12. 自动描述改写（B2：10,630 页）

**问题**：超过一半的页面描述是「《史记》中出现 N 处」的自动生成模板文本，缺乏信息量。

**方案**：
- 对 person：改为"XX，春秋/战国时期 XX 国人，担任 XX 职位，曾在 XX 事件中……"
- 对 event：改为"XX 事件发生于 XX 年，起因是……结果是……"
- 通过 LLM 批量重写（需使用 enrich pipeline）

### 13. ALIAS 补全（B7：16,431 页缺）

**问题**：80% 页面 aliases 为空。

**方案**：
- 从标注文件中提取该页面实体的所有提及变体
- 从 `data/alias_conflicts.json` 中收集已知别名
- 批量写入 frontmatter aliases 字段

## 自动化方案建议

1. **质量监控**：将 `quality_audit.py` 加入 CI，每次变更后生成增量报告，监控关键指标变化趋势（A1/D1/B3）
2. **批量修复脚本**：
   - `scripts/fix_missing_tags.py` — 按类型补默认 tag
   - `scripts/fix_canonical_names.py` — person 页补 canonical_name
   - `scripts/cleanup_orphan_files.py` — 清理孤儿文件
   - `scripts/fix_broken_links.py` — 修复反斜杠后缀等模式化断链
3. **Butler 工作流集成**：将 P0-P1 的修复任务纳入 butler W1 任务选择器
