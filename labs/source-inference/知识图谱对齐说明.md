# 溯源推理案例与知识图谱对齐说明

> 本文档说明如何将溯源推理案例分析与《史记》知识图谱的结构化数据进行对齐和验证

**更新时间**: 2026-03-26
**涉及案例**: 案例11(长平之战)、案例12(冯谖烧券买义)

---

## 一、对齐目标

将溯源推理案例分析与知识图谱建立系统性关联,实现:

1. **事件映射**: 案例分析的历史事件 ↔ 知识图谱事件索引
2. **实体标注**: 案例中的人物/地点/时间 ↔ 知识图谱实体标注规范
3. **关系验证**: 案例分析的因果逻辑 ↔ 知识图谱事件关系网络
4. **常识库验证**: 案例的量化计算 ↔ 常识库数据基准

---

## 二、知识图谱资源

### 2.1 事件索引

**位置**: `kg/events/data/`

**结构**:
- 130个章节事件索引文件: `{NNN}_{章节名}_事件索引.md`
- 每个事件有唯一ID: `NNN-XXX` (章节号-序号)
- 事件包含: 类型、时间、地点、人物、描述、原文引用、年代推断

**示例**:
```
073-009: 长平之战坑杀赵卒 (前260年)
- 事件类型: 战争
- 主要人物: 白起、赵括
- 地点: 长平
```

### 2.2 事件关系网络

**位置**: `kg/events/data/event_relations.json`

**关系类型**:
- `causal`: 因果关系 (A是B的直接原因)
- `sequel`: 延续关系 (B是A的后续)
- `cross_ref`: 跨章互见 (不同章节记述同一事件)
- `concurrent`: 同期关系 (同年跨章事件)
- `co_person`: 共享人物
- `co_location`: 共享地点

### 2.3 实体标注规范

**位置**: `kg/README.md`

**标注符号** (18类):
- `〖@name〗` - 人名
- `〖=place〗` - 地名
- `〖;title〗` - 官职
- `〖◆state〗` - 邦国
- `〖$qty〗` - 数量
- 等

### 2.4 常识库

**位置**: `kg/common-sense/README.md`

**常识类型**:
- 人口规模 (户均5-7人)
- 财政收入 (田赋十税一)
- 粮价利率 (30-50钱/石, 10-30%利率)
- 军队消耗 (每人每日1升米)
- 运输损耗 (长距离30-40%)
