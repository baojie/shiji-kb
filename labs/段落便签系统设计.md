# 段落便签系统设计

> 状态：设计阶段 | 创建：2026-03-14

## 1. 目标

为每个正文段落提供一个简短摘要便签，显示在页面右侧，帮助读者：
- 快速浏览章节内容结构（不读正文也能知道每段在说什么）
- 将段落与已抽取的事件索引关联（点击跳转）
- 辅助长文定位和导航

## 2. Markdown 扩展语法

### 2.1 基本格式

中文全角括注 `【】` 独占一行，紧接正文段落：

```markdown
【黄帝战蚩尤·001-005】
@黄帝@者，=少典=之子，姓$公孙$，名曰%轩辕%。
```

### 2.2 语法规则

| 格式 | 含义 | 示例 |
|------|------|------|
| `【摘要文本】` | 纯摘要，无事件关联 | `【黄帝族系谱】` |
| `【摘要文本·事件号】` | 关联单个事件 | `【黄帝战蚩尤·001-005】` |
| `【摘要文本·001-005,001-006】` | 关联多个事件 | `【封禅巡游·028-012,028-013】` |

- 摘要文本：2-8 字，概括本段核心内容
- 事件号格式：`NNN-NNN`（章节号-事件序号），与事件索引一致
- `·` 为中文间隔号（U+00B7），分隔摘要与事件号
- 便签行与正文之间**不空行**（属于同一段落）

### 2.3 特殊段落

```markdown
::: taiShiGong
【太史公论赞】
太史公曰：学者多称五帝，尚矣。
:::

::: zan
【赞诗】
维昔黄帝，法天则地...
:::
```

对话段落：
```markdown
【舜问四岳·001-008】
@帝尧@曰："谁可顺此事？"
```

### 3.3 便签颜色方案

按段落性质区分颜色（背景色）：

| 类型 | 背景色 | 识别方式 |
|------|--------|---------|
| 叙事（默认） | `#fff8dc` 米黄 | 无特殊标记 |

## 4. 渲染器改动

`render_shiji_html.py` 需要：

1. **解析**：识别 `【...】` 行，提取摘要文本和事件号
2. **关联**：将便签与紧随的正文段落绑定
3. **生成**：输出 `div.paragraph-with-note` 包裹结构
4. **事件链接**：事件号生成到事件索引页的 `<a>` 链接

### 4.1 解析逻辑（伪代码）

```python
import re

NOTE_PATTERN = re.compile(r'^【(.+?)(?:·([\d\-,]+))?】$')

def parse_note_line(line):
    """解析便签行，返回 (summary, [event_ids]) 或 None"""
    m = NOTE_PATTERN.match(line.strip())
    if not m:
        return None
    summary = m.group(1)
    events = m.group(2).split(',') if m.group(2) else []
    return summary, events
```

## 5. 生成工作流

为 130 章批量生成便签标注：

1. **LLM 批量生成**：对每章 tagged.md，让 LLM 为每个段落生成 2-8 字摘要
2. **事件匹配**：自动将段落编号（pn）与事件索引中的 `para_num` 字段匹配
3. **人工审核**：抽查摘要质量
4. **写回**：在每段前插入 `【摘要·事件号】` 行
