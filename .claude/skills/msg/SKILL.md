# /msg — 生成 Git 提交消息草稿

## 铁律

1. **只看缓存区**：只分析 `git diff --cached` 的内容，完全忽略未缓存的修改
2. **不执行 git add**：绝对不添加任何文件到缓存区
3. **不执行 git commit**：只输出消息草稿，等待用户决定是否提交
4. **中文**：提交消息全程使用中文

## 执行步骤

1. 运行 `git diff --cached --stat` 查看缓存文件列表
2. 运行 `git diff --cached` 查看具体改动内容
3. 运行 `git log --oneline -5` 了解本项目的 commit message 风格
4. 撰写中文提交消息草稿，以代码块形式展示

## 消息格式

```
首行：一句话总结（≤50字，说明做了什么）

模块A:
- 新增/更新/修复/删除 具体内容

模块B:
- ...
```

- 首行说"做了什么"，不写版本号
- 按目录/模块分组
- 区分"新增"、"更新"、"修复"、"删除"

## 输出后

展示草稿，不主动询问是否执行 commit。
