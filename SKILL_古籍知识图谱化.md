# SKILL: 古籍知识图谱化 — 从原始文本到结构化知识

> 从《史记》项目中提炼的可重用方法论，适用于二十四史、资治通鉴等大规模古籍处理。

---

## 一、总体方法论

### 核心理念

**"语法高亮是当代的标点符号"** — 通过AI驱动的命名实体识别和语义标注，将古籍从"专家研究对象"变为"大众认知工具"。

### 人机协作分工

| 角色 | 职责 |
|------|------|
| **人（决策者）** | 定义愿景、设计标注体系、审美判断、纠正语义错误、质量把控、任务调度 |
| **AI（执行者）** | 大规模NER、代码开发、批量文件操作、渲染管线、文档生成 |

### 关键设计决策

1. **用大模型做NER，不用关键词匹配** — 古文多义词极多，同一字在不同上下文有不同语义角色
2. **内容与样式分离** — Token标记系统（`@人名@`）让原文可读，渲染交给CSS
3. **先做完再做好** — 先快速覆盖全部章节，再迭代提升质量（消歧、别名合并等）

## 二、处理管线（Pipeline）

```
阶段0: 准备
  原始文本获取 → 按章拆分为独立文件(.txt)

阶段1: 语义标注（AI NER）
  .txt → 大模型(Claude) → .tagged.md
  （含11类实体标注 + 段落编号 + 小节划分）

阶段2: HTML渲染
  .tagged.md → render_shiji_html.py（正则替换）→ .html
  （含CSS样式、段落锚点、导航链接）

阶段3: 实体索引
  130×.tagged.md → build_entity_index.py → entity_index.json + 索引页面
  （含别名合并、拼音排序、搜索过滤）

阶段4: 语义增强
  a. 别名合并（auto_detect_aliases.py → entity_aliases.json）
  b. 歧义消歧（disambiguate_names.py → disambiguation_map.json）
  c. 年份映射（build_year_map.py → year_ce_map.json）

阶段5: 知识工程
  a. 知识单元（SKU）生产 — Factual/Procedural/Relational
  b. 实体关系提取（三元组）
  c. 词表/术语表生成

阶段6: 发布
  publish_to_docs.sh → GitHub Pages
```

## 三、阶段0：文本准备

### 获取原始文本

- 来源：ctext.org（中国哲学书电子化计划）等公开古籍数据库
- 格式：纯文本，每章一个文件
- 命名：`{编号}_{篇名}.txt`（如 `001_五帝本纪.txt`）

### 拆分规则

- 按原书卷/篇/章拆分
- 保持原文完整性，不做任何修改
- 编号补零到3位（001-130），便于排序

## 四、阶段1：AI语义标注

### 4.1 标注体系设计

**11类实体Token标记**：

| 标记 | 类型 | CSS类 | 说明 |
|------|------|-------|------|
| `@人名@` | 人名 | .person | 历史人物、传说人物 |
| `=地名=` | 地名 | .place | 城邑、山川、国名 |
| `$官职$` | 官职 | .official | 职衔、封号 |
| `%时间%` | 时间 | .time | 年号纪年、季节月份 |
| `&朝代&` | 朝代 | .dynasty | 朝代、氏族、国号 |
| `^制度^` | 制度 | .institution | 典章制度 |
| `~族群~` | 族群 | .tribe | 民族、部落 |
| `*器物*` | 器物 | .artifact | 礼器、兵器、书籍 |
| `!天文!` | 天文 | .astronomy | 星象、历法 |
| `?神话?` | 神话 | .mythical | 传说人物、神话事件 |
| `〘动植物〙` | 动植物 | .flora-fauna | 名词性动植物 |

**标注原则**：
- 标记名词性用法，不标记动词/形容词用法（"蚕食"的"蚕"不标记）
- 不标记比喻用法（"龙颜"不标记）
- 嵌套标注允许：`$@安国君@$`（官职包裹人名）
- 同一实体在同段落中首次出现必须标注，后续可选

### 4.2 文档结构规范

```markdown
# [0] 篇名

## 二级标题：主要时期/人物/事件

### 三级标题：细分主题

[1] 第一段正文...

[1.1] 子段落...

> "对话内容使用引用块"

[NOTE] 长篇论述标记
```

**段落编号**：圣经式编号 `[章.节]`（如 [1.1], [1.2], [2.1]）

**小节划分原则**：
- 按内容语义划分，不按固定字数
- 标题必须有意义（"鸿门宴"而非"段落40"）
- 长小节（>2000字）应继续拆分

### 4.3 批量处理脚本模式

```python
# 核心模式：批量调用Claude API处理章节
SYSTEM_PROMPT = """你是《史记》专家...
## 输出格式要求
### 1. 文档结构（一级/二级/三级标题）
### 2. 实体标注规则（11类Token标记）
### 3. 特殊注意事项（按体裁分类）
### 4. 段落分段规则
### 5. 输出要求（直接输出Markdown，不加解释）
"""

def process_chapter(client, chapter_name, input_text):
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=16000,
        temperature=0,        # 确保一致性
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": f"请处理：{chapter_name}\n\n{input_text}"}]
    )
    return response.content[0].text

# 进度管理：保存/恢复已完成的章节
def load_progress(file): ...
def save_progress(file, progress): ...
```

**按体裁定制提示词**：
- 本纪：按帝王/时期划分，关注纪年
- 表：保留表格结构，标注单元格实体
- 书：按制度主题划分
- 世家：按诸侯国/家族划分
- 列传：按人物划分，合传需区分每人
- 类传：总论在前，每人独立标题
- 特殊篇（太史公自序）：特别细致处理

### 4.4 质量控制

- **Lint检查**：`lint_markdown.py`（标注格式）、`lint_html.py`（渲染输出）
- **人工抽检**：每批次抽检2-3章，关注实体遗漏和误标
- **渐进修正**：发现系统性错误后，编写脚本批量修复

## 五、阶段2：HTML渲染

### 5.1 渲染器核心逻辑

`render_shiji_html.py` 的核心是**有序正则替换**：

```python
ENTITY_PATTERNS = [
    # 顺序至关重要！
    (r'\*\*(.+?)\*\*', 'bold'),      # 1. 粗体先于*器物*
    (r'\^(.+?)\^', 'institution'),    # 3-10. 各类实体
    ...
    (r'@(.+?)@', 'person'),           # 11. 人名最后（常作内层）
]
```

**处理顺序规则**：
- `**粗体**` 必须在 `*器物*` 之前
- 外层标记先于内层标记
- `@人名@` 放最后（最常作为内层嵌套）
- 排除已生成的HTML标签内容

### 5.2 CSS样式设计原则

- **克制优雅**：深沉暗色系，低透明度背景
- **字体**：思源宋体（Noto Serif SC），行高2.0
- **段落编号**：淡米色小框，可点击生成永久链接
- **对话**：斜体 + 极淡褐色底色 + 虚线边界
- **表格**：全视口宽度、sticky表头、交替行背景

## 六、阶段3-4：实体索引与语义增强

### 6.1 实体索引构建

```
扫描 tagged.md → 正则提取实体 → 别名合并 → 拼音排序 → 生成HTML索引页
```

- 为每类实体生成独立索引页（按拼音首字母分组）
- 每个实体条目显示：出现次数、章节分布、别名列表
- 正文中实体可点击跳转到索引

### 6.2 别名合并

`entity_aliases.json` 结构：
```json
{
  "person": {
    "刘邦": ["沛公", "高祖", "汉王", "高帝"],
    "项羽": ["项王", "项籍", "西楚霸王"]
  }
}
```

别名来源：
- 人工整理常见别名
- `auto_detect_aliases.py` 自动检测（同章共现的长名-短名对）
- 帝号统一（孝文帝 ← 孝文皇帝/孝文/文）

### 6.3 歧义消歧

4层启发式策略（`disambiguate_names.py`）：

1. **共现全名**：同段落出现全名，短名映射到该全名
2. **附近国名**（80字窗口）："武王"附近有"周"→"周武王"
3. **前置国名**："秦昭王...王曰"→"王"指秦昭王
4. **章节主题国**：周本纪中的"武王"默认为周武王

### 6.4 年份映射

```
叙事年份（"元光六年"）→ 公元年份（前129年）
```

- 从年表章节提取君主在位年份数据库
- 支持288种年份格式
- 生成时间线可视化页面

## 七、阶段5：知识工程

### 7.1 知识单元（SKU）体系

三类知识单元：
- **Factual（事实知识）**：人物传记、诸侯国、军事、思想等
- **Procedural（技能知识）**：军事战略、治国理政、外交谋略等
- **Relational（关系知识）**：标签体系、术语表、实体关系三元组

### 7.2 实体关系提取

```
(主语实体, 关系类型, 宾语实体)
如：(刘邦, 击败, 项羽), (张良, 辅佐, 刘邦)
```

### 7.3 词表生成

按11类实体分别生成词表，每个条目含：
- 规范名、别名列表
- 出现频次、章节分布
- 简要说明

## 九、扩展到其他古籍

### 9.1 适配清单

应用本方法到新的古籍时，需要调整：

| 项目 | 需要调整的内容 |
|------|--------------|
| 实体类型 | 可能需增减类别（如佛经需增加"佛教术语"类） |
| 体裁分类 | 不同史书有不同的卷体（纪传体/编年体/纪事本末体） |
| 提示词 | SYSTEM_PROMPT需根据具体文本特点定制 |
| 别名规则 | 每部书的人物称谓习惯不同 |
| 年份体系 | 不同朝代的纪年方式不同 |
| CSS配色 | 可保持统一，也可按书系调整 |

## 十、经验教训

1. **先粗后细**：先用AI快速标注全部章节（允许错误），再迭代修正
2. **人工纠错最有价值**：AI标注90%正确，人工修正10%的价值远超AI自动修正
3. **Token标记 > 内联样式**：轻量标记让原文可读，渲染交给程序
4. **进度管理必不可少**：批量处理必须支持断点续传（progress.json）
5. **消歧比NER更难**：同名人物的消歧需要深度语义理解
6. **表格是特殊挑战**：年表等表格数据需要专门的渲染管线
7. **别名是冰山一角**：古人一生有名、字、号、谥号、庙号、封号多种称谓
8. **克制的设计更持久**：视觉样式宁可淡雅也不要花哨
