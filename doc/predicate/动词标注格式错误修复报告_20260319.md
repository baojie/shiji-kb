# 动词标注格式错误修复报告

**日期**: 2026-03-19
**版本**: v3.3
**问题类型**: 新旧格式混合

## 问题发现

### 触发场景
在重新生成121章（儒林列传）HTML时，用户报告"新的实体渲染不work"。

### 问题表现
HTML中出现异常渲染：
```html
<span class="verb-penalty" title="刑罚动词">诛</span>[诛〗
```

### 根本原因
源文件中存在**新旧动词标注格式混合**的错误模式：
```
⟦◉诛⟧[诛〗
```

其中：
- `⟦◉诛⟧` - 新格式（v3.x标准）
- `[诛〗` - 旧格式残留（v2.x遗留）

### 1. 手动修复（121章）

### 2. 自动化修复工具
创建批量修复脚本：`kg/entities/scripts/fix_mixed_verb_format.py`

**核心逻辑**：
```python
pattern = r'(⟦[◈◉○◇]([^⟧]+)⟧)\[([^\〗]+)〗'

def replace_func(match):
    new_format = match.group(1)  # ⟦TYPE动词⟧
    verb = match.group(2)         # 动词
    old_verb = match.group(3)     # 旧格式中的动词

    # 验证一致性
    if verb != old_verb:
        print(f"⚠️  警告：动词不匹配 - 新:{verb} vs 旧:{old_verb}")

    return new_format  # 只保留新格式
```

**使用方法**：
```bash

# 修复指定章节

```

## 问题分析

### 错误来源推测
1. **版本迁移不完整**：从v2.x到v3.x迁移时，部分脚本未完全清理旧格式
2. **手动编辑残留**：在迁移过程中手动调整时留下混合格式
3. **脚本Bug**：早期的migrate_verb_tags.py可能在某些边界情况下产生混合格式

### 影响范围
- **渲染层面**：HTML显示异常，用户体验受损
- **数据质量**：格式不一致影响后续自动化处理
- **检索效率**：混合格式干扰正则匹配和统计

### 为何之前未发现
- Lint工具主要检查嵌套和类型问题，未覆盖此类格式混合
- HTML生成器容错性较强，部分错误被静默处理
- 用户主要使用实体标注功能，动词渲染问题不明显

## 改进措施

### 1. Lint工具增强
在 `lint_verb_tagging.py` 中新增检查规则：

```python
def check_mixed_format(self, text):
    """检查新旧格式混合"""
    pattern = r'⟦[◈◉○◇][^⟧]+⟧\[[^\〗]+〗'
    matches = re.finditer(pattern, text)

    for match in matches:
        self.add_error(
            error_type='mixed_format',
            severity='ERROR',
            line=self.get_line_number(match.start()),
            content=match.group(0),
            message='新旧动词格式混合'
        )
```
