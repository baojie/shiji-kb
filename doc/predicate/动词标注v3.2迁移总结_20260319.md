## 一、任务背景

用户要求"基于v3.2规范重新标注001-130全部章节"。经过分析，发现主要问题是：

1. **动词标注格式混用**：部分章节存在旧格式〖[verb〗，需要迁移到新格式⟦TYPE动词⟧
2. **日常动词过度标注**：部分日常动词（如"得"、"归"）被标注，不符合v3.2标注范围
3. **刑罚制度名词需保留**：2字及以上的制度名词（如〖[斩首〗）按规范应保持不变

## 二、执行策略

### 2.1 现状分析

初步扫描发现旧格式标注共913处：

| 类型 | 数量 | 处理策略 |
|-----|------|---------|
| 刑罚制度名词（2字+） | 491处 | ✅ 保留（符合规范） |
| 核心词表单字动词 | 2处 | ✅ 迁移到新格式 |
| 日常动词 | 41处 | ❌ 删除标注 |
| 扩展/未知动词 | 379处 | ⚠️ 保留待人工审核 |

### 2.2 工具开发

开发了 `scripts/clean_old_verb_tags.py` 智能清理工具：

**核心功能**：
- 识别v3.2核心词表53个动词（军事28+刑罚23+政治2）
- 自动迁移核心词表动词：〖[斩〗 → ⟦◉斩⟧
- 自动删除日常动词标注：〖[得〗 → 得
- 保留刑罚制度名词：〖[斩首〗（2字+）
- 生成审核报告：标记需人工判断的动词

**运行模式**：
```bash
python scripts/clean_old_verb_tags.py --dry-run     # 预览模式
python scripts/clean_old_verb_tags.py --execute    # 执行清理
python scripts/clean_old_verb_tags.py --report     # 生成报告
```
