# Issue #1 时间与数量标注混淆修复报告

## 问题描述

发现部分数量词被错误标注为时间类型（`〖%X〗`），实际应为数量类型（`〖$X〗`）。

来源：https://github.com/baojie/shiji-kb/issues/1

## 问题分析

### 根本原因

时间标注中混入了含数量单位的词语，这些词语虽然可能包含数字，但其核心语义是数量而非时间。

### 问题类型

1. **数量+度量单位**：如"千石"（容量）、"八尺馀"（长度）、"二百蹄"（马匹数）
2. **数量+物品**：如"八篇"（书篇）、"十驷"（车马）、"千社"（社稷）
3. **数量+行政单位**：如"十五郡"（郡县数）

## 修复方案

### 检测规则

含有以下数量单位字的标注应归类为数量（`〖$X〗`）而非时间（`〖%X〗`）：

- **度量单位**：蹄、足、石、章、畦、里、步、亩、顷、丈、尺、寸、斤、两、钱、贯
- **物品单位**：匹、驷、双、辈、篇、卷、社、车、马、人、户、家、郡

### 排除规则

以下词语虽含上述字符，但确为时间或其他类型，不做修复：
- 战国（历史时期）
- 国中、国家、郡国（地名/政治概念）

## 技术实现

### 修复脚本

[scripts/fix_time_quantity_confusion.py](scripts/fix_time_quantity_confusion.py)

**核心逻辑**：
```python
# 时间特征字
TIME_CHARS = set('年月日岁世元载春夏秋冬子丑寅卯辰巳午未申酉戌亥...')

# 数量单位字
QUANTITY_UNIT_CHARS = set('蹄足石章畦里步亩顷丈尺寸斤两钱贯...')

def should_be_quantity(text):
    """判断是否应该是数量类型"""
    if text in EXCLUDE_WORDS:  # 排除特殊词
        return False
    if any(c in QUANTITY_UNIT_CHARS for c in text):
        return True
    return False
```
