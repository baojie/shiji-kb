# 实体渲染规划 v5.0

**文档目标**：定义《史记》18类实体的HTML/CSS渲染方案，平衡美观性、可读性与语义区分度。

**设计参考**：
1. **古籍专名线**：中华书局等古籍出版中的人名线、地名线传统
2. **IDE语法高亮**：VSCode、JetBrains等现代编辑器的语法高亮设计
3. **可访问性**：WCAG 2.1 AA标准，不依赖单一颜色区分

---

## 一、设计原则

### 1. 视觉层级（3层）

```
第1层：动词（关系核心）  - 鲜明背景 + 加粗 + 独立配色
第2层：实体（叙事要素）  - 统一淡底 + 分组配色 + 下划线变化
第3层：正文（普通文本）  - 无装饰
```

### 2. 配色方案：6色 + 独立数量色

基于**语义分组**而非逐个类型配色：

| 颜色 | 语义分组 | 包含实体 | 下划线样式 |
|------|---------|---------|----------|
| **褐色** `#8B4513` | 空间叙事要素 | 人名、地名、官职 | 实线/虚线 |
| **青色** `#008B8B` | 时间维度 | 时间（独立） | 点线 |
| **紫色** `#9370DB` | 政治组织 | 朝代、封国、族群 | 实线/虚线 |
| **蓝色** `#4682B4` | 社会制度 | 制度、刑法、礼仪、身份 | 实线/虚线/点线/无 |
| **墨绿** `#2F4F4F` | 文化思想 | 典籍、思想 | 波浪线/点线 |
| **橙褐** `#CD853F` | 物质世界 | 器物、生物、天文、神话 | 虚线/点线/波浪线/无 |
| **海绿** `#2E8B57` | 数量度量 | 数量 | 等宽字体 + 实线 |

### 3. 下划线设计

参考古籍专名线传统 + IDE错误/警告线：

| 下划线类型 | CSS属性 | 适用场景 | 示例 |
|----------|---------|---------|------|
| **实线** | `border-bottom: 1px solid` | 高频核心实体 | 人名、地名 |
| **虚线** | `border-bottom: 1px dashed` | 中频重要实体 | 官职、朝代 |
| **点线** | `border-bottom: 1px dotted` | 低频或抽象实体 | 思想、礼仪 |
| **波浪线** | `text-decoration: underline wavy` | 特殊/不确定实体 | 神话（可选） |
| **无** | - | 背景色已足够 | 制度、器物等 |

### 4. 背景色统一

- **所有实体**：统一淡黄色 `rgba(255, 248, 220, 0.6)`
- **区分动词**：动词使用红/蓝/金独立背景色

## 五、可访问性考虑

### 1. 色盲友好

- **不依赖单一颜色**：结合下划线样式区分
- **对比度**：所有颜色与淡黄背景对比度 ≥ 4.5:1（WCAG AA）
- **红绿色盲**：生物（绿）vs 神话（品红）有下划线区分

### 2. 屏幕阅读器

- `cursor: help`：提示可交互
- `title` 属性：提供实体类型信息
- 语义化HTML：使用 `<span class="person">` 而非 `<span style="...">`

### 3. 高对比度模式

- 保留下划线：在高对比度模式下颜色可能失效，下划线仍可见
- 字重变化：部分实体加粗，提供额外区分

## 八、已知数据质量问题

### 问题1：`#`符号的历史语义漂移

**问题描述**：
- 大量身份/称谓被错误标注为时间类型`〖#xxx〗`
- 例如：`〖#天子〗`（604次）、`〖#诸侯〗`、`〖#士大夫〗`、`〖#卿相〗`等

**根本原因**：
- 早期标注时`#`可能用于"称谓/身份"
- 后期`#`被重新定义为"时间"类型
- 早期标注未迁移，造成语义混乱

**正确标注应为**：
```
〖#天子〗  → 〖_天子〗   （身份）
〖#诸侯〗  → 〖_诸侯〗   （身份）
〖#士大夫〗 → 〖_士大夫〗 （身份）
〖#卿相〗  → 〖;卿相〗   （官职）
〖#公卿〗  → 〖;公卿〗   （官职）
```

**影响**：
- 当前这些实体会被渲染为青色+点线（时间样式）
- 应该渲染为蓝色+虚线（身份样式）或褐色+虚线（官职样式）

**解决方案**（待实施）：
1. 批量查找替换：`〖#天子〗` → `〖_天子〗`
2. 人工审核：区分官职（`;`）vs 身份（`_`）
3. 更新所有130章
4. 重新生成HTML

**临时workaround**：
- 渲染规划v5.0按正确语义设计
- 数据清理后样式会自动正确
