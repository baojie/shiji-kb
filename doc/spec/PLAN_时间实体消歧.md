## 二、消歧方法论设计

### 2.1 数据来源优先级

```
优先级1：年表章节（841 BCE之后，精确到年）
  014 十二诸侯年表（-841 ~ -478）
  015 六国年表（-476 ~ -207）
  022 汉兴以来将相名臣年表（-206 ~ -20）

优先级2：夏商周断代工程（841 BCE之前）
  西周王室（周文王～周厉王）
  商后期（武丁～帝辛）
  鲁国早期（伯禽～献公，由鲁真公-855向前推算）

优先级3：君主数据库（rulers.json，656条）
  登基年、退位年字段
  支持年份范围验证

优先级4：事件定年（events索引，已有公元年标注的事件）
  用于交叉验证和补充

优先级5：人物生卒年（辅助推断）
  当人物担任官职、参与事件时的年份约束
```

### 2.2 五层消歧策略

#### 策略1：年号直接映射（汉代，精确度★★★★★）

```
模式：〖%建元六年%〗 → 查 reign_periods.eras['建元'] → 公元前135年
适用：汉代年号（建元、元光、元朔等32个年号）
优势：无歧义，一一对应
```

#### 策略2：近距离君主（窗口60字符，精确度★★★★☆）

```
模式：...〖@秦昭王〗...诸事...〖%三年%〗...
方法：向前搜索60字符内最近的〖@君主〗或〖$君主谥号〗
验证：计算出的公元年必须在该君主在位范围内（容差±5年）
```

#### 策略3：序列延续（同段落，精确度★★★★☆）

```
模式：〖%元年%〗...事件1...〖%三年%〗...事件2...〖%五年%〗
方法：本段落前面已有已解析年份 → 沿用同一君主，按序递增
约束：不跨越段落边界（## 标题分割）
```

#### 策略4：章节主题国+小节标题（精确度★★★☆☆）

```
模式：005秦本纪 ## 孝公时期 → 该节内〖%X年%〗默认归属秦孝公
方法：CHAPTER_STATE表 + 段落标题解析（## 君主名时期）
限制：仅作为兜底策略，需其他策略验证
```

#### 策略5：原始名称索引（无公元年，精确度★☆☆☆☆）

```
模式：〖%三年%〗 → 无法确定君主 → 记录为 "舜三年"（ruler_key）
用途：上古时期（夏/殷/西周早期）无年表覆盖的年份
渲染：链接到 timeline.html#ancient-舜三年（先秦早期索引区）
```

### 2.3 预过滤规则（排除非绝对年份）


### 2.4 年份范围验证机制

```python
def validate_year_in_reign(ce_year, ruler_name, rulers_db):
    """
    验证计算出的公元年是否在君主在位范围内
    容差：±5年（考虑史料记载差异）
    """
    ruler = find_ruler(ruler_name, rulers_db)
    if not ruler:
        return False

    start = ruler.get('登基年', -9999)
    end = ruler.get('退位年', 9999)

    # 容差5年
    if start - 5 <= ce_year <= end + 5:
        return True

    # 越界恢复：在同国君主序列中查找正确归属
    correct_ruler = find_ruler_by_year(ce_year, ruler['所属国'], rulers_db)
    if correct_ruler:
        return correct_ruler['谥号']

    return False
```

## 五、技术实现要点

### 5.1 近距离君主查找算法

```python
def find_nearby_ruler(content, para_id, pos, window=60):
    """
    在pos位置向前搜索window字符范围内的君主标记
    返回：(ruler_name, distance) 或 None
    """
    start = max(0, pos - window)
    context = content[start:pos]

    # 优先级1：完整君主名 〖@秦昭王〗
    pattern1 = r'〖@([^〗]+)〗'
    matches1 = list(re.finditer(pattern1, context))

    # 优先级2：君主谥号 〖$昭王〗
    pattern2 = r'〖\$([^〗]+)〗'
    matches2 = list(re.finditer(pattern2, context))

    candidates = []
    for m in matches1 + matches2:
        name = m.group(1)
        if is_known_ruler(name, rulers_db):
            distance = pos - m.end()
            candidates.append((name, distance))

    # 返回最近的君主
    if candidates:
        candidates.sort(key=lambda x: x[1])
        return candidates[0][0]

    return None
```

### 5.2 序列延续推理算法

```python
def sequential_inference(chapter_id, para_id, year_text, prev_mappings):
    """
    在同一段落内，根据前文已解析年份推断当前年份
    """
    # 查找本段落前面最近的已解析年份
    prev_entries = [
        e for pid, e in prev_mappings.items()
        if pid.startswith(para_id.split('.')[0] + '.')
    ]

    if not prev_entries:
        return None

    # 取最近的一条
    last_entry = prev_entries[-1]
    last_ruler = last_entry['ruler']
    last_year_num = cn_to_int(last_entry['year_text'])
    current_year_num = cn_to_int(year_text)

    if not last_year_num or not current_year_num:
        return None

    # 验证：当前年份数 > 前一年份数
    if current_year_num <= last_year_num:
        return None

    # 计算公元年
    ce_year = last_entry['ce_year'] - (last_year_num - current_year_num)

    # 验证：落在君主在位范围内
    if validate_year_in_reign(ce_year, last_ruler, rulers_db):
        return {
            'ce_year': ce_year,
            'ruler': last_ruler,
            'method': 'sequential',
            'confidence': 'high'
        }

    return None
```

### 5.3 越界恢复机制

```python
def recover_from_overflow(ce_year, wrong_ruler, rulers_db):
    """
    当年份超出原君主在位范围时，尝试在同国君主序列中恢复
    """
    state = rulers_db[wrong_ruler]['state']

    # 查找该国所有君主
    state_rulers = [
        r for r in rulers_db.values()
        if r.get('state') == state
    ]

    # 找到在位时间包含ce_year的君主
    for ruler in state_rulers:
        if ruler['start_bce'] <= ce_year <= ruler['end_bce']:
            return ruler['name']

    return None
```

## 六、质量控制

### 6.1 验证规则清单

| 规则ID | 描述 | 检查逻辑 |
|--------|------|---------|
| V1 | 年份在君主在位范围内 | start_bce - 5 ≤ ce_year ≤ end_bce + 5 |
| V2 | 同段落年份递增 | para内所有年份排序后 ce_year 单调递增 |
| V3 | 君主交替合理性 | 相邻两个不同君主的年份，后者登基年 ≤ 前者退位年 + 3 |
| V4 | 与事件定年一致 | 若该段落有事件索引，年份应匹配 |
| V5 | 年号年份范围 | 建元六年 → 应为建元年号范围内（-140~-134） |
| V6 | 元年=登基年 | 若标注为"元年"，ce_year应等于君主登基年 |

### 6.2 质量指标

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| 覆盖率 | ≥85% | 有ce_year的条目数 / 总时间实体数 |
| 准确率 | ≥95% | 人工验证100条随机样本，正确数 / 100 |
| 置信度分布 | high≥60% | 统计 confidence 字段分布 |
| 上古时期覆盖 | ≥50% | 001-003章的覆盖率（允许较低） |
| 841BCE后覆盖 | ≥95% | 有年表支持的时期覆盖率 |

### 6.3 问题分类体系

| 类别 | 描述 | 典型案例 | 解决方案 |
|------|------|---------|---------|
| E1 | 无法确定君主 | 上古章节、多国并提 | 降级为ruler_key |
| E2 | 年份超出在位范围 | 计算错误、跨君主叙述 | 越界恢复机制 |
| E3 | 同段落多个君主 | 列传章节、外交记述 | 缩小窗口、增加候选权重 |
| E4 | 年号错误 | 史料差异、异体字 | 手动修正表 |
| E5 | 君主别名未识别 | "孝公"未映射到"秦孝公" | 扩充ruler_aliases.json |
