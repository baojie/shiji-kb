> 日期：2026-03-09

> 基于自动标注系统（LLM生成）输出的首次系统性复审
> 耗时：约8小时（09:51—17:42，含Agent执行+流程调试）

## 一、研究方法

### 1.1 总体流程

第一轮反思是对130章事件索引（共约3,092个事件）的首次系统性年代审查。事件索引由LLM自动标注系统生成，包含事件摘要、时间标注、年代推断理由等字段。第一轮的核心目标是**发现并修正自动标注系统的系统性错误**。

流程四步：

1. **提示词生成**：`run_review_pipeline.py --prompt NNN` 为每章生成审查提示词，嵌入年表数据和该章事件索引
2. **Agent反思执行**：以5章为一批，调用Agent工具逐章审查，对每个事件进行年表交叉验证、reign-year换算校核、格式规范检查
3. **结果吞入**：`run_review_pipeline.py --ingest NNN reflect_NNN.json` 更新pipeline状态，提取新错误模式追加到SKILL
4. **修正应用**：`apply_reflect_fixes.py NNN` 将修正写入事件索引文件

### 1.2 审查策略

第一轮面对的是"冷启动"状态——SKILL文档几乎为空，对自动标注系统的错误模式尚无认知。因此第一轮的核心策略是：

- **全量覆盖**：130章逐章审查，无跳过
- **模式发现优先**：每章除修正错误外，还提取新发现的错误模式（new_patterns），积累到SKILL文档
- **以年表为ground truth**：凡年表有明确记载的事件，以年表年份为准

### 1.3 技术实现

- 130章分26批（每批5章同步Agent），总耗时约4小时
- 每章Agent独立执行，输出JSON格式的corrections数组和new_patterns数组
- 第一轮JSON字段统一使用`suggested_year`

### 3.2 分析

**第一轮最大发现：自动标注系统的确定性标注普遍不足（43.6%）。** 年表中有明确记载的事件，自动标注系统往往给出正确的年份但使用了过于保守的`[约公元前XXX年]`格式，未能识别为精确纪年。这一系统性问题贯穿几乎所有章节。

**第二大类（23.5%）是综合修正。** 这类修正需要结合年表查证、reign-year换算、人物活跃期推断等多种方法才能确定，反映了自动标注系统在复杂推理场景中的局限。

**年表验证修正（19.8%）** 是真正的年份错误。这些错误中，偏差从1-2年到数百年不等，最极端的案例出现在001五帝本纪（偏差可达数百年）和028封禅书（偏差可达千年级别）。

**单锚点塌缩（4.6%）** 虽然修正数不多，但每处修正的偏差幅度极大，是自动标注系统最严重的结构性缺陷。

## 四、典型案例分析

### 案例1：单锚点塌缩——五帝本纪全章塌缩（001）

**问题**：001章66个事件全部标注为前2290年或前2230年（黄帝和颛顼的生卒中点），完全丢失了五帝长达数百年的时间跨度。

**修正**：44处修正，按帝王世系重新分配年代：
- 黄帝事件：[约公元前2600年]（传统推算）
- 颛顼事件：[约公元前2500年]
- 帝喾事件：[约公元前2430年]
- 尧事件：[约公元前2350—前2250年]
- 舜事件：[约公元前2250—前2200年]

**特征**：自动标注系统仅找到一个锚点（黄帝生卒年），将全章事件塌缩到该锚点，忽略了五帝时间跨度约400年。

### 案例2：通论章节多时代塌缩——封禅书（028）

**问题**：028章从上古封禅传说到汉武帝封禅，横跨数千年。自动标注以秦二世元年（前209年）为唯一锚点，将上古、殷商、西周、春秋、战国事件全部标注为前209年左右。

**修正**：41处修正，偏差从几十年到上千年：
- 028-001 古封禅传说：前209年 → [约公元前2600年]（偏差2400年）
- 028-018 儋见秦献公：前393年 → [约公元前384年]
- 028-027 高祖置祠祝官：前208年 → [约公元前200年]

**特征**：通论性质章节（封禅书、儒林列传等）因跨越多个时代，自动标注系统无法处理多时段结构，产生极端塌缩。

### 案例3：《史记》与年表系统性偏差——田齐威王（046）

**046-010 齐威王选贤治国**
- 原标注：`（公元前370年）`
- 修正为：`（公元前348年）`，偏差22年

**推理链**：
1. 原文纪年：%九年%
2. 按《史记》自身纪年：威王元年=前378年，九年=前370年
3. 按年表：威王元年=前356年，九年=前348年
4. 交叉验证年表一致性：
   - 前353年条注"齐威王四年"（桂陵之战）→ 元年=353+3=前356年
   - 前350年条注"齐威王七年" → 元年=350+6=前356年
   - 前334年条注"齐威王二十三年" → 元年=334+22=前356年
5. 三个独立年表条目交叉印证同一基准年，《史记》与竹书纪年/年表存在22年系统性偏差

**特征**：两套权威史料体系的系统性分歧识别，多点交叉验证。

### 案例4：出土文献推翻传世文献——苏秦系年（069）

**系列修正**：069-001~069-021，苏秦全传21个事件

**推理链**：
1. 《史记》将苏秦活动定在前334—前333年（与张仪同时代）
2. 年表前311年条记苏秦赴燕昭王处
3. 关键证据：1973年马王堆帛书《战国纵横家书》出土后，学界认定苏秦活跃于前311—前284年
4. 矛盾检验：
   - 069-008"说韩宣王"：韩宣惠王元年=前332年，前333年韩宣惠王尚未即位
   - 069-009"说魏襄王"：魏襄王元年=前318年，前333年魏襄王尚未即位
   - 069-010"说齐宣王"：齐宣王元年=前319年，前333年齐宣王尚未即位
5. 三位君主在位期均与《史记》原标注年份产生系统性矛盾
6. 结论：按年表+出土文献将整传事件下移约25年

**特征**：出土文献推翻传世文献→多位君主在位期矛盾检验→系统性年代修正。

### 案例5：人物世系倒推——晋穆侯命子之谶（039-004）

**原标注**：`[约公元前808年]`
**修正为**：`[约公元前805年]`

**推理链**：
1. 文侯死于前746年，在位35年 → 元年前780年
2. 殇叔在位4年 → 前784—前781年
3. 穆侯在位27年 → 元年前811年
4. 穆侯七年生太子仇 = 前805年
5. 穆侯十年生成师 = 前802年
6. 事件核心"命子之谶"发生在成师出生时，取前805年（太子仇出生年为谶言起因之年）

**特征**：从已知终点（文侯卒年）沿世系链逐步回推，跨越三代君主定位事件年代。

### 案例6：前后君主纪年衔接处的归属判断——田常弑简公（032-032）

**原标注**：`（公元前485年）`
**修正为**：`（公元前481年）`，偏差4年

**推理链**：
1. 原文纪年标记：%四年%
2. 年表前481年条明确记载："齐简公四年，简公任用阚止...田常杀阚止及简公"
3. 错误归因分析：前485年是悼公四年（田乞杀悼公、立简公之年），并非简公四年
4. 换算验证：简公元年=前484年，简公四年=前481年
5. 错误原因：将前任君主（悼公）被杀之年误作后任君主（简公）的reign-year基准

**特征**：弑君-立新君叙事中"N年"归属的精确判断，前后两任君主纪年边界辨析。

### 案例7：《史记》与《左传》著名年代分歧——下宫之难（043-006）

**原标注**：`（公元前574年）`
**修正为**：`[约公元前583年]`

**推理链**：
1. 赵世家原文："景公三年"，换算：景公元年=前599年，三年=前597年
2. 年表前583年条记载"讨赵氏，杀赵同、赵括"，即景公十七年
3. 《史记》"景公三年"与年表/《左传》"景公十七年"相差14年
4. 内部一致性检验：
   - 043-008赵氏复位已标前583年
   - 若下宫之难在前597年，则"居十五年"后=前582年，与复位约前583年基本吻合
5. 结论：采用年表前583年，但因《史记》自身记载分歧，使用`[约]`推断标记

**特征**：《史记》vs《左传》史料分歧判断、内部时间跨度一致性验证（"居十五年"）、标注精确度等级的审慎选择。

## 六、推理规则总结

### 6.1 第一轮建立的核心推理规则

第一轮从无到有建立了12条推理逻辑规则，写入SKILL文档：

1. **年表直接查证**：年表有载即为精确纪年，这是最可靠的定年方法
2. **Reign-year精确换算**：`公元前年份 = 基准元年 - (N - 1)`
3. **前后锚点插值**：在两个已知年份之间线性插值
4. **人物活跃期约束**：事件年份不应超出涉及人物的合理活跃期
5. **世系链回推**：从已知终点沿世系逐步回推未知年代
6. **多源交叉验证**：同一事件在多章出现时取精确度最高标注
7. **确定性分级标注**：精确`（）` > 确定推算`[]` > 近似`[约]`
8. **概览-详情一致性**：概览表和详情section必须同步
9. **前元/中元/后元区分**：改元帝王必须区分纪年时段
10. **秦历岁首十月校正**：秦及汉初跨年事件需注意历法差异
11. **通论章节多锚点处理**：跨多时代章节需为各时段独立设锚
12. **出土文献优先原则**：当出土文献与传世文献矛盾时，优先采信出土文献

### 6.2 规则使用频率

| 规则 | 第一轮使用频率 |
|------|---------------|
| 年表直接查证 | 极高（~640次，含确定性升级和年表验证） |
| Reign-year换算 | 中（~50次） |
| 确定性分级标注 | 极高（~440次） |
| 概览-详情一致性 | 中（~49次） |
| 多源交叉验证 | 中-低（按需） |
| 其余规则 | 低（特定场景触发） |

### 7.1 从零构建SKILL文档

这份SKILL文档在第二轮中得到全面验证（仅新增1条模式），证明第一轮的模式发现基本饱和。

### 7.2 自动标注系统缺陷全景

第一轮揭示了LLM自动标注系统在年代推断中的5类核心缺陷：

1. **锚点处理缺陷**：单锚点塌缩、多时代章节无法分段处理
2. **换算逻辑缺陷**：改元混淆、同名混淆、即位年/元年边界
3. **确定性判断不足**：几乎不区分精确/推算/近似三级标注
4. **叙事结构理解不足**：复合段落、追叙、总结性叙述无法正确处理
5. **史料分歧盲区**：不知道《史记》与年表/出土文献存在系统性偏差

### 7.3 修正偏差幅度分布

| 偏差量级 | 占比 | 典型场景 |
|----------|------|----------|
| 仅格式变更（年份不变） | ~44% | 确定性标注升级 |
| 1-10年 | ~20% | Off-by-one、reign-year换算 |
