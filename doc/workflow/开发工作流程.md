## 一、总览：全流程管线

```
原始文本
  │
  ▼
[01 校勘]  archive/ → docs/original_text/
  │
  ▼
[02 结构分析]  chapter_md/*.tagged.md
  │   18类实体标注（〖〗格式v2.5）+ 段落编号 + 小节划分 + 语义区块
  │
  ├──▶ [03 实体系统]  kg/entities/
  │       build_entity_index.py     → entity_index.json + docs/entities/
  │       disambiguate_names.py     → disambiguation_map.json
  │       auto_detect_aliases.py    → entity_aliases.json
  │
  ├──▶ [04 事件系统]  kg/events/
  │       extract_events.py         → *_事件索引.md（130篇）
  │       build_year_map.py         → reign_periods.json + year_ce_map.json
  │       annotate_ce_years.py      → 事件公元纪年标注
  │       run_review_pipeline.py    → 五轮反思审查
  │       extract_event_relations.py→ event_relations.json（7,652条）
  │       build_metro_map_data.py   → metro_map_data.json
  │
  ├──▶ [05 关系构建]  kg/relations/, kg/facts/

  │       extract_facts.py          → 原子事实三元组（~10万，kg/facts/）
  │
  ├──▶ [06 本体构建]  kg/rdf/
  │       build_taxonomy.py         → *_taxonomy.md + *.ttl
  │
  ├──▶ [08 SKU构造]  ontology/skus/
  │
  └──▶ [09 应用]
          render_shiji_html.py      → docs/chapters/（认知辅助阅读器）
          app/metro/                → 事件地铁图
          app/game/                 → 史记争霸游戏
```

## 二、文本标注

### 2.1 标注规范（v2.5，〖〗格式）

18类实体使用〖TYPE content〗非对称格式标注：

| 类型 | 符号 | 示例 |
|------|------|------|
| 人名 | `〖@X〗` | `〖@秦始皇〗` |
| 地名 | `〖=X〗` | `〖=咸阳〗` |
| 官职 | `〖;X〗` | `〖;丞相〗` |
| 时间 | `〖%X〗` | `〖%元年〗` |
| 邦国 | `〖'X〗` | `〖'汉〗` |
| 氏族 | `〖&X〗` | `〖&刘氏〗` |
| 身份 | `〖#X〗` | `〖#天子〗` |
| 数量 | `〖$X〗` | `〖$三万人〗` |
| 制度 | `〖^X〗` | `〖^郡县制〗` |
| 族群 | `〖~X〗` | `〖~匈奴〗` |
| 器物 | `〖•X〗` | `〖•鼎〗` |
| 天文 | `〖!X〗` | `〖!彗星〗` |
| 神话 | `〖?X〗` | `〖?黄帝〗` |
| 生物 | `〖+X〗` | `〖+龙〗` |
| 典籍 | `《X》` | `《春秋》` |
| 礼仪 | `〖:X〗` | `〖:宗庙〗` |
| 刑法 | `〖[X〗` | `〖[腰斩〗` |
| 思想 | `〖_X〗` | `〖_仁义〗` |

段落编号：`[1]` `[1.1]` `[1.1.1]`（不重用，永久引用锚点）

详见 [skills/SKILL_03a_实体标注.md](../../skills/SKILL_03a_实体标注.md)

### 2.2 格式校验

```bash
# 检查单个文件
python3 scripts/lint_markdown.py chapter_md/001_五帝本纪.tagged.md

# 检查整个目录
python3 scripts/lint_markdown.py chapter_md/
```

### 2.3 格式修复工具

```bash
# 修复引号（直引号→弯引号）
python3 scripts/fmt_fix_quotes.py chapter_md/001_五帝本纪.tagged.md

# 批量修复引号
python3 scripts/fmt_convert_quotes_batch.py

# 修复诗赋格式
python3 scripts/fmt_fix_verse.py chapter_md/117_司马相如列传.tagged.md

# 修复太史公曰格式
python3 scripts/fmt_fix_zan.py chapter_md/

# 拆分对话
python3 scripts/fmt_split_dialogues.py chapter_md/007_项羽本纪.tagged.md
```

### 2.4 小节划分

```bash
# 自动生成小节建议（AI辅助）
python3 scripts/section_auto_generate.py chapter_md/061_伯夷列传.tagged.md

# 手动添加小节后提取数据
python3 scripts/section_extract_all.py

# 更新 Index 页面小节链接
python3 scripts/section_update_index.py
```

### 2.5 标注验证

```bash
# 验证标注未改变原文（去标记后与原文逐字比对）
python3 kg/entities/scripts/validate_tagging.py

# 检查破损标记
python3 kg/entities/scripts/fix_broken_tags.py --scan
python3 kg/entities/scripts/fix_broken_tags.py --fix  # 自动修复
```

详见 [skills/SKILL_02a_章节切分与编号.md](../../skills/SKILL_02a_章节切分与编号.md)

### 4.3 事件年代反思审查管线

```bash
# 三步管线（每章）
python3 kg/events/scripts/run_review_pipeline.py --prompt NNN   # 1. 生成提示词
# （Agent执行反思，输出JSON到/tmp/reflect_NNN.json）
python3 kg/events/scripts/run_review_pipeline.py --ingest NNN /tmp/reflect_NNN.json  # 2. 更新状态
python3 kg/events/scripts/apply_reflect_fixes.py NNN            # 3. 写入事件索引

# 辅助工具
python3 kg/events/scripts/lint_ce_years.py              # 校验公元年合法性
python3 kg/events/scripts/batch_fix_collapsed_dates.py   # 批量修复塌缩日期
python3 kg/events/scripts/fix_by_chronology.py           # 按年表交叉验证
python3 kg/events/scripts/cross_validate_dates.py        # 跨章验证
```

详见 [skills/SKILL_04d_事件年代审查.md](../../skills/SKILL_04d_事件年代审查.md)

## 七、HTML生成与发布

### 7.1 渲染单个章节（调试用）

```bash
python3 render_shiji_html.py chapter_md/001_五帝本纪.tagged.md
```

渲染器加载三个JSON元数据层：
- `kg/entities/data/disambiguation_map.json`（人名消歧）
- `kg/year_ce_map.json`（年份→公元年）
- `kg/entities/data/entity_aliases.json`（别名→规范名）

详见 [skills/SKILL_09a_认知辅助阅读器.md](../../skills/SKILL_09a_认知辅助阅读器.md)

### 7.2 批量生成所有章节

```bash
python3 generate_all_chapters.py
# 输出：docs/chapters/*.tagged.html（130篇）
```

### 7.3 一键发布到 GitHub Pages

```bash
./publish_to_docs.sh
```

步骤：准备目录 → 复制CSS/JS → 生成HTML → 移除`.tagged`后缀 → 修复路径 → lint检查

### 7.4 完整知识图谱重建（全量流程）

```bash
# 1. 词表
python3 kg/vocabularies/scripts/build_vocabularies.py

# 2. 实体消歧
python3 kg/entities/scripts/disambiguate_names.py

# 3. 年份消歧（含编年索引页）
python3 kg/events/scripts/build_year_map.py

# 4. 实体索引页
python3 kg/entities/scripts/build_entity_index.py

# 5. HTML生成+发布
./publish_to_docs.sh
```

## 十、故障排除

| 问题 | 解决方案 |
|------|---------|
| CSS/JS找不到 | `./publish_to_docs.sh` 重新发布 |
| 消歧映射不更新 | 删除 `disambiguation_map.json` 后重跑 `disambiguate_names.py` |
| Index页面被覆盖 | `git checkout HEAD -- docs/index.html` |
| Purple Numbers点击无效 | 检查 `docs/js/purple-numbers.js` 是否存在，重新发布 |
| 年份消歧异常 | 重跑 `build_year_map.py` + `lint_ce_years.py` |
| 实体索引缺少条目 | 重跑 `build_entity_index.py` |
