## 一、总览：全流程管线

```
原始文本
  │
  ▼
[文本标注]  chapter_md/*.tagged.md
  │   11类实体标注 + 段落编号 + 小节划分
  │
  ├──▶ [词表构建]  kg/vocabularies/
  │       build_vocabularies.py
  │
  ├──▶ [实体系统]  kg/entities/
  │       disambiguate_names.py     → disambiguation_map.json
  │       auto_detect_aliases.py    → entity_aliases.json（人工审核）
  │       build_entity_index.py     → entity_index.json + docs/entities/
  │
  ├──▶ [事件系统]  kg/events/
  │       extract_events.py         → *_事件索引.md（130篇）
  │       build_year_map.py         → reign_periods.json + year_ce_map.json
  │       annotate_ce_years.py      → 事件公元纪年标注
  │       run_review_pipeline.py    → 五轮反思审查
  │       extract_event_relations.py→ event_relations.json（7,652条）
  │       build_metro_map_data.py   → metro_map_data.json
  │
  ├──▶ [年代系统]  kg/chronology/
  │       build_year_map.py         → reign_periods.json + year_ce_map.json
  │
  └──▶ [HTML生成]  docs/

          build_entity_index.py     → docs/entities/（11类索引）
          publish_to_docs.sh        → 一键发布到 GitHub Pages
```

## 二、文本标注

### 2.1 标注规范

参见 [doc/spec/标注格式规范.md](../spec/标注格式规范.md)，核心要点：

| 类型 | 符号 | 示例 |
|------|------|------|
| 人名 | `@...@` | `@秦始皇@` |
| 地名 | `=...=` | `=咸阳=` |
| 官职 | `$...$` | `$丞相$` |
| 时间 | `%...%` | `%元年%` |
| 朝代 | `&...&` | `&秦&` |
| 制度 | `^...^` | `^郡县制^` |
| 族群 | `~...~` | `~匈奴~` |
| 器物 | `*...*` | `*鼎*` |
| 天文 | `!...!` | `!彗星!` |
| 神话 | `?...?` | `?黄帝?` |
| 动植物 | `🌿...🌿` | `🌿龙🌿` |

段落编号：`[1]` `[1.1]` `[1.1.1]`（不重用，永久引用锚点）

### 2.2 格式校验

```bash
# 检查单个文件
python3 scripts/lint_markdown.py chapter_md/001_五帝本纪.tagged.md

# 检查整个目录
python3 scripts/lint_markdown.py chapter_md/
```

### 2.3 格式修复工具

```bash
# 修复引号（直引号→弯引号）
python3 scripts/fmt_fix_quotes.py chapter_md/001_五帝本纪.tagged.md

# 批量修复引号
python3 scripts/fmt_convert_quotes_batch.py

# 修复诗赋格式
python3 scripts/fmt_fix_verse.py chapter_md/117_司马相如列传.tagged.md

# 修复太史公曰格式
python3 scripts/fmt_fix_zan.py chapter_md/

# 拆分对话
python3 scripts/fmt_split_dialogues.py chapter_md/007_项羽本纪.tagged.md
```

### 2.4 小节划分

```bash
# 自动生成小节建议（AI辅助）
python3 scripts/section_auto_generate.py chapter_md/061_伯夷列传.tagged.md

# 手动添加小节后提取数据
python3 scripts/section_extract_all.py

# 更新 Index 页面小节链接
python3 scripts/section_update_index.py
```

### 2.5 标注验证

```bash
# 验证标注一致性（实体存在于词表中）
python3 kg/entities/scripts/validate_tagging.py

# 检查破损标记（奇数@配对等）
python3 kg/entities/scripts/fix_broken_tags.py --scan
python3 kg/entities/scripts/fix_broken_tags.py --fix  # 自动修复
```

## 三、实体系统

### 3.1 构建词表

```bash
# 从130篇标注文件提取11类实体词表
python3 kg/vocabularies/scripts/build_vocabularies.py
# 输出：kg/vocabularies/data/*_词表.md（11个文件）
```

### 3.2 人名消歧

```bash
# 四层启发式消歧（国名前缀→附近国名→章节主题国→共现全名）
python3 kg/entities/scripts/disambiguate_names.py
# 输出：kg/entities/data/disambiguation_map.json
#       71章 644条映射

# 质检消歧结果
python3 kg/entities/scripts/review_disambiguation.py
```

方法论参见 [SKILL_实体消歧.md](../../SKILL_实体消歧.md)

### 3.3 别名管理

```bash
# 自动检测候选别名（刘邦→沛公→汉高祖 等）
python3 kg/entities/scripts/auto_detect_aliases.py
# 输出候选列表，需人工审核后写入 entity_aliases.json

# 报告差异（新标注与现有别名的冲突）
python3 kg/entities/scripts/report_differences.py
python3 kg/entities/scripts/find_differences.py
```

### 3.4 构建实体索引

```bash
# 生成 entity_index.json + 11类实体 HTML 索引页
python3 kg/entities/scripts/build_entity_index.py
# 输出：kg/entities/data/entity_index.json（11,069条）
#       docs/entities/*.html（12个页面，含编年索引）
```

### 3.5 SKU 实体扩充

```bash
# 为 ontology/skus/ 中的知识单元关联实体
python3 kg/entities/scripts/augment_sku_entities.py
```

## 四、事件系统

### 4.1 事件提取

```bash
# 从标注文本提取结构化事件（AI Agent辅助）
python3 kg/events/scripts/extract_events.py --chapter 007
# 输出：kg/events/data/007_项羽本纪_事件索引.md

# 验证事件格式
python3 kg/events/scripts/validate_events.py
```

方法论参见 [SKILL_事件识别.md](../../SKILL_事件识别.md)

### 4.2 年份消歧与公元纪年

```bash
# Phase 1: 从年表章节提取在位年表
# Phase 2: 全文年份消歧
# Phase 3: 生成编年索引页
python3 kg/events/scripts/build_year_map.py
# 输出：kg/chronology/data/reign_periods.json（380位君主）
#       kg/chronology/data/year_ce_map.json（1,498条映射）
#       docs/entities/timeline.html

# 为事件标注公元纪年
python3 kg/events/scripts/annotate_ce_years.py
```

方法论参见 [SKILL_年份消歧.md](../../SKILL_年份消歧.md)

### 4.3 事件年代反思审查（五轮迭代）

```bash
# 生成审查提示词
python3 kg/events/scripts/generate_review_prompts.py

# 运行 AI Agent 批量审查管线
bash kg/events/scripts/agent_review_batch.sh

# 应用反思修正
python3 kg/events/scripts/apply_reflect_fixes.py

# 校验公元年合法性
python3 kg/events/scripts/lint_ce_years.py

# 批量修复塌缩日期
python3 kg/events/scripts/batch_fix_collapsed_dates.py

# 按年表交叉验证
python3 kg/events/scripts/fix_by_chronology.py

# 补标已知无年份事件
python3 kg/events/scripts/fix_undated_known_events.py

# 跨章验证（第五轮）
python3 kg/events/scripts/cross_validate_dates.py
```

五轮反思结果参见 [doc/events/](../events/)

### 4.4 事件关系提取

```bash
# 提取9种事件关系（延续/因果/跨章因果/包含/对立/互见/共人/共地/同期）
python3 kg/events/scripts/extract_event_relations.py
# 输出：kg/events/data/event_relations.json（7,652条）
```

方法论参见 [SKILL_古籍事件提取与关系发现.md](../../SKILL_古籍事件提取与关系发现.md)

### 4.5 事件地铁图

```bash
# 生成地铁图数据
python3 kg/events/scripts/build_metro_map_data.py
# 输出：app/metro/data/metro_map_data.json（130条线路）
```

## 五、HTML生成与发布

### 5.1 渲染单个章节（调试用）

```bash
python3 render_shiji_html.py chapter_md/001_五帝本纪.tagged.md
# 输出：chapter_md/001_五帝本纪.tagged.html（本地预览）
```

渲染器会加载：
- `kg/entities/data/disambiguation_map.json`（人名消歧）
- `kg/chronology/data/year_ce_map.json`（年份→公元年）
- `kg/entities/data/entity_aliases.json`（别名→规范名）

### 5.2 批量生成所有章节

```bash
python3 generate_all_chapters.py
# 输出：docs/chapters/*.tagged.html（130篇）
```

### 5.3 一键发布到 GitHub Pages

```bash
./publish_to_docs.sh
```

脚本执行顺序：

| 步骤 | 操作 |
|------|------|
| 1 | 准备 docs/ 目录结构 |
| 2 | 复制 `css/shiji-styles.css` → `docs/css/` |
| 3 | 复制 JS 文件 |
| 4 | `generate_all_chapters.py` 生成所有章节 |
| 5 | 移除文件名中的 `.tagged` 后缀 |
| 6 | 修复 HTML 内的 CSS/JS/导航路径引用 |
| 7 | 更新 `docs/index.html` 中的章节链接 |
| 8–10 | 运行 lint 质量检查（章节/Index/实体索引） |

发布后推送：

```bash
git add docs/
git commit -m "Update GitHub Pages content"
git push
```

### 5.4 完整知识图谱重建（全量流程）

```bash
# 1. 词表
python3 kg/vocabularies/scripts/build_vocabularies.py

# 2. 实体消歧
python3 kg/entities/scripts/disambiguate_names.py

# 3. 年份消歧（含编年索引页）
python3 kg/events/scripts/build_year_map.py

# 4. 实体索引页
python3 kg/entities/scripts/build_entity_index.py

# 5. HTML生成+发布
./publish_to_docs.sh
```

## 八、故障排除

### HTML路径错误（CSS/JS找不到）

```bash
ls css/shiji-styles.css          # 源文件
ls docs/css/shiji-styles.css     # 发布版本
./publish_to_docs.sh             # 重新发布修复
```

### 消歧映射不更新

```bash
rm kg/entities/data/disambiguation_map.json
python3 kg/entities/scripts/disambiguate_names.py
```

### Index 页面被覆盖

`generate_all_chapters.py` 有保护机制，若意外覆盖：

```bash
git checkout HEAD -- docs/index.html
```

### Purple Numbers 点击无效

```bash
ls docs/js/purple-numbers.js
grep "purple-numbers.js" docs/chapters/001_五帝本纪.html
./publish_to_docs.sh
```

### 年份消歧结果异常

```bash
python3 kg/events/scripts/build_year_map.py
python3 kg/events/scripts/lint_ce_years.py
python3 kg/entities/scripts/audit_interpolated_dates.py
```

### 实体索引页缺少条目

```bash
python3 kg/entities/scripts/build_entity_index.py
```
