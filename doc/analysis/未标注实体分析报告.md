## 四、词性分析规划

> **目的**：进一步分析44万未标注字中，有多少"肯定不是实体"，有多少是候选实体。

### 4.1 工具选择：字级规则分析（无外部依赖）

**不使用jieba等现代分词工具**，原因：
- jieba词典基于现代汉语，文言文分词准确率低
- 文言文以**单字词为主**（古汉语约90%是单音节词）
- 虚词高度封闭（之乎者也等约230字），适合词表驱动

**方法**：维护文言虚词表 + 动词表 + 形容词表，对未标注汉字按字分类。

### 4.2 分类体系

| 分类 | 说明 | 是否可能是实体 |
|------|------|---------------|
| 虚词（Level 1） | 之乎者也、连词、副词、代词 | 肯定不是 |
| 动词（Level 2） | 曰、伐、封、杀等常见动词 | 通常不是（注意动名活用） |
| 形容词 | 贤、暴、善、强等 | 通常不是 |
| 数词/量词 | 一二三、年月里等 | 通常不是 |
| **候选实体** | 余下汉字 | **可能是**（名词/专名） |

### 4.3 两遍分析流程

**第一遍**：字级分类（按词表归类每个汉字）

**第二遍 Lint**：检测"夹心"模式
- 识别 `[候选字][非候选字][候选字]` 组合
- 这类夹在候选字中间的虚词/动词可能实为复合实体的组成字（如"无忌"中的"无"）
- 输出高频夹心三字组合供人工审核，决定是否将某字从词表移除

### 4.4 脚本与输出

- **脚本**：`scripts/pos_analysis.py`（已创建，零外部依赖）
- **输出**：
  - `doc/analysis/pos/NNN_章名_pos.json`（130章，每章词性分布）
  - `doc/analysis/pos_summary.md`（汇总报告 + 候选实体TOP词表 + Lint警告表）
- **运行方式**：
  ```bash
  python scripts/pos_analysis.py --chapter 001   # 单章测试

  ```

## 五、实体与词性的关系

史记标注系统的实体**不都是名词**，按语法功能分三类：

| 类别 | 例子 | 语法特征 |
|------|------|----------|
| **纯名词实体** | 人名、地名、器物、族群 | 典型名词，做主宾语 |
| **动名两用实体** | 礼仪（封禅、祭祀）、刑法（弃市、腰斩） | 可作动词，也可作名词 |
| **概念型实体** | 思想（天命、王道）、典籍书名 | 常作主题论述 |

**标注原则**：系统是**语义驱动**，不是词性驱动。判断依据是"是否指称某类可独立命名的对象"。因此封禅、弃市、天命等动名两用词/概念词均应标注。

### 6.2 新增5种（当前草稿）

基于现有分析，为4类高价值新实体和1类修订设计语法，全部采用 **CJK标点符号**：

| 实体类型 | 符号 | 示例 | Unicode | HTML class |
|----------|------|------|---------|------------|
| **典籍/书名** | `《...》` | `《春秋》` | U+300A/300B | `book` |
| **礼仪** | `〈...〉` | `〈封禅〉` | U+3008/3009 | `ritual` |
| **刑法** | `【...】` | `【弃市】` | U+3010/3011 | `legal` |
| **思想/概念** | `〔...〕` | `〔天命〕` | U+3014/3015 | `concept` |
| **音乐** | `〖...〗` | `〖五音〗` | U+3016/3017 | `music` |

**符号选择说明**：
- `《》`：中文书名号，已天然出现在文本中，零学习成本
- `〈〉`：单角括号（U+3008/3009），视觉清晰，IME可输入；区别于书名号
- `【】`：黑方括号，直觉联想"法条"，与白方括号`〔〕`形成对比
- `〔〕`：白方括号（U+3014/3015），与`【】`有明确视觉差异
- `〖〗`：白六角括号，与`【】`/`〔〕`有明确视觉差异

> ⚠️ **已废弃**：`「」`（U+300C/300D）和 `『』`（U+300E/300F）曾作为礼仪/思想符号候选，
> 但因 `render_shiji_html.py` 的 QUOTE_PATTERNS 已将其识别为引号 span，存在渲染冲突，改为 `〈〉`/`〔〕`。

**为何完全安全（不冲突）**：
- 全部为CJK标点，非ASCII
- CommonMark、GFM、Obsidian 等主流Markdown解析器均不处理这些字符
- 与现有11种标注符号无重叠
- `〈〉`/`〔〕` 不在 `render_shiji_html.py` 的 QUOTE_PATTERNS 中

**原方案问题**（已废弃）：
- `#刑法#` → `#` 是Markdown标题符号（严重冲突）
- `|思想|` → `|` 是Markdown表格符号（冲突）
- `+礼仪+` → 部分解析器识别为列表（潜在冲突）

## 附：边界说明

### 礼仪 vs 制度 的区别

- **制度 `^`**：制度性安排、政治体制（郡县制、封禅制度作为制度）
- **礼仪 `〈〉`**：具体礼仪动作/仪式（封禅作为具体行为、祭祀、朝聘）

### 刑法 vs 事件描述 的区别

- **刑法 `【】`**：法律术语本身（弃市、腰斩、族诛）
- **不标注**：描述性动词短语（"将其斩首"中"斩首"视情况而定）

### 动植物符号修改说明

- `〘...〙` → `〘...〙`
- 修改原因：emoji符号需特殊输入，且多字节处理复杂；CJK标点更统一
