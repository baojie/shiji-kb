# 项目初衷、愿景与未来方向

## 一、初衷：为什么做这个项目

### 让"皓首穷经"不再必需

传统历史研究的核心困境是**人力瓶颈**。一位学者穷其一生，能精读的古籍不过数十万字；而中国古典文献的总量以亿字计。《史记》57.7万字、130篇，一个人细读尚且需要数月，遑论系统性地标注实体、提取事件、发现关系、检验矛盾。

项目的出发点是一个简单的问题：**AI能否把这个过程从数十年压缩到数周？**

答案是肯定的。一个人加一群AI Agent，在约120小时内完成了：18类实体的99,214次标注、3,185个历史事件的提取与纪年推断、7,652条事件关系的发现、五轮Agent反思审查的2,119处修正。这不是替代历史学家的判断力，而是把历史学家从重复性劳动中解放出来，让他们的时间花在真正需要人类智慧的地方——阐释、质疑、发现。

### 让古文不再是阅读障碍

> 现在没有人再会读没有标点符号的文章，未来也不会有人愿意读没有语法高亮的文章。

语法高亮是认知辅助的第一步。为18类实体赋予不同颜色，读者无需精通古文就能一眼看出谁是人、哪是地、什么官。但高亮只是表层——背后的结构化数据才是真正的价值。当实体、事件、关系被机器理解后，就能进行人力不可能完成的事：跨130篇的矛盾检测、跨越2600年的模式发现、从57万字中自动归纳治乱兴衰的规律。

### 方法比结果更重要

《史记》是第一个试验田，但项目真正要输出的不只是一个知识库，而是**一套可复用的方法论**。26个SKILL文档、九阶段管线架构、四层语义模型——这些方法经过实战验证，可以直接应用于其他古籍。项目的野心不在《史记》本身，而在于证明：AI Agent驱动的古籍数字化是一条可规模化的道路。

## 二、愿景：我们在构建什么

### 四层语义递进模型

项目的知识架构建立在四层语义递进模型上，每一层是下一层的前提：

```
结构语义 → 图谱语义 → 知识语义 → 应用语义
```

- **结构语义**（第1层）：文本怎么组织的？句子间什么关系？——校勘、切分、编号、句间关系识别
- **图谱语义**（第2层）：谁做了什么？在哪里？什么时候？——实体标注、事件提取、关系发现
- **知识语义**（第3层）：怎么分类？遵循什么规律？——本体构建、逻辑推理、知识单元化
- **应用语义**（第4层）：历史真相是什么？有哪些矛盾？——矛盾检测、规律归纳、交互式探索

当前项目已基本完成前两层，正在深入第三、四层。

### 从"细读"到"远观"的范式转换

项目实践了数字人文的核心理念转换：

- **传统路径**：史料阅读 → 提出观点 → 选择性引证 → 形成论述
- **数字人文路径**：数据采集 → 结构化标注 → 计算分析 → 模式发现 → 假设提出 → 统计验证

这不是用机器取代人的阅读，而是让**远观（Distant Reading）发现宏观模式，细读（Close Reading）验证具体解释**，两者互补。机器能从3,185个事件中发现"边缘崛起"的结构性规律，人来判断这个规律在什么历史语境下成立。

### Agentic Ontology：AI时代的知识构建范式

项目实践了一种与传统本体工程根本不同的构建方式：

| | 传统本体构建（2001） | AI Agent本体构建（2026） |
|--|---------------------|---------------------------|
| 起点 | 专家在Protégé中从空白开始 | AI从原始文本中提取 |
| 方向 | 自上而下设计 | 自下而上涌现 |
| 规模 | 数十个类、数百个实例 | 数千个类、数万个实例 |
| 迭代速度 | 数周一轮 | 数小时一轮 |
| 瓶颈 | 专家时间用于设计 | 人类精力用于质量控制 |

核心转变：**2001年的难点是"设计"，2026年的难点是"策展"**。本体不再是专家预先设计的蓝图，而是从数据中生长出来、由人类修剪和校准的有机结构。

### 知识不只是存储，而是可以推理的

项目在知识提取过程中意外发现了20+个跨章洞察（eureka.md），这些不是预设的研究假设，而是知识图谱结构化后自然涌现的模式：

- **征服-治理倒转**：打天下的手段恰恰是治天下的障碍
- **边缘优势**：成功王朝一致从边缘地区起源
- **宽恕悖论**：对下属的宽恕创造忠诚，对敌国的宽恕制造灾难
- **功臣清洗的结构性必然**："飞鸟尽，良弓藏"不是个别暴君的选择，而是权力结构的内在逻辑

这证明了一个关键假设：**当知识被充分结构化后，机器辅助的推理能发现人类难以通过线性阅读察觉的深层模式**。

## 三、未来方向

### 近期：深化《史记》的知识层

**社会网络分析**
- 基于3,797个人名和已有关系数据，构建跨朝代人物网络
- 发现隐藏的政治派系、家族集团、师承谱系
- 用PageRank等指标量化人物影响力

**时空分析**
- 对接CHGIS（中国历史地理信息系统）历史坐标
- 构建可按朝代切换底图的交互式历史地图
- 实现事件的时空联动可视化

**推理与矛盾检测**
- 系统化跨章交叉验证（如007项羽本纪 vs 008高祖本纪的死伤数矛盾）
- 沉默证据推理（如秦始皇无皇后记载的异常）
- 因果链路径分析与影响力评估

**十表结构化**
- 将十表（013-022）从叙述文本转为机器可读格式
- 补充提取表内事件（预计每章20-30事件）
- 建立与叙事章节的交叉链接

### 中期：从一部书到一个文库

项目的方法论（管线架构、SKILL文档、Agent工作流）设计之初就考虑了可移植性。下一步扩展：

| 目标 | 规模 | 适配要点 |
|------|------|---------|
| 二十六史 | ~4,000万字 | 实体类型扩展、朝代特有制度 |
| 资治通鉴系列 | ~600-700万字 | 编年体结构、与纪传体互参 |
| 先秦诸子 | ~100万字 | 思想概念为核心实体、论证结构分析 |

关键：不是重复劳动，而是**复用已验证的SKILL，让每一部新古籍的处理成本递减**。

### 远期：30亿字级的古籍数字化基础设施

- **四库全书、佛道经典**：总规模约30亿字
- **Agent驱动、Skills可复用**：方法论本身成为可分发的知识资产
- **从文献到知识到发现**：机器不只是帮助阅读，而是帮助**发现**——跨文献的交叉验证、矛盾识别、规律归纳
- **开放协作**：代码和方法论开源，数据以CC BY-NC-SA 4.0发布，让全世界的研究者都能站在这个基础上工作

最终愿景：**建立一个由AI Agent维护和持续进化的中国古典文献知识网络，让任何人都能以问答、可视化、推理的方式探索数千年的历史智慧。**
