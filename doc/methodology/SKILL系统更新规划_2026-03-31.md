## 四、SKILL组织优化建议

### 4.1 三级文档结构完善

**当前问题**：
- references/目录下仅有2个文档使用三级SKILL命名规范
- 其他支撑性文档缺少明确的加载时机说明

**建议操作**：

1. **为高复杂度SKILL创建配套references/**：
   ```
   SKILL_03a_实体标注.md (主文档 < 500行)
     → references/SKILL-03a-entity-types.md (18类实体详细定义)
     → references/SKILL-03a-examples.md (典型案例库)

   SKILL_03c_按章反思.md (主文档)
     → references/SKILL_03c1-rules.md (已存在，72条规律库)

   SKILL_04c_事件年代推断.md (主文档)
     → references/SKILL-04c-era-tables.md (年号表、在位年表)
   ```

2. **为每个三级文档添加元数据**（参考SKILL_03c1-rules.md的格式）

3. **在主SKILL中明确指出何时读取references/**：
   ```markdown
   ## 步骤2：类型判断

   如遇复杂边界情况，读取 `references/SKILL-03a-entity-types.md` 中的详细定义。
   ```

### 4.2 渐进载入优化

根据`00-META-06`的三层载入模型，建议：

**Layer 1（总是加载）**：
- name + description（< 100词）
- 当前：所有SKILL已符合

**Layer 2（触发时加载）**：
- SKILL_XX主文档（< 500行）
- 需要检查：是否有SKILL超过500行？

**Layer 3（执行时按需）**：
- references/下的支撑文档
- 需要完善：大部分SKILL尚未拆分出references/

**具体操作**：
1. 统计所有SKILL文档行数，识别超过500行的文档
2. 将详细内容（如规则库、示例库、数据表）迁移到references/
3. 在主文档中添加明确的读取指针

### 4.3 草稿文档处理

**draft/目录现状**：
- 包含若干META-SKILL草稿
- 命名规范：META-SKILL-*.md

**建议操作**：
1. 评估每个草稿的成熟度（是否已在实践中验证）
2. 成熟的草稿转为正式00-META-XX文档
3. 未成熟的保留在draft/，但添加"状态"标记（实验阶段/待验证/已废弃）

## 五、质量检查清单

根据`00-META-06`附录中的检查清单，建议对所有SKILL执行以下检查：

### 5.1 结构质量检查

| 检查项 | 检查方法 | 工具 |
|--------|---------|------|
| 文档长度<5000字 | 统计字数 | `wc -m` |
| 规则数量<15条 | 人工计数 | 目视检查 |
| 嵌套层次<3层 | 检查Markdown标题层级 | `grep -E '^#{4,}'` |
| 有导航地图 | 检查是否有"文档导航"章节 | 人工检查 |
| 有交叉引用 | 检查是否有链接到其他SKILL | `grep -E '\[.*\]\(SKILL'` |

### 5.2 命名与组织规范检查

| 检查项 | 检查方法 |
|--------|---------|
| 一级SKILL命名规范 | 正则：`^SKILL_\d{2}_[\u4e00-\u9fa5]+\.md$` |
| 二级SKILL命名规范 | 正则：`^SKILL_\d{2}[a-z]_[\u4e00-\u9fa5]+\.md$` |
| 三级SKILL命名规范 | 正则：`^SKILL-\d{2}[a-z]-[a-z-]+\.md$` |
| 元数据完整性 | 检查YAML frontmatter中name和description字段 |

### 5.3 内容质量检查

| 检查项 | 说明 |
|--------|------|
| 规则来源明确 | 每条规则应说明来自哪次反思/试点 |
| 规则可执行 | Agent能理解并执行 |
| 示例充分 | 每条规则至少1个示例 |
| 例外明确 | 什么情况下规则不适用 |
| 质量标准量化 | 准确率/召回率目标 |

## 六、执行计划

### 第一阶段：核心文档完善（优先级P0-P1）

**目标**：确保高频使用的SKILL文档完整且规范

1. **验证关键SKILL存在性**（P0）
   - 检查CLAUDE.md中引用的SKILL是否都存在
   - 检查SKILL_00中提到的子SKILL是否都存在

2. **完善元数据**（P0）
   - 为所有SKILL添加/检查YAML frontmatter
   - 确保description符合触发规范

3. **同步内容一致性**（P1）
   - SKILL_00 ↔ README.md
   - SKILL_10系列 ↔ CLAUDE.md
   - SKILL_01系列 ↔ CLAUDE.md（标注铁律）

### 第二阶段：文档结构优化（优先级P2）

**目标**：建立清晰的三级文档结构

1. **识别需要拆分的SKILL**
   - 统计行数，找出超过500行的文档
   - 评估复杂度，确定是否需要拆分

2. **创建references/子文档**
   - 按照`SKILL-NNx-name.md`规范命名
   - 添加完整元数据
   - 在主SKILL中添加读取指针

3. **渐进载入优化**
   - 精简主文档到核心规则
   - 详细内容移至references/
   - 添加清晰的导航地图

### 第三阶段：质量提升与版本管理（优先级P3）

**目标**：建立SKILL演化追踪机制

1. **添加版本历史**
   - 为核心SKILL添加版本历史章节
   - 记录主要变更和原因

2. **质量自检**
   - 执行完整的质量检查清单
   - 修正发现的问题

3. **草稿文档正式化**
   - 评估draft/中的文档成熟度
   - 将成熟文档转为正式元技能

## 九、核心价值

这个规划的核心价值在于：**不破坏已有的优秀实践，而是系统性地补充缺口、优化结构、建立演化机制**，使SKILL系统更符合其自身定义的最佳实践（尤其是00-META-06中的方法论）。

**关键原则**：
- 保持向后兼容，不破坏现有工作流
- 渐进式改进，分阶段实施
- 自动化优先，减少手工维护负担
- 文档驱动，所有改进都有清晰的文档支持
