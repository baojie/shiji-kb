# 史记人物本体构建过程详录

> 记录从 1,825 个人名实体出发，经 10+ 轮反思迭代，构建含 142 个类、1,800 个实例的 OWL/RDF 人物本体的完整过程。

---

## 一、起点与背景

### 1.1 输入数据

- **来源**：《史记》130 章标注文本中的人名实体（`〖@X〗`），经 `build_entity_index.py` 提取
- **规模**：3,539 个人名词条，经别名合并（586 组）后得到 1,825 个独立人物
- **已有信息**：每个人名附带出现次数和出现章节列表

### 1.2 目标

构建一棵人物分类树（taxonomy），使每个人物归入至少一个语义类（如帝王、诸侯、刺客），并生成 OWL/RDF 本体文件（`person.ttl`）。

### 1.3 与生物本体的对比

生物本体是先行试验品（388 种，20 类），构建顺利。但人物本体有三个根本差异：

| | 生物 | 人物 |
|---|---|---|
| 类的来源 | 词表中预设（动物/植物/家畜/野兽） | 需要从实例中归纳 |
| 实例-类关系 | 1:1（"马"→马属） | 1:N（韩信→将领+臣） |
| 分类信号 | 词义本身（"虎"→野兽） | 章节位置+名称模式+历史知识 |

---

## 二、第一轮：初始分类（章节信号 + 名称模式）

### 2.1 分类方法

采用"章节信号"作为主要分类依据——《史记》体例本身就是分类系统：

- 本纪（卷1-12）→ 帝王
- 世家（卷31-60）→ 诸侯重臣
- 列传标题 → 列传标题人物类型（刺客列传→刺客）

对每个人物，取其出现次数最多的章节作为"主章"，按主章体裁分类。

### 2.2 初始分类结果

| 类 | 人数 | 说明 |
|---|---|---|
| 帝王 | 437 | 主章为本纪的所有人物 |
| 诸侯重臣 | 609 | 主章为世家的所有人物 |
| 策士 | 168 | 列传中的策士相关章节 |
| 刺客/游侠/佞幸等 | 各 5-30 | 有明确列传标题的 |
| 未分类 | ~600 | 无法归入上述类别 |

### 2.3 问题：帝王类 88% 误分率

**发现过程**：列出"帝王"类 437 人，逐一审查。

**误分根因**：`主章=本纪 ≠ 此人是帝王`。韩信（333次）主要出现在高祖本纪（ch008），被自动分为帝王——但韩信是将领，不是帝王。类似的，萧何、张良、范增、项伯、樊哙……大量臣子因与帝王共同出现在本纪中而被误分。

**核心教训**：`单信号分类不可靠`。仅凭章节信号，帝王类准确率只有 12%（49/437）。

## 三、第二轮：帝王类反思（三信号叠加）

### 3.1 修正方法

设计三信号叠加验证：

1. **帝王名册**：明确的帝王列表（五帝、夏商周秦汉各朝帝王）
2. **帝号模式**：名称匹配 `X帝/X祖/X宗/X王`（但需排除诸侯王）
3. **本纪标题人物**：12 篇本纪的标题人物

三个信号均不满足 → 判定为误分类。

### 3.2 结果

437 → 90 人。但仍有问题：

- **秦国诸侯被当作帝王**：秦穆公、秦孝公等是诸侯（秦国），不是帝王（秦朝）。
  - 修正原则：`秦国的是诸侯，秦朝的才是帝王`
- **周公旦被当作帝王**：摄政不是天子
  - 修正：移入臣类

### 3.3 帝王子类拆分

按朝代拆分为子类：

```
帝王 [90人]
├── 五帝时代 [7人]（黄帝/颛顼/帝喾/尧/舜等）
├── 夏 [17人]（禹/启/太康…）
├── 商 [31人]（汤/纣…）
├── 周 [33人]（武王/文王/成王…）
├── 秦朝 [5人]（始皇帝/二世…）
├── 秦末 [7人]（陈涉/项梁…）
├── 汉 [12人]（高祖/惠帝/文帝…）
└── 上古 [3人]（神农/伏羲/帝挚）
```

### 3.4 反复出现的问题

- **帝武乙/帝太丁** 出现在"商代(本级)"而非"商帝王"→ 因为分类数据用的 sub-code 没有正确映射到帝王子类
- **周威烈王/周康王** 残留在帝王(本级) → 补充分入周帝王
- **帝挚** 分类困难 → 归入上古帝王

**教训**：`大类修正后，检查"(本级)"残留`。每次拆分子类后，必须检查本级是否还有遗漏。

## 四、第三轮：诸侯类反思

### 4.1 初始问题

诸侯类 609 人中包含了大量帝王和臣子：

- 帝王出现在世家中（高祖在某些世家中被提及）
- 后妃出现在世家中
- 臣子是世家的附属人物

### 4.2 修正方法

排除法：已被分为帝王/后妃的人物，不重复归入诸侯。

609 → 432 人。

### 4.3 诸侯子类拆分

按国家拆分：

```
诸侯
├── 齐国诸侯 [22人]
├── 晋国诸侯 [20人]
├── 楚国诸侯 [18人]
├── 秦国诸侯 [15人]
├── 鲁国诸侯 [15人]
├── ...其他国家
└── (本级) [18人] — 楚汉之际的诸侯，难以归国
```

### 4.4 诸侯里的臣子问题

反思发现：齐国诸侯中有"无知"（27次出现）——查证后发现是齐国的小诸侯（公孙无知），不是臣子。但"晏孺子"（14次）也在齐国诸侯中——同样是齐国君主。

**教训**：`不要被名字误导`。"无知""晏孺子"看起来不像君主名，但在史记语境中确实是。

## 五、第四轮：臣类展开（最大的挑战）

### 5.1 规模

排除帝王、诸侯、后妃、策士等已分类人物后，"臣"类约 800+ 人，是最大的类。

### 5.2 分类维度设计

采用`时代 × 国家 × 性质`三级嵌套：

```
臣
├── 先秦早期
│   ├── 五帝时代
│   ├── 夏代
│   ├── 商代 → 商代名臣
│   ├── 西周 → 周族先祖 / 西周朝臣
│   └── 三代
├── 春秋
│   ├── 齐国臣子 → 齐国贤臣 / 齐国权臣
│   ├── 鲁国臣子
│   ├── 晋国臣子 → 六卿 / 世族 / 宗室 / 将领 / 卿大夫 / 谋臣
│   ├── ...其他国家
│   └── 孔门弟子
├── 战国
│   ├── 秦国臣子 → 秦国先祖 / 秦国将领 / 秦国相臣
│   ├── 赵国臣子 → 赵国将领 / 赵国政治
│   ├── 楚国臣子 → 楚国权贵
│   ├── ...其他国家
├── 秦臣子 → 秦朝权臣 / 秦朝将领 / 秦朝重臣 / 秦末起义
├── 楚汉 → 刘邦阵营(武将/文臣) / 项羽阵营
├── 汉初 → 汉初诸将 / 吕氏集团
└── 汉中后 → 文帝朝臣 / 景帝朝臣 / 文景朝臣 / 文景武朝臣 / 武帝朝臣(将领/文臣) / 汉其他
```

### 5.3 遇到的问题清单

#### 问题1：韩康子分入魏国

韩康子主要出现在魏世家（ch044），因为三家分晋的叙事在魏世家中展开。按章节信号分类，被归入魏国臣子。

**修正**：改用人物的实际所属国家而非出现章节。

**教训**：`章节信号 ≠ 所属国家`。三家分晋、合纵连横等跨国叙事导致人物出现在非本国章节。

#### 问题2：国王混入臣子类

吴楚臣子中出现白公胜、平王、叶公——前两个是楚国君主，后者是封地之主。

**修正**：白公/平王移入诸侯/楚国，叶公保留臣子（封地≠诸侯）。

**教训**：`每次拆分子类后，必须逐一审查实例`。

#### 问题3：汉初诸将中有楚汉诸侯

黥布（英布）、魏豹、张耳等在汉初诸将中出现——但他们是楚汉之际的诸侯王，不是刘邦的将领。

**修正**：移入诸侯类。

#### 问题4：匈奴/大宛西域中有汉人

张骞、赵破奴出现在匈奴/大宛西域相关章节，被分入外邦。实际是汉武帝时期出使或征战的汉臣。

**修正**：移入臣/武帝朝臣。

#### 问题5：类名歧义

- `燕 [7人]` — 是诸侯/燕国？还是臣/春秋/燕国臣子？
- `春秋未分国` — 暗示"应该分但还没分"，实际是"无法归入具体国家"
- `战国·秦` — 用中间点不规范

**修正**：所有类名必须自明（`燕国臣子`）、不依赖树结构上下文。

## 六、第五轮：20人以上叶子类强制拆分

### 6.1 拆分标准

所有 20 人以上的叶子类必须尝试继续拆分。拆分方式：

1. 按国家拆分（如春秋→齐/鲁/晋/…）
2. 按性质拆分（如晋国臣子→六卿/世族/将领/谋臣）
3. 按时期拆分（如汉臣→文帝朝/景帝朝/武帝朝）

### 6.2 晋国臣子（42人）拆分为例

晋国臣子是春秋最大的子类。按性质分出六个子类：

| 子类 | 人数 | 说明 |
|---|---|---|
| 六卿 | 12 | 知/韩/赵/魏/范/中行六家 |
| 世族 | 8 | 非六卿的世家大族 |
| 宗室 | 5 | 晋国公族 |
| 将领 | 7 | 军事将领 |
| 卿大夫 | 5 | 朝廷重臣 |
| 谋臣 | 5 | 谋略类人物 |

### 6.3 反复出现的树渲染错误

在生成 Markdown 分类树时，反复出现树形连线错误：

**错误1：平铺而非嵌套**

```
├── 秦国臣子  [25人]
├── 秦国先祖  [8人]     ← 应该是秦国臣子的子类！
├── 秦国将领  [10人]    ← 应该是秦国臣子的子类！
├── 秦国相臣  [7人]     ← 应该是秦国臣子的子类！
```

应该是：
```
├── 秦国臣子  [25人]
│   ├── 秦国先祖  [8人]
│   ├── 秦国将领  [10人]
│   └── 秦国相臣  [7人]
```

**错误2：`└──` 后的 `│` 缺失**

当一个子类是最后一个（用 `└──`），其后续兄弟节点的 `│` 连线会消失，导致树形断裂。

**根因**：`build_person_taxonomy.py` 中的树渲染函数没有正确跟踪父节点的"最后一个子节点"状态。

**修正**：重写为递归渲染器 `render_tree()`，每层正确维护 `is_last` 状态和 `prefix` 字符串。

## 七、第六轮：中间层与顶层结构调整

### 7.1 引入中间层分组

最初的顶层类过多（帝王/诸侯/后妃/策士/儒生/法家/兵家/商贾/刺客/游侠/…），需要引入分组：

| 分组 | 下属类 |
|---|---|
| 王室 | 帝王、诸侯、后妃、汉宗室 |
| 诸子百家 | 儒生、法家、兵家、思想家、文学家、隐士、史家 |
| 社会人物 | 刺客、游侠、佞幸、滑稽、商贾 |

### 7.2 合并决策

**诸侯重臣 vs 将相**：最初两者分立，但边界模糊。合并为统一的"臣"类，用`时代×国×性质`三维拆分。

**功臣侯爵**：最初独立，后合并入"臣"（功臣侯爵的核心身份是臣子，侯爵只是封赐）。

### 7.3 引入"虚构人物"类

史记中有"牛哀"（变为虎的故事）等文学虚构角色，不应与历史人物混分。新增 `虚构人物` 类。

### 7.4 引入"疑似误标"类

有些"人名"实际不是人名：
- "而以"（54次）— 连词被误标为人名
- "开封"— 地名被误标为人名
- 单字名（"开""相"等）— 无法确认是否为人名

归入 `疑似误标` 类，待后续清理。

## 八、第七轮：URI 与命名规范

### 8.1 拼音碰撞问题

最初用拼音做 URI：`per:Han`（汉）、`per:Han`（韩）——碰撞！

**修正**：全部改用中文 URI。类名 `per:帝王`，实例名 `per:韩信`。

### 8.2 类名后缀冗余

脚本生成的类名出现 `per:刘邦武将臣`、`per:齐国臣子臣`——"臣"后缀是 bug（脚本在所有臣子子类名后自动追加了"臣"）。

需要人工检查修正，但当前版本保留了这一历史遗留。

### 8.3 多重分类表达

在 RDF 中，用 `a per:帝王 , per:将相` 表达多类型归属。约 10.6% 的高频人物属于两个以上分类。

## 九、第八轮：(本级)残留清理

### 9.1 什么是(本级)

每次拆分子类后，无法归入任何子类的实例留在父类的"本级"。例如：

```
帝王  [90人]
├── 周  [33人]
├── 商  [31人]
├── (本级)  [3人]  ← 这3人未分入任何朝代
```

### 9.2 发现的典型问题

| 类 | 本级人数 | 问题 |
|---|---|---|
| 帝王(本级) | 3 | 周威烈王/周康王/帝挚未分入朝代 |
| 商代(本级) | 5 | 帝武乙/帝太丁等商王未分入帝王/商 |
| 春秋(本级) | 45 | 跨国活动的臣子无法归入具体国家 |
| 文景武朝臣(本级) | 28 | 横跨多朝的臣子 |

### 9.3 处理原则

- 能明确归类的 → 移入正确子类
- 确实跨越多个子类的 → 保留在父类本级
- 本级人数 < 5 → 可接受，不强制拆分

## 十、第九轮：分类数据格式演进

### 10.1 从 pickle 到 JSON

最初分类数据存储在 pickle 文件（`/tmp/person_classified_final10.pkl`）中，格式为 `{name: (sub_code, count)}`，sub_code 是拼音编码字符串。

问题：
- sub_code 需要 SUB_ZH 映射表转中文
- 多层嵌套用下划线拼接（`WarringStates_Jin_SixFamilies`）→ 不直观
- pickle 不可读、不可 diff

**改进**：设计 JSON 格式，直接用中文路径：

```json
{
  "韩信": {
    "classes": ["臣"],
    "count": 333,
    "path": ["楚汉", "刘邦阵营", "刘邦武将"]
  }
}
```

### 10.2 脚本抽象化

`build_person_taxonomy.py` 硬编码了人物特有的分组逻辑（王室/诸子百家/社会人物），不能复用于其他实体类型。

**最终方案**：废弃 `build_person_taxonomy.py`，新建 `build_taxonomy.py`。关键设计决策：

- **输入不再是 JSON 分类数据，而是 TTL 本身**。TTL 已经包含类层次（`rdfs:subClassOf`）、实例归类（`a <class>`）、标签（`rdfs:label`）、计数（`:count`），无需中间格式
- 用法：`python kg/rdf/scripts/build_taxonomy.py kg/rdf/person.ttl`
- 自动推断单位（人/种/个）、自动推断根类
- 支持 `--order` 参数控制顶层子类排列顺序
- 对人物和生物均通过验证（person: 130类/1821实例, biology: 20类/70实例）

## 十二、方法论总结

### 12.1 关键发现

1. **文献体例即分类学**。《史记》的本纪/世家/列传体例是天然的分类信号源，但`体例信号 ≠ 分类断言`——需要多信号叠加验证。

2. **单信号分类灾难性不可靠**。帝王类仅凭"主章=本纪"分类，准确率 12%。至少需要两个独立信号叠加。

3. **分类从边缘案例中涌现**。不是先设计分类树再填充实例，而是实例的误分暴露分类规则的缺陷，修正规则 → 改进分类树。

4. **反思收敛**。每轮反思发现的问题越来越少。10 轮后，剩余问题集中在 20-50 人的叶子类，需要词条简介（per-entity summary）才能进一步判断。

5. **"章节信号 ≠ 所属国家"是最大陷阱**。韩康子出现在魏世家（因三家分晋叙事），张骞出现在匈奴列传（因出使叙事），都不意味着他们属于该国。

6. **类名必须自明**。不能用"燕"——到底是燕国诸侯还是燕国臣子？必须用"燕国臣子"。

7. **拆分后必须清理(本级)残留**。每次拆分子类，必须检查是否有实例被遗忘在父类。

### 12.2 典型错误模式

| 错误模式 | 出现次数 | 示例 |
|----------|---------|------|
| 章节信号误导 | 50+ | 韩信→帝王（因出现在本纪） |
| 国王混入臣子 | 10+ | 白公→吴楚臣子（实为楚国君主） |
| 汉人混入外邦 | 5+ | 张骞→匈奴（因出现在匈奴列传） |
| 诸侯混入将领 | 5+ | 黥布→汉初诸将（实为楚汉诸侯） |
| 类名歧义 | 10+ | "燕"→燕国诸侯 or 燕国臣子？ |
| 树渲染平铺 | 5+ | 子类显示为同级而非嵌套 |
| (本级)残留 | 10+ | 拆分后遗忘在父类的实例 |
| 拼音URI碰撞 | 1 | Han=汉=韩 |

### 12.3 工作量估计

- 分类反思（10+轮）：约 8 小时对话
- 脚本开发和调试：约 4 小时
- 树渲染修复：约 2 小时
- 总计：约 14 小时 + 约 5 亿 tokens
