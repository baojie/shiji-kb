# v2 提示词迭代分析

基于 5 章（002/007/067/087/129）共 100 个高置信度 PN 的三方对齐（source / ours / hunterhug）人工比对，归纳 LLM 翻译 Agent 的系统性缺陷，形成可落入 `SKILL_01h` 提示词的规则。

## 1. 采样统计

| 章 | 题 | 全章 PN | 高置信池 (conf≥0.7，去重叠) | 采样 | 问题 PN（主观） |
|----|----|--------|--------------------------|------|----------------|
| 002 | 夏本纪 | 69 | 23 | 20 | 14 |
| 007 | 项羽本纪 | 588 | 46 | 20 | 17 |
| 067 | 仲尼弟子列传 | 152 | 144 | 20 | 11 |
| 087 | 李斯列传 | 69 | 34 | 20 | 18 |
| 129 | 货殖列传 | 43 | 40 | 20 | 15 |
| 合计 | — | — | 287 | 100 | ~75（75%） |

> 说明：hunterhug 经常把"下一段/整节"的段落翻译错对齐到当前 PN（spillover），为此脚本取"同一 hunterhug 块首次出现"，避免重复比对；"问题 PN"指至少存在一类我们认为提示词可改进的缺陷。

主要发现（简述）：
- **代词/单字简称不展开**：几乎所有章节出现；是最刺眼的可读性缺陷。
- **文言动词照抄**：`崩/立/曰/畔/卻/夷三族/具五刑/拜X为Y` 等普遍不转白话。
- **时间词语义错改**：年龄语境的 `岁` 被误改为 `年`（067 整批弟子生卒）。
- **"礼 → 礼记" 误渲染**：067 中把"礼之用/礼后乎"里的"礼"（抽象礼义）渲染成"礼记"（典籍），属 `translate_surface.py` 规则误伤 + LLM 未纠偏。
- **半角标点混用**：`002/007/087` 可见半角逗号、分号，违反全角约定。
- **古地名歧义**：`山东/山西/河南` 等未做古今消歧。

## 2. 问题模式分类（按严重性/频次倒序）

### 模式 A：单字代称/简称不展开（出现 ~30+ 次，severity: **high**）

**描述**：白话中保留原文"姓的单字"或"名的单字"作主语/宾语，读者无法识别人物。最典型是 `斯`（李斯）、`高`（赵高）、`梁`（项梁）、`籍`（项羽自称）、`商`（卜商）、`通`（会稽郡守殷通）。hunterhug 统一展开为双字名或加头衔。

**正例（hunterhug 好处理）**：
- 087#9.3 源 `斯曰："安得亡国之言！"` → H：`李斯说："你怎么能说出这种亡国的话呢！"`
- 067#51 源 `孔子曰："商始可与言诗已矣。"` → H：`孔子说："卜商啊，现在可以和你讨论《诗经》了。"`
- 007#7.2 源 `会稽守通谓梁曰` → H：`会稽郡守殷通对项梁说`（补人名"殷"+全名"项梁"）

**反例（本库欠佳）**：
- 087#9.3 我：`高乃谓丞相斯曰 → 高于是对丞相斯说`（`高`/`斯`两主语都是单字）
- 087#4.1 我：`李斯也在被议论驱逐之列。斯于是上书说`（第二次出现即退化为"斯"）
- 067#51 我：`孔子说："商可以开始和他谈诗经了。"`（`商`=卜商，未展开）
- 007#179.1 我：`籍与江东子弟八千人渡长江西进`（项羽自称，宜加"我"或保留"籍"）

**建议规则**：
> **人名展开**：
> - 人物首次出现用**全名**（姓+名，或依消歧 `|规范名`）。
> - 同一对话段内重复引用时，允许用双字名（如"李斯"/"赵高"/"项梁"），**禁止退化为单字**（如"斯"/"高"/"梁"）。
> - 对话中的第二人称 `公/君/子/若` 指代明确时，译为"你"或"您"。
> - 第一人称自称（如项羽说"籍与江东子弟…"）保留"籍"或译为"我"，**不可译为"项籍"**（自称悖逻辑）。
> - 若 `〖@X|Y〗` 已消歧，以 Y（规范名）决定展开形式。

### 模式 B：常见文言动词/句式未转白话（~25 次，severity: **high**）

**描述**：`崩/立/曰/毋/卻/畔/夷N族/具N刑/拜X为Y/下其议X/收士卒/具以N刑` 等动词或句式被直接保留为文言。

**正例**：
- 002#26 源 `中康崩,子帝相立` → H：`中康逝世以后，他的儿子帝相继位`
- 087#17.1 源 `具斯五刑` → H：`李斯被判处五刑` ＞ 本库：`对斯具以五刑`
- 087#18.1 源 `二世拜赵高为中丞相` → H：`二世任命赵高任中丞相` ＞ 本库：`二世拜赵高为中丞相`

**反例**：
- 002#26 我：`中康崩,儿子帝相立。帝相崩,儿子帝少康立…`（整段"崩/立"未译）
- 087#13.1 我：`兵至鸿门而卻`→`兵马达到鸿门才被击退`（"卻"译OK，但"兵马达到"不自然）
- 087#20.1 我：`子婴与妻子用组带系颈`（"系颈"保留文言，应为"将丝带系在脖子上"）

**建议规则**：
> **文言动词必改写**（译后通读，凡不像现代汉语的**必须**替换）：
> - `崩/薨/卒` → "逝世/去世"
> - `立/即位/嗣` → "继位"
> - `曰` → "说"（参见模式 G：可按语气选词）
> - `毋/勿` → "不要"
> - `畔` → "反叛/背叛"
> - `卻/郤` → "退却/撤退/击退"
> - `夷N族` → "诛灭N族"
> - `具N刑/论X刑` → "判处N刑"
> - `拜X为Y` → "任命X担任Y"
> - `下其议X / 议下X` → "把这个议题交给X讨论"
> - `收其士卒` → "收拢/收集他的士卒"

---

### 模式 C：年龄"岁"被误改为"年"（~10 次，severity: **high**）

**描述**：`少孔子三十九岁` 类年龄表达，`岁` 必须保留为"岁"（现代汉语相同），本库一律改为"年"（误）。这是 `translate_surface.py` 中"岁→年"规则错误波及到年龄语境。

**正例**：
- 067#61 源 `少孔子三十九岁` → H：`比孔子小三十九岁`
- 067#93 源 `少孔子三十六岁` → H：`比孔子小三十六岁`

**反例（本库 5+ 处）**：
- 067#61/93/98/103/104 本库：`比孔子小三十九年/三十六年/四十二年/四十六年/五十年`（均误）

**建议规则**：
> **时间词语境消歧**：
> - **年龄语境**（"X岁"作为年龄）：`岁` 保留为"岁"，不得改为"年"。
> - **时长/纪年语境**（"岁馀/三岁/十馀岁"表示过了X年）：`岁` 译为"年"；`X馀` 展开为"X多/X几"。
> - **翻译时先辨别语境**：若"岁"前接"少/长/年"（比较年龄）→ 年龄；若接时长上下文（"居N岁/岁馀"）→ 时长。
>
> **脚本层协同**：`translate_surface.py` 的"岁→年"应仅在 `〖% 岁|时长〗` 消歧明确为"时长"时才触发（目前似乎无条件触发）；LLM 层亦应主动保护年龄"岁"。

---

### 模式 D：半角标点混用（~30% 章节可见，severity: **medium**）

**描述**：本库译文大量使用半角逗号 `,`、分号 `;`、冒号 `:`。`CLAUDE.md` 已禁止半角引号，但半角逗号/分号同样风险。

**正例**：hunterhug 全部全角。

**反例**：
- 002#2 本库：`…他的仁爱使人亲近,他的言语值得信任;他的声音合乎音律,…`（连续半角）
- 002#10/12/14.7/14.8/14.9/15/15.3 同病
- 087#4.4、087#13.4 混用

**建议规则**：
> **标点全角化**：译文中的逗号、句号、分号、冒号、感叹号、问号**必须**使用全角中文标点：`，。；：！？"""''（）`。禁止出现半角 `,.;:!?"'()`。生成前后必须自查。
>
> **脚本兜底**：建议在 `generate_translation_json.py` 前加一步 `python scripts/lint_halfwidth_punct.py`（新脚本），检测半角并报错。

---

### 模式 E：古地名/职官歧义未消歧（~10 次，severity: **medium**）

**描述**：`山东/山西/关中/河南/河内/河外/中国` 等古代指称，白话直译易被读者误读为今地。`会稽守通` 的"通"是人名（殷通），直译为"郡守通"像动词。

**正例**：
- 087#13.1 H：`起兵于崤山以东`（"山东"→"崤山以东"）
- 007#7.2 H：`会稽郡守殷通对项梁说`（补人名）

**反例**：
- 007#54.1 本库：`沛公在山东时`（→今山东省？）
- 087#13.1 本库：`起于山东`
- 129#25 本库：`山东食海盐，山西食盐卤`
- 007#7.2 本库：`会稽郡守通对梁说`（"通"像动词）

**建议规则**：
> **古今歧义地名**（"山东/山西/关东/关中/河南/河内/河外/中国/江南/北地"等）：若原文已消歧 `〖= 山东|关东〗`，遵从；未消歧时，**首次出现或易误读处补括号**如"山东（崤山以东）"。
>
> **职官+单字名**（`会稽守通/沛公/韩王信`）：单字名作主语时补全，或保留消歧 `〖@通|殷通〗` 的规范名。

---

### 模式 F：`礼` 被误渲染为 `礼记`（系统性 bug，severity: **high/but 脚本层**）

**描述**：`translate_surface.py` 的 `{ 典籍` 规则 `礼→礼记|礼` 本应只对标注为 `〖{ 礼〗`（典籍类）的"礼"生效；但实际处理后 067 大量"礼之用/礼后乎/设礼"这些"礼义/礼节"语境的"礼"都被渲染为"礼记"，意义错误。

**正例**：
- 067#96 H：`礼的应用，以恰到好处为可贵。`
- 067#19 H：`孔子用礼乐慢慢地诱导他`

**反例**：
- 067#96 本库：`礼记的应用，以和为贵，先王的道以此为美`（"礼"=礼义，被误译为"礼记"）
- 067#19 本库：`孔子设置礼记仪慢慢诱导子路`（"礼"=礼仪，被误为"礼记"+"仪"）
- 067#51 本库：`礼记也在仁之后吗？`

**建议规则**（三层）：
> 1. **脚本修复**（主要）：`translate_surface.py` 必须根据 tag 前缀判定——仅 `〖{ 礼〗`（典籍）才替换为"礼记"；`〖^ 礼〗`（制度/礼仪）、`〖: 礼〗`（礼仪）保持"礼"。其他典籍同理（诗/书/易/乐）。
> 2. **提示词配合**：**LLM 层禁止自行把"礼"改写为"礼记"**，除非原文明确标注为 `〖{ 礼〗` 典籍。
> 3. **回查**：已渲染的 067 等章需批量扫描 `礼记(?![:：])` 是否应回退为"礼"。

---

### 模式 G：对话语气一律"说"（~15 次，severity: **low**）

**描述**：`曰` 字被机械译为"说"。原文含 `怒曰/笑曰/问曰/对曰/谏曰/让曰/辞曰/谢曰` 等语气词时，白话一律"说"显得单调。

**正例**：
- 087#14.2 H：`二世非常生气地说`
- 007#179.1 H：`项王笑着说`

**反例**：
- 087#9.3 本库对话全段：`高说 / 斯说 / 高说 / 斯说 / 高说 / 斯说`（六连"说"）

**建议规则**：
> **对话动词语气化**：根据原文线索选词
> - `曰` → "说"（默认）
> - `怒曰` → "生气地说 / 怒道"
> - `笑曰` → "笑着说 / 笑道"
> - `问曰` → "问道"
> - `对曰 / 答曰` → "回答说"
> - `谏曰` → "劝谏说"
> - `让曰 / 责曰` → "责备说"
> - `谢曰` → "道歉说 / 答谢说"

---

### 模式 H：议论/说理段落文言句式保留（~10 次，severity: **medium**）

**描述**：《货殖》《督责书》等说理段落，本库保留 `夫…者…也 / …所以…者…也 / 则…则…` 等句式，读起来仍是文言。

**正例**：
- 129#4 H：`农、工、商、虞这四个方面，是人民衣食的来源` 对比 本库 `这四者，是民众衣食的本源`
- 087#13.4 H：`君主和臣子的职分一经确定，上下关系的准则也明确了`

**反例**：
- 087#13.7 本库议论整段保留 `必能违逆世俗而改变风气，废除他所厌恶的，建立他所喜欢的，所以生有尊重之势`（"有…之势"文言）
- 129#27 本库：`所以清廉的官吏做得久了，越久就越富有`（"越久越富"OK，但周围"贤人深谋于廊庙"等直译"廊庙"未改）

**建议规则**：
> **议论段落句式白话化**：
> - `夫 X 者，Y 也` → "X，是 Y"
> - `…之所以…者也` → "…的原因是…"
> - `於是/乃` → "于是/就"
> - `既已` → "已经"
> - `X则Y，Y则Z` 可保留对偶，但每分句内部换现代词。
> - 拆长句：遇到连续的 `者/也/而/则` 串联 4 个以上分句，改为两到三个短句。

### 模式 I（低）：添译克制 — 继承，但"补足主语/指代"除外

本库相对 hunterhug **添译更少**，这是优点，应维持。唯一例外是**指代消歧**（补说话人主语、补代词所指），不是"扩写"，应执行。

`仓廪实而知礼节 / 天下熙熙 / 藉寇兵而赍盗粮` 等名句本库处理良好，保持原句引用，可继续这样。

## 3. 建议对 SKILL_01h 提示词的总体增补

在 `Translation Agent提示词` 章节 **"翻译要求"** 列表中：

### 3.1 修改现有条目

- **第 1 条"准确性"**：增加"**代词/简称必须展开**：单字姓（斯/高/梁/籍）禁止作主语；若原文有消歧 `|规范名`，以规范名展开。"
- **第 2 条"流畅性"**：增加"**文言词必改**：`崩/立/曰/畔/卻/夷族/具刑/拜…为/下其议` 等一律改写为现代表达；译完通读，不像现代汉语的**必须**替换。"
- **第 3 条"简洁性"**：增加"**不添译背景**（除消歧/指代补足）；hunterhug 式'注上括号注释'的扩写风格不采纳。"

### 3.2 新增条目（插在"保留实体标注"之前）

- **第 X 条 · 标点全角**：译文的所有标点一律全角中文（`，。；：！？"""''（）`），严禁半角 `,.;:!?"'()`。
- **第 X 条 · 对话动词语气化**：根据原文 `怒/笑/问/对/谏/让/谢` 选择"怒道/笑着说/问道/回答说/劝谏说/责备说/答谢说"等，不要一律"说"。
- **第 X 条 · 时间词语境区分**：年龄语境"岁"保留为"岁"（绝不可改"年"）；时长语境可译"年"。`X馀` → "X多/X几"。
- **第 X 条 · 古今歧义地名**：`山东/山西/关中/河南` 等古地名首次出现且易误读时，补括号如"山东（崤山以东）"，或遵循原文 `〖=〗` 消歧。
- **第 X 条 · `礼` 不擅改**：除非原文明确标注为 `〖{ 礼〗`（典籍），**严禁**把"礼"改写为"礼记"；礼义/礼节/礼仪语境保持"礼"。
- **第 X 条 · 议论段落白话化**：`夫…者…也` 句式需拆为现代短句；`於是/乃/既/则/而/以/者也` 等虚词必须改写或删除。

### 3.3 脚本/流程配合（提示词无法单独解决的）

- `translate_surface.py`：
  - **典籍规则收紧**：`礼/诗/书/易/乐 → 礼记/诗经/…` 仅在 `〖{ 〗` 标注下触发，不得影响其他类型。
  - **时间"岁→年"规则**：仅在 `〖% X|时长〗` 明确为时长时触发；年龄语境禁触发。
- 新增 `scripts/lint_halfwidth_punct.py`：扫描译文中半角标点（排除英文数字内语境），在 `generate_translation_json.py` 前强制执行。
- 回修批次：对已发布的 067/087/129 按上述规则重跑一遍（可分两步：脚本修复 + LLM 微调）。

## 4. 开放问题

1. **hunterhug spillover**：hunterhug 常把下一段/整节的翻译对齐到同一 PN（见 002 中大段泛滥），需要提前过滤采样才能用作"金标准"。当前脚本用"同一 hunterhug 块只保留首次出现"做了兜底，但这意味着我们只比对了约 3–10% 的 PN。
2. **白话参考（百话史记）利用不足**：本报告主要以 hunterhug 为对照。白话版的"消失率"集中在高频虚词，可作为补充训练信号，但需单独审计。
3. **LLM 自然化 vs 脚本兜底的边界**：提示词能否让 LLM 一次就写出自然白话，还是需要"生成 → surface 脚本 → 再校对"管线？目前倾向混合方案；若提示词修改后质量上升，可降低 surface 脚本依赖。
4. **年龄/时长歧义消歧**：未来可以在原文标注阶段对 `岁` 做 `|年龄` vs `|时长` 的消歧（现仅有 `|时长/纪年`），从源头根治模式 C。
5. **对话主语追踪**：是否需要在提示词中加入一段"对话追踪示例"（few-shot），让 LLM 学会在对话切换时补主语？可在 v3 尝试。

## 核心摘要（80 字内）

核心缺陷集中在**单字简称不展开、文言动词/句式照抄、半角标点混用、"礼→礼记"脚本误伤**。关键改动：强制展开单字名、常见文言动词清单必改、全角标点约束、收紧 `translate_surface.py` 典籍与时间规则。
