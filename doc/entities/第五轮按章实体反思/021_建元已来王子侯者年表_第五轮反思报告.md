## 四、典型案例

### 案例 1：全章父国 `〖&〗` → `〖◆〗` 系统修正（167 处）

**发现**：本章 162 个王子侯国均以"父亲 = 汉诸侯王"立宗；父亲所属邦国如"长沙""中山""城阳" 等全部用 `〖&〗` 氏族标记，但这些是**封建邦国**（分封制下的政治实体），而非血缘氏族（如"崔氏""若敖氏"）。

**规则依据**：
- **A107**：年表主国/世家主国名一律 `〖◆〗`
- **B7**：春秋以降参与分封的政治实体 → 邦国 `〖◆〗`；血缘/族姓集团（陶唐氏、有虞氏）→ 氏族 `〖&〗`
- **跨章对齐**：017 汉兴以来诸侯王年表 100% 使用 `〖◆长沙〗`/`〖◆中山〗`/`〖◆城阳〗`；059 五宗世家、052 齐悼惠王世家、060 三王世家亦均用 `〖◆〗`

**修正规模**：17 个邦国 × 平均约 10 次 = 167 处

### 案例 2：A116 省称强制消歧（129 处）

**背景**：王子侯首次袭封标注为 `〖@刘X〗`（"刘"姓显式），但本行后续省称裸用 `〖@X〗`。这违反 A116 功臣传承链父子孙单字名强制消歧原则。

**典型三代世袭链**（r16 剧侯国，菑川懿王子）：
```
原：五。〖%二年〗...原侯〖@刘错〗〖%元年〗。六。一。五。〖%二年〗，〖;孝侯〗〖@广昌〗〖%元年〗。
改：五。〖%二年〗...原侯〖@刘错〗〖%元年〗。六。一。五。〖%二年〗，〖;孝侯〗〖@广昌|刘广昌〗〖%元年〗。
```

**特殊 r102 广陵侯国**（城阳共王子）：
```
原：常侯〖@刘表〗〖%元年〗。四。二。〖%五年〗，侯〖@成〗〖%元年〗。
改：常侯〖@刘表〗〖%元年〗。四。二。〖%五年〗，侯〖@成|刘成〗〖%元年〗。
```
第二代"成"是"刘成"（省称），与首次"刘表"同一家族。补 `|刘成` 统一。

### 案例 3：r134 衡山王赐的父+子特殊拆分（A113.2 + A116）

**背景**：r134 是唯一采用 `〖&国〗〖;王〗〖@本名〗〖#子〗` 格式的条目，其余 161 行均为 `〖&国〗〖@谥号王|全名〗〖#子〗`。原因：衡山王刘赐造反自杀（元狩元年），没有谥号。

**修正**：

改：〖◆衡山〗〖;王〗〖@赐|刘赐〗〖#子〗

**说明**：
- `〖&衡山〗` → `〖◆衡山〗`（A107 邦国）
- `〖;王〗` 保留（A113 爵位）
- `〖@赐〗` → `〖@赐|刘赐〗`（A116/C5，017 canonical `〖@赐|刘赐〗`）

### 新发现建议（回写 SKILL_03c1-v2-rules.md）

#### 建议 1：**年表 `〖&国〗` 批量 lint 扫描**

汉代年表类章节（017-021）中，"父亲所属邦国"应一律 `〖◆〗`。本章初始全部误用 `〖&〗`，说明 AI 标注器在**"血缘传承"语境下易误将邦国当氏族**（因为"父亲=汉诸侯王"确实是血缘链接）。

**扫描命令**：
```bash
grep -nP '〖&(长沙|中山|城阳|菑川|江都|河闲|河间|齐|济北|赵|代|鲁|广川|楚|胶东|衡山|淮南|汉|燕|吴|淮阳|梁|清河|临江|真定|常山|广陵)〗' chapter_md/017_*.tagged.md chapter_md/018_*.tagged.md chapter_md/019_*.tagged.md chapter_md/020_*.tagged.md chapter_md/021_*.tagged.md
```

**判定原则**：
- 若 `〖&X〗` 的 X 是**分封王国**（长沙王/中山王/城阳王 等）→ 应改 `〖◆X〗`
- 若 `〖&X氏〗` 或 `〖&X〗` 指**血缘族姓**（刘氏/吕氏/若敖氏）→ 保留 `〖&〗`

#### 建议 2：**汉代王子侯省称 `|刘X` 消歧模板**

推恩令后汉诸侯王子孙众多，同名率极高（本章单字"福""成""嘉""婴""顺""德" 等均有 3-5 处同名）。同名人物属不同父系时，`|刘X` canonical 不足以区分（都是刘姓）。本章保留 canonical 统一为"刘X"，**同名异人的区分属下一轮 entity_index 总索引阶段任务**。
