## 五、新发现 / 规律候选

### 发现 1：惠景间侯表"谥号+王+子" 结构标注分歧

本章出现两类"封国+谥号+王子"结构：
- **2 字谥号省称**（悼惠、齐悼惠）：`〖&齐〗〖@悼惠|齐悼惠王〗〖@王子〗`（"王子"整词）
- **整谥号含王**（楚元王、淮南厉王）：`〖&楚〗〖@元王|楚元王〗〖@子〗`（"王"归谥号，子作亲属词）

建议在 SKILL_03c1-rules A105 条目补充"年表'谥号+王子'结构"注解：优先整合谥号+王为整体人名，"子"单独作亲属词。

### 发现 2：r70 侯功栏"布/越"双同名异人消歧

r70（栾布封俞侯）侯功栏涉及栾布（主）、彭越（前主人）、黥布（作乱者）三人，其中"布"作栾布或黥布均出现，必须强制消歧（A109 同名异人 + A116 传承链）。本轮处理完毕。

建议在 SKILL_03c1-rules A109 条目补充"惠景间侯表 r70 栾布-黥布-彭越三者叙事消歧"案例。

### 发现 3：`〖@燕王〗卢绾` 误合

r86 原标 `〖@燕王〗卢绾` 将"燕王卢绾"误合（"卢绾"本应独立作 `〖@卢绾〗`）。同类模式应扫描全库：

```bash
grep -nE '〖@[^〗]+王〗[^〖]{1,3}' chapter_md/*.tagged.md
```

本轮仅修复 019 本章，跨章扫描建议下一轮进行。

### 发现 4：侯号/封号"为〖&X〗王"合并策略未定

A113.1 规定"`〖◆X〗` 紧跟裸王 → `〖;X王〗`"，但本章大量 `〖&X〗` 紧跟"王"（15+ 处）。`〖&〗` 是氏族/邦国前缀，`〖◆〗` 是政治实体。A113.1 是否扩展至 `〖&〗`+王，需项目级定稿。本轮保守保留。
