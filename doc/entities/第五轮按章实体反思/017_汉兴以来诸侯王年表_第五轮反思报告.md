**背景**：第四轮报告 §B 只处理了 5 行 `〖@王X〗` 错误黏连（r25/r31/r78/r80/r99），但全章另有 40 处 `〖@X王〗本名` 模式（本名未加 canonical）需按 A105 拆分。

**修正模板**：
```
原：〖@X王〗本名Y〖%元年〗
改：〖@X王〗〖@Y|刘Y〗〖%元年〗
```

**典型案例（r52 一行 5 处）**：
```
原: 初王〖@献王〗〖@德〗...王〖@彭祖〗...初王〖@阏于〗...初王〖@非〗...初王〖@余〗...〖@定王〗〖@发〗
改: 初王〖@献王〗〖@德|刘德〗...王〖@彭祖|刘彭祖〗...初王〖@阏于|刘阏于〗...初王〖@非|刘非〗...初王〖@余|刘馀〗...〖@定王〗〖@发|刘发〗
```
（对照 059 五宗世家所有 canonical）

### 案例 2：格式错误"六安国" 误标为"安国"（D 类）

**发现过程**：扫描 `〖@[^〗|]+〗` 时，发现 `〖@安国〗` 单次出现于 r86（武帝元狩二年）。按原文"置六安国，以故陈为都"，"六安"是邦国名（汉武帝削胶东王刘寄，立其子庆为六安王），而非"六（数量）+ 安国（人名）"。这是典型的 AI 标注器**误分词错误**。

**修正**：
```
原: 置六〖@安国〗，以故陈为都。
改: 置〖◆六安〗国，以故陈为都。
```

### 案例 3：跨朝代时代错位（F 类，对标 015 样板）

**发现**：r71（元狩二年，前121）第17列（广川王列）出现 `〖@缪王|周缪王〗`。

**错误分析**：周穆王（亦作周缪王）在位前976-前922，与前121 差距近 800 年。此处应是**广川缪王**（广川惠王越之后继立者）。AI 标注器做 token-level 消歧时盲选"最著名"的周缪王，严重时代错位。

**修正**：`|周缪王` → `|广川缪王`（按列所属邦国+谥号）。

**新陷阱类型**：跨朝代名同。除 015 的"秦王/魏王/齐王"战国朝代切换外，汉年表出现"谥号省称同字"导致误匹配到周代王——属 C5 跨朝代陷阱。

### 案例 4：邦国表头"荆" 的补标（C 类）

**发现**：表头 L65 有 `〖◆楚〗 | | | 〖◆齐〗 | ... | 荆 | 〖◆淮南〗...`——楚/齐/淮南/燕/赵/梁/淮阳/代/长沙 均已标 `〖◆〗`，唯独"荆"未标。r1 表首行同样"楚。齐。荆。燕。赵。梁。代。" 七处全部未标。

**修正**：按 A107，本章主国名一律 `〖◆〗`，L65 表头 + r1 全部补齐（共 8 处）。

### 新发现建议（回写 SKILL_03c1-v2-rules.md）

#### 建议 1：A105 批量扫描模板（"谥号王+本名" 模式）

对汉代年表类章节，应编写自动扫描脚本：
```python
pattern = r'〖@(\w+王)〗〖?@?(\w+)〗?'  # 匹配 〖@X王〗Y 或 〖@X王〗〖@Y〗
# 对每处检查 Y 是否带 canonical；否则按行所属邦国列查询本姓（刘/吴/张/韩 等），补 |姓Y
```

#### 建议 2：新增陷阱"同字跨朝代 AI 消歧"

扩展 015 报告的陷阱表：
- **汉·谥号王误配周代王**：汉年表"缪王/穆王/厉王/幽王/康王/武王/成王/安王" 省称，AI 可能消歧到周同谥王。需按年份 + 邦国列双重判定。

**典型陷阱**：
| 字/词 | 周代候选 | 汉代候选 | 判定依据 |
|-------|---------|---------|---------|
| 缪王 | 周穆王 | 广川缪王 / 城阳缪王 | 按年份 / 所属邦国列 |
| 武王 | 周武王 | 胶西武王胡 / 齐悼惠王 | 语境（太史公曰 vs 表格）|
| 康王 | 周康王 | 燕康王嘉 / 胶东康王寄 / 长沙康王庸 | 按邦国列 |
| 厉王 | 周厉王 | 淮南厉王长 / 胶西厉王次昌 | 按邦国列 |
| 幽王 | 周幽王 | 赵幽王友 | 按邦国列 |

#### 建议 3：邦国漏标的分词陷阱——"六安"

"六"作为数量词与"六安/六国" 的邦国前缀字形混淆，AI 易误分为"六 + 安国"（其中"安国"误作人名）。本章 r86 即典型案例。

**扫描方法**：对任何 `〖@[X]\{1,2\}国〗` 标注做合法性检查：若前字为数字（六/三/七）+ 可识别邦国（六安/三晋/七国），应重分词为 `〖◆X〗国`。
