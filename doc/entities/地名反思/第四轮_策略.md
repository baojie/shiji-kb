# 第四轮地名反思 · 分级资源策略

> 核心原则：**置信度越低，资源投入越多**。

---

## 一、四级梯度

| 档位 | 置信度 | 条数 | 允用资源 | 分析深度 | 目标 |
|------|-------|------|---------|---------|------|
| **T1 高置信** | ≥0.90 | 919 | 仅规则检验 | **抽样 5%** | 捕抽样错误 |
| **T2 中置信** | 0.70-0.89 | 665 | 规则 + 本地证据包 | **全扫一遍** | 发现显式白名单漏网 |
| **T3 低置信** | 0.50-0.69 | 779 | + **轻量 LLM 推理** | 每条 ≤1 轮问答 | 批量确认/否定 |
| **T4 极低置信** | <0.50 | 134 | + **大模型 + 网络搜索**（维基/百科/历史地理志）| 每条深度研究 | 逐条查证 |

---

## 二、T1 抽样（高置信度捕错）

**方法**：从 919 条随机抽 46 条（5%），人工对 context_windows 做 sanity check。

**预期发现**：显式白名单中的个别错误（如我之前把 "大行" 错归非地名）。

**产出**：若发现错误则修正显式清单，无错误则"high-confidence validated"。

---

## 三、T2 本地证据包扫描（中置信度）

**方法**：对 665 条走 [`analyze_evidence.py`](../../../kg/entities/scripts/analyze_evidence.py) 的**严格模式**：
- 仅采信 `X，Y名` 这种三家注显式陈述
- 仅采信"同句官职"（太守→郡, 令丞→县, 亭长→乡里）

**预期产出**：补充 L1 白名单 10-30 条；提升 T2→T1 的条数 50-100。

---

## 四、T3 轻量 LLM 批量（779 条）

**方法**：
1. 每条构造紧凑 prompt：
   ```
   《史记》地名"{name}"，当前分类标签：[县, 城邑]。
   上下文（{N} 次出现）：
     - 章节A: ...{name}...
     - 章节B: ...{name}...
   请从 [水域/山脉/原野/关隘/区域/国家/州/郡/县/城邑/建筑/陵墓/乡里/道桥/误标/虚构]
   中选择最可靠的**单一**首选分类，并给出 0-1 置信度。
   若当前分类合理即回答"confirm"。
   ```
2. 调用轻量模型（claude-haiku / claude-sonnet）
3. 批处理：20 条一批 → ~40 次调用

**成本估算**：每条 prompt ≈ 500 tokens，返回 50 tokens → 每批 ≈ 11K tokens × 40 批 = 440K tokens。Haiku 价格合理。

**产出**：每条得到 LLM 推荐 + 置信度。推荐与当前分类不一致的条目进入**人工复核队列**。

---

## 五、T4 深度研究（134 条）

**方法**：对 conf<0.50 的 134 条地名，每条：
1. **证据收集**：从 place_evidence_bundles.json 取本条全部证据
2. **外源查询**：
   - WebSearch 维基百科 / 百度百科 "史记 {name}" 条目
   - 查 "《汉书·地理志》 {name}" 相关条目
   - 查中国历史地理数据库（如有）
3. **大模型综合判断**：调用 Opus 给出详细分类依据
4. **结果保存**：每条一个 markdown 片段，含 evidence + 查询结果 + 最终推荐

**资源预算**：134 个 Agent 调用，每个用 general-purpose + WebSearch。

**分批执行**：20 条一批 → 7 批。

**产出**：`doc/entities/地名反思/第四轮_T4_深研.md` 每条含证据链与外部信源。

---

## 六、执行顺序 & 输出

按倒序执行：**T4 → T3 → T2 → T1**，高价值优先。

每个 Tier 完成后：
- 提炼规则 → 追加到 [`classify_places.py`](../../../kg/entities/scripts/classify_places.py)
- 重跑 `classify + confidence_report + export_csv`
- 更新 `第四轮_反思报告.md`

---

## 七、成本与中止条件

- 若 T4 平均每条修复成本 > 5 agent 调用，考虑缩减 T4 范围到 top-50 最关键
- 若 T3 LLM 建议与当前分类一致率 > 80%，说明我们规则已趋近最优，T3 可提前停止
- **硬停**：本轮总 agent 调用 ≤ 50 次；若超过，收尾产出部分结果

---

## 八、未来方向

- **图谱聚类**：未完全利用的 `place_evidence_bundles.json` 的 `cooccur_tags`，可以做同现图谱，相同"邻居指纹"的地名可能属同类
- **历史地理 KB**：接入开源的中国历史地理信息系统（CHGIS）API 做权威查证
- **多轮校对**：每半年重新跑本策略，捕获新证据（如源文件标注修正后）
