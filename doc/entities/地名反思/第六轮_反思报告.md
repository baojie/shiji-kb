## 一、决策：为什么选 T1

由 Claude 自主决策（用户授权"自己决策"）。理由：

1. **从未抽样**：前五轮都在推低置信度，T1 像"黑箱"

4. **Agent 递减**：前两轮已见 40%→20% 纠错率，继续 T4 边际收益低

## 三、发现的错误

### 3.1 误白名单（3 条显著）

| 条目 | 原分类（错） | 修正分类 | 依据 |
|------|------------|----------|------|
| **阙** | 误标（因单字泛称）| **建筑** | "斩使者于阙下" 指宫阙 |
| **颍谷** | 山脉（误归山谷）| **城邑** | "颍谷之考叔有献於公" 郑邑 |
| **鄂** | 县（HAN_VASSAL_KINGDOM）| **县 + 国家**（多标签）| 楚熊渠封次子为鄂王 |

### 3.2 `HAN_VASSAL_KINGDOMS` 类混淆（9 条）

这是**最大系统性错误**：把**侯国（县级）**错归为**王国（郡级）**。

**背景**：
- 诸侯王国（郡级）：楚国、齐国、赵国、梁国…— 有王室宗庙
- 侯国（县级）：安国侯、昌国侯、充国… — 只有侯爵

**移除（9 条）**，重归为 **县 / 国家 / 古邑**：
- `安国` → 县（樊哙等所封高祖功臣侯国）
- `昌国` → 县（乐毅封地）
- `充国` → 县（汉充国县）
- `东国` / `南国` / `北国` / `西国` → 县（史记散见小邑/侯国）
- `晋国` → 国家（春秋大国，非汉王国）
- `耆国` → 国家（古耆国）

### 3.3 不作修改（确认正确）

67 条抽样确认分类合理：
- `玉堂` / `脽` / `燕邸` —建筑 ✓
- `苍梧之野` / `易下` —原野 ✓

## 六、反思

### 6.1 T1 抽样揭示的系统性问题

HAN_VASSAL_KINGDOMS 的误归是**分类概念混淆**，不是简单的类别选择错。
- 高置信度（L1 = 0.90）反映"规则稳定"，不反映"规则正确"
- **抽样是质量保证不可或缺的一环**

### 6.2 不对称成本

- 低置信度修错：单条影响 1-2 条（仅自身）
- **高置信度修错：单条影响 5-10 条（通过 peer / split 传播）**
- 因此 T1 抽样边际价值远高于继续 T4 扫描

### 6.3 下一步建议

**T2（0.70-0.89，703 条）抽样**：
- 用同样方法抽 30-40 条
- 预期发现更少系统性错误（因为有多重规则交叉验证）

**规则审计**（更根本）：
- 对每个 `EXPLICIT_*` 集合做一次全量"静态审查"
- 看是否有内部冲突、重复或分类概念混淆

### 6.4 反思的反思

"置信度 < 0.9 值得质疑" 这是直觉，但 **"置信度 ≥ 0.9 也值得抽样"** 是本轮的核心发现。今后的策略需要补一档：

| 档位 | 行动 |
|------|------|
| T0 **(≥0.90 抽样)** | **每轮 5% 抽样** |
| T1-T4 | 按原策略 |
