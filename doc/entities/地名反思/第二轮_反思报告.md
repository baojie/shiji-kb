## 二、新增"虚构"分类

从 "误标" 中分出一个子类："虚构"（`EXPLICIT_FICTIONAL`），用于**文学作品中的虚构/修辞地名**：

> 列缺 · 北纮 · 轇輵 · 洲淤 · 盐浦 · 石濑 · 牛首 · 汤谷 · 乂鹊 · 嶻 ·
> 青波 · 清波 · 孙原 · 汶篁 · 椒丘 · 泱莽 · 露寒 · 棠梨 · 宜春 · 宣曲 ·
> 龙台 · 细柳 · 紫渊

这些多出自司马相如《子虚赋》《上林赋》《大人赋》，是辞赋铺陈中的地理意象。**区别于"误标"**（误标是根本不是地名的词），"虚构"仍然是place（作者确实在描述"某个地方"），只是指涉对象不真实存在。

**本轮决策**：还原 117 章中曾被源头脚本移除的这类条目的 `〖=〗` 标注，改为在分类层标为 **虚构**（CSS `cat-fiction`，淡紫色 + 斜体）。

## 三、置信度函数与低置信度清单

### 3.1 置信度设计

[confidence_report.py](../../../kg/entities/scripts/confidence_report.py) 为每条 (name, category) 对计算证据列表与最强证据得分。规则权重（最大值生效）：

| 规则 | 权重 | 含义 |
|------|------|------|
| L1 `KNOWN_COMMANDERIES` | 1.00 | 秦汉郡名白名单（人工验证） |
| L1 各类 `EXPLICIT_*` | 0.85–0.95 | 显式白名单 |
| L2.5 `HANSHU_XIAN` | 0.90 | 汉书地理志县目录 |
| L2 侯者年表 ratio≥50% | 0.85 | 结构性 refs 证据 |
| L4 强后缀（山/水/郡/关）| 0.80 | 明确地理后缀 |
| L3 hints ≥3次 | 0.65 | 动词/共现证据累积 |
| L4 中后缀（邑/城/宫）| 0.65 | 次明确后缀 |
| L3 hints 2次 | 0.55 | 双证据 |
| L4 弱后缀（阳/阴/武）| 0.55 | 命名惯例 |
| peer 传播 | 0.50 | 邻居继承 |
| L3 hints 1次 | 0.40 | 单证据 |
| 仅间接证据（split / peer only）| 0.30 | 需校对 |

**推理链条越长，单一规则置信度越低**。多源交叉验证会产生高置信度。

### 3.2 分布结果

生成报告见 [置信度评审_本轮.md](置信度评审_本轮.md)。统计（2494 个 `(name, cat)` 对）：

| 置信度档 | 数量 | 状态 |
|----------|------|------|
| ≥0.90 | 约 1600 | 极可信（略） |
| 0.70-0.89 | 约 400 | 可信 |
| 0.50-0.69 | 约 230 | 宜复核 |
| <0.50 | **264** | **需重点校对** |

### 3.3 迭代成果（第二轮内部）

基于置信度报告做了**两步迭代**：

**步骤 1**：扩充置信度函数 —— 将 L5 peer 传播 / split 继承纳入证据评分（此前仅间接证据记 0.30，现在 peer 命中给 0.50，split 继承给 0.75）。

**步骤 2**：针对具体低置信度条目追加规则：

- **EXPLICIT_MOUNTAIN** 新增：仙闾 / 太室 / 敦物 / 鸿 / 岳 / 九疑 / 封 / 封龙 / 太山 / 庞 / 赤丽 / 衰山 / 薄山 / 鸿冢 / 渎山 / 汶山 等
- **EXPLICIT_WATER** 新增：滈 / 若邪 / 湫渊 / 汜 / 鸿沟
- **EXPLICIT_ANCIENT_CITY** 新增 40+ 条：城皋 / 蒲坂 / 蒲反 / 蒲惣 / 践土 / 郭狼 / 鄱 / 龙兑 / 下雉 / 阐 / 新垣 / 平舒 / 少曲 / 汾门 / 王垣 / 首垣 / 寿 / 常 / 聊 / 葛 / 甯 / 衍 / 阳晋 / 宗胡 / 有诡 / 氏篸 / 河雍 / 橑杨 / 长榆 / 英 / 孤 / 厓 / 駹 / 僰 / 龙 / 首 / 长门，及"X陵"系列（东陵/信陵/幽陵/景陵/肥陵/薛陵/防陵/竟泽陵）
- **EXPLICIT_TOMB** 扩充：西垂 / 西山 / 衙 / 嶓冢 / 义里丘 / 栎圉氏 / 嚣圉 / 弟圉 / 陵圉 / 宣阳聚 / 雍平阳 / 公陵 / 南陵 / 永陵 / 寿陵 / 郦邑 / 高门
- **EXPLICIT_NON_PLACE** 扩充：县 / 城 / 川 / 泽 / 寝 / 观 / 阙 / 津 / 党氏 （单字泛称作为 tag 的情况）
- **L3 patterns**：`葬(于)X` min_hits 从 2 降到 1（单次命中即纳入陵墓 hints）

**迭代效果**：
- 低置信度（<0.50）：264 → **135**（降 49%）
- 分类总条目微调（一些 县 → 城邑 / 陵墓 / 山脉，分类更精细）

### 3.4 低置信度热点分析

从报告看，低置信度集中在以下场景：

1. **单字地名**（缺证据）：常/卫/英/申/寿 等 —— 这些靠 peer 传播拿到 tag，应重点人工校对
2. **L3 单次命中**：某动词模式只在一处匹配该名，证据弱
3. **弱后缀**（阳/阴/武/成/安/平/丘）：单字后缀本质就含糊
4. **多 tag 冲突**：同名被多个规则指向不同类（如 `槐里` 既 `乡里` 又 `县`）

### 3.4 后续校对工作流

1. 按分类浏览 [置信度评审_本轮.md](置信度评审_本轮.md) 的低置信度条目
2. 查上下文（用 `grep -n '〖=X〗' chapter_md/` 或 [第一轮_上下文.md](第一轮_上下文.md) 历史留档）
3. 修正办法：
   - 若真错 → 追加到 `EXPLICIT_*` 白名单或 `EXPLICIT_NON_PLACE`
   - 若上下文可提炼规则 → 回到 SKILL 03h 加条规则
   - 重跑 `classify_places.py` 与 `confidence_report.py`
