## 六、反思

### 策略验证

- **T4（<0.50）投入最大，但回报最高**：134 条找到 20+ 误分类，占比 ~15%
- **T2（0.70–0.89）第三轮已摘低垂果实**：本轮本地扫描几乎无新发现
- **T3（0.50–0.69）本轮跳过**：ROI 低于 T4，待专门一轮做
- **T1（≥0.90）维持抽样监控**：累计已验证 ~50 条无异常

### Agent 使用经验

单次 Agent 调度 5 条地名 ≈ 27K tokens，平均每条 ~5.4K tokens。成本合理，效果好：
- **Agent 利于有定见但需核实的情况**（壤东是城邑还是原野？）
- **不适合完全冷门名字**（没有百科可查的，Agent 也只能基于史记推断）

**最佳组合**：领域知识快筛 + Agent 核实不确定项。

### 未尽事项

- T3（779 条）应在下一轮用批量 LLM 扫一遍
- cooccur_tags 的图谱聚类（第三轮未充分挖掘）可能带来新信号
- 接入 CHGIS（中国历史地理信息系统）做权威查证，突破纯规则上限

### 策略性结论

**conf<0.50 已降至 110 条（占总数 4.4%）**。进一步降低需要权衡边际成本：
- 接受 ~100 条为"证据稀疏稳定"条目，标记为"待人工验证"
- 或继续用 Agent 批处理 3-5 轮直至天花板
