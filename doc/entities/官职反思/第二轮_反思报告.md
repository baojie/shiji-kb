## 六、经验教训

1. **"人名误标"是最大的误标子类**：106 条（6.9%），远超"纯误标"的 119 条中真正的非人名部分。这说明 `〖;X〗` 标注对于"官号 + 具体人"的复合体（丞相何/太宰嚭/萧相国/老上单于）界限不清，**源头标注应统一拆分为 官职 + 人名 两个实体**。
2. **简称识别是高 ROI 步骤**：汉代中央官有大量简称（大农/光禄/大鸿/水衡/属国/大内/贰师/楼船/骠骑/游击），一个简称白名单扩充就能解锁十几条官职。
3. **春秋"X子"尊称的陷阱**：末字"子"的 2 字词多为春秋大夫尊称（桓子/武子），但极少数是古官名（如"司职吏"不以子结尾就无歧义）。简单后缀规则会误判，必须人工审查。
4. **EXPLICIT_* 必须全连接到 classify_l1_whitelist**：添加新白名单集时若忘记在 L1 函数中检查，会静默失败。本轮发现 `EXPLICIT_FOREIGN` 漏连接导致 `归义` 一条死角；后续添加任何 EXPLICIT_* 必须同步更新 L1。

## 七、第三轮候选

**0 条未分类** → 无需第三轮冷启动反思。

后续迭代方向应转向：
- **T1 (≥ 0.90 置信度) 抽样质量保证**：人工审查高置信度条目，找出系统性误判（类似地名分类第六轮反思）
- **低置信度 (< 0.50) 条目深研**：目前很多多标签是 L1+L4 叠加触发的伪自信，需要用 Agent + 共现语境验证
- **白名单审计**：检查 900+ 条 EXPLICIT_* 项是否均有史记证据对齐（类似地名第七轮 audit_explicit_sets.py）
- **诸侯王国官体系的时代消歧**：汉初诸侯王国设相/内史/中尉/郎中令/廷尉，与中央同名但级别差别大，需按时代 + 语境细分
