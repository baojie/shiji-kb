## 五、经验教训

1. **细分误标 > 大类误标**：将 `误标` 拆为 `误标`/`人名误标`/`身份误标`/`谥号误标` 四子类，让"源头标注修正"有明确的操作对象（按子类批量改标）。
2. **诸侯国官的时代层级问题**：汉初诸侯王国官员与中央同名但级别不同（相=郡守级），不能简单按"三公九卿"字面匹配。引入 `HAN_VASSAL_STATES` 常量 + 动态匹配函数，兼顾静态白名单的精确度和规则扩展性。
3. **置信度透明度公式的学问**：`opacity = 0.40 + 0.60*conf` 保底 0.4 让低置信度也"看得见"；改为 `opacity = conf` 后低置信度几乎不可见，**视觉强度与置信度强烈对齐**，用户一眼就能锁定需要审查的条目。代价：需要手动悬停才能读到 0.1-0.3 的徽章文字，但这正是"低置信度应花更多精力审查"的诱导信号。
4. **早返回（return）vs 累加（append）的架构决策**：误标类（4 类）采用 early-return 单标签，因为一旦被识别就不应再多标签；分类类（16 类）采用 append 多标签，允许"丞相"同时挂三公 + 泛称。这种混合模式简化了下游逻辑，也避免了冲突类别（如"丞相"不会同时是三公和误标）。
