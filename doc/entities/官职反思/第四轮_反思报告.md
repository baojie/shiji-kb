## 二、关键 Bug 修复：L2/L3 过度应用

### 2.1 L2 兜底化（不再中层叠加）

**原问题**：L2 "侯者年表→爵位"/"诸侯王年表→王号" 在 `final = ['泛称']` 时会加 `爵位`，导致像 `子`/`男`/`卿`/`期门` 等 泛称条目被误叠 爵位。

**修复**：L2 现在只在 `final` **完全为空** 时才应用，并放到 merge 链路最后（兜底）。

```python
# 之前：只要 L1 是 泛称/空 就加 L2
_l1_weak = not final or set(final) <= {CAT_GENERIC}

# 现在：严格只有 final 为空才加 L2
if l2 and l2 not in final and not final:
    final.append(l2)
```

**效果**：
- `园厩啬夫`：['爵位', '县乡吏'] → ['县乡吏']
- `复土将军/徇北将军`：['爵位', '军职'] → ['军职']
- `子`/`男`/`期门`：不再误叠爵位
- `男`/`期门` 现已单标 泛称

### 2.2 L3 context 不得覆盖具体类别

**原问题**：L3 上下文规则 `〖;X〗将兵|领兵` 会把 `丞相`/`相国` 等具体三公官误加 `军职` 标签（如"萧丞相将兵..."）。

**修复**：L3 结果只在 `final` 无具体类别（即只有 泛称 或 空）时生效。

```python
_has_specific = final and not (set(final) <= {CAT_GENERIC})
if l3_cat and l3_cat not in final and not _has_specific:
    final.append(l3_cat)
```

**效果**：
- `丞相`：['三公', '军职'] → ['三公']
- `相国`：['三公', '军职'] → ['三公']

### 2.3 merge 优先级重排

新顺序（从强到弱）：
```
L1 (白名单) > L2.5 (汉书百官表) > L3 (context 上下文) > L4 (后缀) > L2 (章节兜底)
```
