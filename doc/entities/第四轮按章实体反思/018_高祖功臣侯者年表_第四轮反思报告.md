## 三、章节特有发现

### 发现 1：年表复合侯号易被空格 / 字间切分

本章多处出现「X侯 Y(人名) 元年」的列格式。在排版压力下，`文成侯张良` / `乐成侯` 等 **跨修饰词 + 谥号 + 侯** 结构易被 AI 切成 `文〖@成侯〗` / `乐〖@成侯〗` 形式（将修饰前缀裸露于标注外）。建议对年表类章节加入复合侯号完整性扫描：

```bash
grep -nE '[一-鿿]〖@[一-鿿]+侯〗' chapter_md/NNN_*.tagged.md
```

匹配到「前字+@侯〗」即可能是复合侯号被切。

### 发现 2：年号 + 时长混合

`太初百年` 类混合表达在年表 / 本纪的太史公论赞段落中出现，需在 A8（时长消歧）之外补一项「年号 + 时长」分离检查。

### 侯国谥号消歧 A87/C1 情况说明

本次**不对已标 `〖@X侯〗` 批量注入规范名后缀 `|Y侯X`**，原因：

1. **数据结构风险**：侯表同侯国多代袭封（如平阳侯 r1：懿侯曹参→靖侯窋→简侯奇→夷侯时→恭侯襄→今侯宗），若统一写 `〖@懿侯|平阳懿侯〗`，需逐行绑定 (国名, 侯号, 代序)，裸脚本批量易错
2. **A87 默认原则**：当同段内紧邻全名（如 `〖@懿侯〗〖@曹参〗`）已经消歧，可不再注入规范名
3. **C1-C5**：单字名已补 `〖@〗` 基础标注（如 `〖@伉〗`/`〖@彊〗`/`〖@灶〗`/`〖@穰〗`），若后续发现同章同名冲突再按 A87 加 `|` 消歧
