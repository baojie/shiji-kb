## 反思总结

### 第四轮新增价值

1. **E3 规律首次覆盖本章**：发现第三轮"0 处标点差异"报告的盲区——半角/全角引号属 CLAUDE.md §4 铁律范畴，但第三轮的 `lint_text_integrity.py` 仅做纯文本字符比对，不校验引号形态。E3 规律（2026-04-16 新增）正是针对此类"幽灵修复"的系统性修补。
2. **A4 侯号职官化深度推进**：本章集中揭示了 `〖=封地〗〖;侯〗` 及 `〖@封号〗〖;侯〗` 两大合并模式——封号作名词修饰侯位时应整体标为 `〖;X侯〗`（职官/爵位），而非拆分为地名+侯位。
3. **A87+A108 配合**：本章"陵/勃/平"三主人公省称在赞诗段（L104）被强制消歧，与非赞诗段的豁免规则形成对比，印证 A108 规律的独立价值。

### 规律库反馈建议

- **A4 新模式**：`〖@封地名〗〖;侯〗` → `〖;封地侯〗`（本章 L61/L63 "安国侯"、L36/L38 "淮阴侯"、L43 "户牖侯"、L45 "曲逆侯"）。此模式广泛存在于汉初世家/列传，建议补入 A4 第三节"封地+侯合并规则"。
- **E3 执行时机**：必须作为第四轮 Step 0 **首先执行**，因为半角引号会干扰后续 Python 脚本的上下文字符串匹配（本轮即因半角→全角转换后，导致 1 处 `〖@信〗知之乎` 的 pattern 匹配失效，需二次修复）。

### 剩余可能问题

- `〖@孝惠〗` 仅出现 1 处且已消歧；`〖@惠帝〗`（L59/L63）上下文明确无歧义，未消歧。
