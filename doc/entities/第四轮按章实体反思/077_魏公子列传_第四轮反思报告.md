## 二、最严重违规：E3 半角引号泛滥（60 处）

### 2.1 问题描述

本章 `.tagged.md` 文件中**共有 60 个半角引号 `"`（U+0022）**，分布于以下行：

- L9, L13, L15, L19, L21, L23, L25, L29, L31, L33, L41

而对应的原文（`docs/original_text/077_魏公子列传.txt`）使用的是**全角引号 `“`（U+201C）+ `”`（U+201D）**，各 30 个，共计 30 对完整对话引号。

这是对 **CLAUDE.md §4 "*.tagged.md 编辑规范"** 和 **SKILL_03c1-rules E3** 的严重违规：

> ❌ **绝对禁止的引号**：半角引号：`"` `'` `` ` ``（U+0022, U+0027, U+0060）
> ✅ **只允许使用中文全角引号**：左双 `“`（U+201C）+ 右双 `”`（U+201D）

### 2.2 违规发现过程

**第三轮报告隐患**（2026-04-15）：第三轮报告 §5.4 声称"无半角引号（检出的引号均在对话引述标记中，非实体标注内）"——这是**错误判断**。原文用全角引号，标注文件引入了 60 个半角引号，属于实质违规，并非"仅在对话引述中可以豁免"。类似 SKILL_03c1-rules E3 所述"048章第三轮报告声称已修但实际未改动"的"幽灵修正"模式。

**第四轮复查命令**：
```bash
python3 -c "open('chapter_md/077_魏公子列传.tagged.md').read().count(chr(0x22))"
# 返回: 60
```

### 2.3 修复方法（Python 脚本批量处理）

```python
# 成对替换：第1/3/5...个 " → “；第2/4/6... → ”
result = []
quote_index = 0
for char in content:
    if char == '"':
        result.append('\u201C' if quote_index % 2 == 0 else '\u201D')
        quote_index += 1
    else:
        result.append(char)
```

**修复后验证**：
- U+201C（左双）= 30
- U+201D（右双）= 30
- U+0022（半角）= 0
- `lint_text_integrity.py 077` → 0 处实质差异 ✓

## 三、A5/A4: `〖@赵惠文王弟平原君〗` 黏连分拆

### 3.1 问题位置与原文

**位置**：L19
**原文**：「公子姊为**赵惠文王弟平原君**夫人」

### 3.2 错误标注

```
〖@公子〗〖#姊〗为〖@赵惠文王弟平原君〗夫人
```

该标注将 "赵惠文王弟平原君"（**赵惠文王**是人，**弟**是血缘身份，**平原君**是封号人）整体作为一个人名实体，违反：

- **A75 / A86**：亲属附加称谓"弟"应独立标 `〖#〗`
- **A87**：封号"平原君"独立成人名

### 3.3 修正后

```
〖@公子〗〖#姊〗为〖@赵惠文王〗〖#弟〗〖@平原君〗夫人
```

- `〖@赵惠文王〗`：谥号人名（赵国惠文王，按 B7 规则 1 邦国+谥号合并）
- `〖#弟〗`：亲属关系词（A75 / A86 保留 `〖#〗`）
- `〖@平原君〗`：封号人名（赵胜）

**语义验证**：`赵惠文王之弟平原君`（赵胜），三者分别承担不同语义功能，原文 "弟" 字不再被错误地吞入人名内部。
