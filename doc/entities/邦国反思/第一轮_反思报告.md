## 六、规则沉淀

### 6.1 rulers.json 类型→类别映射需 guard

**问题**：rulers.json 的 `诸侯王` 类型既包含战国诸侯王（秦楚齐燕等），也包含外邦王（南越/东越/闽越）和秦末项羽分封王（雍塞翟等）。直接映射会让外邦/秦末条目多余加上 `周代诸侯` 标签。

**方案**：当 L1 已给出更具体类别（外邦/秦末列国/汉诸侯王国/上古方国）时，L2.5(rulers) 的 `周代诸侯` 映射跳过。

**效果**：多标签从 22 → 9，全部合理。

### 6.2 X国 与 X 同名冗余 → split

**问题**：秦国/齐国/楚国/燕国/魏国/赵国/宋国/晋国/菑川国 与 `〖◆秦〗` 等单字条目指同一邦国，是源头标注的冗余。

**方案**：`_is_split_xguo()` 动态规则 + `EXPLICIT_NEED_SPLIT` 显式列入。

### 6.3 X国（X 非邦国）保持独立

**原则**：荆国/身毒国/滇国 等 — X 不是独立 feudal-state 条目 — 不归 split，按语义归对应类。
