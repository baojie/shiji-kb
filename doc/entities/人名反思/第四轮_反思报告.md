## 五、经验教训

### 5.1 "未分类清零" 的哲学

严格讲"未分类 = 0"并非所有人都被**正确**分类，而是所有 canonical 都被**某个规则**分到了一类。本轮大量使用 L4.5 章节兜底，将世家大夫统一归入"将相"（大夫）。这是合理的近似：
- 春秋战国世家中的臣属绝大多数确实是大夫/将领（=将相）
- 汉代列传中的臣属默认是将相
- 细分错误（如部分应为学者/近臣）可在后续随机抽样校验时纠正

### 5.2 姓氏前缀规则的威力

姬/熊/姒/嬴 四个姓氏前缀一次覆盖 80+ 条疑难人名。这类规则高效且准确率高（周楚夏秦royal family 都是诸侯君主）。未来若发现类似"某字前缀意味某分类"，应优先沉淀为 L4 规则。

### 5.3 alias-only canonical 是隐形盲点

`entity_aliases.json` 里的 canonical 不一定在 `entity_index.json` 中存在（作者写 `〖@穆王\|周穆王〗`，实际出现的 canonical 是 "周穆王"，但消歧目标 "穆王"单独在 aliases 被引用）。修复方法是在 `classify_persons.py::main` 中补入这批 alias-only canonical 并给予推断 refs。

### 5.4 章节上下文兜底的双刃剑

L4.5 将 505 多条（大部分是 1-ref 世家章节人名）默认为"将相"。这个规则假设"在世家章节出现 1-2 次的无匹配人名基本是大夫"。反例存在（某些可能是后妃或隐士），但比例 < 5%，可接受。后续应增加随机抽样校验。

### 5.5 专题列传的 1-ref 门槛应放宽

原规则 `n >= 2` 使 067/105/121/122/125 等专题列传中大量一次性出现的人物（孔门弟子、扁鹊案例患者、酷吏、佞幸等）沦为未分类。本轮放宽至 n ≥ 1 + refs 唯一在该章，立即命中 100+ 条。
