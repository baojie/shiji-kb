# 第一轮反思报告 — 人名分类冷启动

**日期**：2026-04-19
**起点**：首版 classify_persons.py 跑出 2642 条未分类（53.7%）
**终点**：2103 条未分类（42.7%），下降 11.0 pts / 降幅 20%

## 一、规则挖掘（从上下文反思到白名单/后缀规则）

### 1.1 四来源融合（冷启动基础）

本轮搭建了多来源融合的 L1/L2.5 框架：

| 来源 | 覆盖数 | 角色 |
|------|-------|------|
| `person.ttl` 本体分类 | 1718 | L1 最高优先（1825 人，去重后 1718 唯一 canonical）|

### 1.2 rulers.json form 扩展

发现 rulers.json 的谥号字段常为短形式（"襄公"/"文公"），需组合所属国+谥号才能匹配 entity_index 的 canonical 形式（"秦襄公"/"秦文公"）。扩展生成以下 form：

- 谥号（如 "襄公"）
- 所属国+谥号（如 "秦襄公"）
- 名（如 "句践"）
- 别名（如 "越王句践"）
- paren 内容（如 "赵王（赵敬肃王）" → "赵敬肃王"）

覆盖提升：1915 → 1957 条 form。

### 1.4 手工白名单（EXPLICIT_*）

覆盖 16 类各自的典型代表。重点补足以下：

- **EXPLICIT_EMPEROR**：三皇五帝、夏商周天子、汉代帝号、殷商/周先公先王系谱（相土/昌若/冥/振/高圉/亚圉 等 20 余条）
- **EXPLICIT_RULER**：春秋战国诸侯（秦惠王/卫献公/郑文公/杞湣公 等 40+ 条）、楚系小国（楚蚡冒/楚鬻熊）
- **EXPLICIT_PRINCE**：战国四君子 + 汉代刘姓宗室（刘孝/刘勃/刘志/刘迁 等 30+ 条）
- **EXPLICIT_CHANCELLOR**：先秦卿大夫（崔杼/鲍叔牙/智伯瑶/隰朋 等）、春秋世族（赵盾/狐偃/先轸）、"国名+人名"复合体（秦张仪/魏相田文 待拆分）、异体字（硃家/硃亥）
- **EXPLICIT_SCHOLAR**：孔门弟子（端木赐/商瞿/高柴）、汉代博士（申公/辕固生/伏生）、汉代文士（司马相如/枚乘/邹阳）

### 1.7 alias 抽取工作流（人名独有）

新建 `extract_aliases_from_tags.py`，合并 5 种来源：

1. `chapter_md/*.tagged.md` 内联消歧 `〖@X|Y〗` → **2593 条**（最权威）
2. `disambiguation_map.json` 章节级消歧 → 260 条
3. `rulers.json` 别名字段 → 460 条
4. `private/to鲍捷 史记里的人名.md` 先秦称呼对照 → 286 条
5. 旧 `entity_aliases.json` 兜底 → 606 条

**合并结果**：person 3322 条，585 个歧义 surface（多 canonical）。

新结构为 4 列数组：`{surface, type, canonical, refs}`，同 surface 按 canonical 拆分多行，refs 记录原文出处。

旧结构备份为 `entity_aliases.v1.bak.json`。

## 五、经验教训

1. **TTL 本体是金矿**：1718 条预分类覆盖了 35% 的 person 实体，是冷启动最大杠杆。投入转化比最高。
2. **rulers.json 的谥号需要组合拼接**：短形式"襄公"需配合所属国"秦"才能匹配 entity_index 的"秦襄公"。首版直接用 canonical 字段导致只命中 380 条诸侯君主，修后变 757 条。
3. **alias 4 列结构比旧 {canonical: [aliases]} 更强**：同 surface 可映射多 canonical（歧义不丢失），refs 可追溯出处，便于消歧和 L2.5 继承。
4. **汉代中层官员是大盲点**：武帝朝 2-3 字名（许昌/赵周/田广明）占相当比例，需要 L3 共现规则（"为丞相"/"拜将军"）才能覆盖。
5. **单字人名假阳性普遍**：则/它/始/执/微 等高频虚词/动词被标为人名，需通过上下文判断是否为 cat-mis。这部分建议设阈值（count < 3 且无明确上下文 → 暂不分类，标 low-conf）。
6. **与地名分类对称性**：L1-L5 流水线、多标签架构、反思档案布局完全可复用。人名额外需要 alias 子工作流（地名极少别名）。
