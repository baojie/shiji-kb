#### 2.2.4 主人公省称全覆盖原则

**发现章节**：001、057章（第一批）+ 091黥布列传（第二批）

**规律**：世家/列传主人公的姓名省称应全覆盖，不论首次或后续出现。

**升级案例**：
- 091黥布列传："上"字（指刘邦）遗漏18处，占修正总数16.5%
- 主人公高频省称（单字）极易遗漏，需系统grep检查

**常见遗漏位置**：
- 动词前主语：`勃为人` → `〖@勃〗为人`
- 介词后宾语：`说勃曰` → `说〖@勃〗曰`
- 被动句施事：`告勃欲反` → `告〖@勃〗欲反`
- **君主省称**：`上闻之` → `〖@上〗闻之`

**检查方法**：
```bash
# 通用检查
grep -n '[^〗]主人公省称[^〗〖]' chapter_md/NNN_*.tagged.md

# 君主省称检查
grep -n '[^〗]上[^〗〖]' chapter_md/NNN_*.tagged.md | grep -v '上下\|以上\|上古'
```

### 8.4 分类树增强方案（2026-03-21提出）

**核心思想**：
> **"规律中的分类知识可以转化为分类树节点，分类树可以反哺规律判断"**

**双向增强机制**：
1. **反思→分类树**：从反思规律中提取分类知识，更新taxonomy
2. **分类树→反思**：按章反思时读取分类树，辅助类型判断

- **Step 0增强**：读取`kg/taxonomy/person_taxonomy.json`（人物群体分类）

- **完成后操作**：发现新群体词→追加到`kg/taxonomy/data/taxonomy_updates.jsonl`
