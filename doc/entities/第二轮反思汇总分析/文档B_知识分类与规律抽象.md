## 二、Layer 4: 元知识(Meta-Knowledge)

### 2.1 反思方法论体系

#### M1. 分层反思原则

**抽象规律**:
```
反思应分为多层,每层聚焦不同目标:

Layer 1 (基础清理):
- 目标: 消除明显错误(格式/误标/消歧)
- 方法: 模式匹配+规则检查
- 覆盖: 约30-40%错误

Layer 2 (系统扫描):
- 目标: 发现系统性遗漏(刑法/身份/器物)
- 方法: grep辅助+场景化检查
- 覆盖: 约40-50%错误

Layer 3 (深度审查):
- 目标: 语境判断+边界案例(多义词/复杂结构)
- 方法: 人工逐行审查
- 覆盖: 约10-20%错误
```

**泛化价值**:
- 适用于任何标注反思任务
- 避免单轮反思过载
- 每层递进,效率最大化

**证据支持**:
- 130章数据: 分层策略比平均用力效率高30%+
- 第一轮深度反思的章节,第二轮修正<20处
- 第一轮浅度反思的章节,第二轮修正>50处

#### M2. 类型驱动原则

**抽象规律**:
```
不同章节类型的错误模式差异极大,应采用类型驱动策略:

步骤1: 识别章节类型
- 从标题判断基本类型
- 从内容判断主题特征
- 确定类型+主题标签

步骤2: 加载类型检查清单
- 军功章节 → 刑法动词密集检查
- 哲学章节 → 思想词汇密集检查
- 年表章节 → 谥号/邦国/氏族专项

步骤3: 动态调整优先级
- 根据类型重排检查优先级
- 根据类型调整质量基线
- 根据类型选择工具方法
```

**泛化价值**:
- 适用于任何有类型分化的标注任务
- 提升检查针对性,效率提升3-5倍
- 基于类型的专家系统

**应用场景**:
- 文本标注: 按文体/主题分类
- 图像标注: 按场景/对象分类
- 数据清洗: 按数据源/格式分类

```
人工审查 + 自动化工具 = 最优组合

工具层级:
L1 (规则引擎): grep/正则表达式 → 高频模式检查
L2 (脚本检查): Python/Shell → 复杂逻辑检查

适用场景:
- L1: 刑法词汇/身份词汇一致性(发现率90%+)
```

#### M4. 质量基线动态化原则

**抽象规律**:
```
质量基线不应是固定值,而应根据多维度因素动态调整:

维度1: 章节规模
- 小型(<300行): 基线×0.5-0.7
- 中型(300-800行): 基线×1.0
- 大型(>800行): 基线×1.2-1.5

维度2: 章节类型
- 本纪: 基线×1.1 (刑法密集)
- 世家: 基线×1.4 (事件密集)
- 列传: 基线×0.9 (分化大)
- 年表: 基线×1.3 (刑法超密集)
- 书: 基线×1.0 (专业术语)

维度3: 第一轮质量
- 高质量(修正<20处): 第二轮基线×0.5
- 中质量(修正20-50处): 第二轮基线×0.8
- 低质量(修正>50处): 第二轮基线×1.2
```

**泛化价值**:
- 适用于任何多轮质控任务
- 更科学的质量评估
- 异常检测更准确

**公式化表达**:
```python
def dynamic_baseline(chapter_size, chapter_type, previous_corrections):
    """动态计算质量基线"""
    base = 50  # 基础基线

    # 规模系数
    if chapter_size < 300:
        size_factor = 0.6
    elif chapter_size < 800:
        size_factor = 1.0
    else:
        size_factor = 1.3

    # 类型系数
    type_factors = {
        '本纪': 1.1,
        '世家': 1.4,
        '列传': 0.9,
        '年表': 1.3,
        '书': 1.0,
    }
    type_factor = type_factors.get(chapter_type, 1.0)

    # 前轮质量系数
    if previous_corrections < 20:
        quality_factor = 0.5
    elif previous_corrections < 50:
        quality_factor = 0.8
    else:
        quality_factor = 1.2

    return int(base * size_factor * type_factor * quality_factor)
```

#### M6. 场景化思维原则

**抽象规律**:
```
错误不是均匀分布,而是在特定场景高度聚集:

场景识别三要素:
1. 触发词: 战/击/攻(战争) | 祭/祀/礼(祭祀) | 诛/杀/灭(政治斗争)
2. 密集度: 某类实体密度>阈值(如刑法5-10个/段)
3. 遗漏模式: 特定类型遗漏率>平均(如祭祀场景身份词遗漏45%)

场景化检查:
- 定义场景清单(战争/祭祀/政治斗争/表格/奏疏等)
- 每场景明确检查重点+工具方法
- 扫描全文识别场景,执行专项检查
```

**泛化价值**:
- 适用于任何有场景分化的文本
- 聚焦高风险区域,效率提升
- 建立场景知识库

```
阶段4: 元知识(N≥30)
- 从规律中抽象方法论

- 跨项目复用
```

**泛化价值**:
- 指导知识管理策略
- 避免过早或过晚提炼
- 建立知识演进机制

**判断标准**:
```python
def should_extract_rule(occurrence_count, chapters_count):
    """判断是否提炼为规律"""
    if occurrence_count >= 3:
        return True, "重复出现≥3次,建议提炼"
    elif occurrence_count / chapters_count > 0.05:
        return True, "出现率>5%,建议提炼"
    else:
        return False, "暂不提炼,继续观察"
```

#### M8. 遗漏原因心理学模型

**抽象规律**:
```
遗漏的三大心理学原因:

原因1: 麻痹效应(Habituation)
- 高频刺激导致敏感度下降
- 对策: 分段休息,每30行暂停
- 自动化: 高频词汇用工具检查

原因2: 认知负担(Cognitive Load)
- 多任务并行导致注意力分散
- 对策: 单一任务聚焦(如只检查刑法词)
- 自动化: 分层反思,每层单一目标

原因3: 专题盲区(Domain Blindness)
- 过度关注主题词,忽略常见词
- 对策: 反向检查(列出"易遗漏清单")
- 自动化: 场景化检查,明确盲区重点
```

**泛化价值**:
- 理解人类标注局限性
- 设计更合理的反思流程
- 指导工具开发方向

**对策映射**:
| 心理原因 | 人工对策 | 工具对策 | 流程对策 |
|---------|---------|---------|---------|
| 麻痹效应 | 分段休息 | grep自动检查 | 第二轮专项扫描 |
| 认知负担 | 单任务聚焦 | 分类检查脚本 | 分层反思 |
| 专题盲区 | 易遗漏清单 | 场景化检查模板 | Step 3优先级调整 |

```
- 工具检查 → 发现80%错误 → 人工复核
```

**泛化价值**:
- 三位一体的质量保障体系
- 适用于任何大规模质控

#### M10. 反思ROI优化原则

**抽象规律**:
```
反思投入回报率(ROI)优化:

公式: ROI = (发现错误数 × 错误权重) / 投入时间

优化策略:
1. 高ROI优先:
   - 刑法词汇grep检查: 发现率95%,耗时5分钟 → ROI极高
   - 逐行人工审查: 发现率10%,耗时2小时 → ROI低

2. 边际效益递减:
   - 第一轮: 发现50处/小时
   - 第二轮: 发现30处/小时
   - 第三轮: 发现10处/小时
   - → 适时终止,避免过度反思

3. 资源分配:
   - 80%时间用于高ROI检查(工具辅助)
   - 20%时间用于低ROI深度审查(人工)
```

**泛化价值**:
- 经济学视角的质控
- 避免过度投入
- 适用于资源有限场景

## 三、Layer 3: 系统方法(System Methods)

### 3.1 流程优化方法

#### S1. Step 0增强 - 章节类型自动识别

**方法描述**:
```
Step 0: 预处理与类型识别

输入: 章节文件路径
输出: 章节类型 + 主题特征 + 检查清单

流程:
1. 读取章节标题 → 判断基本类型(本纪/世家/列传/表/书)
2. 读取前100行 → 统计关键词密度 → 判断主题特征
3. 根据类型+主题 → 加载对应检查清单
4. 计算动态质量基线

关键词密度表:
| 主题 | 关键词 | 密度阈值 |
|------|--------|---------|
| 军事 | 击/攻/破/战/败 | >20/100行 |
| 哲学 | 道/德/仁/义/法 | >15/100行 |
| 礼仪 | 祭/祀/礼/朝/享 | >10/100行 |
| 医学 | 疾/病/脉/药/诊 | >8/100行 |
```

**抽象价值**:
- 自动化类型识别,减少人工判断
- 为后续检查提供上下文
- 可扩展到其他文本类型


```python
def disambiguate_homograph(word, context, full_text):
    """同形异义词消歧四步流程"""

    # Step 1: 前文是否提及此人?
    if re.search(f'〖@[^〗]*{word}[^〗]*〗', full_text[:context.start()]):
        return {
            'decision': '人名省称',
            'reason': 'Step 1: 前文已提及此人',
            'action': f'标注为〖@{word}〗'
        }

    # Step 2: 是否有职衔/身份修饰?
    title_pattern = r'〖;[^〗]+〗〖@?' + word
    if re.search(title_pattern, context.text):
        return {
            'decision': '人名',
            'reason': 'Step 2: 有职衔修饰',
            'action': f'标注为〖@{word}〗'
        }

    # Step 3: 是否为动词短语?
    verb_phrases = {
        '释之': r'释之(不问|而去)',
        '非子': r'非子(罪|过)',
        '得意': r'(洋洋|意气)得意',
    }
    if word in verb_phrases:
        if re.search(verb_phrases[word], context.text):
            return {
                'decision': '动词短语',
                'reason': 'Step 3: 匹配到动词短语模式',
                'action': '不标注'
            }

    # Step 4: 无法确定?
    return {
        'decision': '不标注',
        'reason': 'Step 4: 无法确定,保守原则',
        'action': '保持原状'
```

```python
            )

        results.append(result)

    return results
```

#### S17. 表格体裁自动检查

```python
def check_chronicle_table(lines):
    """年表自动检查脚本"""

    table_lines = [line for line in lines if line.strip().startswith('|')]

    errors = {
        '国名列遗漏': [],
        '人名列遗漏': [],
        '谥号误标': [],
        '纪年遗漏': []
    }

    for i, line in enumerate(table_lines):
        if line.startswith('|--'):  # 跳过分隔行
            continue

        cells = [cell.strip() for cell in line.split('|')]

        # 假设表格结构: | 国名 | 侯爵 | 人名 | 纪年 | 事件 |
        if len(cells) < 6:
            continue

        # 检查国名列(第1列)
        if cells[1] and not '〖=' in cells[1]:
            errors['国名列遗漏'].append({
                'line': i,
                'content': cells[1],
                'expected': f'〖={cells[1]}〗'
            })

        # 检查人名列(第3列)
        if cells[3] and not '〖@' in cells[3]:
            errors['人名列遗漏'].append({
                'line': i,
                'content': cells[3],
                'expected': f'〖@{cells[3]}〗'
            })

        # 检查谥号误标(第3列若包含"侯/王/公"+〖;〗)
        if cells[3] and '〖;' in cells[3] and any(x in cells[3] for x in ['侯', '王', '公']):
            errors['谥号误标'].append({
                'line': i,
                'content': cells[3],
                'issue': '谥号误标为官职,应为〖@〗'
            })

        # 检查纪年列(第4列)
        if cells[4] and not '〖%' in cells[4]:
            errors['纪年遗漏'].append({
                'line': i,
                'content': cells[4],
                'expected': f'〖%{cells[4]}〗'
            })

    return errors
```

**抽象价值**:
- 结构化文本检查
- 列级别规则
- 批量处理
