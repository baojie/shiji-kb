## 一、研究理念

### 1.1 从"细读"到"远观"

**细读（Close Reading）**
- 传统人文研究方法
- 深入分析具体文本细节
- 注重语境、修辞、含义
- 适用范围：单篇或少数篇章

**远观（Distant Reading）**
- 数字人文研究方法
- 聚合大规模数据分析
- 发现宏观模式和趋势
- 适用范围：整部史记或跨章节

**结合策略**
- 远观发现模式 → 细读验证解释
- 细读提出假设 → 远观量化验证
- 宏观与微观互相补充

### 1.2 数据驱动的历史研究

**传统历史研究**
```
史料阅读 → 提出观点 → 选择性引证 → 形成论述
```

**数字人文研究**
```
数据采集 → 结构化标注 → 计算分析 → 可视化展示 →
模式发现 → 假设提出 → 统计验证 → 学术阐释
```

**优势**
- 全面性：覆盖全文，减少选择性偏差
- 系统性：统一标准，可重复验证
- 发现性：揭示隐藏模式和关联
- 客观性：数据支撑，减少主观臆断

### 1.3 跨学科融合

本研究融合以下学科方法：

| 学科 | 贡献方法 | 应用场景 |
|------|----------|----------|
| 计算语言学 | 分词、词性标注、依存分析 | 文本处理、语言特征分析 |
| 数据科学 | 统计分析、机器学习、数据挖掘 | 模式发现、预测建模 |
| 网络科学 | 图论、网络分析、社区发现 | 人物关系、社会结构 |
| 地理信息科学 | GIS、时空分析、地图可视化 | 历史地理、事件分布 |
| 知识工程 | 本体建模、知识图谱、语义推理 | 知识组织、智能问答 |
| 信息可视化 | 图表设计、交互设计、视觉编码 | 数据呈现、探索分析 |
| 历史学 | 史料考证、历史解释、批判思维 | 结果验证、学术阐释 |

## 二、数据准备与标注方法

### 2.1 文本获取与预处理

#### 2.1.1 文本来源
- 《史记》标点本（中华书局版）
- 数字化文本：130篇完整文本
- 文件组织：docs/original_text/ 目录

#### 2.1.2 预处理步骤
1. **格式统一**
   - 编码：UTF-8
   - 格式：纯文本（.txt）
   - 命名：序号_篇名.txt（如 001_五帝本纪.txt）

2. **文本清洗**
   - 去除页码、注释
   - 统一标点符号
   - 保留原始分段

3. **质量检查**
   - 字数统计验证
   - 缺字检查
   - 格式一致性检查

### 2.2 语义标注方法

#### 2.2.1 实体标注（11类）

《史记》实体分类体系：

| 序号 | 类别 | 标记 | 示例 | 说明 |
|------|------|------|------|------|
| 1 | 人名 | `@人名@` | @秦始皇@ | 历史人物姓名 |
| 2 | 地名 | `=地名=` | =咸阳= | 地理位置名称 |
| 3 | 官职 | `$官职$` | $丞相$ | 官员职位名称 |
| 4 | 时间 | `%时间%` | %元年% | 时间表达 |
| 5 | 朝代 | `&朝代&` | &秦& | 朝代/国家名 |
| 6 | 制度 | `^制度^` | ^郡县制^ | 政治经济制度 |
| 7 | 族群 | `~族群~` | ~匈奴~ | 民族/部落名称 |
| 8 | 器物 | `*器物*` | *鼎* | 器具、物品 |
| 9 | 天文 | `!天文!` | !彗星! | 天文现象、星宿 |
| 10 | 神话 | `?神话?` | ?黄帝? | 神话传说人物 |
| 11 | 动植物 | `🌿动植物🌿` | 🌿桑🌿 | 动植物名称 |

#### 2.2.2 标注原则

**一致性原则**
- 同一实体在全文中使用相同标记
- 人物别名、字号统一为最常用名

**完整性原则**
- 标注完整实体（如"秦始皇"不拆分）
- 包含修饰词（如"大将军"而非"将军"）

**准确性原则**
- 根据语境判断类别
- "秦"可能是朝代&秦&或地名=秦=
- "王"可能是官职$王$或人名@某王@

**边界情况处理**
- 复合实体：官职+人名 → $丞相$@李斯@
- 嵌套实体：地名+建筑 → =咸阳=宫
- 模糊实体：神话/历史 → 标注为主要类别

#### 2.2.3 段落编号

采用圣经式编号系统（Biblical notation）：

```
# [0] 章节名

## [1] 一级标题
### [1.1] 二级标题
#### [1.1.1] 三级标题

[1.1.1] 段落内容第一句...
[1.1.2] 段落内容第二句...
```

**编号规则**
- [0]：保留给章节标题
- [1], [2], [3]...：一级标题
- [1.1], [1.2]...：二级标题
- [1.1.1], [1.1.2]...：段落句子

**优势**
- 精确定位：可引用具体句子
- 便于索引：支持交叉引用
- 结构清晰：层级关系明确

### 2.3 标注工具与流程

#### 2.3.1 半自动标注流程

```
原始文本 → 分词标注 → 实体识别 → 人工校对 → 标注文件
```

**工具链**
1. **分词**：jieba / pkuseg
2. **NER**：基于规则 + BERT模型
3. **校对**：人工审核修正
4. **输出**：tagged.md 文件

#### 2.3.2 质量控制

**三级检查**
1. 自动检查：格式、完整性
2. 交叉验证：一致性比对
3. 专家审核：准确性判断

**标注一致性指标**
- Cohen's Kappa > 0.8
- 实体边界F1 > 0.90
- 分类准确率 > 0.95

## 四、网络分析方法

### 4.1 社会网络分析（SNA）

#### 4.1.1 网络构建

**节点（Node）**
- 人物（1820个人名）

**边（Edge）**
- 关系类型：
  1. 君臣关系

  3. 师徒关系

**节点指标**

| 指标 | 含义 | 应用 |
|------|------|------|
| Degree Centrality | 度中心性 | 识别关系最多的人物 |
| Betweenness Centrality | 中介中心性 | 识别关键桥梁人物 |

**网络指标**

| 指标 | 含义 | 应用 |
|------|------|------|
| Density | 密度 | 网络紧密程度 |

| Modularity | 模块度 | 社区结构明显程度 |

### 6.2 语义推理

#### 6.2.1 推理规则

**传递性**
```
父子关系 + 父子关系 → 祖孙关系
A父of B AND B父of C → A祖父of C
```

**继承性**
```
人物 IS-A 历史人物
秦始皇 IS-A 帝王 IS-A 人物
```

**时序推理**
```
事件A发生在前 AND 事件B发生在后
→ A可能导致B（需进一步验证）
```

#### 6.2.2 知识补全

**链接预测**
- 预测缺失关系
- 识别隐含联系

**实体对齐**
- 同名异人消歧
- 异名同人识别

## 七、统计分析方法

### 7.1 描述统计

#### 7.1.1 集中趋势

**均值（Mean）**
- 平均篇幅：4,834字/篇
- 平均句长：20-30字

**中位数（Median）**
- 篇幅中位数
- 人物出现频次中位数

**众数（Mode）**
- 最常见字数区间
- 最频繁的关系类型

#### 7.1.2 离散程度

**标准差（SD）**
- 篇幅标准差
- 词频标准差

**四分位数（Quartile）**
- Q1, Q2（中位数）, Q3
- 箱线图展示

**变异系数（CV）**
- CV = SD / Mean
- 比较不同指标的离散程度

### 7.2 相关性分析

#### 7.2.1 Pearson相关

**分析对象**
- 篇幅 vs 人物重要性
- 对话比例 vs 人物个性鲜明度
- 地名频次 vs 地理重要性

**相关系数解释**
- |r| > 0.7：强相关
- 0.4 < |r| < 0.7：中等相关
- |r| < 0.4：弱相关

#### 7.2.2 Spearman相关

**非参数方法**
- 适用于序数数据
- 不要求线性关系

### 7.3 假设检验

#### 7.3.1 t检验

**应用场景**
- 本纪 vs 列传平均句长差异
- 战国 vs 汉朝人物数量差异

**显著性水平**
- α = 0.05
- p < 0.05：拒绝原假设

#### 7.3.2 卡方检验

**应用场景**
- 不同体裁对话比例差异
- 不同朝代官职分布差异

### 7.4 回归分析

#### 7.4.1 线性回归

**模型**
```
Y = β0 + β1*X1 + β2*X2 + ... + ε
```

**应用**
- 预测篇幅基于主题
- 预测人物重要性基于多指标

#### 7.4.2 逻辑回归

**应用**
- 预测人物正负面评价
- 分类篇章体裁

### 7.5 聚类分析

#### 7.5.1 K-means

**步骤**
1. 选择K值
2. 随机初始化质心
3. 分配样本到最近质心
4. 更新质心
5. 迭代至收敛

**应用**
- 人物分群
- 篇章分类

#### 7.5.2 层次聚类

**类型**
- 凝聚式（Bottom-up）
- 分裂式（Top-down）

**树状图（Dendrogram）**
- 可视化聚类过程
- 选择最佳分割
