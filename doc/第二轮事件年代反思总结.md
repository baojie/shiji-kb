## 一、研究方法

### 1.1 总体流程

第二轮反思采用与第一轮相同的自动化Agent管线，但在SKILL文档（已含第一轮积累的25条错误模式+12条推理逻辑）和更新后的事件索引基础上运行：

1. **提示词生成**：`run_review_pipeline.py --prompt NNN` 为每章生成审查提示词，嵌入最新SKILL规则和该章事件索引
2. **Agent反思执行**：以5章为一批，同步调用Agent工具，逐章对每个事件进行年表交叉验证、reign-year换算校核、格式规范检查
3. **结果吞入**：`run_review_pipeline.py --ingest NNN reflect_NNN.json` 更新pipeline状态，提取新错误模式追加到SKILL
4. **修正应用**：`apply_reflect_fixes.py --all` 将431处修正写入130个事件索引文件

### 1.2 审查策略

第二轮的审查重心与第一轮不同：

| 维度 | 第一轮 | 第二轮 |
|------|--------|--------|
| 主要问题 | 系统性错误（单锚点塌缩、锚点计算错误、帝王混淆） | 精细调校（确定性升级、年表验证、格式修正） |
| 修正量级 | 1,010处（平均7.8/章） | 431处（平均3.3/章） |
| 典型偏差 | 数十至数百年 | 1-5年或仅格式变更 |
| 新模式发现 | 25条错误模式 | 1条新模式（#26裸年份缺少确定性标记） |

### 1.3 技术实现

- **字段兼容**：第二轮Agent输出的JSON结构有三种变体（`suggested_year`/`corrected_time`/`new`），`apply_reflect_fixes.py` 已增加自动兼容逻辑
- **批量执行**：130章分26批（每批5章同步Agent），总耗时约4小时
- **质量保障**：每章Agent独立执行，逐事件对照年表验证，输出带置信度的corrections数组

## 四-B、深度时间交叉推理案例集

以下案例展示了Agent反思中最具学术价值的推理过程——通过多源证据交叉、时间窗口约束、史料分歧判断等方法定年。

### 深度案例1：人物共处窗口约束——卫灵公闻濮水琴声（024-013）

**原标注**：`[约公元前530年]`
**修正为**：`[约公元前533年]`

**推理链**：
1. 原文明确说卫灵公"将之晋"见**晋平公**，师涓在**平公**面前演奏
2. 年表查证：卫灵公元年=前534年
3. 年表查证：晋平公卒于前532年（"在位二十六年"）
4. **时间窗口约束**：两人共处时期仅 **前534—前532年**（3年窗口）
5. 原标注前530年时晋平公已去世2年，与原文"平公"出场**矛盾**
6. 结论：取窗口中间值约前533年

**推理特征**：双人物在位期交集 → 约束为极窄时间窗口 → 排除矛盾标注

### 深度案例2：《史记》与年表系统性纪年分歧——齐威王选贤治国（046-010）

**原标注**：`（公元前370年）`
**修正为**：`（公元前348年）`，偏差22年

**推理链**：
1. 原文纪年：%九年%
2. 按《史记》自身纪年：威王元年=前378年，九年=前370年
3. 按年表：威王元年=**前356年**，九年=356-(9-1)=**前348年**
4. 交叉验证年表一致性：
   - 前353年条注"齐威王四年"（桂陵之战）→ 元年=353+3=前356年 ✓
   - 前350年条注"齐威王七年" → 元年=350+6=前356年 ✓
   - 前334年条注"齐威王二十三年" → 元年=334+22=前356年 ✓
5. 结论：《史记》与竹书纪年/年表对田齐威王有22年系统性偏差，以年表为准

**推理特征**：两套权威史料体系的系统性分歧识别，3个独立年表条目交叉印证同一基准年

### 深度案例3：出土文献推翻传世文献系年——苏秦说六国（069章）

**系列修正**：069-001~069-014，苏秦全传9个事件系统性修正

**推理链**：
1. 《史记》将苏秦活动定在前334—前333年（与张仪同时代）
2. 年表前311年条记苏秦赴燕昭王处
3. **关键证据**：1973年马王堆帛书《战国纵横家书》出土后，学界认定苏秦活跃于前311—前284年
4. **矛盾检验**：
   - 069-008"说韩宣王"：韩宣惠王元年=前332年，前333年韩宣惠王**尚未即位**
   - 069-009"说魏襄王"：魏襄王元年=前318年，前333年魏襄王**尚未即位**
   - 069-010"说齐宣王"：齐宣王元年=前319年，前333年齐宣王**尚未即位**
5. 三位君主的在位期均与《史记》原标注年份产生**系统性矛盾**
6. 结论：按年表+出土文献将整传事件下移约25年

**推理特征**：出土文献→推翻传世文献→多位君主在位期矛盾检验→系统性年代修正

### 深度案例4：前后君主纪年衔接处的归属判断——田常弑简公（032-032）

**原标注**：`（公元前485年）`
**修正为**：`（公元前481年）`，偏差4年

**推理链**：
1. 原文纪年标记：%四年%
2. 年表前481年条明确记载："齐简公四年，简公任用阚止...田常杀阚止及简公"
3. **错误归因分析**：前485年是**悼公四年**（田乞杀悼公、立简公之年），并非简公四年
4. 换算验证：简公元年=前484年，简公四年=484-(4-1)=**前481年** ✓
5. 错误原因：将前任君主（悼公）被杀之年误作后任君主（简公）的reign-year基准

**推理特征**：弑君-立新君叙事中"N年"归属的精确判断，前后两任君主纪年边界辨析

### 深度案例5：原文"明年"链的歧义消解——始郊雍见五畤（012-006）

**第一轮标注**：`（公元前133年）`
**第二轮修正为**：`（公元前134年）`

**推理链**：
1. 纪年标记：%元光元年%，年表确认元光元年=前134年
2. 第一轮推理：原文"後六年窦太后崩(前135)。其明年征文学(前134)。明年上初至雍"，将两个"明年"理解为**链式递推**：前135→前134→前133年
3. 第二轮反思：
   - 发现纪年标记%元光元年%=前134年，与链式推理的前133年**矛盾**
   - 重新分析"明年"语义：两个"明年"可理解为**并列叙述**（均指前135年之明年=前134年），非链式递推
   - 交叉验证：012-011"祠泰一"有独立标记%元光二年%=前133年，若012-006=前134年则构成 **前134→前133年的合理时间序列**
4. 结论：恢复为前134年，与纪年标记一致

**推理特征**：两轮反思的自我修正、古汉语"明年"语义歧义分析、纪年标记与年值一致性校核、相邻事件时间序列验证

### 深度案例6：《史记》与《左传》的著名年代分歧——下宫之难（043-006）

**原标注**：`（公元前574年）`
**修正为**：`[约公元前583年]`

**推理链**：
1. 赵世家原文："景公三年"，换算：景公元年=前599年，三年=**前597年**
2. 年表前583年条记载："讨赵氏，杀赵同、赵括"，即景公**十七年**
3. **史料分歧**：赵世家"景公三年" vs 年表/《左传》"景公十七年"，相差14年
4. **内部一致性检验**：
   - 043-008赵氏复位已标前583年（景公十七年）
   - 若下宫之难也在前583年，则与原文"居十五年"（程婴匿孤十五年）矛盾
   - 若下宫之难在前597年，则"居十五年"后=前582年，与赵氏复位前583年基本吻合
5. 结论：采用年表前583年，但因《史记》自身记载存在分歧，使用`[约]`推断标记而非精确标记

**推理特征**：《史记》vs《左传》史料分歧判断、内部时间跨度一致性验证（"居十五年"）、标注精确度等级的审慎选择

### 深度案例7：子产卒年的纪年归属辨析（042-022）

**原标注**：`[约公元前496年]`（声公五年换算）
**修正为**：`（公元前522年）`，偏差26年

**推理链**：
1. 原文纪年标记含"声公五年"
2. 换算：声公元年=前500年，声公五年=500-(5-1)=**前496年**
3. **矛盾检验**：子产活跃在简公时期（前565—前530），前496年时子产若在世已超过100岁，**不可能**
4. 年表前522年条直接记载"子产死"
5. 正确归属：子产卒于**郑定公八年**（定公元年=前529年，八年=前522年）
6. "声公五年"为原文记载有误或指另一事件

## 五、推理规则总结

### 5.1 第二轮确认有效的核心规则

以下规则在第二轮中继续发挥作用，按使用频率排序：

1. **年表直接查证**（~114次）：仍是最可靠的定年方法。年表有载即为精确纪年。
2. **确定性回查升级**（~153次）：标注后回查年表，凡年表有载的从`[约]`升级为`（）`。这是第二轮最频繁使用的规则。
3. **Reign-year精确换算**（~9次）：`公元前年份 = 基准元年 - (N - 1)`，注意前元/中元/后元区分。
4. **概览-详情一致性校核**（~25次）：概览表时间列与详情`**时间**`行、`**年代推断**`行必须同步。
5. **跨章节交叉定位**：同一事件在多章出现时，取精确度最高的标注。

### 5.2 第二轮新增推理规则

**规则13（新增）：裸年份格式回查**

当时间字段为不带`（）`或`[]`标记的裸"公元前XXX年"时，需判断确定性等级并添加正确标记。实践中这类裸年份几乎都可查到年表依据，应统一标注为`（公元前XXX年）`。

## 八、方法论贡献

### 8.1 Agent反思管线的可复用性

本项目建立的Agent反思管线（SKILL文档 + 提示词生成 + Agent执行 + 结果吞入 + 修正应用）具有以下可推广特征：

1. **自增长SKILL**：每轮反思自动发现新错误模式并追加到SKILL，下一轮所有章节均受益
2. **全自动批量执行**：130章无需人工干预，脚本自动管理提示词生成、状态跟踪、结果吞入
3. **收敛性验证**：第二轮修正量为第一轮的42.7%，证明该流程具有自然收敛特性
4. **跨领域适用**：同一管线架构可用于其他古籍的年代标注审查（如《汉书》、《资治通鉴》）

### 8.2 SKILL文档作为"可执行规范"

26条错误模式 + 12条推理逻辑构成了一份**古籍事件年代标注的可执行规范**。与传统的文档规范不同，SKILL文档：
- 附带真实案例和识别方法
- 可直接嵌入Agent提示词作为审查规则
- 经130章 x 2轮 = 260次实战验证
