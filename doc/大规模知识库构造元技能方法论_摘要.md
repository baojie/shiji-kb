# 《大规模知识库构造元技能方法论——以史记知识库为例》摘要

## 主持稿/新闻稿版本（280字）

本文系统总结了用AI构建大规模知识图谱的核心方法论，以《史记》130篇、57.7万字的知识库项目为实践案例。作者将复杂的知识工程过程提炼为可复用的14个"元技能"——从OTF（On-The-Fly，边做边总结）、JIT（Just-In-Time，即时响应）的快速启动方法，到Agent驱动的质量反思，再到跨项目的技能迁移，形成了一套完整的通用工程流程。

全文分为三大部分：元元技能（好东西都是总结出来的）奠定方法论基础，核心方法论（反思、迭代、柳叶刀方法等）解决MVP到生产的工程化问题，工序相关技能（标注体系、提示词工程、质量控制等）提供具体操作指南。这套方法论高度通用，不仅适用于历史文献（《汉书》《资治通鉴》），更可迁移到法律文书、医学文献、企业档案等任何需要结构化知识提取的领域。

书中所有方法均在GitHub开源（github.com/baojie/shiji-kb），包含3,198个事件、15,190个实体的完整知识图谱，为AI驱动的知识工程提供了可落地的实践范式。

## 精简版（230字）

本书总结了用AI构建大规模知识图谱的核心方法论，以57.7万字《史记》项目为实践案例，提炼出14个可复用的"元技能"。从OTF+JIT的快速启动，到Agent驱动的质量反思，再到跨项目的技能迁移，形成完整的通用工程流程。

全书分为三层：**元元技能**"好东西都是总结出来的"奠定方法论基础；**核心方法论**（反思、迭代、柳叶刀方法等）解决MVP到生产的工程化问题；**工序技能**（标注体系、提示词工程、质控等）提供具体操作指南。

这套方法论以《史记》为例，已成功提取3,198个事件、15,190个实体，形成完整知识图谱。更重要的是，这些元技能高度通用，可迁移到法律文书、医学文献、企业档案等任何需要结构化知识提取的领域。全部方法在GitHub开源，为AI知识工程提供可落地的实践范式。
