## 一、问题背景

### 1.1 问题发现
用户在查看121章《儒林列传》HTML渲染时发现`〖#士大夫〗`等标注错误地以时间样式（青色+点线）显示。

### 1.2 根本原因
**符号语义漂移**：`#`符号在项目演进过程中发生语义变化
- **早期**：`#`用于标注称谓/身份（如 `〖#天子〗`、`〖#臣〗`）
- **后期**：`#`重新定义为时间标注（如 `〖#春秋〗`、`〖#战国〗`）
- **结果**：大量历史身份/官职标注被错误保留为`#`，导致渲染错误

### 1.3 问题规模
系统性扫描130章发现：
- **影响章节**: 129/130章（仅1章无错误）
- **总错误数**: **8,774** 处
  - 身份/官职错误: 8,249 处
  - 人名嵌套错误: 525 处

### 2.2 人名嵌套错误
格式：`〖@人名前缀〖#/身份〗人名后缀〗`

**错误示例**：
```
〖@澹台〖#子〗羽〗 → 应为 〖@澹台子羽〗
〖@仲〖#孙〗何忌〗 → 应为 〖@仲孙何忌〗
```

**总计**: 525处嵌套错误

### 2.3 复合词拆分错误
**错误示例**：
```
〖#弟〗〖#子〗 → 应为 〖_弟子〗
〖#夫〗〖#子〗 → 应为 〖_夫子〗
〖#士〗〖#大夫〗 → 应为 〖_士大夫〗
```

## 三、修复策略

### 3.1 修复优先级
```python
# 优先级1: 复合词（避免干扰后续处理）
〖#弟〗〖#子〗 → 〖_弟子〗
〖#夫〗〖#子〗 → 〖_夫子〗
〖#士〗〖#大夫〗 → 〖_士大夫〗

# 优先级2: 身份称谓 (#→_)
天子、诸侯、士、臣、民、夫、子、孙、弟、父、
匹夫、缙绅、庶民、万民、百姓、庶人

# 优先级3: 官职称谓 (#→;)
君、卿相、公卿、大夫、太后、太子、陛下、
王、帝、上、主、天王

# 优先级4: 人名嵌套展平
〖@xxx〖_/# /;/^yyy〗zzz〗 → 〖@xxxyyzzz〗
```

### 3.2 符号映射规则
| 错误符号 | 实体类型 | 正确符号 | 示例 |
|---------|---------|---------|------|
| `#` | 身份称谓 | `_` | `〖#臣〗` → `〖_臣〗` |
| `#` | 官职称谓 | `;` | `〖#君〗` → `〖;君〗` |
| 嵌套 | 人名内嵌 | 展平 | `〖@x〖#y〗z〗` → `〖@xyz〗` |
