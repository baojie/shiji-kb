# SKILL: 古籍文本语义化 — 实体标注、对话格式化与语义增强

> 从《史记》130篇文本语义化实践中提炼，适用于史传类古籍的内容标注流程。
>
> **注意**：篇章拆分、段落编号（Purple Numbers）和小节划分属于结构化流程，
> 详见 [SKILL_古籍章节切分与编号.md](SKILL_古籍章节切分与编号.md)。
> 本文档聚焦**内容标注**部分。

---

## 一、问题定义

古籍文本语义化分两个独立阶段：

```
archive/chapter_numbered/NNN_*.txt
  │
  ▼（结构化 → 参见 SKILL_古籍章节切分与编号）
  ├── 篇章拆分
  ├── 段落编号（Purple Numbers）
  └── 小节划分（H2/H3 标题）
  │
  ▼（内容标注 ← 本文档）
  ├── 11类实体 Token 标注
  ├── 对话格式化
  ├── 语义区块（→ SKILL_区块与韵文处理）
  └── 语义增强（并列结构等）
  │
  ▼
chapter_md/NNN_*.tagged.md
```

**本文档范围**：从已有结构（H1/H2/H3 标题 + 段落编号）的文本，添加内容层语义标注。

## 二、实体标注 Token

### 2.1 设计原则

- **最小侵入**：保留原文字符，用短符号包裹，便于人工阅读与编辑
- **语义明确**：每种实体对应一个 CSS 类，样式由 `css/shiji-styles.css` 控制
- **内容与样式分离**：标注文件只记录语义，渲染器负责转换为 HTML

### 2.2 Token 与 CSS 映射

| 类型 | Token | CSS 类 | 示例 |
|------|-------|--------|------|
| 人名 | `@人名@` | `.person` | `@秦始皇@` |
| 地名 | `=地名=` | `.place` | `=咸阳=` |
| 官职 | `〖;官职〗` | `.official` | `〖;丞相〗` |
| 时间/纪年 | `%时间%` | `.time` | `%元年%` |
| 朝代/国号 | `〖&朝代〗` | `.dynasty` | `〖&汉〗` |
| 封国 | `〖$封国〗` | `.feudal-state` | `〖$齐〗` |
| 制度/典章 | `^制度^` | `.institution` | `^郡县制^` |
| 族群/部落 | `~族群~` | `.tribe` | `~匈奴~` |
| 器物 | `*器物*` | `.artifact` | `*鼎*` |
| 天文/历法 | `!天文!` | `.astronomy` | `!彗星!` |
| 传说/神话 | `?神话?` | `.mythical` | `?黄帝?` |
| 动植物 | `〘名称〙` | `.flora-fauna` | `〘龙〙` |

引号内容（对话）由渲染器自动识别，应用 `.quoted` 样式——无需手动标记。

示例行：
```markdown
[1] @黄帝@居=轩辕之丘=，娶於&西陵&之女。$天子$受命，%元年%始。
```

### 2.3 标记处理顺序

渲染器内部处理顺序（`ENTITY_PATTERNS` 列表）：
1. `**`（加粗）必须在 `*`（器物 token）之前处理，避免冲突
2. 段落编号在所有实体标记之后处理
3. 正则表达式排除 `<>` 字符：`=([^=<>]+)=`（防止匹配 HTML 属性中的 `=`）

## 三、推荐标注方法：大模型 NER

### 3.1 为什么不用关键词匹配

基于关键词列表的字符串匹配无法处理：
- **多义词**："和"既是人名（`@羲@、@和@`）也是动词（`律和声`，不标注）
- **复合实体**："夏后帝启"、"西伯昌"需要理解上下文才能确定边界
- **词性判断**："蚕食"中"蚕"是动词不标注，但 `〘蚕〙` 作名词应标注
- **比喻用法**："龙颜"（帝王容貌）不标注，但 `〘龙〙`（神兽）应标注

### 3.2 大模型工作流

1. 大模型阅读完整章节，建立语义理解和人物关系
2. 识别各类实体，根据上下文判断词性，**只标记名词性实体**
3. 生成带标记的 `*.tagged.md` 文件
4. 人工审核关键段落和边界情况（重点：多义词、动植物、复合人名）

## 四、边界情况

### 4.1 动植物标记

| 情况 | 示例 | 处理 |
|------|------|------|
| 名词（应标注） | `〘柳〙`、`〘龙〙`（神兽）、`〘稻〙` | 标注 |
| 动词用法（不标注） | `土既蚕`（养蚕）、`蚕食`（侵占） | 不标注 |
| 比喻用法（不标注） | `龙颜`（帝王容貌）、`龙飞凤舞`（成语） | 不标注 |
| 并列名词 | `〘草〙〘木〙渐包` | 各自标注 |
| 神兽 | 龙、凤、麒麟等 | 标注 |

### 4.2 氏族 vs 朝代

- **方案A（兼容）**：渲染后处理把确认的氏族名替换为 `.dynasty`
- **方案B（语义化）**：新增 `.clan` 类，氏族名单外置为 `config/clans.json`（推荐，尚未实施）

### 4.3 多义词

高频多义词不自动标注，需依上下文判断：
- `@羲@、@和@`（人名，标注） vs `律和声`（动词，不标注）
- 建议：自动标注脚本中排除已知高频多义词，改为人工或大模型辅助

## 五、对话格式化

### 5.1 规则

将包含多个"X曰"的长段落拆分为独立对话行：

1. 识别 `X曰：` 模式（X 为人名或官职标注）
2. 在每个"X曰"之前断行（第一个除外）
3. 保留引号内容完整性（不在引号中间断开）
4. 拆分后每行转为列表项（`- ` 开头）
5. 非对话的叙述内容单独成行

### 5.2 示例

原文（一行包含多人对话）：
```markdown
[1.1] 尧求能治水者，群臣四岳皆曰鲧可。尧曰："鲧为人负命毁族，不可。"四岳曰："等之未有贤於鲧者，原帝试之。"於是尧听四岳，用鲧治水。
```

拆分后：
```markdown
- [1.1] 尧求能治水者，群臣四岳皆曰鲧可。
- 尧曰："鲧为人负命毁族，不可。"
- 四岳曰："等之未有贤於鲧者，原帝试之。"
- 於是尧听四岳，用鲧治水。
```

### 5.3 对话引号渲染

渲染器自动识别中文引号（`""` `''` `「」` `『』`）内的内容，应用 `.quoted` 样式：
- 斜体显示，极淡褐色背景，褐色虚线下划线
- 引号内的实体标记保持各自样式不受影响

## 六、段落长度控制

| 内容类型 | 建议上限 | 处理方式 |
|----------|----------|----------|
| 叙事段落 | 150 字符 | 按语义单元拆分，用子编号 `[N.1]` `[N.2]` |
| 论述性内容（谏言、策论） | 可较长 | 用 `:::` 区块标记，保持完整语义单元 |

拆分时保持语义完整性，不在句中断开；子编号规则见 [SKILL_古籍章节切分与编号.md](SKILL_古籍章节切分与编号.md)。

## 七、语义区块与韵文

语义区块（`::: 标签` / `:::`）用于标注论述性内容（谏言、策论、制度）和篇末评论（太史公曰、赞）。

区块相关的完整内容已独立为专题文档，包括：
- Fenced Div 语法与所有标签类型
- 渲染规则（标签为 tooltip，非标题）
- 区块不跨节规则
- 太史公曰/赞的标注模式
- 韵文检测算法与格式修复脚本

**详见 [SKILL_区块与韵文处理.md](SKILL_区块与韵文处理.md)**。

## 八、语义增强

### 8.1 并列结构

古汉语列举性内容用 Markdown 列表提升结构化程度（血缘/世系、官制/职责、地理/方位、历法/节气）：

```markdown
- 其一曰**玄嚣**，是为青阳；
- 其二曰**昌意**，降居若水。
```

## 十、关键经验

1. **大模型 NER 优于关键词匹配** — 古文多义词密集，只有语义理解才能正确判断词性
2. **对话拆分大幅提升可读性** — 史记多人对话段落，拆分后结构清晰，利于后续事件提取
3. **保守标注优于过度标注** — 有疑问的词宁可不标，避免误标动词和成语
4. **先结构后标注** — 先完成段落编号再做实体标注；若反序操作，每次调整编号都需重新关联实体引用
