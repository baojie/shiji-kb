# 时间投入统计 - 快速入门

## 概述

基于 Claude Code 对话记录推断你在项目上的实际时间投入。

**核心原理**：
- ✅ **用户频繁发送指令** → 你在主动工作
- ❌ **长时间无用户输入** → Claude 自动执行，你可能在做其他事

## 快速开始

### 1. 首次配置

```bash
# 如果尚未配置，创建配置文件
cp .claude_cost_config.example.json .claude_cost_config.json

# 编辑配置文件，添加项目路径
nano .claude_cost_config.json
```

### 2. 查看时间统计

```bash
# 查看所有时间的统计
python scripts/analyze_time_investment.py

# 生成本周报告
python scripts/generate_time_report.py --period weekly

# 生成本月报告
python scripts/generate_time_report.py --period monthly

# 生成指定时间段报告
python scripts/generate_time_report.py --start 2026-03-01 --end 2026-03-31
```

### 3. 报告保存位置

默认报告保存在 `logs/time_reports/` 目录：

```
logs/time_reports/
├── weekly_report_202612.md      # 周报
├── monthly_report_202603.md     # 月报
└── archive/                     # 历史报告归档
    └── 2026/
        ├── 03/
        └── 04/
```

## 核心指标说明

### 会话识别

相邻用户消息间隔 > 30分钟 → 视为新会话

```
用户消息:  M1 ---------- M2 --- M3 -------------- M4 ------ M5
间隔:          35分钟        8分钟      40分钟         6分钟

会话划分:  [--会话1--]   [--------会话2--------]   [--会话3--]
```

### 有效工作时间

基于用户消息时间戳计算：
- 每条用户消息前后有 ±5分钟缓冲时间
- 两条用户消息间隔 > 10分钟，只计10分钟（Claude自动执行的时间不算）

### 主动参与率

```
主动参与率 = 有效工作时间 / 会话总时长 × 100%
```

**解读**：
- **> 70%**：高度交互模式（频繁指导）
- **30-70%**：混合模式（指导+自动化）
- **< 30%**：自动化模式（主要是Claude自动执行）

```bash

  --start 2026-03-01 \
  --end 2026-03-31 \
  --output logs/time_reports/

python scripts/generate_cost_report.py \
  --start 2026-03-01 \
  --end 2026-03-31

# - 时间投入：217小时
# - 成本投入：$45.32
# - 效率：每小时成本 $0.21
```

### 场景3：工作模式优化

```bash
# 查看整体统计

# 分析参与率：
```

## 参数调优

如果觉得统计不准确，可以调整参数（编辑脚本开头的配置）：

```python
# scripts/analyze_time_investment.py

SESSION_TIMEOUT = 30      # 会话超时（分钟）
                         # 建议：高强度项目 20，轻度维护 60

MAX_INTERVAL_COUNT = 10   # 最大间隔计时（分钟）
                         # 建议：快速交互 5，长任务 15

MESSAGE_BUFFER = 5        # 消息缓冲时间（分钟）
                         # 建议：熟练用户 3，学习探索 8
```

## 常见问题

### Q: 有效时间为什么比总时长少很多？

A: 这是正常的！有效时间只计算你主动参与的时段，不包括 Claude 长时间自动执行的时间。低参与率说明你在有效利用 AI 自动化。

### Q: 参与率超过100%怎么办？

A: 偶尔出现是正常的（多个会话的缓冲时间重叠导致）。如果经常出现，可以减小 `MESSAGE_BUFFER` 参数。

### Q: 如何统计多台机器的总时间？

A: 需要合并多台机器的对话记录。参考 SKILL_10h 文档的"Q4: 如何统计多台机器的总时间？"章节。

### Q: 数据准确吗？

A: 这是基于对话模式的**推断**，不是精确追踪。适合用于：
- ✅ 了解大致的时间投入
- ✅ 对比不同时期的工作量
- ✅ 评估工作模式和效率
- ❌ 不适合精确计费或考勤
