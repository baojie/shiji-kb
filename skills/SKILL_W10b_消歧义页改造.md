# SKILL W10b: 消歧义页改造

> "消歧义不是删除内容，是给内容找到精确的家。"

---

## 一、何时执行

| 触发场景 | 动作 |
|---|---|
| 某词在《史记》中指代两个或以上完全不同的事物 | 改造为消歧义页 |
| 现有页面 type 为 place/concept/person，但页面名是泛称 | 评估是否需要改造 |
| 搜索该词时，用户可能想找到不同的目标页 | 触发 H16 任务 |
| 已存在 `X（限定词）` 格式的多个相关页面 | 说明歧义已存在，整理入消歧义页 |

**不需要改造的情形：**
- 该词在《史记》中只有一个主要含义（偶尔借代算同义）
- 歧义程度很低（如专名偶尔用作通名）

---

## 二、核心原则

### 2.1 内容必须先迁移，再改原页

```
❌ 错误流程：直接把原页内容改成消歧义格式 → 原内容消失
✅ 正确流程：
   Step 1: 建立 "X（限定词）" 新页，把原内容移入
   Step 2: 原页改为消歧义页，列出所有含义
```

**血泪教训**：直接在原页写消歧义格式，会导致大量引文、历史内容永久丢失。

### 2.2 新页命名规范

| 原页 | 主含义类型 | 新页名示例 |
|---|---|---|
| 京 | 郑国地名 | 京（郑地） |
| 梁 | 已是消歧义 | 不需要 |
| 商 | 商朝（朝代） | 保留原页，只加 hatnote |
| 平原 | 多义 | 平原（地名）、平原（平原君） |

命名原则：`X（限定词）`，限定词优先用**时代+类型**，如：
- `X（地名）`、`X（郑地）`、`X（汉代）`
- `X（人物）`、`X（官职）`、`X（概念）`

### 2.3 消歧义页的内容规范

消歧义页**只做导航**，不做内容详解：
- 每个义项一行，格式：`N. **[[目标页]]**（简短说明）`
- 如果目标页不存在，先建 stub 或写红链，不要把内容堆在消歧义页里
- 消歧义页本身**不需要** `## 史记引文` 节（引文属于具体含义的页面）

---

## 三、操作步骤

```
分析歧义 → 命名新页 → 建新页（迁内容）→ 建其他缺失页 → 改原页为消歧义 → 更新反向链接
```

### Step 1：分析该词的所有含义

```bash
# 看现有页面引文，判断引文分属哪些不同含义
cat "wiki/public/pages/X.md"

# 看是否已有相关页面
ls wiki/public/pages/ | grep "^X"

# 看史记引文中出现的上下文
grep -A2 "X" wiki/public/pages/X.md
```

列出含义清单：
```
含义1: X作为...（出现在哪些章节引文）→ 目标页：[[XX]]（已有/需建）
含义2: X作为...（出现在哪些章节引文）→ 目标页：[[XY]]（已有/需建）
含义3: ...
```

### Step 2：确定原页主含义 → 建新内容页

原页的现有内容属于哪个含义，就建对应的新页：

```bash
# 1. 读原页内容存入临时文件
cat "wiki/public/pages/X.md" > /tmp/X_original.md

# 2. 修改新页的 frontmatter（id、label 改为新页名，type 保留）
# 3. 用 edit_page.py 建立新页
python3 wiki/scripts/butler/edit_page.py "X（限定词）" /tmp/X_new.md \
    --summary "w10b: 从X迁移内容，建立X（限定词）独立页" \
    --author "butler"
```

**新页 frontmatter 修改要点**：
- `id:` 改为新页名（如 `京（郑地）`）
- `label:` 改为新页名
- `type:` 保持原类型（place/person/concept）
- `aliases:` 可保留原页别名
- `tags:` 可保留

### Step 3：为其他含义建 stub（如有缺失）

若其他含义已有独立页面（如 `京师`、`京索`）→ 直接在消歧义里链接。
若没有独立页面 → 按需建 stub，或先在消歧义页写红链，后续 H5 扩展。

### Step 3.5：验证引文迁移完整性（必须通过才能执行 Step 4）

**⛔ 禁止跳过此检查**：在将原页改为消歧义格式之前，必须逐条核查原页 `## 史记引文` 节中的每条引文：

```bash
# 查看原页所有引文（含义分配表）
grep -A200 "## 史记引文" /tmp/X_original.md
```

对每条引文，确认它已出现在某个子页面（含义1页、含义2页等）中：

| 引文 | 已迁移至 | 确认方式 |
|---|---|---|
| 引文1 | [[X（限定词1）]] | `grep -l "引文关键词" wiki/public/pages/X（限定词1）.md` |
| 引文2 | [[X（限定词2）]] | 同上 |

**若某条引文尚未归属**：必须先将其加入对应子页面，再执行 Step 4。不得遗漏任何引文。

### Step 4：将原页改为消歧义页

```markdown
---
id: X
type: disambiguation
label: X
aliases: []
tags: []
---

# X（消歧义）

**X** 在《史记》中指代多个不同概念：

1. **[[X（限定词1）]]**：简短说明（一句话）。见[[章节]]。
2. **[[X（限定词2）]]**：简短说明。见[[章节]]。
3. **[[已有相关页]]**：简短说明。

---

## 相关页面

- [[X（限定词1）]]
- [[X（限定词2）]]
- [[已有相关页]]
```

```bash
python3 wiki/scripts/butler/edit_page.py "X" /tmp/X_disambig.md \
    --summary "w10b: X改造为消歧义页，原内容已迁至X（限定词）" \
    --author "butler"
```

### Step 5：更新反向链接

```bash
# 找所有链接到原页的页面
grep -rl "\[\[X\]\]\|\[\[X|" wiki/public/pages/ | head -20

# 判断每处链接的语义：
# - 如果那处链接明显指向某一具体含义 → 改为指向对应的具体页面
# - 如果无法确定 → 保持 [[X]]（消歧义页会导航）
```

---

## 四、引文的归属处理

原页中的 `## 史记引文` 节，按每条引文所属含义分配：

| 引文上下文判断 | 归属到 |
|---|---|
| 明确涉及含义1的地名/人物/概念 | 迁移到新页（含义1页面） |
| 明确涉及含义2 | 留在含义2页面（如已有），或建新页 |
| 模糊/难判断 | 保留在消歧义页的 `## 引文参考` 节（简短） |

消歧义页本身可保留一个**极简引文参考**节，仅列出章节分布，不展开原文。

---

## 五、消歧义页字段规范

```yaml
---
id: X                    # 原词，不加括号
type: disambiguation     # 固定值
label: X
aliases: []              # 通常为空，歧义词别名意义不大
tags: []                 # 通常为空
---
```

**禁止在消歧义页 frontmatter 中出现**：
- `coords:` / `coords_name:` / `coords_source:` （地理坐标归具体地名页）
- `auto_generated: true`（消歧义页是手工构建的）
- `source_table:`（不适用）

## 六、实例：京（2026-04-25）

### 背景

`京` 原为 `type: place` 页，记录郑国地名。但 `京` 在《史记》中有多个含义：

| 含义 | 说明 | 相关页面 |
|---|---|---|
| 京（郑地） | 郑庄公封弟段于京；申不害故里 | 需新建 |
| 京师 | 都城通称 | [[京师]]（已有） |
| 京索 | 楚汉战争战场区域（京、索之间） | [[京索]]（已有） |
| 京兆 | 汉代行政区划 | [[京兆]]（已有） |

### 执行记录

1. **建 `京（郑地）`**：迁移原 `京.md` 的 place 内容（坐标、引文等）
2. **改 `京`**：改为 `type: disambiguation`，列出4个含义
3. **引文归属**：
   - 042_郑世家、063_老子韩非列传引文 → `京（郑地）`
   - 007/008/092 中"京、索间"引文 → `京索`（已有页面）
4. **反向链接**：检查 `[[京]]` 的引用，按语义更新

## 七、常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 直接把原页内容替换为消歧义格式 | 原内容永久丢失 | 先建新页迁移内容 |
| 跳过 Step 3.5 引文迁移检查直接改原页 | 引文段落永久消失 | 必须逐条核查引文已归属子页面后才能改原页 |
| 消歧义页堆放大量引文 | 消歧义页变成内容页，失去导航功能 | 引文放到具体含义页 |
| 不更新反向链接 | 旧链接语义模糊或错误 | Step 5 必须执行 |
| 新页命名冲突 | 覆盖已有页 | 建页前先 `ls` 确认 |
| 消歧义页保留坐标等地理字段 | 字段语义混乱 | 字段归具体地名页 |
