# SKILL W10g: 随机页面优化（H6）

> "随机抽样是发现系统性问题的最好方式。每次抽5页，不修复，只诊断，把问题归类。"

---

## 一、何时执行

| 触发场景 | 说明 |
|---|---|
| 质量迭代主循环中当前轮无 P0/P1 任务 | 空闲时执行 |
| 需要了解 wiki 整体质量现状 | 随时可执行 |
| 每 10 轮迭代周期的最后1-2轮 | 常规抽样 |

**注意**：H6 是**诊断工序**，不是修复工序。本轮发现的问题加入队列，由后续轮次处理。

---

## 二、抽样方法

```bash
# 随机抽取 5 个页面
ls wiki/public/pages/*.md | shuf | head -5

# 若没有 shuf 命令（macOS）
ls wiki/public/pages/*.md | sort -R | head -5
```

---

## 三、诊断流程

对每个抽到的页面，快速浏览并逐项检查：

### 诊断清单

```
□ 1. 有无 infobox / 基本 frontmatter（id、type、label）
□ 2. 正文中人名/地名是否有 [[wikilink]]（无链→H2）
□ 3. tags 字段是否为空或 < 2 个（→H8）
□ 4. 有无史记引文节，引文是否有 PN 引注（无引注→H4）
□ 5. 有无指向不存在页面的红链（→H5）
□ 6. 是否有残留标注符号如 〖◆〗 ⟦⟧（→H15）
□ 7. 是否是 stub 但内容很少（→H18）
□ 8. 是否缺少与其他页面的交叉链接
```

### 问题分诊表

| 发现的问题 | 归入队列 | 优先级 |
|---|---|---|
| 人名/地名未链接 | H2（W10c） | P1 |
| tags 为空 | H8（W10i） | P2 |
| 引文无 PN 引注 | H4（W10e） | P1 |
| 有红链目标页不存在 | H5（W10f） | P1 |
| 残留标注符号 | H15（W10p） | P1 |
| stub 长期未更新 | H18（W10r） | P2 |
| 内容重复另一页 | H1（W10a） | P0 |

---

## 四、写入队列

每次随机抽样后，最多生成 **3 个新队列条目**（不是 5 页 × 所有问题，要择优）：

```bash
# 在 housekeeping_queue.md 中追加
# 格式示例：
# - [ ] H2 | P1 | [[页面名]] | 正文有5处人名未链（张仪、苏秦、魏王、范雎、昭王）
# - [ ] H8 | P2 | [[页面名]] | tags 为空
# - [ ] H5 | P1 | 目标词 | 在3页中引用但无对应页（出现在页面A、B、C）
```

**选择策略**：优先写入优先级高（P0/P1）的问题；P2 问题积累到 3 条以上再写入。

---

## 五、成功标准 / 完成条件

- [ ] 抽取了 5 个随机页面
- [ ] 每页完成诊断清单检查
- [ ] 新增队列条目 ≤ 3 条
- [ ] 本轮未直接修改任何页面
