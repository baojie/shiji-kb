# SKILL 03e1: 跨类型迁移

> 按类型反思的两种场景之一（另一：[SKILL_03e2 按类型反思纠错](SKILL_03e2_按类型反思纠错.md)）。
> 两者总入口：[SKILL_03e 按类型反思](../SKILL_03e_按类型反思.md)。

---

## 一、适用场景

**问题形式**：某类实体（源类型）整体搬到另一类（目标类型）。

**典型例**：
- `〖^制度〗` 中的 礼/宗庙/社稷/封禅 → `〖:礼仪〗`
- `〖=地名〗` 中的 秦/汉/齐（指政治实体时）→ `〖◆邦国〗`
- `〖;官职〗` 中的 天子/太后/皇后 → `〖#身份〗`（已完成，v2.3）

**改动对象**：`chapter_md/*.tagged.md` 中的类型标记。

**与按类型反思纠错（03e2）的区别**：
- 本 skill 改的是**源头标注**（标错了类）
- 03e2 改的是**分类结果**（标对了但需要细分子类）
- 两者遇到同一实体时，**先做本 skill（源头清理）再做 03e2（细分）**

## 二、即用提示词

```
你是《史记》实体标注质量审查员。执行一轮按类型反思（跨类型迁移）：

- 源类型：SOURCE_TYPE（如 〖^制度〗）
- 目标类型：TARGET_TYPE（如 〖:礼仪〗）
- 迁移描述：DESCRIPTION（如"礼制词汇从制度类型迁出"）

工作目录：/home/baojie/work/knowledge/shiji-kb
章节文件：chapter_md/NNN_*.tagged.md（130章）

## Step 1：侦察（统计现有分布）

grep -roh '〖SOURCE_MARKER[^〗]*〗' chapter_md/ | sort | uniq -c | sort -rn

（SOURCE_MARKER 为类型标记的字符，如 ^、;、=、&、% 等）

列出：全部不同词汇及出现次数、总出现章节数、总出现次数。

## Step 2：词表分类

分三类（always/context/exclude）：

### always_list（无歧义，直接替换）
在任何上下文中都属于 TARGET_TYPE，不存在歧义。
格式：词汇 | 出现次数 | 理由

### context_list（需上下文判断）
同一词汇在不同语境可能属于不同类型，需逐条审核。
格式：词汇 | 出现次数 | 歧义说明

### exclude_list（保留在源类型）
确实属于源类型，不应迁移。
格式：词汇 | 理由

## Step 3：Phase 1 执行（always_list 自动替换）

复用现有 migrate 脚本（见 §四）或编写新脚本，批量替换 always_list。

替换后必做：
- python scripts/lint_text_integrity.py（确认原文字符未变）
- python scripts/lint_markdown.py（确认格式无破坏）

## Step 4：Phase 2 准备（context_list 上下文报告）

生成 TSV 审查报告（含前后 20 字上下文）：
章节 | 段落 | 词汇 | 前文 | 后文 | decision（keep/change）
decision 列留空，人工或下一轮 LLM 审查填写。

## Step 5：Phase 3 新词扫描（遗漏发现）

在未标注裸文本中搜索 TARGET_TYPE 的典型词汇：
python scripts/tag_new_entity_types.py --candidates new_candidates.txt

## 输出
- A：词表分类结果（always/context/exclude 各自数量）
- B：Phase 1 执行报告（替换章节 + 处数）
- C：Phase 2 TSV（等待人工审阅）
- D：Phase 3 新候选词及建议
- E：本轮发现的新规律（写入 §五 已知存量错误）
```

## 三、三阶段迁移法

每轮跨类型迁移固定三步：

### Phase 1：自动替换（always_list，高置信度）

构建 `always_list`：该类型中无歧义、可直接替换的词。

- 收录标准：该词在任何语境下都属于 TARGET_TYPE，不存在歧义
- 批量应用：复用 scripts/migrate_*.py（见 §四）
- 验证：
  - `python scripts/lint_text_integrity.py`（确认原文字符未变）
  - `python scripts/lint_markdown.py`（确认格式无破坏）

### Phase 2：上下文审查（context_list，歧义词）

构建 `context_list`：同一词汇在不同上下文分属不同类型。

- 生成 TSV 报告（含前后文窗口）
- 人工或 LLM 填 decision 列（keep/change）
- 批量应用已决定的条目

### Phase 3：新词扫描（new_candidates，遗漏发现）

在未标注裸文本中搜索 TARGET_TYPE 的典型词汇。

脚本：`scripts/tag_new_entity_types.py`

## 五、常见误标方向速查

高影响、仍在使用的迁移方向：

| 源类型 | 目标类型 | 典型词汇 | 状态 |
|-------|---------|---------|------|
| `〖=地名〗` | `〖◆邦国〗` | 秦/汉/齐/楚/赵 等指政治实体时 | **仍有残余**：`〖=齐〗` 65处、`〖=汉〗` 22处等待清理 |
| `〖^制度〗` | `〖:礼仪〗` | 礼/宗庙/社稷/封禅/郊/巡狩/方士/酎金 等 | **待执行**：`〖^礼〗`173 + `〖^宗庙〗`84 + `〖^社稷〗`78 + `〖^封禅〗`73 + `〖^郊〗`35 |
| `〖%时间〗` | `〖$数量〗` | X里/X人/X众/百里-三百里 | ✓ v2.5 完成 |
| `〖%时间〗` | 不标注 | 年龄（年X十）/时长（数十年） | ✓ v2.5 完成 |
| `〖&氏族〗` | `〖◆邦国〗` | 汉/周/秦/齐 等统一王朝/封国 | ✓ v2.1 完成 |
| `〖&氏族〗` | `〖~族群〗` | 匈奴/月氏/大宛 等外邦政权 | ✓ v2.1 完成 |
| `〖;官职〗` | `〖#身份〗` | 天子/太后/皇后/食客/处士 | ✓ v2.3 完成 |
| `〖;官职〗` | `〖@人名〗` | 高祖/沛公/元年（作人名） | ✓ v2.3 完成 |

**已过时或无效的条目**（勿使用）：
- `〖^制度〗→〖[刑法〗`：**过时**。`〖[` 现在是**动词 marker**（击/杀/坐/战/破等），非刑法类型。制度→名物 已通过 commit 930267e1 重命名处理。
- `〘X〙→〖+X〗` 旧生物格式：**已清零**。`〘〙` 现仅用于 `〘※成语〙`。
- `〖^制度〗→〖!天文〗`：**大致完成**。`〖!` 已广泛使用（日月/太白/彗星等），残余可按需零星处理。

- **新增实体类型前必须先扫描存量误标**——v2.5 新增数量类型时，存量迁移 1,894 处，远多于新标注 690 处。
