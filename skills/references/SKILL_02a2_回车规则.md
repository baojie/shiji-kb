# SKILL 02a2: 回车规则 — 标注文件段落内换行标准

> 从《史记》130篇标注文件的可读性优化实践中提炼。
>
> **前置**: 段落编号已完成，详见 [SKILL_02a_章节切分与编号.md](SKILL_02a_章节切分与编号.md)。

---

## 一、核心原则

**一个 Purple Number `[N]` = 一个语义段落**，段落内部可按句号（。）换行以提升可读性，换行不改变段落归属。

### 1.1 换行 vs 不换行

| 类型 | 换行规则 | 说明 |
|------|----------|------|
| 平行世系 | ✅ 每句一行 | "X卒，子Y立。"重复出现的密集世系链 |
| 平行列举 | ✅ 每句一行 | 结构相同的并列短句（如封国概述） |
| 密集命名/短对话 | ✅ 每句一行 | "一曰X，二曰Y"命名列举或密集短对话 |
| 纪年编年 | ✅ 每句一行 | "X年，…。Y年，…。"密集时间序列 |
| 叙事段落 | ❌ 保持单行 | 有情节推进、对话、因果关系的连续叙事 |
| 表格单元格 | ❌ 不回车 | 含 `\|` 的表格行内部不拆行 |
| 韵文/赞 | ✅ 每句一行 | 已在 SKILL_02b 中定义 |

### 1.2 判定标准

脚本通过三条独立路径检测：

**路径A：骨架平行结构**（`parallel`）
1. 提取每句骨架（标注类型+功能词），≥ 3 句骨架相同 → 平行
2. 平均句长 ≤ 80字，骨架长度 ≥ 3

**路径B：密集列举**（`enumeration`）
1. "曰" ≥ 5次，曰密度 ≤ 40字/曰
2. 句数 ≥ 5，平均句长 ≤ 35字
3. 总长 ≤ 500字（排除长叙事对话如"触龙说赵太后"）

**路径C：纪年编年**（`chronicle`）
1. 时间标注 `〖%...〗` ≥ 5个
2. 句数 ≥ 5，平均句长 ≤ 50字
3. "曰" ≤ 2（排除含对话的叙事段落）

**路径D：极短句列举**（`short-list`）
1. 句数 ≥ 6，平均句长 ≤ 15字
2. 典型：龟策列传占卜条目、五脏对应、星象描述等无标注的密集短句

**全局排除**：表格行（含 `|`）不拆行

## 二、需要拆行的典型模式

### 2.1 世系链（最常见）

密集的君主传承记录，每句记录一次即位或死亡：

**拆行前**（难以阅读）：
```
[23] 〖@微子开〗卒，立其弟〖@衍〗，是为〖@微仲〗。〖@微仲〗卒，子〖@宋公〗〖@稽〗立。〖@宋公〗〖@稽〗卒，子〖@丁公〗〖@申〗立。
```

**拆行后**（清晰的世系脉络）：
```
[23] 〖@微子开〗卒，立其弟〖@衍〗，是为〖@微仲〗。
〖@微仲〗卒，子〖@宋公〗〖@稽〗立。
〖@宋公〗〖@稽〗卒，子〖@丁公〗〖@申〗立。
```

### 2.2 侯表世系

年表中的侯位传承：

```
[4] 〖@缪靖侯〗卒，子〖;康侯〗〖@遂成〗立。
〖@遂成〗卒，子〖;怀侯〗〖@世宗〗立。
〖@世宗〗卒，子〖;侯〗〖@终根〗立，为〖;太常〗，〖[坐法〗，国除。
```

### 2.3 并列短事件

多个独立事件按时间排列，彼此无因果关系：

```
[14] 〖@穆公〗〖%五年〗，〖@齐桓公〗卒。
〖%十六年〗，〖@晋文公〗败〖◆楚〗师于〖=城濮〗。
是岁，〖@穆公〗卒，子〖@共公朔〗立。
```

### 2.4 短平行结构共享PN

短对话、短列举如果每项很短（均长 ≤ 60字），**不应给每项单独分配子PN**，而应合并为一个段落内的多行：

**错误**（每项独立PN，浪费编号空间）：
```
[12]

- [12.1] 〖@尧〗曰："谁可顺此事？"
- [12.2] 〖@放齐〗曰："〖#嗣子〗〖@丹硃〗开明。"
- [12.3] 〖@尧〗曰："吁！顽凶，不用。"
```

**正确**（合并为一个段落，每句一行）：
```
[12] 〖@尧〗曰："谁可顺此事？"
〖@放齐〗曰："〖#嗣子〗〖@丹硃〗开明。"
〖@尧〗曰："吁！顽凶，不用。"
```

**判定标准**：
- 列表项 ≥ 3个，且每项平均 ≤ 60字 → 合并为一个PN
- 列表项各自较长（均 > 60字）或有独立语义单元 → 保留独立PN
- 合并后，如果列表后面还有后续段落（原来编号较大），需重新编号

**典型合并场景**：
- 短对话（X曰："..."Y曰："..."，每句 ≤ 30字）
- 世系链（X卒，子Y立。每句 ≤ 20字）
- 平行列举（X主Y，Z主W。每项 ≤ 30字）

## 三、不应拆行的段落

### 3.1 叙事段落

有情节发展、对话、因果推理的段落保持单行：

```
[30] 〖%九年〗，〖;大司马〗〖@孔父嘉〗妻好，出，道遇〖;太宰〗〖@华督〗，〖@督〗说，目而观之。〖@督〗利〖@孔父〗妻，乃使人宣言国中曰："〖@殇公〗即位〖%十年〗耳，而十一战，〖#民〗苦不堪……"
```

### 3.2 长篇引文/议论

箕子洪范、太史公曰等连续论述：

```
[15] "〖^皇极〗：皇建其有极，敛时〖^五福〗，用傅锡其〖#庶民〗……"
```

## 四、格式规范

### 4.1 换行格式

- `[N]` 编号**只出现在第一行**
- 后续行**顶格**，不缩进
- 段落之间保持空行分隔（与未拆行时一致）

```
[N] 第一句。
第二句。
第三句。

[N+1] 下一段落。
```

### 4.2 禁止事项

- ❌ 不得在后续行重复 `[N]` 编号
- ❌ 不得在换行处添加空行（空行表示段落分隔）
- ❌ 不得为了拆行而改变原文标点

## 五、工具与脚本

### 5.1 自动拆行脚本

```bash
# 预览将要拆分的段落
python scripts/split_lineage_paragraphs.py --dry-run

# 执行拆分
python scripts/split_lineage_paragraphs.py
```

**判定逻辑（骨架相似度算法）**：

1. 按"。"拆分句子
2. 对每句提取"骨架"：标注内容替换为类型标记（`@`→`P`, `%`→`T`, `◆`→`S` 等），只保留功能词（卒/立/生/封等）和标点
3. 归一化骨架：合并连续同类标记（`PP`→`P+`）
4. 统计骨架频次，最大频次 ≥ 3 则判定为平行结构
5. 额外约束：平均句长 ≤ 80字、"曰" ≤ 2次、骨架长度 ≥ 3（排除空骨架误报）

**骨架示例**：
| 原句 | 骨架 | 归一化 |
|------|------|--------|
| `〖@微仲〗卒，子〖@宋公〗〖@稽〗立` | `P卒，子PP立` | `P卒，子P+立` |
| `〖@共公〗〖%八年〗卒，子〖@德公〗立` | `PT卒，子P立` | `PT卒，子P立` |
| `〖!角〗、〖!亢〗、〖!氐〗，〖=兗州〗` | `H、H、H，L` | `H、H、H，L` |

```bash
# 带骨架详情的预览
python scripts/split_lineage_paragraphs.py --dry-run --verbose
```

### 5.2 手动判断流程

对于脚本未覆盖的边界情况：

1. 该段是否有3个以上结构相同的短句？→ 是则拆行
2. 该段是否有对话或因果推理？→ 是则保持单行
3. 拆行后每行是否独立可理解？→ 是则拆行
