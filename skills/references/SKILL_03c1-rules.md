### A.0 红线（最高优先级）

#### A70. 严禁修改动词标注格式（2026-03-24新增，最高优先级）

> **红线提示**（2026-04-16 标记）：A70 为"最高优先级格式红线"，建议读者把此条视为 §0 前置规则优先阅读；保留 A 区编号以免破坏外部引用。


**问题描述**：
在实体反思任务中，**严格禁止**修改动词标注的格式。动词和实体使用不同的标注符号，两者不可互相转换。

**错误模式**（严重违规）：

| 错误转换 | 说明 | 严重程度 |
|---------|------|---------|
| `⟦◈伐⟧` → `〖[伐〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦◉杀⟧` → `〖[杀〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦◉诛⟧` → `〖[诛〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `⟦○废⟧` → `〖[废〗` | 动词格式→实体格式 | ❌ 严重错误 |
| `〖[攻〗` → `⟦◈攻⟧` | 实体格式→动词格式 | ⚠️ 仅使用专用工具 |

**正确做法**：

✅ **保持格式不变**：
- 看到`⟦◈⟧`/`⟦◉⟧`/`⟦○⟧`/`⟦◇⟧`格式 → 保持不变
- 看到`〖[〗`格式 → 保持不变（不在反思任务中迁移）

✅ **第三轮反思任务聚焦内容，不涉及格式**：
- 检查实体标注遗漏
- 检查时长消歧
- 检查人名省称规范
- 检查原文完整性
- **不修改动词标注格式**

**规范依据**：

根据`doc/spec/SPEC_动词标注.md`（v3.3版）：
- **动词标注**：必须使用`⟦TYPE动词⟧`（数学双方括号 U+27E6/U+27E7）
- **实体标注**：必须使用`〖TYPE内容〗`（白色龟甲括号 U+3016/3017）
- **动词TYPE符号**：
  - `◈` (U+25C8) = 军事动词（伐、攻、击、围等）
  - `◉` (U+25C9) = 刑罚动词（杀、诛、弑、族等）
  - `○` (U+25CB) = 政治动词（废、立、封等）
  - `◇` (U+25C7) = 经济动词（贸、贡、赋等）

**历史遗留说明**：

项目中存在两种刑法标注格式：
- **旧格式**（v2.8及之前）：`〖[verb〗`（使用实体标注符号）
- **新格式**（v3.0及之后）：`⟦◈verb⟧`/`⟦◉verb⟧`（使用动词标注符号）

**正确的格式迁移方向**：`〖[verb〗` → `⟦◈verb⟧`/`⟦◉verb⟧`

**⚠️ 常见错误理解（2026-03-30新增）**：

❌ **错误理解**：将 `⟦◈⟧`/`⟦◉⟧` 误认为"旧格式"
- 错误表述："章节使用⟦◈⟧/⟦◉⟧旧格式"
- 错误来源：混淆了格式演进的方向

✅ **正确理解**：
- `⟦◈⟧`/`⟦◉⟧`/`⟦○⟧`/`⟦◇⟧` = **动词的正确格式**（新规范，v3.0+）
- `〖[〗` = **用实体符号标注动词的历史遗留格式**（旧格式，v2.8-）
- 反思时看到 `⟦◈⟧`/`⟦◉⟧` → **这是正确的**，保持不变
- 反思时看到 `〖[〗` → **这是待迁移的旧格式**，保持不变（等待专用工具）

**记忆要诀**：
- 数学双方括号 `⟦⟧` (U+27E6/U+27E7) = 动词专用符号（新/正确）
- 龟甲括号 `〖〗` (U+3016/3017) = 实体专用符号
- 动词和实体各有专用符号，不可混用

**错误案例分析**（2026-03-24发现）：

在021-030章和031-040章第三轮反思中，Agent错误地进行了**反向转换**：
- 025_律书：7处 `⟦◈◉⟧` → `〖[〗` ❌
- 028_封禅书：10处 `⟦◈◉⟧` → `〖[〗` ❌
- 029_河渠书：1处 `⟦◈◉⟧` → `〖[〗` ❌
- 031-045章：多处错误（已恢复）❌

**影响**：
- 破坏动词标注体系的一致性
- 导致动词-实体混淆
- 影响后续知识图谱构建
- 需要使用git恢复所有错误修改

**格式迁移的正确方式**：

❌ **禁止**：在反思任务中手动转换格式
✅ **正确**：使用专门的迁移工具：
```bash
python kg/entities/scripts/migrate_verb_tags.py --all --backup
```

**违反后果**：

1. 立即停止任务
2. 使用`git restore`恢复所有被修改文件
3. 删除错误的反思报告
4. 重新执行反思任务（正确理解）

**适用范围**：所有反思任务（第一轮、第二轮、第三轮）
**严重程度**：❌❌❌ 严重（违反基础规范）

### A.1 人物与身份类（@/#/;）

涵盖人名、身份、官职、群体称谓、亲属/敬谦称、帝号/庙号等与人物实体相关的判断规则。

**本子区条目**（32条）：A1, A2, A3, A14, A15, A16, A21, A48, A54, A55, A56, A59, A71, A72, A73, A74, A75, A76, A77, A85, A86, A88, A89, A90, A92, A93, A94, A95, A96, A97, A99, A101

#### A1. 帝号/谥号/庙号→人名（009-010章归纳）

帝号/谥号/庙号指代具体某人→`〖@人名〗`，不是身份`〖#〗`或官职`〖;〗`。

**皇室称谓归类总表**（本主题相关：A1, A3, A77, A90, A99；2026-04-16 汇总）：

| 类型 | 标注 | 示例 |
|------|------|------|
| 具体帝号/谥号/庙号 | `〖@〗` | 高帝、高后、孝武皇帝、幽王、厉王 |
| 通称（活帝王） | `〖#〗` | 天子、皇帝、太后、陛下、上、**今上/圣上**（见 A99） |
| 通称（已故帝王） | `〖#〗` | **先帝/先王**（见 A3） |
| 帝王自称 | `〖#〗` | **朕、寡人、孤、予一人、不穀**（见 A90） |
| 敬称/谦称 | `〖#〗` | **夫子、先生、阁下、足下、小子、愚人**（见 A77） |
| 泛称 | `〖#〗` | 今王、人主、单于 |

**判断标准**：能替换为具体人名→`〖@〗`；泛指一类身份→`〖#〗`。

#### A2. 特指后妃→人名（012章首发现）

特指某具体后妃的称号→`〖@〗`，泛称→`〖#〗`。
- `康后`（胶东康王后）→`〖@〗`；`太后`/`皇后`（泛称）→`〖#〗`

#### A3. 先帝/先王→身份（006章首发现）

"先帝""先王"是追述已故帝王的称号→`〖#身份〗`，不是官职`〖;〗`。

#### A14. 庙号消歧（013章首发现）

殷商庙号加消歧前缀：中宗→`〖#中宗|帝太戊〗`。

#### A15. 撰述者→人名（013章首发现）

#### A16. 稷字语境判断（001第二轮）

#### A54. 年表侯号+人名组合检查（021章第二轮）

年表类章节中，`〖;侯号〗刘X〖%元年〗` 格式的人名容易遗漏，需系统性扫描。

**检查命令**：
```bash
grep -n '〖;.*侯〗刘[^〗]' chapter_md/NNN_*.tagged.md
```

**修正模式**：
```
〖;X侯〗刘Y  →  〖;X侯〗〖@刘Y〗
```

**021章案例**：
- L147-151：长沙定王子分封集中区域，4处人名遗漏
- `〖;敬侯〗刘义` → `〖;敬侯〗〖@刘义〗`
- `〖;靖侯〗刘狗彘` → `〖;靖侯〗〖@刘狗彘〗`

**易错点**：
1. 首次出现时标注完整
2. 集中分封区域（同一父王的多个儿子）后续行遗漏
3. 复合名（如"狗彘"）更易遗漏

**适用章节**：年表类（014-021）

#### A55. 年表"侯X"结构词性判断（021章第二轮）

`侯X不...` 结构中X的词性判断：人名 vs 副词 vs 动词。

**判断流程**：
1. 查找前文是否有"X元年" → 有：人名 / 无：继续
2. 检查X是否为常见副词 → 是：副词，不标注
3. 检查X是否为动词 → 是：不标注
4. 无法确定 → 保守不标注

**常见副词清单**：
| 副词 | 意义 | 示例 |
|------|------|------|
| 阴 | 暗中、私下 | 侯阴不使人为秋请 |
| 阳 | 公开地 | 侯阳拒之 |
| 明 | 明显地 | 侯明示其意 |
| 暗 | 暗中 | 侯暗通款曲 |

**021章案例**：
- L126: `侯〖@阴〗不使人为秋请` → `侯阴不使人为秋请`
- 分析：
  1. 前文无"阴元年"记录
  2. "阴"为常见副词（暗中）
  3. 语法："侯+副词+否定+动词"符合汉代语法
- 结论："阴"是副词，不标注

**误判原因**：
- 将"阴"理解为继承人名字
- 忽略时间逻辑：刘摇元年后仅一年即国除，无时间给继承人

**类似易混淆词**：
- 阳（副词/人名/地名）
- 明（副词/人名/形容词）

#### A56. 人名边界切分错误检查（021章第二轮）

`〖;侯号〗刘〖@X〗` 格式中，"刘"被拆分到标注外的错误。

**错误模式**：
```
〖;康侯〗刘〖@不害〗  →  错误
〖;康侯〗〖@刘不害〗  →  正确
```

**检查命令**：
```bash
grep -n '〖;.*侯〗刘〖@' chapter_md/NNN_*.tagged.md
```

**021章案例**：
- L75: `〖;康侯〗刘〖@不害〗` → `〖;康侯〗〖@刘不害〗`

**规律**：汉代刘姓人名，"刘"必须包含在人名标注内。

**适用章节**：所有含刘姓人名的章节（尤其年表类）

#### A59. 年表侯号官职用法判断（022章新增）

年表职官列中，侯号作为官员身份使用时标注为〖;官职〗。

**判断标准**：

| 句式 | 判断 | 标注 | 示例 |
|------|------|------|------|
| 侯号+人名+为+官职 | 侯号作官衔 | 〖;侯号〗 | 〖;绛侯〗〖@周勃〗为〖;太尉〗 |
| 封+人名+侯号 | 侯号作封号 | 〖@侯号〗 | 封〖@平津侯〗 |
| 侯号+人名（主语） | 整体是人名 | 〖@侯号人名〗 | 〖@魏其侯窦婴〗为丞相 |

**关键区分**：
- 年表职官列 → 侯号是身份标识 → 〖;〗
- 封赏/初封 → 侯号是封号 → 〖@〗
- 主语整体 → 侯号+人名不拆分 → 〖@〗

**检查方法**：
```bash
grep -E '侯[为|曰|，]' chapter_md/NNN_*.tagged.md  # 找侯号任官
```

**错误案例**：
- ❌ `〖=曲逆〗侯〖@陈平〗为左丞相` → 误作地名
- ✅ `〖;曲逆侯〗〖@陈平〗为〖;左丞相〗` → 侯号是官衔

**补充A54、A55**：与021章发现的规律一致，022章提供更多实证案例。

**适用章节**：014/018/019/020/021/022等年表类章节

#### A71. 法家辩论对比群体（2026-03-30新增，068章）

**问题描述**：
变法/法家理论辩论段落中，对比性群体称谓（知者vs愚者、智者vs常人等）呈现系统性遗漏。

**典型场景**：
- 商鞅变法立信段落（068_商君列传）
- 韩非法术辩论（063_老子韩非列传）
- 其他变法改革相关章节

**遗漏案例**（068章）：
```
原文："民不可与虑始，而可与乐成。论至德者不和於俗，成大功者不谋於众。是以圣人苟可以彊国，不法其故"

遗漏的群体称谓：
- 民（已标）
- 圣人（已标）
- 知者（遗漏）→ 应标注〖#知者〗
- 愚者（遗漏）→ 应标注〖#愚者〗
- 智者（遗漏）→ 应标注〖#智者〗
- 常人（遗漏）→ 应标注〖#常人〗
- 学者（遗漏）→ 应标注〖#学者〗
```

#### A88. 大段主语连续切换的漏标陷阱（2026-04-16新增，005章）


**审查清单**：对每个连接词「於是/乃/遂/复/又」开启的长句，逐句核对邦国/人名是否齐标。

### A.2 时间类（%）

涵盖纪年、时长、干支、日月多义等时间实体的标注规则。

**本子区条目**（3条）：A8, A57, A91

#### A8. 时长vs纪年（008章归纳，001章第三轮细化）

**消歧标注规范**：

时间类型〖%〗包含两大子类，必须通过消歧标注区分：

| 子类型 | 标注格式 | 说明 |
|--------|---------|------|
| **时长/持续时间** | `〖%XX年\|时长〗` | 必须加 `\|时长` 消歧 |
| **纪年格式（年月日）** | `〖%XX年〗` | 无需消歧标注 |

**默认规则**：没有消歧标注时，默认为**纪年格式**（年月日）

**时长类型（6类，均需标注 `|时长`）**：

1. **任期时长**：在位XX年、立XX年、践帝位XX年、摄政XX年
   - `在位〖%七十载|时长〗`、`〖@尧〗立〖%七十年|时长〗得〖@舜〗`
2. **经过时长**：XX年而老/崩、XX年得、功用不成
   - `〖%二十年|时长〗而老`、`〖@尧〗於是听岳用〖@鲧〗。〖%九岁|时长〗，功用不成`
3. **守丧时长**：三年丧毕、三年四方莫举乐
   - `〖%三年|时长〗丧毕`、`〖%三年|时长〗，四方莫举乐`
4. **制度周期**：三岁一考功、五岁一巡狩
   - `〖%三岁|时长〗一考功`、`〖%五岁|时长〗一〖^巡狩〗`
5. **发展阶段**：一年成聚、二年成邑、三年成都
   - `〖%一年|时长〗而所居成聚，〖%二年|时长〗成邑，〖%三年|时长〗〖=成都〗`
6. **已经时长**：三年矣、数月矣
   - `女谋事至而言可绩，〖%三年|时长〗矣`
7. **年龄**（A8-7，2026-04-16新增；004章 N2、001章实操）：年X岁、春秋已五十、年二十以孝闻
   - 标注格式：`〖%XX|年龄〗`（不带"年/岁"字时也保留数字主体）
   - 示例：`年〖%二十|年龄〗以孝闻`、`〖%春秋〗已〖%五十|年龄〗矣`、`年〖%六十一|年龄〗代〖@尧〗践帝位`
   - 触发词：「年X」「春秋已X」「X岁以…」（紧跟动词或描述）
   - 与时长区分：年龄是某时点的累计岁数，不可替换为"过了X年"

**纪年格式类型（3类，无需消歧）**：

1. **编年纪年**：元年、二年、三年（单独使用表某年）
   - `〖%元年〗`、`〖%三年〗`
2. **月份日期**：正月、八月、正月上日、八月丙子
   - `〖%正月上日〗，〖@舜〗受终於〖=文祖〗`
   - `〖%岁二月〗，东〖^巡狩〗；〖%五月〗，南〖^巡狩〗`
3. **完整纪年**：XX年XX月XX日
   - `〖%八月丙子〗`

**判断关键三问**：

1. **能否替换为"第X年"且不改变语义？** → 是：时长（需`|时长`）
   - "立七十年得舜" ≈ "立了七十年后得到舜" → 时长
   - "元年正月" ≠ "第元年第正月" → 纪年
2. **有语法助词"而/矣/毕"修饰？** → 是：通常是时长
   - "二十年而老"、"三年矣"、"三年丧毕"
3. **是历法/编年体的时间坐标点？** → 是：纪年格式（无需消歧）
   - "元年"、"正月上日"是时间坐标 → 纪年

**典型案例对比**：

| 原文 | 错误标注 | 正确标注 | 类型 |
|------|---------|---------|------|
| 朕在位七十载 | `朕在位〖%七十载〗` | `朕在位〖%七十载\|时长〗` | 任期时长 |
| 二十年而老 | `〖%二十年〗而老` | `〖%二十年\|时长〗而老` | 经过时长 |
| 三年丧毕 | `〖%三年〗丧毕` | `〖%三年\|时长〗丧毕` | 守丧时长 |

#### A28. "释之"消歧规则（014章第二轮）

"释之"有两种含义，需根据语境判断：

**情况A - 人名**：张释之（汉朝廷尉，见其他章节）
- 标注：`〖@张释之〗`
- 判断：前文明确提及此人

**情况B - 动词短语**：释放之义
- 标注：不标注
- 判断：释放/赦免语境，如"肉袒谢，释之"

**014章实例**：

- L389: 类似语境 → 去标注

**判断流程**：
1. 前文是否提及"张释之"等具体人物 → 人名
2. 语境是"释放/赦免" → 动词短语，不标注
3. 无法确定 → 保守不标注

#### A29. "不礼"动词短语误标（014章第二轮）

**本主题相关（动词短语/虚词结构误标人名清单）**：A28（释之）, A29（不礼）, A45（子将等复合结构）, A53（出於/用事）, A95（他人+N）, A97（贤/戚等对仗抽象词）, A98（得意）, B2（起/杀/攻等高频动词；见 B 区）。公共判断逻辑：前文无此人名 + 语义为动作/虚词 → 去标。


**错误**：`王〖@不礼〗`、`蔡〖@不礼〗`

**正确**：去标注

**说明**：
- "不礼"：不以礼待之（动词短语）

- "蔡不礼" = 蔡国不以礼待之

#### A45. "子将"等复合结构的语法识别（062章第二轮）

**错误**：`〖@子将〗隐矣`（误作人名）
**正确**：子将隐矣（"子"=您，"将"=将要）

**识别规则**：
- "子+动词"结构 → "子"是尊称代词，不是人名
- 常见模式：子欲、子所、子言、子独、子将

**对比真实人名**：
- 孟子、韩非子、老莱子 → "子"是敬称后缀，前有明确人名
- 判断依据：前文是否已引入此人物

**类似易误标**：出于、得意、不礼等动词短语

#### A53. "出於"/"用事"动词短语误标（004章第二轮）

**"出於"误标问题**：
- ❌ "所弃妖子〖@出於〗路者"
- ✅ "所弃妖子出於路者"（动词短语"出于路"）

**"用事"误标问题**：
- ❌ "以X为卿，〖@用事〗"
- ✅ "以X为卿，用事"（执政/掌权之义）

**判断标准**：
- "出於X"：出自/来自X → 动词短语，不标注
- "用事"：执政掌权 → 动词短语，不标注

**004章案例**：
- L233/L235: "出於路""出於襃" → 去标注
- L197/L242: "用事" → 去标注

#### A98. "得意"判断规则（006第二轮，原误编 A19，2026-04-16 重编为 A98）

"得意"需根据语境判断：

**人名语境**：`〖@张得意〗`（明确指某个叫"得意"的人）

**动词短语语境**：不标注（表示满意、称心、实现意图）

#### A61. 嵌套刑法标注错误（2026-03-23新发现）

**问题描述**：刑法制度名词被错误地拆分成多个独立标注，导致嵌套。

**错误模式**：

| 错误标注 | 正确标注 | 说明 |
|---------|---------|------|
| `〖[当〖[斩〗〗` | `〖[当斩〗` | "当"="应当"+"斩"被错误分别标注 |
| `〖[当〖[死〗〗` | `〖[当死〗` | "当死"="应当处死"是完整法律用语 |
| `〖[腰〖[斩〗〗` | `〖[腰斩〗` | "腰斩"是特定刑罚名称，不可拆分 |
| `〖[〖[斩〗首〗` | `〖[斩首〗` | "斩首"是完整刑罚动词 |
| `〖[系狱〗〖[当〖[死〗〗` | `〖[系狱〗〖[当死〗` | 连续嵌套错误 |

**错误根源**：
1. 标注者将"当斩"、"当死"理解为两个独立动词："当"（应当）+"斩/死"（刑罚）
2. 实际上"当斩"、"当死"是完整的法律判决用语，应作为整体标注
3. "当"字本身不是刑法动词，只是助动词

**语义分析**：
- "当斩" = "应当被斩首"（法律判决语）
- "当死" = "应当处死"（法律判决语）
- "腰斩" = 特定刑罚名称（从腰部斩断）
- "斩首" = 特定刑罚动词（斩断头颅）

**检测命令**：
```bash
grep -n '〖\[.*〖\[' chapter_md/NNN_*.tagged.md
```

#### A24. 旗名判断规则（004第二轮）

旗帜名称常见格式："颜色+旗"/"形状+旗"/"用途+旗"

**器物标注**：
- 大白旗、小白旗、罕旗、常旗 → `〖•器物〗`
- 黄旗、赤旗、黑旗、青旗 → `〖•器物〗`

- 大白本是旗名，不是太白金星
- 区分方法：带"旗"字明确指旗帜→器物；单独"太白"且谈天象→天文


#### A25. 器物省称识别（004第二轮）

在简洁语境（赞文、诗句、列举）中，器物常用省称。

**常见省称**：
- 乐 → 乐器（"太师抱乐"）
- 车 → 戎车/兵车/常车（"载以车"）
- 兵 → 兵器/甲兵
- 旗 → 某某旗（"县之旗"）

**判断方法**：
1. 查看同章其他段落是否有完整表达
2. 根据语境推断（祭祀→乐器，战争→兵器）
3. 简洁文体（赞、颂、铭）中省称频率更高

**检查方法**：在赞文段落特别关注单字器物词

#### A26. 礼器类词汇表（004第二轮）

礼仪场景（祭祀、封建、朝会）常见礼器，容易与制度混淆。

**礼器分类**：

| 类别 | 器物 | 说明 |
|------|------|------|
| 青铜器 | 鼎、彝、爵、觚、尊、卣、簋 | 祭祀用青铜礼器 |
| 玉器 | 玉、璧、圭、璋、琮、琥 | 礼仪用玉器 |
| 祭品容器 | 木主、神主、牌位 | 祭祀用牌位 |
| 祭祀用品 | 明水、牲、粟、酒、血 | 祭祀供品 |
| 仪仗 | 旗、车、剑、节 | 礼仪场合的器物 |

**易混淆点**：
- 彝 vs 礼制：祭祀用的"宗彝"→器物；"典章彝器"→制度
- 明水 vs 概念：祭祀用的清水→器物；抽象"明德"→不标注
- 木主 vs 神祇：具体牌位→器物；神灵本身→神祇

**检查方法**：


#### A30. 年表"公"的标注豁免（014章第二轮）

**现象**：年表中"公"（指诸侯）大量出现，理论上属身份类`〖#〗`，但密度极高。

**决策**：在年表/表类章节中，"公"作为通称可不标注，避免过度标注。

**适用条件**：
- 文体：年表/表类章节
- 密度：单行出现3次以上"公"字
- 语境：明确指诸侯国君的通称

**说明**：014章每行可能包含12国多个事件，"公"字密集出现（几乎每行5-10处），全部标注会导致标注噪音过大。

#### A44. 著述语境典籍名称识别（062章第二轮）

讨论著作的章节（如老子韩非列传）中，典籍名称识别规则：

**固定搭配**：
- "著书X篇" / "作X" → X是典籍
- "X之书" / "X曰" → X是典籍

#### A50. 典籍篇名vs制度概念辨析（034章第二轮）

需区分具体典籍名与抽象制度概念：

| 类型 | 标注 | 示例 |
|------|------|------|
| 典籍/篇名 | 〖{〗 | 君奭(《尚书》)、甘棠(《诗经》) |

**判断方法**：
1. "作X" / "X曰" / "X之诗" / "传X" 语境中的X → 典籍类

**034章案例**：

- "作〖{甘棠〗之诗" (《诗经》篇名)
- "为〖^三公〗" (官职制度)
- "为〖^从长〗" (合纵制度)

#### A33. 主人公省称全覆盖原则（057章第二轮）

世家/列传主人公的姓名省称应全覆盖，不论首次或后续出现。

**常见遗漏位置**：

- 介词后宾语：`说勃曰` → `说〖@勃〗曰`
- 被动句施事：`告勃欲反` → `告〖@勃〗欲反`
- 叙事句主语：`勃既定燕` → `〖@勃〗既定燕`
- 动词宾语：`拜勃为太尉` → `拜〖@勃〗为太尉`

**检查方法**：
```bash
grep -n '[^〗]主人公省称[^〗〖]' chapter_md/NNN_*.tagged.md
```

**057章统计**：22处"勃"字省称遗漏。

**注意事项**：
- 远古单名（尧/舜/禹等）同样适用全覆盖原则（参考001章第二轮）
- 同章多人物时，主人公省称优先级最高

#### A49. "天下"政治地理性质判断（034章第二轮，001章+002章第二轮补充）

"天下"需根据语境判断标注类型：

| 语境 | 标注 | 示例 |
|------|------|------|
| 政治地理实体（授受对象） | 〖=天下〗 | "让天下於X"、"传天下於X"、"以天下授益"、"天下归X" |
| 抽象概念/修辞 | 不标注 | "天下大乱"、"天下皆知"、"天下属意" |

**判断标准**：
- 能否作为具体对象进行授受/传递 → 地名类（禅让/授受语境）
- 能否替换为具体地理/政治范围（如"九州""四海"）→ 地名类
- 泛指天下人/局面/状态 → 不标注

**典型语境**：
- 禅让对话（"让天下"/"传天下"/"授天下"/"以天下授X"）→ 必定是政治地理实体
- 归附描述（"天下归X"/"天下咸服"）→ 政治地理实体
- 名誉传播（"天下谓之X"/"天下恶之"）→ 政治地理实体
- 战国合纵（"天下诸侯"）→ 政治地理实体
- 抽象描述（"天下大乱""天下归心""天下属意""天下咸朝"）→ 不标注

**章节案例**：
- 001章（五帝禅让）：15+处"天下"全部为政治地理实体
- 002章（精细案例）：
  - ✓ "以〖=天下〗授〖@益〗" → 禅让对象，政治地理实体
  - ✗ "天下皆以舜之诛为是" → 抽象（泛指天下人）
  - ✗ "天下於是太平治" → 抽象（天下局面）
- 034章：L76段落（禅让对话）中7次"天下"均为政治地理实体

**五帝时期特殊性**：五帝本纪（001-002章）中"天下"多指具体政治地理范围（禅让对象），但也有抽象用法需精细辨别

#### A62. 氏族名+谥号+王 结构标注错误（2026-03-23新发现）

**问题描述**："氏族名+谥号+王"结构被错误拆分成氏族和官职，应标注为完整人名。

**错误模式**：

| 错误标注 | 正确标注 | 说明 |
|---------|---------|------|

#### A87. 人名消歧标注规则（2026-04-15新增）

**核心规则**：

- **单字名 / 别名 / 字号 / 省称 / 封号** → **必须消歧**：使用 `〖@显示名|规范名〗` 格式
- **全名/规范名 / 仅一处出现 / 上下文无歧义** → 按上下文判断，**默认不消歧**

**必须消歧的场景**：

| 场景 | 错误 | 正确 |
|------|------|------|
| 单字名 | `〖@政〗` | `〖@政\|秦始皇〗` |
| 字号 | `〖@子房〗` | `〖@子房\|张良〗` |
| 别名/旧名 | `〖@子楚〗` | `〖@子楚\|秦庄襄王〗` |
| 省称 | `〖@桓公〗`（多国均有桓公） | `〖@桓公\|齐桓公〗` |
| 封号 | `〖@安国君〗` | `〖@安国君\|秦孝文王〗` |
| 同名异人 | `〖@信〗`（同章多个） | `〖@信\|韩信〗` / `〖@信\|韩王信〗` |

**默认不消歧的场景**：

- 全名规范且全书唯一所指（如 `〖@司马迁〗`、`〖@吕不韦〗`）
- 段落内首次全名后紧随的省称（不超过 2-3 行内）
- 历史共识无歧义的远古单名（`〖@尧〗`/`〖@舜〗`/`〖@禹〗`/`〖@汤〗`/`〖@纣〗` 等，参 C3）

**判断标准**：

1. 名称片段去除上下文后，能否唯一指向一人？→ 否，必须消歧
2. 同章/同书中是否存在同名/同字号他人？→ 是，必须消歧
3. 名称形式是字/号/封号/谥号/省称等非本名？→ 是，必须消歧

**与既有规律的关系**：
- C1-C4：单字人名消歧的处理流程与统计
- A4：侯号引用
- A5：职官+人名拆分
- A87（本条）：统一消歧标注的判断标准

#### B4. 梁魏国名转换（2026-03-30新增，070章）

**问题描述**：
"梁"在战国语境中常指魏国（都城大梁），需根据语境判断是地名（梁邑）还是国名（魏国别称）。

**历史背景**：
- 魏国定都大梁（今河南开封）后，常以都城"梁"代指魏国
- 类似于"秦"既可指秦国，也可指秦地；"齐"既可指齐国，也可指齐地

**语境判断规则**：

| 语境 | 标注 | 示例 |
|------|------|------|
| **国家主体** | `〖◆梁〗` | 从郑至梁、梁南与楚、秦驱韩梁攻齐 |
| **地理范围** | `〖◆梁〗` | 梁之地势、梁之亡可立而须 |
| **具体地名** | `〖=梁〗` | 攻大梁、围梁城 |
| **氏族** | `〖&梁〗` | 梁氏 |

**典型案例**（070章）：

```
✓ 正确：从〖◆郑〗至〖◆梁〗二百馀里（从郑国到魏国）
✓ 正确：〖◆梁〗南与〖◆楚〗境（魏国南边与楚国接壤）
✓ 正确：秦驱〖◆韩〗〖◆梁〗攻〖◆齐〗（秦国驱使韩国魏国攻打齐国）
✓ 正确：〖◆秦〗折〖◆韩〗而攻〖◆梁〗（秦国打败韩国后攻打魏国）

❌ 错误：从〖◆郑〗至〖=梁〗（梁不是地点，是国家）
❌ 错误：〖=梁〗南与〖◆楚〗境（梁是魏国，不是地名）
```

**检查方法**：
```bash
# 查找所有"梁"的标注
grep -n '[〖〗]梁[〖〗]' chapter_md/NNN_*.tagged.md
```

**判断技巧**：
1. **并列判断**：与其他国名并列时（韩梁、楚梁），通常是国名
2. **动词判断**：作为军事/外交动作的主体/受体时，通常是国名
3. **地理判断**：描述疆域范围时（梁之地、梁之北），通常是国名
4. **修饰判断**：前有"大"字修饰（大梁），通常是具体地名

**涉及章节**：070_张仪列传（9处遗漏）

**类似情况**：
- 宋：既指宋国，也指宋地、宋城
- 楚：既指楚国，也指楚地、郢都
- 需要根据语境判断

#### B6. "X氏"/"某某氏" 一律氏族；邦国是分封政治实体（2026-04-16新增，002章）

**核心判断**：
- **氏族 `〖&〗`**：指血缘/族姓集团，名号通常带"氏"字（陶唐氏、有虞氏、夏后氏、豢龙氏、有熊氏、高阳氏、神农氏、缙云氏…）。即使省略"氏"字（如"陶唐既衰"、"高阳嗣位"），只要语境指血缘集团/族姓延续，仍标 `〖&〗`。
- **邦国 `〖◆〗`**：指**分封制度下的政治实体**，有明确国界、君主、臣民、参与会盟征伐（齐、楚、秦、晋、鲁、卫、陈、宋、郑、吴、越…）。

**典型修正**：
- 〖◆陶唐〗既衰 → `〖&陶唐〗`（陶唐氏为尧之族姓集团）
- 〖◆有虞〗 → `〖&有虞〗`
- 〖◆夏后〗 → `〖&夏后〗`（"夏后氏"省称）

**判断流程**：
1. 名号本身或邻近文本含"X氏"→ 一律氏族 `〖&〗`
2. 上古族号（陶唐/有虞/有熊/高阳/高辛/神农/帝鸿/少暤/颛顼/缙云…）→ 氏族 `〖&〗`
3. 春秋以降参与分封、有国界君主的政治实体（齐楚秦晋等）→ 邦国 `〖◆〗`
4. 同名既可指族也可指国时，按当下语境判：
   - "X氏作乱/专政/衰微/兴起" → 氏族
   - "伐X/与X会盟/X之地/X君" → 邦国

#### C5. 跨时代人名消歧校验（2026-04-16新增，004章）

单字人名消歧时，必须校验候选规范名的**生卒年与当前语境年代重叠**。同字不同人（如「满」=周穆王 vs 王孙满；「广」=田广 vs 韩广）应按以下优先级选择规范名：

1. **最近在场人物优先**：当前段落已出现的全名
2. **同段/同篇人物优先**：本章主角及其家族成员
3. **同时代人物优先**：与当前纪年相符的历史人物

**典型错误案例**：
- 004 L147「立〖@昭王〗子〖@满〗」第三轮误消歧为 `|王孙满`（春秋人，相距约400年）→ 应为 `|周穆王`
- 007 L971「立〖@田荣〗子〖@广〗」第三轮误消歧为 `|韩广`（燕王）→ 应为 `|田广`（齐王）

**操作清单**：
- 消歧前 grep 检查全章是否有更近的同字全名
- 消歧后用人物所属国/朝代核验年代区间是否吻合
- 父子相承段落优先消歧为同氏族成员

**自动检查命令**（聚合一章中所有 `〖@X|YZ〗` 的候选规范名）：
```bash
grep -oE '〖@[^〗|]+\|[^〗]+〗' chapter_md/NNN_*.tagged.md | sort -u
```
- 若同一 X 对应多个 YZ 候选 → 必须人工复核每处归属
- 典型陷阱（同字异人）：满（周穆王/王孙满）、广（田广/韩广）、信（韩信/韩王信）、馀（陈馀/张馀）、敖（张敖/共敖；张敖=赵王，共敖=临江王）

**Gazetteer 建议**：项目层维护单字名规范名映射表（如 `scripts/entity_gazetteer`），按"姓氏+父名/封国+时代"三元组唯一标识。

"有过"在脉学语境（"络脉有过"、"蹶阴有过"）表异常/偏盛，是医学术语，**不标 `〖&氏族〗`**。

#### D3. 工程/经济章数量词高密度扫描（2026-04-15新增，029/030章）

河渠书/平准书/货殖列传等工程或经济章节，数量词密度 ~11%（高于平均 5-7%），需系统性扫描所有数字（工程数量、面积/产量、距离/时长）。

#### D4. 货殖/经济类章"车"字优先器物（2026-04-15新增，129章）

货殖列传、平准书、河渠书等章节中的「轺车/牛车/千车」等车辆量词列举，一律 `〖•车〗`；仅"车骑之众"等已成惯用军事表达且未加骑修饰时考虑去标。**严禁** `〖~车〗`。

#### D5. 本纪类时长高频（2026-04-15新增，004章；2026-04-16补充触发词清单）

本纪类章节任期/经过时长是高频表达，时长消歧（A8）应作为本纪类的**重点审查项**，必须显式 `|时长` 消歧。

**任期/经过时长触发词清单**（命中即审查后接 `〖%XX年/岁/月〗` 是否需 `|时长`）：
- **即位类**：即位XX年、立XX年、在位XX年、践位XX年、嗣位XX年、摄政XX年、摄行XX年
- **统治类**：行政XX年、王XX年、帝XX年、君XX年、相XX年（任相）
- **经过类**：居XX年、後XX年、前XX年、之後XX年、其後XX年、是岁/是年後XX年
- **结束类**：XX年而崩、XX年而薨、XX年而卒、XX年而代、XX年而立、XX年而废、XX年而老、XX年而禅
- **享国类**：享国XX年、有国XX年、传国XX年、世XX年
- **考核类**：XX考、XX岁一考、XX年一巡狩

**操作要点**：
- 凡上述触发词后跟数量+年/岁/月，默认是时长（A8 默认纪年），必须加 `|时长` 显式消歧
- "正月""五月""八月""十一月"等具体月份是纪时（不加 `|时长`）；"五岁一巡狩"中"五岁"是经过时长（须加 `|时长`）
- 配合 grep 扫描：`grep -nE '(立|即位|享国|在位|居|後|摄政|践位|嗣位|行政)〖%[^|〗]*[年岁月]〗'`

#### D6. 年表类军事动词高密（2026-04-15新增，014章）

年表类章节军事动词密度极高（如014章约38处），简洁记事格式容易遗漏标注，需要专门的年表标注检查流程。

#### D7. 抽象语境刑法动词补标（2026-04-15新增，004章）

抽象段落（如刑法理论阐述）中的"伐/征"等动词易遗漏，因非具体事件，需在抽象语境也补标。

#### D8. 侠客传义士尊称按人名（2026-04-15新增，100章）

栾公社、朱家等以"公/家"为尊称的侠客称号，独立出现时按人名 `〖@〗` 处理。

#### D9. 律吕名归 名物（2026-04-15新增，025章）

十二律名（黄锺/大吕/太簇/夹锺/姑洗/仲吕/蕤宾/林锺/夷则/南吕/无射/应锺）、十母（甲乙丙丁等天干）、十二子（子丑寅卯等地支）作为律历名物术语，统一标 `〖^〗`。

- 阴阳气、五行术语在律历语境同归 `〖^〗`

#### D11. 医学名物归 名物（2026-04-15新增，105章）

- 与 D1 互补：脉名/经络名在"足X脉"等纯诊断语境去标；穴位/脏腑/病名/方剂保留 `〖^〗`

#### D12. 神灵名归 思想（2026-04-16新增，003章）

> **建议错位归 A 思想类**（2026-04-16 标记）：D12 是跨章节普适规则（本纪/世家/本书均有神祇），不按章节类型激活，与 A9/A10 同族；保留 D12 编号与正文以免破坏外部引用。


**核心规则**：超自然神祇/灵体名词归 `〖_〗`（思想/抽象类），**不**标 `〖?〗`（旧格式残留）、**不**标 `〖@人名〗`、**不**标 `〖^名物〗`。

**典型词表**：
- 至高神：上帝、皇天、昊天、天帝、玄天
- 神祇：鬼神、神人、百神、群神、山川（作为祭祀对象的"山川之神"）、社稷（神格化）
- 灵体：祖、考、先王（在祭祀语境作为受祭灵体时）

**判断要点**：
- 出现在"祭/禋/祀/祷/事/敬/明/望"等祭祀动词前后 → `〖_〗`
- 已有 `〖?上帝〗`/`〖?鬼神〗`/`〖?神人〗` 等旧格式 → 一律改为 `〖_〗`（参 E1）
- 与人格化人名区分：上古帝号"黄帝/帝喾/帝尧"虽含"帝"字仍是 `〖@人名〗`；纯神格"上帝/天帝/皇天"则 `〖_〗`

**扫描命令**：
```bash
grep -nE '〖[?@^]上帝|〖[?@^]鬼神|〖[?@^]皇天|〖[?@^]昊天' chapter_md/
```

## E. 格式残留扫描

#### E1. `〖?X〗` 旧格式残留（2026-04-15新增，099章）

`?` 非 18 类标注，全库扫描后统一按人名处理：

```bash
grep -rn '〖?' chapter_md/
```

#### E2. `〖{{` 误嵌套扫描（2026-04-15新增，099章）

可能是工具链阶段 `{` 典籍标注被误嵌套到 `^` 礼制：

```bash
grep -rn '〖{{' chapter_md/
```
