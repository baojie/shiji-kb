# SKILL 05c: 实体关系构建 — 人物/地点/制度间的结构化关系

> 从标注文本中抽取实体之间的结构化关系（家族、政治、社会），构建关系网络和家谱世系。

---

## 零、规模估算（2026-03-15 基于前5章词频外推）

| 关系大类 | 代表词 | 全书估计次数 | 抽取难度 |
|---------|--------|------------|---------|
| 家庭（父子/兄弟/长子/少子…） | 父子、兄弟、长子、少子、子孙、苗裔 | ~470词×全书倍率 ≈ **9000–12000** | 中（词表+句法） |
| 依附（舍人/宾客/食客/门下…） | 舍人、宾客、食客、从者、门下 | ~410×倍率 ≈ **7000–9000** | 中 |
| 师徒（弟子/受业…） | 弟子、受业 | ~100×倍率 ≈ **1500–2000** | 低（词表） |
| 君臣 | 君臣、臣子 | ~46×倍率 ≈ **600–900** | 低 |
| 友人/知己 | 故人、知己、旧交 | ~36×倍率 ≈ **400–600** | 高（动态陈述为主） |
| 姻亲 | 外戚、婚姻 | ~18×倍率 ≈ **200–400** | 高（多用婚姻事件描述） |
| 仇敌 | 仇雠、仇人 | ~12×倍率 ≈ **150–250** | 高 |
| **合计（关系词可见部分）** | | **~19000–25000** | |

> **注**：上表为关系词的"显式出现次数"，不等于可抽取的结构化三元组数量。大量关系通过动态陈述表达（"以X为友""从X学""与X善"），需句法框架而非词表匹配，实际三元组抽取量预估为词频的 30–50%，即 **6000–12000 条关系三元组**。
>
> 前5章实测：词频合计约1100次（家庭468、依附412、师徒103等），外推倍率约18×（130章/前5章篇幅比）。

## 一、与事件关系的区别

| 维度   | 事件关系（05a）      | 实体关系（05c）                           |

| 主体   | 事件与事件           | 实体与实体                                |

| 粒度   | 段落/章节级          | 人物/地点级                               |
| 产出   | event_relations.json | all_relations.json, family_relations.json |

## 六、SPO三元组输出

实体关系以主-谓-宾（SPO）三元组形式结构化存储，可直接导入知识图谱：

```json
{
  "subject": "李斯",
  "predicate": "任职",
  "object": "丞相",
  "chapter": "087",
  "evidence": "[6.2] 丞相谬其说..."
}
```

| 关系 | 主语类型 | 宾语类型 | 示例 |
|------|---------|---------|------|
| 君臣 | 人 | 人 | 秦始皇 → 李斯 |
| 父子 | 人 | 人 | 刘邦 → 刘盈 |
| 封于 | 人 | 地 | 韩信 → 齐 |
| 任职 | 人 | 官 | 萧何 → 丞相 |
| 属国 | 人 | 国 | 项羽 → 楚 |
| 发生于 | 事件 | 地 | 垓下之战 → 垓下 |
| 参与 | 人 | 事件 | 韩信 → 垓下之战 |
