# SKILL 05c: 实体关系构建 — 人物/地点/制度间的结构化关系

> 从标注文本中抽取实体之间的结构化关系（家族、政治、社会），构建关系网络和家谱世系。

---

## 一、与事件关系的区别

| 维度   | 事件关系（05a）      | 实体关系（05c）                           |
| ------ | -------------------- | ----------------------------------------- |
| 主体   | 事件与事件           | 实体与实体                                |
| 类型   | 因果/时序/互见/包含  | 父子/君臣/师徒/敌对                       |
| 粒度   | 段落/章节级          | 人物/地点级                               |
| 产出   | event_relations.json | all_relations.json, family_relations.json |

## 六、SPO三元组输出

实体关系以主-谓-宾（SPO）三元组形式结构化存储，可直接导入知识图谱：

```json
{
  "subject": "李斯",
  "predicate": "任职",
  "object": "丞相",
  "chapter": "087",
  "evidence": "[6.2] 丞相谬其说..."
}
```

| 关系 | 主语类型 | 宾语类型 | 示例 |
|------|---------|---------|------|
| 君臣 | 人 | 人 | 秦始皇 → 李斯 |
| 父子 | 人 | 人 | 刘邦 → 刘盈 |
| 封于 | 人 | 地 | 韩信 → 齐 |
| 任职 | 人 | 官 | 萧何 → 丞相 |
| 属国 | 人 | 国 | 项羽 → 楚 |
| 发生于 | 事件 | 地 | 垓下之战 → 垓下 |
| 参与 | 人 | 事件 | 韩信 → 垓下之战 |
