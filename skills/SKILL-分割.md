# 任务说明
把连续的第一人称叙述切成多个话题片段。每个片段内部应围绕同一事件、场景或小主题。

# 切割信号（优先识别）
1. 时间跳跃词：后来、然后、下午、晚上、第二天、过了一会儿、早上、中午。
2. 场景切换词：到了、回到、回家、去到、进了、出了、离开。
3. 话题转折词：对了、说到、顺便、另外、还有一件事、想起来。
4. 明显停顿标记：……、[停顿]、停了一会儿。

# 输入格式
```json
{"lines": ["...", "..."], "line_start_idx": 0}
```

# 输出格式
```json
{"segments": [{"start": 0, "end": 5, "topic_hint": "..."}]}
```

# 严格规则
- 输出必须是合法JSON。
- start 和 end 都是闭区间，使用绝对行号：line_start_idx + 相对位置。
- 所有非空行都必须被片段覆盖，不遗漏。
- 片段按顺序排列，不重叠，不倒序。
- 每个片段至少1行。
- topic_hint 不超过12字。
- 任何片段超过30行时，必须继续切分。
- 宁可切多段，也不要把明显不同话题放进同一段。

# 示例1
输入：
```json
{"lines": ["早上我先去了公司", "开了周会", "然后下午我去了医院看我妈", "医生说问题不大", "对了晚上还要去超市买菜"], "line_start_idx": 0}
```
输出：
```json
{"segments": [{"start": 0, "end": 1, "topic_hint": "公司开会"}, {"start": 2, "end": 3, "topic_hint": "医院看我妈"}, {"start": 4, "end": 4, "topic_hint": "晚上买菜"}]}
```

# 示例2
输入：
```json
{"lines": ["我在家里改PPT", "改到一半停了一会儿", "顺便想起明天要交房租", "后来我回到公司拿电脑", "到了公司又跟同事聊了预算"], "line_start_idx": 10}
```
输出：
```json
{"segments": [{"start": 10, "end": 11, "topic_hint": "在家改PPT"}, {"start": 12, "end": 12, "topic_hint": "想到交房租"}, {"start": 13, "end": 14, "topic_hint": "回公司拿电脑"}]}
```

# 禁止规则区
- [v1.0] start 和 end 不得小于 line_start_idx，禁止从0重新开始计数。
