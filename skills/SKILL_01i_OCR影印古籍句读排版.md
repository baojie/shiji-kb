# SKILL 01i: OCR影印古籍的句读与排版

> **核心理念**：OCR影印古籍有"三重不确定性"——标点缺失、文字误识、排版崩坏。不可一次性解决，应按"清洗→结构→句读→OCR错字标注→校验"的单向流水线逐层降噪，每一层只做一件事，结果可审计、可回退。

---

| 书名号 / 破折号 / 顿号约定 | **主规范** | 引用 |
| 断句引擎调用（gj.cool / LLM） | **主** | 引用 |
| 已标点版本的标点反思 | **主** | ❌ 不适用 |
| 字符完整性（对标注文件） | **主** | ❌ 不适用 |
| ---- 以下为 01i 独有 ---- | | |

## 二、快速开始

### 何时使用

从影印版（四库全书、民国石印本等）经 OCR 得到的纯文本，既无标点又有识别错误。

### 核心流水线

```
原始 OCR .txt（无标点、噪声、排版崩坏）
  │
  ├─ 阶段 0：勘探（Survey）            统计噪声形态、识别结构、列 OCR 错字候选
  ├─ 阶段 1：清洗（Clean）             去重复页标、折叠空行、剥行首缩进
  ├─ 阶段 2：结构化（Structure）       Markdown 层级对应原书卷/篇/段
  ├─ 阶段 3：句读（Punctuate）         遵循 SKILL_01f §2，使用 LLM 逐段处理
  ├─ 阶段 4：OCR 错字标注（Flag）      `〈疑X〉` 标记，不改字
  ├─ 阶段 5：引号作用域修复（Quote）   【】作用域栈算法（见 §六）
  └─ 阶段 6：校验（Verify）            字符完整性、标点配对、结构层级
        ↓
  Markdown 底本 + OCR 可疑字报告 + 整理复盘
```

### 成功标准

- [ ] 噪声清除（无重复页标、>2 连续空行）
- [ ] 结构对应原书目次
- [ ] 句读覆盖正文与【】小注
- [ ] **字符保真**：去除标点与 `〈疑X〉` 后与 OCR 原文逐字一致
- [ ] OCR 可疑字全部标注并入表
- [ ] 引号全角且配对平衡（0 行左右不平衡）
- [ ] 标点规范符合 [SKILL_01f §2](SKILL_01f_句读和标点校勘.md)

## 三、铁律

### 3.1 字符保真

**禁止**：修改任何原文字符（包括明知的 OCR 错字）、繁简转换、异体字归并（属 01e）、合并/拆分【】范围。

**允许**：加全角标点、加段落空行、加 `〈疑X〉` 标注（不参与字符对齐）、加 Markdown 标题符号 `# ## ###`。

**验证**：
```bash
python scripts/curation/verify_ocr_preserve.py <原 OCR txt> <整理后 md>
```
（去除标点、`〈...〉` 与 Markdown 符号后应与原 OCR 逐字相同）

### 3.2 OCR 错字"只标不改"

OCR 错字不在本 Skill 订正，只识别、标注、汇总。订正交由 SKILL_01b（多版本互校）。

**标注语法**：

| 符号 | 含义 |
|-----|-----|
| `字〈疑X〉` | 高置信度怀疑，给出建议订正 |
| `字〈?〉` | 可疑但暂无候选 |
| `字〈OCR:X〉` | 已确认 OCR 误识，待下一版订正 |

## 四、阶段流程

### 阶段 0：勘探

**检查项**：总行数、非空行比例、页标形式、`【】` 长度分布、OCR 一字多义候选、卷/篇结构线索。

**产出**：`doc/curation/reports/<书名>_OCR勘探报告.md`

### 阶段 1：清洗

| 目标 | 操作 |
|-----|-----|
| 去重复页标 | 整行匹配删除（如 `<史部,...>`） |
| 折叠空行 | >2 连续 → 1 |
| 剥行首缩进 | 去 `　　`（全角空格），语义由阶段 2 恢复 |

脚本：`scripts/curation/clean_ocr_raw.py`（待开发）

### 阶段 2：结构化

**Markdown 约定**：`#` 书名 / `##` 卷、提要、总论 / `###` 篇题（读 XX 表 / 补 / 存疑）。段落空行分隔。`【】`夹注保持原位。

**识别线索**：卷号标题（"卷一"、"钦定四库全书"分隔）；篇题关键词（"读"、"补"、"存疑"）；孤立短行（作者签名、篇名）归卷首。

### 阶段 3：句读

**标点规范 100% 遵循 [SKILL_01f §2](SKILL_01f_句读和标点校勘.md)**。本节只列 OCR 场景独有事项：

- **分段处理**：每段 <800 字送 LLM，避免长文本失焦
- **夹注独立断句**：`【...】` 内独立完成断句，不与外文共享标点
- **使用 §七 的前置提示**规避 LLM 的三类系统偏差

### 阶段 4：OCR 错字标注

- **先宽后严**：第一遍只标高置信度怀疑（同字多次语义不通）
- **表驱动**：建立 `<书名>_OCR可疑字.md`，按 OCR 字排序，每条候选订正 + 语境
- **不改字**：订正由 01b 在阶段之外进行

### 阶段 5：引号作用域修复

**专项算法**，见 §六。这一步在阶段 3 之后单独执行。

### 阶段 6：校验

| 检查 | 方法 |
|-----|-----|
| 字符完整性 | `verify_ocr_preserve.py` |
| 标点配对 | 引号左右计数 / 行级平衡扫描 |
| 标点全角 | 扫 `"` `'` `,` `.` `?` `!` 等 |
| Markdown 层级 | 目录预览，无跳级 |

## 五、OCR 错字模式归纳

每本 OCR 古籍都会有"高频多义错字"。以《读史记十表》中"防"字为例，至少涉及 13 种原字：

| OCR 字 | 候选原字 | 典型上下文 | 订正信心 |
|-------|---------|----------|---------|
| 防 | 榷 / 议 | 商防（商议）、宜谓之议 | 高 |
| 防 | 幾 | 刘知防（唐史学家刘知幾） | 高 |
| 防 | 微 | 钩稽防义、其防微矣 | 高 |
| 防 | 会 | 书防盟、防诸侯于泽 | 高 |
| 防 | 灭 | 秦防六国、楚防鲁 | 高 |
| 防 | 弑 | 三晋防其君、项羽防义帝 | 中 |
| 防 | 桓 | 防子（赵/韩世系"桓子"） | 中 |
| 防 | 护 | 回防者（回护） | 中 |
| 防 | 毁 / 堕 | 防封地擅其号 | 中 |
| 防 | 策 / 议 | 偃防（主父偃议）、贾谊遗防 | 中 |
| 防 | 略 / 旨 | 持论大防、封禅书大防 | 中 |
| 防 | 究 / 就 | 诚防本纪世家列传 | 中 |
| 防 | 亡 / 殂 | 义帝既防之后 | 中 |

**工程教训**：
- 一个 OCR 字可以形成跨语境多义词，正则无法订正，须靠**上下文 + 多版本互校**
- 启动新书前，先用 1-5% 抽样跑"单字多义频次统计"，估工作量
- 可疑字表按"OCR 字 → 原字候选 → 语境示例"三列组织，便于多版本互校时逐条比对

## 六、引号作用域修复算法（工程规律）

**背景**：LLM 句读后，引号常出现半角 `"` 残留与左右配对混乱。直接按"偶数索引→左、奇数→右"交替转换会在有 `【】` 夹注的文本上失败——因为夹注内可能自带引号，与外层正文共用一个全局计数器。

**规律**：`【】` 夹注是一个**独立的引号作用域**，其内部的引号对与外部独立计数。同理，句尾 `。` 也不构成引号边界（引号可跨句）。

**实现**：[`scripts/curation/fix_quotes_scoped.py`](../scripts/curation/fix_quotes_scoped.py) — 作用域栈 + 逐行重置。

```bash
# 原位转换并报告统计
python scripts/curation/fix_quotes_scoped.py <path>

# 只报告不改
python scripts/curation/fix_quotes_scoped.py <path> --dry-run

# 只扫描不平衡行（修复后巡检）
python scripts/curation/fix_quotes_scoped.py <path> --verify
```

**成功标准**：每行左右引号数相等（`0 行不平衡`）。若某行不均衡，多为 LLM 句读时漏配闭引号的语义 bug，脚本会报告行号供人工补齐。

**案例验证**：《读史记十表》1426 个半角 `"` → 714 对全角，0 行不平衡（一轮脚本修复 + 2 处手工补齐语义漏闭）。

## 七、LLM 句读的三类系统偏差

经《读史记十表》全书实测，LLM 句读有三类可稳定复现的偏差：

| 偏差 | 表现 | 对策（前置提示） |
|-----|------|-----|
| **夹注吞并** | 把 `【小字注】` 合并入正文句子 | 明示："【】内注独立成段，不与外部共享标点" |
| **引号层级错配** | 嵌套引用外层内层都用双引号 | 明示："三层嵌套：外 `" "` / 内 `' '` / 再内 `" "`" |
| **专名连读** | 把 "周"、"鲁"、"齐" 等单字邦国误并 | 明示："古文多单字邦国，不与下字连读构词" |

**前置提示模板**（用于阶段 3 分段处理）：

```
任务：为下段古文加全角标点，满足：
1. 不修改原文字符（包括明显 OCR 错字）
2. 【】夹注独立断句，不与外文共享标点
3. 引号嵌套层级：外 " " / 内 ' ' / 再内 " "
4. 单字邦国（周、鲁、齐等）不与下字连读
5. 全角标点；遵循 SKILL_01f §2.2 ~ 2.4

原文：
{段落文本}

输出：只返回加标点后的文本，不加解释。
```

## 八、工作量估算

按"清代学者注释体古籍"为基线（每字 ≈《读史记十表》的标注密度）：

| 规模 | 推荐策略 |
|-----|---------|
| <2 万字 | 单对话完成方法论 + 全书处理 |
| 2-10 万字 | 单对话分阶段交付（如《读史记十表》的 10 万字级） |
| 10-50 万字 | 多轮处理，每轮 <3 万字；工具化早期启动 |
| >50 万字 | 工具化优先（gj.cool / 本地模型批处理），对话层只做抽样复核 |

**分阶段交付建议**：
- **阶段 1**（单对话可达）：句读 + 排版 + OCR 可疑字表
- **阶段 2**（需独立任务）：多版本互校订正 + 表格恢复 + 脚本化验证 + 发布接入
