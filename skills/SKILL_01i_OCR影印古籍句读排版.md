# SKILL 01i: OCR影印古籍的句读与排版

> **核心理念**：OCR影印古籍有"三重不确定性"——标点缺失、文字误识、排版崩坏。不可一次性解决，应按"清洗→结构→句读→标注可疑字"的单向流水线逐层降噪，每一层只做一件事，结果可审计、可回退。

---

## 一、快速开始

### 何时使用此Skill

- **典型场景**：从影印版（如四库全书、民国石印本）经OCR获得的纯文本，既无标点又有识别错误
- **与 SKILL_01f 的关系**：
  - 01f 假设输入已有基本结构和字形，只需完善或反思标点
  - 01i 假设输入是"生OCR文本"，需要同时处理噪声、结构、句读
  - 本Skill产物（清洗+句读后的Markdown）可作为 01f 的输入

### 核心步骤（单向流水线）

```
原始OCR .txt（无标点、有噪声、排版崩坏）
  │
  ├─ 阶段0：勘探（Survey）        ← 统计噪声、识别结构、列OCR错字模式
  ├─ 阶段1：清洗（Clean）          ← 去重复页标、折叠空行、保留字符
  ├─ 阶段2：结构化（Structure）    ← Markdown 标题层级，段落边界
  ├─ 阶段3：句读（Punctuate）      ← 全角标点、引号嵌套、【】小注独立断句
  ├─ 阶段4：OCR可疑字标记（Flag）  ← 不改字，只在旁标注 〈疑X〉 或单列报告
  └─ 阶段5：校验（Verify）         ← 字符完整性、标点配对、结构层级
        ↓
  清洗+句读+排版后的 .md（人可读底本）+ OCR可疑字校勘报告
```

### 成功标准

- [ ] **噪声清除**：无重复页标、无冗余空行（>2连续）
- [ ] **结构正确**：Markdown层级对应原书卷/篇/段次
- [ ] **句读覆盖**：正文与【】小注均有标点，句子长度合理
- [ ] **字符保真**：除标点与标注记号外，未修改任何原文字符（见"三、铁律"）
- [ ] **可疑字可溯**：所有OCR怀疑点在旁标注 `〈疑X〉` 或记入校勘表
- [ ] **引号/标点规范**：全角、配对、嵌套层级正确（遵循 SKILL_01f §2）

## 二、与 SKILL_01f 的边界

| 维度 | SKILL_01f（句读与标点校勘） | SKILL_01i（OCR影印古籍） |
|-----|---------------------------|------------------------|
| 输入 | 已清洗、有基础结构的古籍文本 | OCR生文本，含噪声与误字 |
| 侧重 | 标点规范、反思、多版本对照 | 清洗、结构重建、OCR错字标记 |
| 字形处理 | 假设字形已定 | 不改字，但需识别可疑OCR字 |
| 标点引擎 | gj.cool / 本地模型 | 同上（在阶段3调用） |
| 完整性验证 | `lint_text_integrity.py` 对齐原文 | 对齐原OCR txt（保留OCR字） |

**关键差异**：01i 不解决OCR错字，只**识别并标记**。真正的订讹交由 01b（多版本互校）用其他版本订正。

## 三、铁律

### 3.1 字符保真原则

**不得**：
- 修改任何原文字符（即使明知OCR错误）
- 做繁简转换、异体字归并（属 SKILL_01e）
- 合并或拆分【】小注范围

**允许**：
- 在字符之间添加全角标点
- 在段落之间添加空行
- 在OCR可疑字右侧用 `〈疑X〉` 或 `〈当作X〉` 标注（不占换算计数）
- 添加Markdown标题符号 `#`、`##`、`###`

**验证方法**：
```bash
# 去掉标点、〈〉标注、Markdown符号后，应与OCR原文逐字相同
python scripts/curation/verify_ocr_preserve.py \
  corpus/shiji/读史记十表.txt \
  corpus/shiji/读史记十表.md
```

### 3.2 OCR错字处理原则

**不改字，只标注**。常见错误模式应单独整理成表（见本项目 `logs/curation/reports/读史记十表_OCR可疑字.md`）。

**标注语法**（仅在阶段4使用，阶段3句读时避免）：
- `字〈疑X〉`：高度可疑，给出建议订正
- `字〈?〉`：可疑但无明确订正
- `字〈OCR:X〉`：明确是OCR误识，下一版本订正

**示例**（《读史记十表》提要中）：
```
徧求友人商防〈疑榷〉，殊无一人。
刘知防〈疑幾〉考正史例至为详悉。
```

## 四、处理流程详解

### 阶段0：勘探（Survey）

**目的**：在动手前建立全局认识。

**检查清单**：
1. 总行数、非空行数、空行比例
2. 页标/分节标记的出现频率与形式（如 `<史部,...>`）
3. `【】` 夹注的嵌套情况与长度分布
4. 一字多义的OCR错字候选（如《读史记十表》中的"防"在不同语境对应 榷/幾/微/议/护/桓/等）
5. 原书的卷/篇结构线索（"卷一"、"第X"、标题位置）

**输出**：`logs/curation/reports/<书名>_OCR勘探报告.md`

### 阶段1：清洗（Clean）

**操作**：

| 目标 | 操作 | 注意 |
|-----|------|-----|
| 去重复页标 | `<史部,...>` 类模板整行删除 | 保留段落间空行作为段落分隔 |
| 折叠空行 | >2连续空行 → 1行 | 卷/篇之间保留1行 |
| 行首缩进 | 去除行首`　　`（全角空格） | 缩进语义通过结构化阶段恢复 |
| 字符保真 | 不做简繁转换、不改OCR字 | 异体字归 SKILL_01e 处理 |

**脚本建议**（`scripts/curation/clean_ocr_raw.py`）：
```python
# 伪代码示意
def clean_ocr(text):
    text = remove_page_markers(text, pattern=r"^<[^>]+>\s*$")
    text = collapse_blank_lines(text, max_consecutive=1)
    text = strip_leading_fullwidth_space(text)
    return text
```

### 阶段2：结构化（Structure）

**Markdown层级约定**：

```markdown
# 书名（如：读史记十表）

## 提要 / 总论 / 卷X

### 篇题（如：读三代世表 / 读三代世表补 / 存疑）

段落：空行分隔。

【】小注：保持原位。
```

**结构识别线索**：
- 以"钦定四库全书"或卷号（"卷一"）为卷边界
- "读XX表" / "读XX表补" / "存疑" 为三级标题
- 原书排版中孤立短行（"南阳汪越撰"、"三代世表第一"）归入卷首

### 阶段3：句读（Punctuate）

**原则**（承袭 SKILL_01f §2）：
- 全角优先
- 引号嵌套：`"..."` → `'...'` → `"..."`
- 书名号：`《...》` 与 `〈...〉`（篇名）
- 对话："曰：""内容""

**特殊处理**：

1. **【】夹注独立断句**：夹注内部通常是短小的注解，独立完成断句，不与主文共享标点
   ```
   主文〇〇〇〇〇〇【注家曰：某者，某也。】继续主文〇〇〇〇。
   ```

2. **"曰"引语**：多数情况用全角双引号"..."；引语内再引用用单引号'...'

3. **清人或前人议论**：多以"XX曰：'...'"形式出现，按引语规则处理

4. **存疑条目**：每条独立成段，内部按长句断

### 阶段4：OCR可疑字标记

**策略**：**先宽后严**。第一遍只标记高置信度怀疑（如同书多次出现、上下文明显不通的"防"字），第二遍结合参考版本订正。

**标注示例**（《读史记十表》常见模式）：
| OCR字 | 候选订正 | 上下文 |
|------|---------|-------|
| 防 | 榷 | "商防" → "商榷" |
| 防 | 幾 | "刘知防" → "刘知幾" |
| 防 | 微 | "钩稽防义" → "钩稽微义" |
| 防 | 会/議 | "皆防" → "皆会" / "不防" → "不議" |
| 防 | 护 | "回防" → "回护" |
| 防 | 略/约 | "大防" → "大略" |

**输出**：汇总表 `logs/curation/reports/<书名>_OCR可疑字.md`

### 阶段5：校验（Verify）

**自动检查**：
1. 字符完整性：去除新增符号后与OCR原文完全一致
2. 标点配对：引号、书名号、括号左右匹配
3. 标点全角：无半角 `,` `.` `?` `!` `"` `'`
4. Markdown层级：无跳级、无遗漏

**人工抽查**：
- 每卷随机抽3段，对照原影印PDF校句读
- 【】小注处断句正确性
- 存疑条目的条目边界

2. 人工复核
3. 【】内注单独处理

## 六、检查清单

### 执行前

- [ ] 读取完整OCR原文，了解规模与噪声
- [ ] 确认有无对照版本（影印PDF、其他版本）
- [ ] 建立项目目录（输出路径、报告路径）

### 执行中

- [ ] 阶段1完成后人工抽查清洗效果，无字符误删
- [ ] 阶段2完成后结构层级与原书目次一致
- [ ] 阶段3分段处理，避免LLM长文本失焦
- [ ] 阶段4OCR可疑字覆盖已知高频模式

### 执行后

- [ ] 字符完整性脚本通过
- [ ] 标点配对、全角规范
- [ ] 标题层级可导出目录
- [ ] 可疑字表已生成
