# SKILL 11: 关系抽取 — 从事件与实体到结构化三元组

> 从事件索引和标注文本中抽取实体间的结构化关系（SPO三元组），为知识图谱提供边数据。

---

## 一、工序位置

```
事件索引 (kg/events/data/)
标注文本 (chapter_md/*.tagged.md)
        ↓ 关系抽取（本SKILL）
实体三元组 (kg/relations/)
        ↓ 知识图谱化（SKILL_13）
RDF图谱
```

前置依赖：SKILL_06（事件识别）、SKILL_10（事件关系发现）

---

## 二、关系类型

### 2.1 实体间关系

| 关系 | 主语类型 | 宾语类型 | 示例 |
|------|----------|----------|------|
| 君臣 | 人 | 人 | 秦始皇 → 李斯 |
| 父子 | 人 | 人 | 刘邦 → 刘盈 |
| 封于 | 人 | 地 | 韩信 → 齐 |
| 任职 | 人 | 官 | 萧何 → 丞相 |
| 属国 | 人 | 国 | 项羽 → 楚 |
| 发生于 | 事件 | 地 | 垓下之战 → 垓下 |
| 参与 | 人 | 事件 | 韩信 → 垓下之战 |

### 2.2 事件间关系

详见 SKILL_10_事件关系发现.md（9种：sequel/causal/cross_ref/co_person 等）

---

## 三、抽取方法

### 3.1 基于标注的自动抽取

从 `〖@人名〗`、`〖=地名〗`、`〖;官职〗` 等标注的共现关系自动提取。

### 3.2 基于事件的LLM抽取

从事件描述中用LLM提取隐含的语义关系。

### 3.3 输出格式

```json
{
  "subject": "李斯",
  "predicate": "任职",
  "object": "丞相",
  "chapter": "087",
  "evidence": "[6.2] 丞相谬其说..."
}
```
