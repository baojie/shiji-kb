# SKILL 03j: 人名身份分类规则

> 为《史记》4,921 个 person 实体打上"身份类"二级分类（帝王/诸侯君主/将相/谋臣策士/学者文士/…），供 `docs/entities/person.html` 按类筛选。
>
> 原则：**规则沉淀 > 一次拍脑袋**。每发现一批误分类，提炼出规则并记录在本文档里，改进 `classify_persons.py`，重跑即可。
>
> **与 SKILL_03h/03i 完全对称的工作流**：L1 白名单 → L2 章节上下文 → L2.5 rulers.json → L3 共现动词/注释 → L4 后缀启发式 → L5 并列传播。
>
> **人名独有 alias 子工作流**：`extract_aliases_from_tags.py` 扫描全部 tagged.md 的 `〖@X|Y〗` 内联消歧 + `disambiguation_map.json` 章节级消歧 + `rulers.json` 别名字段 + 鲍捷私文档先秦称呼对照，合并产出新 `entity_aliases.json`（4 列结构：surface / type / canonical / refs）。

**多标签规则**：
- 允许一人多类（如周公旦 = `宗室` + `将相`；霍光 = `将相` + `宗室`）
- 主标 = priority 数字最小的类（= 最高位）
- JSON 按优先级升序排列，第一个就是主标

**去掉的小类**（<10 人）：
- 酷吏/循吏 → 合并入 `地方官`
- 佞幸/滑稽/方士/医者/日者/龟策 → 合并入 `近臣奇人`
- 刺客/游侠 → 合并为 `刺客游侠`

## 二、分类流水线（决策顺序）

分类脚本 `kg/entities/scripts/classify_persons.py` 按以下层级打分：

```
L1   显式白名单（帝王/诸侯/将相/学者/外邦/上古/…）
     + rulers.json 自动归入 帝王/诸侯君主/外邦
L2   章节上下文
       - 专题列传 → 对应类：
           086_刺客列传 → 刺客游侠
           105_扁鹊仓公列传 → 近臣奇人（医者）
           110_匈奴列传、113-116 外邦四夷 → 外邦
           119_循吏列传、122_酷吏列传 → 地方官
           121_儒林列传 → 学者文士
           124_游侠列传 → 刺客游侠
           125_佞幸列传、126_滑稽列传、127_日者列传、128_龟策列传 → 近臣奇人
           129_货殖列传 → 货殖
       - 本纪章（001-012）的主人公 → 帝王
       - 世家章（031-060）的主人公 → 诸侯君主
L2.5 shihao_index + rulers.json 批量归类
     + 鲍捷私文档（private/to鲍捷 史记里的人名.md）别名补录
L3   共现模式（读 chapter_md）
       - `⟦○弑⟧〖@X〗`、`〖@X〗即位/崩/薨` → 帝王或诸侯君主
       - `〖@X〗为〖;丞相〗` / `拜〖@X〗为〖;将军〗` → 将相
       - `〖;博士〗〖@X〗` → 学者文士
       - 三家注 `X，人名` 兜底
L4   后缀启发式（弱，兜底）
       - 〖@X单于〗/〖@X贤王〗/〖@X昆邪王〗 → 外邦
       - 〖@X夫人〗/〖@X姬〗/〖@X后〗 → 后妃
       - 〖@X公子〗/〖@公子X〗 → 宗室
L5   并列/亲属传播（兄弟姐妹多类一致）
```

首轮落地顺序：L1 + L2 + L4（基础）→ L2.5 → L3 → L5。

## 五、L2.5：rulers.json + shihao_index 白名单

**数据源**：
- `kg/relations/rulers.json`：656 条君主数据，含 `所属国/谥号/名/别名/类型`。按 `类型` 字段分流：
  - `天子/皇帝` → 帝王
  - `诸侯王/汉诸侯王/诸侯侯君` → 诸侯君主
  - `匈奴单于/越王/朝鲜王` 等 → 外邦
- `kg/entities/data/shihao_index.json`：谥号索引，辅助识别诸侯君主

**加载方式**：
- 启动时 `_load_rulers()` 加载 rulers.json，按 `类型` 分派到白名单
- `_load_shihao()` 加载谥号索引，把"XX公/XX王/XX侯"格式批量归入 `EXPLICIT_RULER`

## 八、L5：并列/亲属传播

**规则**：在 `chapter_md` 正文中，两个 `〖@X〗` 标记之间若只有"、"（可含空白），则视为并列。对多个并列 person 构成的 peer 组，从已分类条目向未分类条目传播（多数决 ≥ 60%）。

**典型触发上下文**：
- `〖@荆轲〗、〖@豫让〗、〖@聂政〗` — 刺客序列
- `〖@申公〗、〖@辕固〗、〖@伏生〗` — 儒林序列
- `〖@邓通〗、〖@韩嫣〗、〖@李延年〗` — 佞幸序列

**约束**：仅对当前为空字符串的条目生效；已分类条目不被改写。

## 九、别名（alias）子工作流 ★ 人名独有

### 9.1 新数据结构

`entity_aliases.json` 从旧的 `{canonical: [alias, ...]}` 改为 **4 列表结构**：

```json
{
  "person": [
    {
      "surface": "昭王",
      "type": "person",
      "canonical": "秦昭王",
      "refs": [["005_秦本纪","20.1"], ["071_樗里子甘茂列传","3"]]
    },
    {
      "surface": "昭王",
      "type": "person",
      "canonical": "燕昭王",
      "refs": [["034_燕召公世家","17"]]
    },
    {
      "surface": "卫鞅",
      "type": "person",
      "canonical": "商鞅",
      "refs": [["068_商君列传","2"], ["005_秦本纪","49"]]
    }
  ]
}
```

**关键特性**：
- 同一 surface 按 canonical 拆分多行（歧义时）
- refs 记录该消歧适用的章节/段落
- 可通过 refs 快速定位原文、验证正确性

### 9.2 数据来源优先级

1. **tagged.md 的 `〖@X|Y〗` 内联消歧** ← 最权威（作者实际使用过，2264 对已存在）
2. `disambiguation_map.json` 的章节级短名→全名（71 章 644 条）
3. `kg/relations/rulers.json` 的 `别名` 字段（656 条君主）
4. `private/to鲍捷 史记里的人名.md` 的先秦称呼对照（人工整理的古书对照表，~100 条）
5. 旧 `entity_aliases.json` 手工维护（保底，586 条）

### 9.3 抽取脚本

`kg/entities/scripts/extract_aliases_from_tags.py`：

```
扫描 chapter_md/*.tagged.md
  对每个 〖@X|Y〗 记 (surface=X, canonical=Y, ref=(chapter, para))
合并 disambiguation_map.json
  对每章 {短名: 全名} → (surface=短名, canonical=全名, ref=(chapter, '*'))
合并 rulers.json
  对每条 ruler 的 aliases → (surface=alias, canonical=谥号, ref=[])
合并 private/to鲍捷 史记里的人名.md
  解析 "canonical：alias1、alias2（来源）" 行 → 多条记录
合并旧 entity_aliases.json
  作为保底，但标记 source='legacy'

输出：
  entity_aliases.json（新 4 列结构）
  entity_aliases.v1.bak.json（旧结构备份）
  alias_conflicts.json（同 surface 同 canonical 的冲突报告）
```

### 9.4 冲突处理

- **同 surface → 多 canonical**：正常（如"昭王"→3 种），保留为多行
- **同 (surface, canonical) → 多 refs**：合并 refs
- **rulers.json 与 tagged.md 冲突**：以 tagged.md 为准，rulers.json 的别名追加进 refs=[]（表示未在标注中出现，仅来自历史文献）
- **旧 entity_aliases.json 条目在新数据中找不到 canonical**：归入 `alias_conflicts.json`，需人工审

## 十、已登记的疑难与待办

### 10.1 单字人名假阳性

"象""益""会""代""舍""段"等单字既是人名也是常见动词/副词/形容词。
- 已知假阳性：`象天`（效法天）、`皆益笃`（更加）、`日益`（逐渐）等
- 对策：L3 模式匹配时排除这些上下文；最终归入 `cat-mis`

### 10.2 "X君"歧义

- `信陵君/孟尝君/平原君/春申君/魏公子` → 宗室
- `商君/武安君/应侯/穰侯/文信侯` → 将相
- 需用 rulers.json 或白名单区分

### 10.3 "X侯"歧义

- 汉代列侯（开国功臣）→ 将相
- 春秋小国侯君（曹侯/许侯）→ 诸侯君主
- 按上下文与 rulers.json 决定

### 10.4 诸侯王兼任将相

部分诸侯王起家是将领（如韩信、英布、彭越），后被封王。
- 多标签：主标 `诸侯君主`（最终地位），次标 `将相`（起家身份）

### 10.5 待拆分

- `卫将军青` = 卫青 + 卫将军
- `淮阴侯韩信` = 韩信 + 淮阴侯
- 归入 `cat-split`，源头需重新标注
