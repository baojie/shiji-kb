# SKILL W9: 页面图式反思

> 内容质量的"进化"发生在这里——W5 改进 butler 规则，W9 改进页面结构本身。

---

## 一、触发时机

**每轮 butler 结束后**（即：完成 3 个精品页 + 3 个 stub + W6 检查后），执行一次 W9 反思。

与 W5 不同：
- W5 是系统性反思（每 20 次 action 触发），修订 W1-W4 规则
- W9 是内容性反思（每轮触发），修订页面模板/分类体系
- W9 **不阻塞**下一轮，不写 failures.jsonl，只写 schema_patterns/

---

## 二、反思流程（4 步）

### 步骤 1 · 收集本轮样本

```bash
# 查看本轮新建/升级的 premium 或 featured 页
python3 -c "
import json
data = json.load(open('wiki/public/pages.json'))
pages = data['pages']
premium = [k for k, v in pages.items() if v.get('quality') in ('premium', 'featured')]
print('\n'.join(sorted(premium)[:6]))
"
```

取本轮新建/升级的 **3 个 featured/premium 页** 作为"新样本"。

### 步骤 2 · 识别细分类型

对每个新样本，提取其 `tags` 字段，匹配**细分类型表**（见§四）。

- 若匹配到已有类型 → 找同类型历史页面（≥3 页）进行横向对照
- 若未匹配 → 判断是否应新建类型（≥2 页同构则新建）

**横向对照方法**：
```python
# 从 pages.json 中筛选同类型页面
import json
pages = json.load(open('wiki/public/pages.json'))['pages']
same_type = [p for p in pages.values() 
             if any(t in p.get('tags', []) for t in target_tags)]
```

### 步骤 3 · Schema 差距分析

对同类型的 3-6 个页面，比较：

| 检查项 | 理想状态 | 常见缺失 |
|---|---|---|
| **必有节名** | 细分类型的标准 sections 都在 | 缺对照表、缺典故、缺后世评价 |
| **frontmatter 字段** | 类型特有字段完整 | 汉代诸侯王缺 `封地`、将领缺 `所属阵营` |
| **对照表行** | 类型关键属性都有 | 忠义人物缺"殉主场景"行 |
| **引文比例** | 重要场景有原文 | 只有叙述无引文 |
| **链接密度** | 同类页面互相链接 | 同类人物各自孤立 |

### 步骤 4 · 输出

写 `wiki/logs/butler/schema_patterns/YYYY-MM-DD-R<N>.md`（每轮一个文件）：

```markdown
# 图式反思 2026-MM-DD R<N>

## 本轮样本
- [[页面1]]（细分类型：XXX）
- [[页面2]]（细分类型：YYY）
- [[页面3]]（细分类型：ZZZ）

## 同类横向对照

### 类型: 汉代诸侯王（本轮 +1，累计 12 页）

**当前 schema 覆盖情况**（对 12 页抽样）：
| 节 | 有 | 缺 |
|---|---|---|
| 生平 | 12/12 | - |
| 对照表·封地 | 8/12 | 4 缺 |
| 对照表·结局 | 10/12 | 2 缺 |
| 典故 | 4/12 | 8 缺（大多数无流传典故，正常） |
| 后代/淮南国后续 | 6/12 | 6 缺（但不是每人都有著名后代） |

**发现**：
- 刘长/刘安/刘赐等封国类诸侯王，应增"封国历史"节（封→削→除/平叛）
- 所有被废/流放结局者，应有"流放/废黜经过"子节，而非混入生平

**提案**：
1. 更新 `skills/templates/汉代诸侯王.md` — 增加"封国沿革"节
2. 下轮遇到同类页面时，补充缺失的"结局"行

## 新类型识别

### 忠义殉主者（本轮 +2：纪信、周苛，累计 4 页）

本轮识别出这是一个稳定细分类型，特征：
- tags 包含 `忠义` 或 `烹杀` 或 `殉主`
- 都有"凛然应对"场景（被俘/死前的名言）
- 都与汉楚战争中的具体城池保卫战相关

**推荐 schema**（见 `skills/templates/忠义殉主者.md`）：
- 必有节：生平 > **殉主场景**（含原文引文）> 评析 > 对照表
- 对照表必有行：殉主地点、殉主方式、死前话语（原文）、时间
- 加分：与同一战役中其他牺牲者互链

→ 本轮已创建模板文件。

## 本轮无新发现的类型
- 历史事件（鸿门宴）：已有成熟模板，无需更新
```

若某类型已有 ≥5 页且发现重大 schema 缺口 → **立即写/更新** `skills/templates/<类型>.md`。

## 三、模板文件规范

模板存于 `skills/templates/<细分类型>.md`，格式：

```markdown
---
subcategory: <细分类型名>
tags_pattern: [必须包含的 tag 之一]  
created: YYYY-MM-DD
updated: YYYY-MM-DD
sample_pages: [page1, page2, page3]  # 参考样本
version: 1
---

# 模板：<细分类型名>

## 适用判断
满足以下任意 2 条即适用：
- tags 含 `XXX`
- 人物/事件的核心身份是 YYY
- 主要记载在 NNN 类章节

## 必有节（缺一标红旗）

### 生平
（按时间顺序；重要场景必须有原文引文）

### [细分类型特有节]
（本类型的核心内容，比如"殉主场景"/"封国沿革"/"策谋始末"）

### 对照表
必有行（按类型列出）：
| 父 / 君主 / 所属阵营 | ... |
| 封地 / 活跃地区 / 关键战役 | ... |
| 结局 | ... |
| 史记出现 | X 次 / N 篇 |
| 主要传记 | [[NNN_章节名]] |

### 典故与成语（若有）
### 历史评析
### 相关章节 + 相关人物

## 加分项
- 同类人物互链（如"其他忠义殉主者"）
- 太史公曰 独立引用
- 后世影响/纪念（若有）

## 反面示例（避免）
- ...
```

## 五、积累规则

1. **每轮写一个** `schema_patterns/YYYY-MM-DD-R<N>.md`（即使无新发现，也写"无新发现"）
2. **当某类型积累 ≥3 页**时，如果还没有模板文件，当轮必须创建
3. **当某类型积累 ≥5 页**时，如果模板文件超过 30 天未更新，当轮重新评估
4. **每 10 轮**（即每次 W5 反思前），回顾 `skills/templates/` 目录，更新类型索引表
5. 模板修订**必须**写进 `wiki/logs/butler/skill_changes.md`：
   ```
   YYYY-MM-DD  W9 template/<类型>.md v1→v2  增加"封国沿革"节 [R<N>]
   ```

## KB 写入规则

W9 反思确认某 schema 规则后，将定论写入 `wiki/logs/butler/kb/w9_schemas.md`：

- **写什么**：在 ≥2 轮 W9 中重复发现、已被实际页面验证的结构规则
- **不写什么**：schema_patterns/ 里的单轮发现（那里是草稿，KB 是结论）
- **格式**：`- [R<N> 确认] <规则>` 追加到对应页面类型分组
- **用途**：butler 执行 create-stub / enrich 类动作前读取，确保新页面符合已知最佳结构
