## 二、发现候选（扫描方法）

```bash
# 统计各标签下的页面数，找 ≥5 页的标签
python3 -c "
import json, glob, yaml, re
tag_counts = {}
for f in glob.glob('wiki/public/pages/*.md'):
    content = open(f).read()
    m = re.search(r'^---\n(.*?)\n---', content, re.DOTALL)
    if m:
        try:
            fm = yaml.safe_load(m.group(1))
            for tag in fm.get('tags', []):
                tag_counts[tag] = tag_counts.get(tag, 0) + 1
        except:
            pass
for tag, cnt in sorted(tag_counts.items(), key=lambda x: -x[1]):
    if cnt >= 5:
        print(f'{cnt}\t{tag}')
"
```

**同时检查是否已有对应列表页**：

```bash
# 列表页通常命名为 "{标签}列表" 或 "{标签}索引"
ls wiki/public/pages/ | grep "列表\|索引"
```

**候选条件**：标签下页面数 ≥5，且无 `{标签}列表.md` 或类似命名的汇总页。

## 三、执行步骤

### Step 1：统计标签分布

获得标签→页面数的映射表。

### Step 2：识别缺列表页的标签

对比标签列表与现有列表页，找出缺口。

### Step 3：评估列表页价值

| 评估项 | 建列表页的条件 |
|---|---|
| 页面数量 | ≥5 页 |
| 标签语义明确 | 标签能明确概括该组页面（不是宽泛如"人物"） |
| 用户查找需求 | 用户可能想浏览该类别的所有页面 |

### Step 4：写入 W12 执行队列

```markdown
<!-- 在 housekeeping_queue.md 中追加 H9 条目 -->
- [ ] H9 | P2 | 战国人物列表 | 标签"战国"+"人物"下共47页，无汇总列表页
  - 页面示例：张仪、苏秦、廉颇、蔺相如、白起...
  - 建议列表页名：战国人物列表
  - W12 任务：建立按时代/国别分组的列表页
```

**列表页命名规范**：
- `{标签}列表`（如"战国人物列表"、"军事事件列表"）
- `{类型}索引`（如"本纪索引"、"世家索引"）
