## 核心功能

### 1. 标准拼音注释

为所有汉字自动添加带声调的拼音注释，使用HTML5标准的 `<ruby>` 标注：

```html
<ruby>
  <span class="hanzi">黄</span>
  <rt class="pinyin-rt">huáng</rt>
</ruby>
```

**覆盖范围**:
- 标题（h1, h2, h3）
- 正文段落（p）
- 引用块（blockquote）
- 列表项（li）

### 2. 特殊读音处理

自动识别并应用特殊读音词表中定义的读音。

**处理流程**:
1. 加载 `docs/data/special-pronunciation.json`
2. 按词长降序排序（优先匹配长词）
3. 从左到右扫描文本
4. 完全匹配时应用特殊读音
5. 未匹配时使用标准读音（pinyin-pro库）

**匹配示例**:
```
原文: "冒顿单于"
标准读音: mào dùn dān yú (错误)
特殊读音: mò dú chán yú (正确✓)
```

### 3. 用户控制

通过设置面板控制拼音显示：
- 默认开启拼音注释
- 可随时关闭/开启
- 状态持久化到 localStorage

## 从《史记正义》中提取特殊读音

### 方法论概述

《史记正义》（唐代张守节撰）是提取特殊读音的最权威来源。正义中使用中古反切和直音法标注了大量古今读音差异的字词。

### 提取流程

#### 1. 识别读音注释模式

《史记正义》中有4种常见读音标注模式：

**模式1: 反切注音**
```
格式: "某音甲乙反"
示例: "阏，于达反" → yān
      "与音预" → yǔ
      "亢音刚，父音甫" → gāng fǔ
```

**模式2: 直接注音**
```
格式: "某音某"
示例: "音焉与" → yān yǔ
      "音潮" → cháo
      "令音铃" → líng
```

**模式3: 通假注音**
```
格式: "某读曰某" 或 "某读为某"
示例: "驯读曰训"
      "施读为移"
```

**模式4: 类比注音**
```
格式: "某如某"
示例: "朝音潮（如'潮汐'之潮）"
```

#### 2. 搜索策略

在《史记集解三家注索隐正义》中搜索：

```bash
# 方法1: 直接搜索地名+音注
grep -E "地名.*音|音.*地名" archive/史记集解三家注索隐正义.txt

# 方法2: 搜索"正义"段落中的读音标注
grep -A 3 "正义.*音" archive/史记集解三家注索隐正义.txt

# 方法3: 搜索反切模式
grep -E "音.{1,3}反|读曰|读为" archive/史记集解三家注索隐正义.txt
```

#### 3. 反切转现代拼音

**反切原理**: 取上字声母+下字韵母

常见反切示例：
```
于达反 → yān/è (取"于"的y声母+"达"的韵母)
巨用反 → gòng (取"巨"的g声母+"用"的韵母)
丘袁反 → quān (取"丘"的q声母+"袁"的韵母)
况于反 → xū (取"况"的k/x声母+"于"的韵母)
```

**注意事项**:
- 中古音与现代音有差异，需对照现代读音确认
- 同一反切可能对应多个现代音（如"于达反"可能是è或yān）
- 优先参考《史记正义》给出的直音（如"音焉与"）

#### 4. 验证在史记中出现

```bash
# 验证词条是否在史记原文中出现
grep "词条" archive/史记.简体.txt

# 统计出现次数
grep -c "词条" archive/史记.简体.txt
```

**重要原则**: 只收录在《史记》原文中确实出现的词条。

#### 5. 分类整理

按以下类别整理：
- **地名**: 阏与(yān yǔ)、亢父(gāng fǔ)、卷县(quān xiàn)
- **官职**: 仆射(pú yè)、大行(dà háng)
- **人名**: 樗里(chū lǐ)、盱眙(xū yí)
- **多音字**: 王(wàng动词)、将(jiàng名词)、相(xiàng名词)

## 特殊读音词表设计原则（v3.0铁律）

### 核心原则

⚠️ **禁止使用常见字单字词条** ⚠️

**问题根源**：单字词条会导致大规模误标

当一个常见汉字有多个读音时，如果使用单字词条标注特殊读音，会导致该字的所有出现都被标注为特殊读音，包括那些应该使用常见读音的情况。

**实证数据**（v2.8问题分析）：
- "王"字出现51,495次，仅3.7%应读wàng → 单字标注会误标48,000+次
- "中"字出现14,198次，仅0.5%应读zhòng → 单字标注会误标14,000+次
- "将"字出现14,158次，约50%应读jiàng → 单字标注会误标7,000+次
- "相"字出现13,552次，约40%应读xiàng → 单字标注会误标8,000+次
- **总计潜在误标**: ~129,000次

### 必须使用的方法

#### 方法1: 固定词组（推荐）

使用2字及以上的固定词组来标注特殊读音：

```json
// ✅ 正确：使用固定词组
{
  "text": "将军",
  "pinyin": ["jiàng", "jūn"],
  "context": "官职",
  "note": "官职名，'将'读jiàng（去声）"
}

{
  "text": "丞相",
  "pinyin": ["chéng", "xiàng"],
  "context": "官职",
  "note": "官职名，'相'读xiàng（去声）"
}

{
  "text": "称王",
  "pinyin": ["chēng", "wàng"],
  "context": "动词",
  "note": "称王，动词，'王'读wàng（去声）"
}
```

**优点**：
- ✅ 精确匹配，不会误标
- ✅ 上下文明确，语义准确
- ✅ 可穷举高频固定搭配

**案例对比**：

| 错误做法 ❌ | 正确做法 ✅ | 效果对比 |
|------------|------------|---------|
| `{"text": "王", "pinyin": ["wàng"]}` | `{"text": "称王", "pinyin": ["chēng", "wàng"]}` | "大王"不误标，"称王"正确标 |
| `{"text": "中", "pinyin": ["zhòng"]}` | `{"text": "射中", "pinyin": ["shè", "zhòng"]}` | "其中"不误标，"射中"正确标 |
| `{"text": "将", "pinyin": ["jiàng"]}` | `{"text": "将军", "pinyin": ["jiàng", "jūn"]}` | "将要"不误标，"将军"正确标 |

#### 方法2: 正则表达式（规律性模式）

对于有规律性的用法，可以使用正则表达式模式匹配：

```javascript
// 正则模式示例（在 heading-pinyin.js 中实现）

// 模式1: 数词 + [余/馀] + 骑 (量词用法，读jì)
const qiPattern = /^([0-9一二三四五六七八九十百千万数]+[余馀]?)骑/;
// 匹配："十骑"、"百骑"、"千骑"、"数十骑"、"十余骑"等

// 模式2: 数词 + [余/馀] + 乘 (量词用法，读shèng)
const chengPattern = /^([0-9一二三四五六七八九十百千万数]+[余馀]?)乘/;
// 匹配："千乘"、"万乘"、"数千乘"等
```

**优点**：
- ✅ 一条规则覆盖无限变化
- ✅ 适用于量词、数词等规律性用法
- ✅ 减少词表规模

**适用场景**：
- 数词+量词组合（骑、乘、石、斗等）
- 官职系列（X将军、X相等）
- 地名系列（王+国名等）

#### 方法3: 专有名词姓氏（唯一例外）

**仅限姓氏可以使用单字词条**：

```json
// ✅ 例外：姓氏专有名词可用单字
{
  "text": "华",
  "pinyin": ["huà"],
  "context": "姓氏",
  "note": "华姓读huà"
}

{
  "text": "缪",
  "pinyin": ["miào"],
  "context": "姓氏",
  "note": "姓缪读miào，不读móu"
}

{
  "text": "解",
  "pinyin": ["xiè"],
  "context": "姓氏",
  "note": "姓解读xiè"
}
```

**为什么姓氏可以例外**：
- ✅ 姓氏在史记中几乎100%使用特殊读音
- ✅ 不存在与常见读音混淆的风险
- ✅ 人名组合多样，无法用固定词组穷尽

**v3.0保留的9个姓氏单字**：
华(huà)、解(xiè)、区(ōu)、仇(qiú)、种(chóng)、尉(yù)、缪(miào)、盖(gě)、单(shàn)

#### 方法4: 单字词条例外（v3.1完善）

v3.0原则是"严格禁止单字词条"，但在两种特殊情况下可以例外：

##### 情况1：修正默认库错误（v3.0.2）

**仅当默认拼音库存在系统性错误时**，可添加单字词条修正：

```json
// ✅ v3.0.2例外：修正默认库错误
{
  "text": "於",
  "pinyin": ["yú"],
  "context": "介词",
  "note": "介词，相当于'在'、'到'、'对于'，读yú（阳平），不读wū。此为例外单字词条，因默认拼音库错误标注为wū"
}
```

**准入条件（必须全部满足）**：
1. ✅ **默认库存在系统性错误**：pinyin-pro将"於"标注为wū，而不是yú
2. ✅ **正确读音使用率 >99%**："於"在史记中99%+都是介词yú
3. ✅ **固定词组无法经济覆盖**："於秦"、"於齐"、"於楚"等上千种搭配
4. ✅ **不会引入新的误标风险**：标注为yú不会误标任何场景

**实际效果（v3.0.2）**：
- 修复误标：1,555次（於秦177、於齐149、於故塞1等）
- 准确率提升：92.7% → 98.8%（+6.1%）
- 词表增加：仅1个例外单字词条

##### 情况2：低风险姓氏单字（v3.1）

**仅当姓氏使用率极高时**，可保留单字词条：

```json
// ✅ v3.1保留：低风险姓氏
{
  "text": "缪",
  "pinyin": ["miào"],
  "context": "姓氏",
  "note": "姓缪读miào，不读móu"
}
```

**准入条件（必须全部满足）**：
1. ✅ **姓氏使用率 >60%**："缪"在史记中68.7%作为姓氏
2. ✅ **误标风险 <40%**：仅31.3%会被误标
3. ✅ **该字主要作为姓氏使用**：182次出现中125次是姓氏（缪公、缪侯等）

**v3.1清理结果**：
- 删除8个高风险姓氏单字：华（87.9%误标）、解（98.1%）、区（86.8%）、仇（98.1%）、种（92.3%）、尉（94.0%）、盖（92.3%）、单（96.1%）
- 保留1个低风险姓氏：缪（31.3%误标）
- 新增15个姓名组合替代：华元、华督、解狐、解扬、盖公等
- 减少误标：~241次（-83%）

**为什么这是合理的标准**：
- **"於"字**：修正第三方库错误，正确读音使用率 >99%
- **"缪"字**：该字在史记中主要作为姓氏，姓氏使用率68.7%
- **对比**：其他姓氏单字姓氏使用率<15%，误标风险>85%，必须删除

**注意**：
- ⚠️ 此例外**极其严格**，v3.1最终仅保留2个单字词条
- ⚠️ 不符合准入条件的姓氏单字必须用"姓+名"固定词组替代
- ⚠️ 优先使用固定词组，确认符合例外条件后才使用单字

### 禁止使用的方法

#### ❌ 常见字单字词条

```json
// ❌ 错误：常见字单字词条
{
  "text": "将",
  "pinyin": ["jiàng"],
  "context": "将军、大将",
  "note": "将军读jiàng"
}
```

**为什么禁止**：
1. 会覆盖所有"将"字的读音
2. "将要"、"将至"等动词用法会被错误标注为jiàng
3. 误标率可达50%以上（7,000+次误标）

**正确做法**：
```json
// ✅ 使用固定词组
{"text": "将军", "pinyin": ["jiàng", "jūn"]},
{"text": "大将", "pinyin": ["dà", "jiàng"]},
{"text": "为将", "pinyin": ["wéi", "jiàng"]}
```

## 添加特殊读音

当发现史记中的特殊读音词未被正确标注时：

### 步骤1: 确认读音

查阅权威资料确认正确读音（优先级从高到低）：
1. **《史记正义》**（最权威）
2. **《史记索隐》**
3. **《史记集解》**
4. 《汉语大字典》
5. 学术论文

### 步骤2: 编辑词表

编辑 `docs/data/special-pronunciation.json`：

```json
{
  "text": "冒顿",
  "pinyin": ["mò", "dú"],
  "context": "匈奴单于名",
  "note": "冒顿单于，匈奴历史上著名单于，'冒'读mò，'顿'读dú"
}
```

**必填字段**:
- `text`: 词汇原文
- `pinyin`: 每个字的拼音数组（带声调）
- `context`: 词汇类别
- `note`: 详细注释

### 步骤3: 验证读音

```bash
# 重新生成包含该词的HTML
python render_shiji_html.py chapter_md/110_匈奴列传.tagged.md docs/chapters/110_匈奴列传.html

# 在浏览器中打开，验证读音正确
```

### 步骤4: 更新文档

更新 `lastUpdated` 字段：
```json

  "lastUpdated": "2026-03-31"
}
```

### JavaScript匹配算法

```javascript
// 特殊读音匹配（从长到短）
function matchSpecialPronunciation(text, startIndex) {
  // 遍历词表（已按长度降序排序）
  for (let entry of specialPronunciations) {
    const substr = text.substring(startIndex, startIndex + entry.text.length);
    if (substr === entry.text) {
      return {
        text: entry.text,
        pinyin: entry.pinyin,
        length: entry.text.length,
        note: entry.note
      };
    }
  }
  return null;
}
```

## 故障排除

### 问题1: 拼音不显示

**检查项**:
1. 浏览器控制台是否有错误
2. 网络是否能访问CDN（pinyin-pro）
3. 拼音开关是否被关闭

### 问题2: 特殊读音不生效

**检查项**:
1. JSON格式是否正确（使用验证工具）
2. `pinyin`数组长度是否等于`text`长度
3. 浏览器控制台是否有加载失败警告

### 问题3: 拼音间距太紧/太松

**调整方案**:
编辑 `docs/css/shiji-styles.css`：

```css
ruby { margin: 0 0.03em; }           /* Ruby元素间距 */
rt.pinyin-rt {
    letter-spacing: 0.03em;          /* 字母间距 */
    padding: 0 0.08em;               /* 拼音padding */
}
```

### 处理策略

#### 策略1：穷举高频固定词组（推荐）

**适用**：明确的官职名、专有名词、固定搭配

**示例**：
```json
{
  "text": "丞相",
  "pinyin": ["chéng", "xiàng"],
  "note": "'相'在官职中读xiàng"
},
{
  "text": "列传",
  "pinyin": ["liè", "zhuàn"],
  "note": "'传'在传记文体中读zhuàn"
},
{
  "text": "游说",
  "pinyin": ["yóu", "shuì"],
  "note": "'说'在游说义时读shuì"
}
```

**优点**：精确可控，HTML生成时已确定
**缺点**：需要逐个添加

#### 策略2：正则模式匹配（未来增强）

**适用**：有规律的量词用法

**示例**（伪代码）：
```javascript
// 匹配：数词 + [余] + 骑
if (/^[0-9一二三四五六七八九十百千万]+[余]?骑/.test(text)) {
  return 'jì';  // 量词用法
}
```

**优点**：一条规则覆盖所有变化
**缺点**：需要修改JavaScript，有性能考虑

#### 策略3：单字标注（不推荐）

**问题**：会覆盖所有上下文，造成误读

**反例**：
```json
{
  "text": "骑",  // ❌ 错误
  "pinyin": ["jì"],
  "note": "跨骑义时读jì"
}
```
结果："兒能骑羊"被错误标注为jì

### 优先级排序

根据误读风险和使用频率，建议优先处理：

**P0（高风险，高频率）**：
1. 传（列传zhuàn vs 传说chuán）- 180次
2. 说（游说shuì vs 说话shuō）- 17次游说
3. 间（间谍jiàn vs 中间jiān）- 各约15次

**P1（中风险，高频率）**：
4. 数（屡次shuò vs 数量shù）- 7次shuò用法
5. 降（投降xiáng vs 下降jiàng）
6. 少（少年shào vs 少数shǎo）

**P2（低风险或已有默认读音）**：
- 为、更、恶等（pinyin-pro默认读音通常正确）

### 质量保证

添加新词条前必须：
1. **统计实际使用频率**（使用grep统计）
2. **确认误读风险**（默认读音是否正确）
3. **穷举关键词组**（如"列传""世家传"等）
4. **验证显示效果**（在浏览器中测试）

### 技术创新

#### 1. 正则表达式模式匹配
```javascript
// 数词 + [余/馀] + 骑 → jì
const qiPattern = /^([0-9一二三四五六七八九十百千万数]+[余馀]?)骑/;

// 数词 + [余/馀] + 乘 → shèng
const chengPattern = /^([0-9一二三四五六七八九十百千万数]+[余馀]?)乘/;
```

一条规则覆盖无限变化：十骑、百骑、千骑、数十骑、十余骑...

#### 2. Longest-Match-First 算法优化
```python
# 词表按长度降序排序
data['entries'].sort(key=lambda x: len(x['text']), reverse=True)

# 确保优先匹配长词
# "骠骑将军"(4字) > "大将军"(3字) > "将军"(2字)
```

#### 3. 姓氏专有名词例外
保留9个姓氏单字，原因：
- ✅ 姓氏特殊读音使用率>95%
- ✅ 不存在误标风险
- ✅ 人名组合多样，无法穷尽
