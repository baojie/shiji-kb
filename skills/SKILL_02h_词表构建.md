# SKILL 02h: 词表构建（通用原则）

> 词表（glossary）= 面向某一领域的**受控词汇表**，为知识图谱、索引、消歧提供基础数据。
> 本 Skill 只讲**通用原则与流程框架**，不展开任何具体领域。领域指引见 §七 交叉索引。

---

## 一、任务定义

词表构建是从史记全文中系统提取与组织特定类型词汇：

1. **成语典故** — 固化语言单元（语言层）
2. **专有名词** — 人名/官职/地名/族群（指称层）
3. **术语体系** — 礼制、历法、度量衡（概念层）
4. **文化词汇** — 器物、服饰、音乐、祭祀（领域层）

**与邻近 Skill 的分工**：
- **02e 词法分析**：字词级统计、词性标注（面向语言学）
- **02h 词表构建**：领域词汇提取、受控词表组织（面向知识图谱） ← 本篇
- **03a 实体标注**：在原文中用 `〖〗` 标注实体
- **03b 实体消歧** / **03j 人名分类**：surface → canonical 的消歧与分类体系

---

## 二、词条的通用字段

无论领域，词条基本字段一致：

```json
{
  "surface": "原文形式",
  "canonical": "规范形式",
  "type": "领域类型（成语/官职/地名/...）",
  "meaning": "释义",
  "refs": [
    ["chapter_num", "paragraph"],
    ...
  ],
  "notes": "备注（异体、争议、学术引用）"
}
```

**四列结构**：`surface / type / canonical / refs` 已成为本项目词表的通用骨架。首次在人名别名（2026-04-19）中启用，现推广为统一字段。

## 三、通用工作流程

### 3.1 词条收集

三种来源：

1. **基于已标注实体**：扫描 `chapter_md/*.tagged.md` 提取特定类型标记（例如所有 `〖;〗` 官职）
2. **基于候选词表**：从 02e 的统计候选中筛选
3. **基于外部知识**：对照《汉书》《说文解字》《索隐》《正义》等典籍

### 3.2 定位与匹配

**核心约束**：词条必须锚定到《史记》原文段落，否则归为"叙事性总结"需单独处理。

**技术要点**：
- 去标注纯文本 + 位置映射（`build_strip_map` 模式，见 `scripts/reflect_chengyu_round2c.py`）
- 异体字归并（常见：`脣→唇`、`甕→瓮`、`於→于`、`穀→谷`、`説→说`、`飜→翻`）
- 全半角标点规范化

### 3.3 释义与标注

**释义来源**：
1. 现代工具书（《汉语大词典》《辞海》）
2. 史记三家注（集解/索隐/正义）
3. AI 生成初稿 + 人工校对

**标注同步**：如果词条对应一个原文可标注的片段，**必须在 tagged.md 中加标注**（`〖〗`/`〘〙`/`⟦⟧`）后回填词表。未能标注的叙事性条目单独存于 `*_summarized.json`。

### 3.4 发布与集成

**输出格式**：
- JSON（供程序）
- Markdown（供人）
- HTML（专项索引页，集成到 `docs/special/special_index.html`）

## 四、质量控制

### 4.1 定位率

- 目标：≥95% 的词条定位到原文段落
- 手段：精确匹配 → 规范化匹配 → 分句匹配三级兜底
- 例外：括号叙事性总结（后人概括）作为 `narrative` 单独保留

### 4.2 验证分类

对每条词表条目运行"原文可证性"分类（参考成语识别 02k §6）：

| 状态 | 含义 |
|------|------|
| tagged | 章节中已有直接标注 |
| verbatim | 原文栏直接出现在纯文本中 |
| partial | 分句/省略号拆分后部分出现 |
| narrative | 原文栏是括号叙事 |
| unverified | 在纯文本中完全找不到 → **必须处理**（修正或删除）|

### 4.3 数据一致性

- 章节编号：三位数（001–130）
- 段落编号：`[N]` 或 `[N.M]`（Purple Numbers）
- 规范名：与 `kg/entities/` 的 canonical 字段同步

## 五、双轨数据架构（通用模式）

对任意领域的词表，推荐分两个文件独立生成，互不冲掉：

- **`<topic>.json`**：已标注条目，由扫描 tagged.md 的脚本生成
- **`<topic>_summarized.json`**：未标注但相关的条目，由扫描源词表的脚本生成

源词表（`kg/vocabularies/data/`）保持**人工维护**，两个 JSON 都是派生视图；源词表修改后重跑脚本即可。

首个完整落地：成语领域（`data/chengyu.json` + `data/chengyu_summarized.json`），详见 [SKILL_02k](SKILL_02k_成语识别.md)。

## 六、近期 surface 索引工作（2026-04）

**人名别名 4 列升级**（Issue: 人名分类三件套）

`entity_aliases.json` 从旧版 2 列（canonical + alias 数组）升级为 4 列结构：`surface / type / canonical / refs`。

- 人名别名：591 条 → **3,322 条**（覆盖 2,137 短名 → 2,097 canonical）
- **550 个短名存在歧义**（同一 surface 对应多个 canonical）
- 章节级消歧映射：`disambiguation_map.json` 覆盖 644 条跨 71 章
- 关键脚本：`kg/entities/scripts/extract_aliases_from_tags.py`

每一行代表一个 `(surface, canonical)` 对；同一 surface 按 canonical 拆分多行（歧义时）。`refs` 记录该消歧适用的章节/段落，可快速定位原文验证正确性。

**复用到其他领域**：地名、官职等词表未来的 4 列重构应遵循同构——见下方交叉索引。

实现与统计详见 [SKILL_03b 人名消歧](SKILL_03b_人名消歧.md) §十一、[SKILL_03j 人名分类](SKILL_03j_人名分类.md)。
