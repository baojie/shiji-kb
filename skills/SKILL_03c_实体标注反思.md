# SKILL 03d: 实体标注反思 — 诸章实体质量审查方法

> 本文档总结对标注完成章节进行"实体反思"的方法论，包括反思对象、检查维度、操作流程和已知错误模式。

---

## 一、反思的意义

AI初始标注质量约90%正确，剩余10%的错误往往具有**系统性**：同一类错误在130章中大量重复出现。反思的目标是：

1. **发现系统性误标** — 识别特定词汇或语境下的规律性错误，批量修正
2. **发现遗漏** — AI未标注但应标注的实体
3. **修复格式问题** — 非标准符号、边界断裂等技术性错误
4. **验证边界判断** — 确认18类实体分类决策在该章的应用是否一致

---

## 二、反思维度（六个检查方向）

### 2.1 非标准格式检查

检查是否存在标注系统迭代过程中遗留的旧格式：

| 旧格式 | 正确格式 | 说明 |
|--------|---------|------|
| `〘X〙` | `〖+X〗` | 生物旧括号格式（标准化前的动植物标注） |
| `@X@` | `〖@X〗` | v1.0对称格式 |
| `=X=` | `〖=X〗` | v1.0对称格式 |
| `$X$` | `〖;X〗` 或 `〖$X〗` | v1.0官职格式（$被math渲染占用） |
| `<span style="...">` | `〖TYPE X〗` | v1.0 HTML内联样式 |

**检查方法**：`grep -n '〘' chapter_md/NNN_*.tagged.md`

---

### 2.2 类型边界误标检查

#### 2.2.1 官职 vs 身份

`〖;官职〗` 只收录**朝廷正式册封/任命的职位**。以下常见词汇容易误标为官职：

| 误标为官职 | 正确类型 | 示例 |
|-----------|---------|------|
| 天子、皇帝 | `〖#身份〗` | `〖#天子〗`（社会角色） |
| 太后、皇后 | `〖#身份〗` | 无姓氏时 |
| 诸侯（泛称） | `〖#诸侯〗` | 指一个群体而非职位 |
| 父、母、兄、弟 | 不标注 | 亲属称谓 |

`〖#身份〗` 的识别特征：用于描述社会角色/地位/血缘关系，无法指向具体任命文书。

#### 2.2.2 时间 vs 数量

`〖%时间〗` 中混入数量标注是最常见的系统性误标（v2.5清洗时发现1,894处）：

| 误标为时间 | 正确类型 | 判断依据 |
|-----------|---------|---------|
| 五千里、千里 | `〖$数量〗` | 以"里"结尾 = 距离 |
| 三万人、十万众 | `〖$数量〗` | 以"人/众"结尾 = 数量 |
| 年二十、年五十 | **不标注** | 年龄（年+数字，无量词） |
| 二人、三人 | `〖$数量〗` | 人数 |
| 数十年、千馀岁 | **不标注** | 时长（非纪年） |
| 三百六十六日 | `〖$数量〗` 或 `〖%时间〗` | 日历日数 = 数量（计量天数） |
| 百里/二百里/三百里 | `〖$数量〗` | **五服制度距离**（甸服/侯服等语境中的行政区划距离，非人名"百里X"） |

#### 2.2.3 制度 vs 刑法 vs 礼仪

`〖^制度〗` 常被过度使用，将刑法和礼仪也归入其中：

| 误标为制度 | 正确类型 | 示例 |
|-----------|---------|------|
| 刑罚、刑制等级 | `【刑法】` | 五刑、流宥五刑、官刑、教刑、赎刑、典刑、五服（刑制） |
| 祭祀、礼仪行为 | `〈礼仪〉` | 祭祀、朝觐 |
| 天体/天文现象 | `〖!天文〗` | 日月（"历日月而迎送之"/"日月所照"） |
| 五行哲学概念 | `〔思想〕` | 土德（五行之德）、天地（哲学概念） |

辨别关键：
- `〖^封禅〗`（制度）vs `〈封禅〉`（礼仪行为）— 作为"朝廷典章"用制度，作为具体礼仪活动用礼仪
- `〖!日月〗`（天体）vs `〖^日月〗`（制度，仅当指"日月之行"礼制规范时）— 凡"所照/所至/迎送"语境均为天体
- `〔土德〕`/`〔水德〕`等五行之德 — 是哲学/政治合法性概念，不是制度

#### 2.2.4 地名 vs 其他类型

地名常见误标：

| 误标为地名 | 正确类型 | 说明 |
|-----------|---------|------|
| 邦国名称 | `〖'邦国〗` | 秦/汉/齐等政治实体 |
| 天象 | `〖!天文〗` | 太阴、昊天等 |
| 器物 | `〖*器物〗` | 九鼎误标为地名 |
| 典籍书名（"作X"语境） | `《典籍》` | 甘誓/五子之歌/胤征/夏小正等古籍篇章被误标为器物；凡"作X"/"传X"/"学X"语境中的古籍名 |
| 动物 | `〖+生物〗` | 白马（在"乘白马"语境） |
| 通名 | 不标注 | 山川/海/野（非专有地名时） |
| 人名 | `〖@人名〗` | 长安作为人名时 |
| 修辞性地名 | 不标注 | "成都"在"三年成都"中="成为都邑" |
| 东北/西南等方位词 | 不标注 | 方位副词（"又东北会于X"中的"东北"描述方向，非地名） |
| "道X"中的X | `〖=地名〗` | "道九山/道九川"中"道"=疏导（动词），九山/九川是地理概念，不整体标为器物 |

#### 2.2.5 人名 vs 其他类型

人名常见误标：

| 误标为人名 | 正确类型 | 说明 |
|-----------|---------|------|
| 姓氏/氏族 | `〖&氏族〗` | "姓公孙"的"公孙"；"姓姒"的"姒" |
| 描述性词语 | 不标注 | 昭明（"百姓昭明"中作形容词"明达"） |
| 复字副词 | 不标注 | "夔夔"（夔夔唯谨 = 恭谨貌，非人名重复） |
| 官职名（被任命的职位） | `〖;官职〗` | "以垂为共工"中的共工是职位名，非人名共工 |
| 人名+氏族号连写 | 拆分 | "帝颛顼高阳"→`〖@帝颛顼〗〖&高阳〗`（人名+氏族号） |

### 2.3 实体边界检查

标注边界切割错误：

| 错误类型 | 示例 | 说明 |
|---------|------|------|
| 标注吞噬句子 | `〖*器物〗，...后续文字〗` | 结束符号〗放错位置 |
| 标点符号入标注 | `〖=、〗` | 句读符号被包入标注 |
| 实体被截断 | `〖@黄〗帝` | 实体被分割 |
| 多余标注 | `〖@黄帝〗〖@黄帝〗` | 同一词重复标注 |
| 数字前缀漏标 | `三百六〖%十六日〗` | 边界不完整，应为`〖$三百六十六日〗` |
| 人名+氏族号未拆分 | `〖@帝颛顼高阳〗` | 应拆为`〖@帝颛顼〗〖&高阳〗` |

---

### 2.4 语境判断检查

同一词汇在不同语境下可能属于不同类型：

| 词汇 | 语境A | 语境B |
|------|------|------|
| 成都 | `〖=成都〗`（地名，指四川成都） | 不标注（"三年成都"= 成为都邑） |
| 日月 | `〖!日月〗`（天文，指天体） | `〖^日月〗`（制度，如礼法中的"日月之行"） |
| 山川 | `〖=山川〗`（地名，具体山水） | 不标注（泛指山川河流） |
| 封禅 | `〖^封禅〗`（制度，典章） | `〈封禅〉`（礼仪，具体行为） |
| 天子 | `〖;天子〗`（旧标注误） | `〖#天子〗`（正确，身份） |

---

### 2.5 遗漏实体检查

重点检查以下类型的遗漏：

| 类型 | 常见遗漏场景 |
|------|------------|
| 人名省称 | 段落后部对同一人的单字省称（如"舜"在段落后部无标注） |
| 官职 | 任命动词后的职位名（"以X为Y"中的Y未标） |
| 地名 | 方位词后的具体地名（"至于X"中的X未标） |
| 氏族 | "姓X"中的X未标为氏族 |
| 邦国 | 邦国名独立出现时未标（"代神农氏"中的"神农氏"已标但邦国本身未标） |
| 天文 | 天象词汇（星宿名、历法名词）未标 |

#### 2.5.1 段落内一致性缺失（高频遗漏模式）

**描述**：AI 在同一段落中对同一人物标注不一致——有的出现有标注，有的遗漏——尤其集中在两类句法位置：

| 遗漏位置 | 典型句型 | 示例 |
|---------|---------|------|
| **修饰语/所有格位** | `X的Y` / `X大Z` | `尧大祖也`（=尧的大祖），尧未标 |
| **主语嵌入位** | 动词短语中的主语，前有"於是/乃/遂" | `於是尧妻之二女`，尧未标 |

**根本原因**：AI 倾向于在直接引语前后（`X曰`）、句首高亮位置标注人名，但在句中嵌入的主语位或修饰语位容易遗漏。

**检查规则**：反思时，若段落中某人名已出现标注，则在该段中搜索该词的**全部**出现，确认每处均有标注。

> 典型案例：001章 [13.8]
> `〖@尧〗曰` × 2（已标）
> `於是**尧**妻之二女`（漏标，主语嵌入位）
> `**尧**大祖也`（漏标，修饰语位）

**检测方式（自动化思路）**：
```
对于已知人名列表中的每个词 X：
  在段落中找到所有裸出现（无〖@〗包裹）
  若同段中已有 〖@X〗 → 报告为疑似遗漏
```

## 三、两种多轮反思路径

实体标注反思有两种相互补充的路径，适用于不同的问题：

### 3.1 按类型反思（Type-Driven）

**适用**：系统性、跨章重复的同类错误。一轮聚焦一种实体类型，全130章批量修正。

**典型案例**：
- 全部 `〖;天子〗` → `〖#天子〗`（身份类型清洗）
- 全部 `〖%年龄〗` → 去标注（数量类型清洗）
- 全部 `〘X〙` → `〖+X〗`（旧格式迁移）

**流程**：

```
Step 1：词表构建
    识别目标类型的高置信度词（always_list）+ 需上下文判断的词（context_list）

Step 2：自动替换（Phase 1）
    python scripts/migrate_TYPE_fixes.py --always 130章批量

Step 3：上下文审查（Phase 2）
    生成TSV报告（前后文窗口）→ 人工/LLM填写decision列 → 批量应用

Step 4：新词扫描（Phase 3）
    在未标注文本中搜索new_candidates，发现遗漏
```

**优点**：规模效益高，一次修正覆盖全书；脚本可复用。
**缺点**：无法发现该章特有的语境误判；不处理遗漏实体。

---

### 3.2 按章反思（Chapter-Driven）

**适用**：单章内的语境误判、遗漏实体、章节特有错误。人工+LLM逐章审查。

**核心原则**：直接阅读已标注文件，不需要重新提取——逐一审视每个现有标注，再对未被标注的文字做遗漏检查。

**两个审查对象**：

1. **已标注实体** — 每个标注逐一过：类型对不对？字符边界对不对？
2. **未标注文字** — 扫描剩余文字：有没有应该标但没标的实体？

**流程**：

```
Step 1：格式扫描（自动，先于LLM审查）
    grep -n '〘' chapter_md/NNN_*.tagged.md      # 旧生物格式
    python scripts/lint_markdown.py NNN_*.tagged.md  # 未闭合/空标注

Step 2：现有标注审查（LLM逐段阅读）
    对每个 〖TYPE X〗 / 〚X〛 / 《X》 / 〈X〉 / 【X】 / 〔X〕：
    - 类型是否正确（参考第二节类型边界规则）
    - 边界是否完整（是否吞噬了非实体文字，或遗漏了实体的部分字）
    输出 corrections JSON（paragraph + old_text + new_text/action）

Step 3：未标注文字遗漏检查（LLM逐段阅读）
    对无标注的文字，关注高遗漏场景：
    - "以X为Y"中的Y（官职/身份）
    - "至于X"中的X（地名）
    - "姓X"/"氏X"（氏族）
    - 任命句后的职位名
    - 天象词汇（星宿/历法）
    输出 additions JSON（paragraph + text + type + position）

Step 4：应用修正
    python scripts/apply_reflect_fixes.py NNN corrections.json additions.json
    python scripts/lint_markdown.py NNN_*.tagged.md  # 验证
```

**优点**：发现语境特有错误；可捕捉遗漏；适合重点章节精细打磨。
**缺点**：耗时较长；不适合全130章覆盖。

## Step 1：格式扫描（快速检查）

检查以下旧格式残留，列出所有出现位置：
- `〘X〙`（旧生物格式，应改为 `〖+X〗`）
- `@X@`、`=X=`（v1.0 对称格式）
- `〖=、〗`、`〖=。〗`（句读符号被错误标注）

---

## Step 2：现有标注审查（逐段）

逐段阅读已标注文字。对每个标注 `〖TYPE X〗`/`《X》`/`〈X〉`/`【X】`/`〔X〕`/`〚X〛` 判断：

**类型正确性**（重点检查高频误标）：
- `〖;天子〗`/`〖;诸侯〗`/`〖;食客〗` → 应为 `〖#身份〗`
- `〖^刑法词〗`（五刑/赎刑/官刑/教刑等）→ 应为 `【刑法】`
- `〖^祭祀〗`/`〖^朝觐〗` → 应为 `〈礼仪〉`
- `〖^日月〗`（"所照/所至/历X迎送"语境）→ 应为 `〖!天文〗`
- `〖^五行之德〗`（土德/水德等）→ 应为 `〔思想〕`
- `〖@氏族号〗`（"姓X"中的X）→ 应为 `〖&氏族〗`
- `〖@动词性词〗`（动词短语被误标为人名）→ 去标注
- `〖@叠字副词〗`（夔夔/婆娑等形容副词）→ 去标注
- `〖@人名氏族号连写〗`（帝颛顼高阳等）→ 拆分为 `〖@人名〗〖&氏族〗`
- `〖=方位词〗`（东北/西南等泛化方位）→ 去标注
- `〖=山川〗`/`〖=海〗`（泛指地貌非专名）→ 去标注
- `〖*典籍名〗`（甘誓/周礼等书名）→ 应为 `《典籍》`
- `〖%年龄〗`（"年X十"格式）→ 去标注
- `〖%人数/距离〗`（"X人"/"X里"）→ 应为 `〖$数量〗`
- `〖'邦国〗`（指代君主身份时）→ 应为 `〖#身份〗`

**边界正确性**：
- 数字前缀是否完整（`三百六〖$十六日〗` → `〖$三百六十六日〗`）
- 人名是否被截断
- 标点符号是否误入标注

---

## Step 3：未标注文字遗漏检查

扫描无标注的文字，重点关注：
- `以X为Y`：Y 通常是官职 `〖;Y〗`（任命语境）
- `至于X`/`居于X`：X 通常是地名 `〖=X〗`
- `姓X`/`氏X`：X 通常是氏族 `〖&X〗`
- 段落内已出现标注的人名，在同段后文中的未标出现
- 动植物（禽兽鱼鸟）漏标为 `〖+生物〗`
- 典籍引用（"《X》曰"中的X）漏标为 `《典籍》`
- 天象词（星宿/历法名词）漏标为 `〖!天文〗`

### 3.4 组合策略

**按类型反思推荐顺序**（高→低影响）：

### 4.3 001章新发现的规律（已写入§二）

1. **日月三义问题**：日月在史记中有三种用法：天体（→`〖!〗`）、时间纪年（→`〖%〗`）、礼制规范（→`〖^〗`）。凡"所照/所至/历X而迎送"语境均为天体，AI倾向于误标为制度。

2. **制度^过度捕获刑法**：五刑/五服（刑制）/官刑/赎刑等刑罚体系词汇被AI系统性归入制度，需要整体迁移。五帝本纪中即有7处，预计全书规模更大。

3. **五行之德→思想**：土德/水德/木德等五行德运是政治合法性哲学概念，不是典章制度，应标`〔思想〕`。

4. **帝号+氏族号粘连**：上古帝王的"帝颛顼高阳""帝喾高辛"结构由人名（帝+谥号）+氏族号构成，AI倾向整体标为人名，需拆分。

5. **官职任命中的职位名**：原文"以X为Y"句型中，Y是被授予的职位，但Y在文中也可能作人名独立出现（如"共工"）。须根据上下文区分：任命语境中Y是`〖;官职〗`，独立出现时可能是`〖@人名〗`。

6. **叠字副词不标**："夔夔""婆娑"等叠字形容副词，即使与已标人名同字，也不应标注为人名。

## 五、历史清理记录：v2.1→v2.2（2026-03-13～14）

初始AI标注完成后，通过14轮按类型反思，两天内共修正约15,400处标注。这是迄今最大规模的实体反思行动，其经验是本SKILL的主要来源。

### 5.1 清理背景

AI初始标注的系统性错分：
- `〖&朝代〗` 混合了三种语义：统一王朝（汉/周）、诸侯封国（齐/楚）、宗族血缘（刘氏/嬴氏）

## 六、类型演化反思教训

1. **存量误标 > 增量新标** — v2.5新增数量类型时，从时间迁移1,894处，远多于新标注690处。新增类型前必须先扫描存量误标。
2. **类型边界要有明确判断流程** — 数量与时间的4类边界案例（代词性/历法/年龄/时长）。规则分类器处理95%，LLM反思清洗长尾。
3. **按类型反思优先于按章反思** — 系统性错误用按类型批量修正效率最高；按章反思适合语境特有错误。
4. **反思模式可独立迭代** — 每轮反思聚焦一类，避免同时修改多类导致冲突。
