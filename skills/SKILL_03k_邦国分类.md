# SKILL 03k: 邦国国类分类规则

> 为《史记》204 个 feudal-state 实体（标注符 `〖◆X〗`）打上"国类"二级分类（上古方国/周代诸侯/秦末列国/汉诸侯王国/外邦/朝代/合称…），供 `docs/entities/feudal-state.html` 按类筛选。
>
> 原则：**规则沉淀 > 一次拍脑袋**。每发现一批误分类，提炼出规则并记录在本文档里，改进 `classify_feudal_states.py`，重跑即可。
>
> **与 SKILL_03h/03i/03j 完全对称的工作流**：L1 白名单 → L2 章节上下文 → L2.5 rulers.json + feudal_state_wordlist.json → L3 共现动词/三家注 → L4 后缀启发式 → L5 并列传播。
>
> **邦国独有问题**：存在三类真·跨类型误标 — (a) 部族（匈奴/戎狄）被标邦国；(b) 地理大区（关中/山东）被标邦国；(c) 复合邦国（齐秦楚赵/吴楚七国）需拆分。这些通过 `cat-tribe-mis / cat-place-mis / cat-split` 三个误标子类识别，源头交 03e1 跨类型迁移。
>
> **注意**：朝代（夏商周秦汉）和合称（三晋/七国/战国）**不是误标**，而是 feudal-state 下的合法子类 `cat-dynasty` / `cat-collective`——司马迁笔下这些词作为政治实体统称与邦国并列使用。

## 一、分类体系（11 类 + 未分类）

| 2 | 朝代 | 夏、商、殷、周、秦（统一后）、汉、西周、东周、二周、周室 | `cat-dynasty` |

| — | （空） | 无法判断 | `cat-other` |

**汉王国 vs 汉侯国**：两者**都算 feudal-state**（都是列名的封国，有封君有封地）。区别：
- 王国：刘氏同姓王 + 开国异姓王（梁、淮南、代、齐、楚、吴、胶东…）— 二十余
- 侯国：列侯食邑（酂、留、平阳、赤泉、硃虚、东牟…）— 数百
- 两者在汉代均为列名诸侯国，与郡并立（郡国并行），应一并视为邦国

**多标签规则**：
- 允许一国多类（如 `秦` = `周代诸侯`（战国期）+`朝代`（统一后），按章节消歧；`代` = `秦末列国`+`汉诸侯王国`；`楚` = `周代诸侯`+`秦末列国`（西楚）+`汉诸侯王国`）
- 主标 = priority 最小的类
- 跨时代冲突通过章节上下文（L2）消歧

**"战国七雄"不单列的理由**：秦/楚/齐/燕/赵/魏/韩 均为周代诸侯沿续而来，战国阶段只是时间延续，从"政体性质"看仍属 `周代诸侯`。后续若需区分春秋/战国时段，由置信度标签或三级分类承担，不增设一级类。

**"外邦"不再细分外夷/西域**：《史记》115-116/123 列传并称"四夷/西域"，统治形态（有王、有都、有使臣）近似，合为一类更简洁。

**与地名 `cat-nation` 的边界**：
- place 类里的"国家"（大宛/大夏/朝鲜/南越…）与本 skill 的 `外邦` **高度重叠**
- 处置策略：同一 surface 既可以是 `◆邦国` 也可以是 `=地名`。feudal-state 侧关注"政体 + 统治者 + 国名互认"；place 侧关注"地理区域"
- 本 skill 只处理已被标为 `〖◆X〗` 的条目；`〖=X〗` 里的国家归 03h 管

## 二、分类流水线（决策顺序）

分类脚本 `kg/entities/scripts/classify_feudal_states.py` 按以下层级打分：

```
L1   显式白名单（上古/朝代/周代诸侯/秦末/汉王国/外邦/合称/泛称/部族-mis/地名-mis）
L2   章节上下文规则
       - 本纪/世家时代判定：005_秦本纪 内"秦"=周代诸侯；006_秦始皇本纪 内"秦"=朝代
       - 017_汉兴以来诸侯王年表 → 汉诸侯王国
       - 018-021 侯者年表 → 不应出现 feudal-state，出现即 `cat-place-mis`（侯国是县级）
       - 110_匈奴列传 → 部族误标（匈奴本体）
       - 113-116 外邦四夷 → 外邦
       - 123_大宛列传 → 外邦
L2.5 rulers.json + feudal_state_wordlist.json 批量归类
       - rulers.json 按 `类型` × `所属国`：
           诸侯侯君     → 所属国归 上古方国 / 周代诸侯
           诸侯王       → 所属国归 周代诸侯 或 秦末列国
           汉诸侯王     → 所属国归 汉诸侯王国
           天子/皇帝    → 所属国归 朝代
           单于         → 所属国归 部族误标（匈奴）
       - feudal_state_wordlist.json：
           always_feudal  → 直接按该表分派到对应类
           context_dependent → 留给 L2 按章节消歧
           stays_dynasty.unified_dynasties → `cat-dynasty`（正确分类，非误标）
           stays_dynasty.collective_terms  → `cat-collective`（正确分类，非误标）
           stays_dynasty.foreign_states    → `cat-foreign` 或 `cat-tribe-mis`
           stays_dynasty.geographic_admin  → `cat-place-mis`
L3   共现动词与三家注规则
       - `⟦◈伐/攻/击⟧〖◆X〗`、`〖◆X〗⟦◈败⟧` → 邦国（弱确认，不改类）
       - `〖◆X〗灭〖◆Y〗` → X、Y 同为邦国
       - X之君/X之王/X之主 + rulers.json 命中 → 君主所属国类
       - 三家注 `X，国名/封国名/诸侯国/小国` → 周代诸侯 或 上古方国
L4   后缀/字形启发式（兜底）
       - X+国（长度≥2）且 X 与单字国同名 → `cat-split` 候选（秦国/齐国/燕国…）
       - 含"氏" → 朝代/氏族（归 `cat-dynasty` 或建议迁 dynasty 类型）
       - 含数字集体（三/七/六/两/五）→ `cat-collective`
       - 方位+大区（北国/西国/东国/南国）→ `cat-generic`
L5   并列传播（与 03h 同逻辑）
       - `〖◆X〗、〖◆Y〗、〖◆Z〗` 同类传播（已分类 → 未分类）
       - 典型：`秦、楚、齐、燕、赵、魏、韩` → 周代诸侯（战国）序列
```

首轮落地顺序：L1 + L2.5 + L4（基础） → L2 章节上下文 → L3 → L5。

## 四、L2：章节上下文规则

### 4.1 时代消歧（周代诸侯 vs 朝代 vs 秦末 vs 汉诸侯王国）

单字邦国 秦/楚/齐/燕/赵/魏/韩/吴/越/代/梁 在不同章节含义不同：

| 章节 | 典型归类 |
|------|---------|
| 001-005 本纪（五帝/夏/殷/周/秦） | 秦本纪中的"秦"=周代诸侯；"夏""殷""周"=朝代 |
| 006-008 秦始皇/项羽/高祖本纪 | 秦 → `cat-dynasty`（秦朝）；楚 → 秦末列国（西楚）|
| 009-012 汉本纪 | 楚/齐/赵/代/梁 → 汉诸侯王国；秦 → `cat-dynasty` |
| 031-045 世家（楚/齐/晋/秦/赵/韩/魏…） | 周代诸侯（本国传） |
| 068-083 战国列传 | 周代诸侯 |
| 084-085 屈原/吕不韦 | 周代诸侯 |
| 091-094 彭越/黥布/淮阴/韩信 | 汉诸侯王国 |
| 106 吴王濞列传 | 汉诸侯王国（吴=汉吴）|
| 118 淮南衡山列传 | 汉诸侯王国 |

**实现**：对多义单字国，按 `feudal-state.refs` 的章节号区段 vote，投票多数决；跨时代高频（秦/楚/代/吴）保留多标签。

### 4.2 侯者年表 → 汉侯国

汉代列侯的"侯国"是**邦国的一种**（列侯的封国 = 食邑封地）。若 feudal-state 条目的 refs 集中在 018-021 侯者年表，且不在 `EXPLICIT_HAN_VASSAL` 内 → `cat-han-marquis`（保留在 feudal-state，不迁移）。

### 4.3 外邦四夷列传

- 110_匈奴列传：匈奴及其部落 → `cat-tribe-mis`（匈奴本身是部族非邦国）
- 113_南越/114_东越/115_朝鲜/116_西南夷：对应邦国 → `cat-foreign`
- 123_大宛列传：西域诸国 → `cat-foreign`

### 4.4 诸侯王年表 → 汉诸侯王国

017_汉兴以来诸侯王年表 内 feudal-state 实体 → `cat-han-vassal`。

## 六、L3：共现动词与三家注规则

首轮启用模式（`CONTEXT_PATTERNS` in `classify_feudal_states.py`）：

| 模式 | 推断 | 示例 |
|------|------|------|
| `〖◆X〗⟦◈伐/攻/击/侵/袭⟧〖◆Y〗` | X、Y 同为邦国（弱确认） | 〖◆秦〗⟦◈攻⟧〖◆赵〗 |
| `〖◆X〗⟦◈灭⟧〖◆Y〗` | X、Y 同为邦国；Y 国祚终结 | 〖◆秦〗⟦◈灭⟧〖◆韩〗 |
| `〖◆X〗与〖◆Y〗` | X、Y 同类 peer | 〖◆齐〗与〖◆鲁〗 |
| `〖◆X〗王` / `〖◆X〗之王` | 确认 X 是邦国（有王号）| 〖◆楚〗王 |
| `〖◆X〗公`（春秋时代章节）| 周代诸侯 | 〖◆齐〗桓公 |
| `亡〖◆X〗` / `入〖◆X〗` / `奔〖◆X〗` | X 是邦国 | 奔〖◆楚〗 |
| `封…为〖◆X〗王` / `立…为〖◆X〗王` | 汉诸侯王国 | 封刘濞为〖◆吴〗王 |
| `徙〖◆X〗` / `分〖◆X〗` | 汉诸侯王国（徙封/分封）| 徙〖◆梁〗王 |

### 三家注模式

| 注释模式 | 推断类别 |
|----------|----------|
| `X，国名` / `X，国也` | 周代诸侯（兜底） |
| `X，封国` / `X，小国` | 周代诸侯 或 上古方国 |
| `X，古国也` / `X，古之X国` | 上古方国 |
| `X，汉封国` / `X，汉王国` | 汉诸侯王国 |
| `X，西域国名` / `X，胡国` | 西域诸国 |
| `X，匈奴国` / `X，夷狄之国` | 外夷邦国 或 部族误标 |

**冲突处理**：L3 推断仅在 L1+L2+L2.5 给出空或 `cat-generic` 时覆盖；已归 `dynasty-mis / collective-mis / tribe-mis / place-mis` 的不被改写。

## 七、L4：后缀/字形启发式（兜底）

见 `classify_feudal_states.py::SUFFIX_RULES`。初版：

| 模式 | 类别 | 备注 |
|------|------|------|
| `X国`（长度 2，X 为单字邦国） | `cat-split` | 秦国/齐国/燕国/楚国/赵国/魏国/宋国 → 与同名单字重复，建议源头合并 |
| `X国`（X 非邦国同名） | 按语义分派 | 菑川国/身毒国/滇国/荆国 → 按对应类 |
| `X王` 单独出现（如 秦王、楚王） | → 官职迁移 | X王 应归 official (职位 cat-wang)，非邦国。若已被标 ◆，归 `cat-split` |
| 含"氏"后缀 | `cat-dynasty` | 氏族沿续朝代语义 |
| 数字集体（三/七/六/两/五 + 字/词） | `cat-collective` | 三晋/七国/六国/两越（合法分类）|
| 方位 + 国/土/州 | `cat-generic` 或 `cat-place-mis` | 北国/西国 → 泛称；关中/山东 → 地名 |
| 前缀"大"+外国名 | `cat-foreign` | 大宛/大夏/大月氏 |

## 八、L5：并列邦国传播

**规则**：在 `chapter_md` 正文中，两个 `〖◆X〗` 标记之间若只有"、"（可含空白），则视为并列。对多个并列 feudal-state 构成的 peer 组，从已分类条目向未分类条目传播（多数决 ≥ 60%）。

**典型触发上下文**：
- `〖◆秦〗、〖◆楚〗、〖◆齐〗、〖◆燕〗、〖◆赵〗、〖◆魏〗、〖◆韩〗` — 周代诸侯（战国期）序列
- `〖◆鲁〗、〖◆卫〗、〖◆宋〗、〖◆郑〗、〖◆陈〗、〖◆蔡〗` — 周代小国序列
- `〖◆南越〗、〖◆东越〗、〖◆闽越〗` / `〖◆大宛〗、〖◆大夏〗、〖◆康居〗、〖◆乌孙〗` — 外邦序列
- `〖◆梁〗、〖◆淮南〗、〖◆胶东〗、〖◆胶西〗` — 汉诸侯王国序列

**约束**：仅对当前为空字符串的条目生效；已分类条目（含各 mis 子类）不被改写。

## 九、误标识别（邦国独有三条子流程）★

朝代与合称在 §一 已确立为合法分类（`cat-dynasty` / `cat-collective`），不再视为误标。真·跨类型误标仅剩三条：

### 9.1 `cat-tribe-mis` — 部族误标为邦国

**判定条件**：
- 在 `EXPLICIT_TRIBE_MIS` 内（匈奴/戎/狄/荆蛮/犬戎/山戎/三苗/有苗 等）
- 或 rulers.json 类型 = `单于` 且所属国 ∈ {匈奴}

**源头修复**：`〖◆X〗 → 〖&X〗`（tribe 标签），批量交 **03e1 跨类型迁移**。

**边界说明**：
- 南越/东越/闽越/朝鲜/夜郎/滇 — 这些虽然是夷狄，但《史记》视为"国"（有王、有都城、有正式外交），归 `cat-foreign` 非 `cat-tribe-mis`
- 匈奴、戎、狄 — 司马迁视为"部族/游牧联盟"，归 `cat-tribe-mis`

**预期规模**：约 15-25 处（匈奴高频）。

### 9.2 `cat-place-mis` — 地理大区/郡名误标为邦国

**判定条件**：
- 在 `EXPLICIT_PLACE_MIS` 内（关中/山东/北地/河东/上郡/南郡/豫章/益州…）
- **注意**：汉代侯国不归 place-mis，而归 `cat-han-marquis`（侯国也是邦国）
- 只有纯地理大区（关中/山东/河东）和郡名（上郡/南郡）才归 place-mis

**源头修复**：`〖◆X〗 → 〖=X〗`（place 标签），批量交 **03e1 跨类型迁移**。

**预期规模**：约 10-20 处（去掉侯国后规模显著变小）。

### 9.3 `cat-split` — 复合邦国需拆分

**判定条件**：
- 含"X国"后缀且 X 本身已有 `〖◆X〗` 条目（秦国/齐国/燕国/楚国/赵国/魏国/宋国）— 与同名单字冗余
- 含邦国 + 王/主/君（齐王/楚王 — 应归 official 非 feudal-state）
- 源头误合的多国连写（若标注者未按合称录入而是糊成一整个 ◆ 标签）

**说明**：合法的合称（三晋/七国/齐秦楚赵）归 `cat-collective`，不进 split；split 专指**源头标注技术错误**（如漏切分、国 后缀冗余、与官职边界错位）。

**源头修复**：逐条拆分，人工确认。

**预期规模**：约 15-20 处。

## 十、已登记的疑难与待办

### 10.1 `秦` 跨时代消歧

- 005_秦本纪 前半（襄公-孝公）：周代诸侯
- 005_秦本纪 后半（惠文王-始皇统一）：周代诸侯（战国期）
- 006_秦始皇本纪（统一后）：`cat-dynasty`
- 006-008 中的"秦"偶尔也指前朝（如汉初言"秦之弊政"）：上下文判断

**实现**：按章节号 × 段落号 vote；无强信号时保留多标签（`周代诸侯` + `朝代`）。

### 10.2 `楚` 的多重身份

- 周代诸侯（楚子至战国末年）→ 秦末列国（西楚）→ 汉诸侯王国（刘交楚国）
- 三层身份需章节 × 谥号/人名共现判断
- 已有 feudal_state_wordlist.json 注明"西楚单列"

### 10.3 `代` 的三重身份

- 战国赵北边代国（代王嘉）→ 秦末列国（项羽分封）→ 汉诸侯王国（代王刘恒）
- 章节消歧：043_赵世家 = 周代诸侯；006-008 = 秦末列国；009 文帝本纪 = 汉诸侯王国

### 10.4 `X国` 的拆分原则

- `秦国/齐国/燕国/楚国/赵国/魏国/宋国/晋国` 与对应单字重复，源头应合并
- `菑川国/身毒国/滇国/荆国` 是独立条目（合并会丢失"国"后缀信息）
- 决策：只有 `中水/杜衍/吴防/涅阳/射阳/缪` 等前缀在 `EXPLICIT_HAN_VASSAL` 中的保留

### 10.5 汉侯国 vs 汉王国（都算邦国）

- 汉**王国**（同姓王 + 异姓王）：梁、淮南、代、齐、楚、吴、胶东、胶西、济北、济南、江都、长沙、菑川、河间 等 → `cat-han-vassal`
- 汉**侯国**（列侯封地）：酂、留、平阳、舞阳、绛、赤泉、硃虚、东牟… → `cat-han-marquis`
- **两者都保留在 feudal-state 类型**（郡国并行制下的"国"）—— 不迁 place

### 10.6 `dynasty` 实体类型的重叠

- 当前 entity_index 中 `dynasty` 实体类型 和 `feudal-state` 实体类型 存在 74 条重合（夏/商/周/秦/汉/...）
- 本 skill 保留立场：feudal-state 下的 `cat-dynasty` 子类是**正确分类**，用于指这些邦国条目在某些章节语境中指代"朝代"
- 是否在实体层面（type）保留双 type 标签，由 03b/03e1 另议；本 skill 只打二级 cat 标签，不动 type

## 十四、与其他 skill 的交互

### 与 03e1 跨类型迁移
- 本 skill 的 `cat-tribe-mis / cat-place-mis` 两个子类**反向触发 03e1**
- 对应迁移：
  - `〖◆X〗 → 〖&X〗`（邦国→部族）约 15-25 处
  - `〖◆X〗 → 〖=X〗`（邦国→地名）约 20-30 处
- `cat-dynasty` 和 `cat-collective` 不触发跨类型迁移——这些是 feudal-state 内部合法子类

### 与 03h 地名分类
- place 类的"国家" `cat-nation`（大夏/大宛/南越…）与本 skill 的 `cat-foreign` 重叠
- 同一 surface 两类型共存合法：该 surface 同时作为政治邦国和地理区域被标注

### 与 03i 官职分类
- 官职类的"王号" `cat-wang`（齐王/楚王/代王）与本 skill 的邦国条目有派生关系
- X王 应归 official，若被标为 `〖◆X王〗` → `cat-split`（拆成 `〖◆X〗` + `〖;X王〗`）

### 与 03j 人名分类
- person 类的"诸侯君主" `cat-ruler` 与本 skill 的"周代诸侯/战国七雄/汉诸侯王国"有君主-邦国关系
- rulers.json 是两个 skill 的**共享数据源**
