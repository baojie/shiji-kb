# 智能分段功能说明

## 功能概述

在显示设置中新增"智能分段"选项，允许用户控制段落的显示方式。

## 功能特性

### 智能分段（默认开启）

- 段落按原有的编号结构独立显示
- 例如：pn-1、pn-1.1、pn-1.2 各自独立成段
- 保持清晰的层次结构

### 关闭智能分段

- 将共享顶级编号的段落合并显示
- 例如：pn-1、pn-1.1、pn-1.2 合并成一个连续段落
- 隐藏子段落编号（如 1.1、1.2），只保留顶级编号（如 1）
- 阅读体验更接近传统连续文本

## ✅ 功能已完成

智能分段功能已成功实现！当关闭"智能分段"时：
- 段落内的 `<br>` 标签被移除
- 所有子段落的内容被合并到顶级段落中
- 子段落元素被隐藏
- 内容连续显示（浏览器的自动换行是正常的）

## 实现细节

### JavaScript（settings-panel-config.js）

```javascript
{
    id: 'smart-paragraph',
    label: '智能分段',
    storageKey: 'shiji-smart-paragraph',
    defaultValue: true,  // 默认开启
    onChange: function(enabled) {
        updateSmartParagraph(enabled);
    }
}
```

`updateSmartParagraph()` 函数的工作原理：

1. **开启智能分段时**：
   - 移除 `merge-paragraphs` 类
   - 清除所有段落的 `sub-paragraph` 和 `top-paragraph` 标记类

2. **关闭智能分段时**：
   - 添加 `merge-paragraphs` 类到 body
   - 遍历所有 `<p>` 标签，检查其中的 `<a class="para-num">` 的 id
   - 如果 id 包含小数点（如 `pn-1.1`），添加 `sub-paragraph` 类
   - 如果 id 是顶级编号（如 `pn-1`），添加 `top-paragraph` 类

### CSS（shiji-styles.css）

关闭智能分段时，通过 `.merge-paragraphs` 类和段落标记类实现：

1. `.sub-paragraph` - 子段落改为行内显示，隐藏编号
2. `.top-paragraph` - 顶级段落保持块级显示和段落间距
3. 列表项保持原有显示方式

**关键技术点**：由于 HTML 结构中段落编号的 `id` 在 `<a>` 标签而非 `<p>` 标签上，因此使用 JavaScript 动态添加类名来标记段落类型，而不是纯 CSS 选择器。

## 合并规则

段落合并基于**顶级编号**进行：
- 提取段落编号的第一个数字作为顶级编号
- 所有共享相同顶级编号的段落合并为一个段落
- **支持所有层级**：子段落、孙段落、重孙段落等

### 层级说明
- **顶级段落**：`pn-1`, `pn-11`, `pn-100` （无小数点）
- **子段落**：`pn-1.1`, `pn-11.1` （一个小数点）
- **孙段落**：`pn-1.1.1`, `pn-11.1.1` （两个小数点）
- **重孙段落**：`pn-1.1.1.1`, `pn-11.1.1.1` （三个小数点）
- 更多层级...

**合并示例**：
- `pn-1`, `pn-1.1`, `pn-1.2`, `pn-1.3` → 顶级编号都是 `1` → 合并
- `pn-11`, `pn-11.1`, `pn-11.2`, `pn-11.3`, `pn-11.4`, `pn-11.5` → 顶级编号都是 `11` → 合并
- `pn-11`, `pn-11.1`, `pn-11.1.1`, `pn-11.1.2`, `pn-11.2` → 顶级编号都是 `11` → **全部合并**
- `pn-2`, `pn-3` → 不同顶级编号 → 分别独立

## 示例对比

### 示例1：单位数编号

**智能分段开启（默认）**：
```
1 黄帝者，少典之子...

1.1 轩辕之时，神农氏世衰...

1.2 蚩尤作乱，不用帝命...
```

**智能分段关闭**：
```
1 黄帝者，少典之子...轩辕之时，神农氏世衰...蚩尤作乱，不用帝命...
```

### 示例2：两位数编号

**智能分段开启（默认）**：
```
11 乃命羲、和，敬顺昊天...

11.1 分命羲仲，居郁夷...

11.2 申命羲叔，居南交...

11.3 申命和仲，居西土...

11.4 申命和叔；居北方...

11.5 岁三百六十六日，以闰月正四时...
```

**智能分段关闭**：
```
11 乃命羲、和，敬顺昊天...分命羲仲，居郁夷...申命羲叔，居南交...申命和仲，居西土...申命和叔；居北方...岁三百六十六日，以闰月正四时...
```

### 示例3：多层级嵌套（假设存在）

**智能分段开启（默认）**：
```
12 顶级段落内容...

12.1 子段落内容...

12.1.1 孙段落内容...

12.1.2 孙段落内容...

12.2 子段落内容...

12.2.1 孙段落内容...

12.2.1.1 重孙段落内容...
```

**智能分段关闭**：
```
12 顶级段落内容...子段落内容...孙段落内容...孙段落内容...子段落内容...孙段落内容...重孙段落内容...
```

**所有层级的子段落都会被合并**，编号全部隐藏，只保留顶级编号（12）。
