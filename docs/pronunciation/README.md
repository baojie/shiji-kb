## 📝 文档标准结构

每个多音字分析文档包含以下章节：

### 1. 基本信息
- 字、主要读音、分析日期、数据来源

### 2. 读音统计
- **总体分布**：各读音出现次数、占比、主要含义
- **详细分类**：按读音分组，列出典型词组、例句

### 3. 词表处理方案
- **采用方案**：说明采用的词表策略（单字/固定词组/正则）
- **新增词条**：JSON格式的词条示例
- **方案说明**：覆盖率、误标风险分析
- **设计原则依据**：引用SKILL_01d v3.0规范

### 4. 实施状态
- 统计分析完成情况
- 词条添加情况
- 相关提交记录

### 5. 备注
- 历史典故、语言学特征、成语等补充信息

## 🎯 设计原则（SKILL_01d v3.0）

所有词表方案遵循以下原则：

1. **固定词组为主**：优先使用固定词组而非单字词条
2. **单字词条例外**：仅当使用率>99%时才添加单字词条
3. **避免高误标风险**：不添加可能导致大量误标的词条
4. **数据驱动决策**：基于完整的统计分析制定方案
5. **优先标注少数读音**：优先处理占比<50%的特殊读音

## 📈 统计方法

### 分析工具

使用bash脚本进行统计分析：

```bash
# 基础统计
grep -o "字" archive/史记.简体.txt | wc -l

# 上下文分析
grep -C 2 "字" archive/史记.简体.txt > /tmp/context.txt

# 详细分类
grep "字" archive/史记.简体.txt | grep -E "模式"
```

### 判断标准

1. **读音分类**：根据语境、词性、搭配判断
2. **覆盖率计算**：词条覆盖次数 / 该读音总次数
3. **误标风险**：错误标注次数 / 总出现次数
