## 背景与目标

### 问题现状

在《史记》正音与拼音标注工作中，我们发现：

1. **上下文信息不足**：早期统计只记录"王曰"、"言占"等简单词组，缺乏足够的上下文信息来判断读音
2. **词表维护困难**：全局特殊读音词表 (`special-pronunciation.json`) 越来越长，维护成本高
3. **精度不够**：pinyin-pro 对古汉语多音字的处理精度有限，需要大量人工干预

### 解决方案

采用**两步上下文分析法**：

1. **提取上下文**：每个多音字前后各延展2个汉字（遇标点停止），记录所有上下文
2. **抽象规则**：从上下文数据中抽象出词组和单字发音规则
3. **记录发音**：不遗漏任何一个多音字的发音
4. **对比优化**：只保留 pinyin-pro 无法正确处理的发音
5. **分层词表**：长词表 → 拼音词表，上下文单字 → 分章节补充词表
6. **未来训练**：积累语料和标注，训练 AC 自动机

## 工具与脚本

### 1. `extract_polyphone_contexts.py` - 上下文提取器

**功能**：
- 提取指定多音字的所有出现位置
- 向左右各延展2个汉字（遇标点符号停止）
- 统计完整上下文模式、左侧模式、右侧模式
- 输出 JSON 和文本两种格式

**用法**：

```bash
# 单字符提取
python scripts/extract_polyphone_contexts.py 占

# 批量提取（所有多音字）
python scripts/extract_polyphone_contexts.py --all

# 自定义延展范围
python scripts/extract_polyphone_contexts.py 王 --left 3 --right 3
```

**输出文件**：
- `data/polyphone_contexts/{字}_contexts.json` - 结构化数据
- `data/polyphone_contexts/{字}_contexts.txt` - 人类可读格式

**输出示例**（占字）：

```
# 占 - 上下文分析

**总计出现次数**: 79
**完整上下文模式数**: 58

## 完整上下文模式（按频率排序）

| 上下文 | 出现次数 | 说明 |
|--------|---------|------|
| 以占病 | 14 | |
| 以占病者 | 5 | |
| 占曰 | 3 | |
...
```

### 2. `analyze_pronunciation_rules.py` - 发音规则分析器

**功能**：
- 读取上下文 JSON 文件
- 提取高频词组模式（默认 ≥3次）
- 分类为：固定词组、成语短语、上下文模式
- 生成发音标注模板

**用法**：

```bash
# 生成标注模板
python scripts/analyze_pronunciation_rules.py 占

# 批量生成
python scripts/analyze_pronunciation_rules.py --all

# 调整频率阈值
python scripts/analyze_pronunciation_rules.py 王 --min-freq 5

# 解析已标注的模板
python scripts/analyze_pronunciation_rules.py --parse data/pronunciation_templates/占_pronunciation.md
```

**输出文件**：
- `data/pronunciation_templates/{字}_pronunciation.md` - 标注模板
- `data/pronunciation_templates/{字}_pronunciation.dict.json` - 发音词典（解析后）

**模板示例**：

```markdown
# 占 - 发音标注模板
**说明**: 请为每个词组/上下文标注正确的发音（拼音）

## 固定词组（2-3字）
| 词组 | 拼音 | 频率 | 说明 |
|------|------|------|------|
| 以占病 | _____ | 14 |  |
| 占曰 | _____ | 3 |  |
```

### 3. `compare_with_pinyinpro.py` - pinyin-pro 对比工具（TODO）

**功能**：

- 识别 pinyin-pro 处理错误的词组
- 只保留需要特殊处理的词组

### 4. `optimize_pronunciation_dict.py` - 词表优化工具（TODO）

- 生成最终的发音词典文件

## 操作指南

### 处理单个多音字的完整流程

以"占"字为例：

```bash
# 1. 提取上下文
python scripts/extract_polyphone_contexts.py 占

# 2. 查看上下文分析（可选）
cat data/polyphone_contexts/占_contexts.txt

# 3. 生成发音标注模板
python scripts/analyze_pronunciation_rules.py 占

# 4. 人工标注发音
vim data/pronunciation_templates/占_pronunciation.md

# 示例标注：
# | 以占病 | zhān | 14 | 占卜疾病 |
# | 占曰 | zhān | 3 | 占卜后说 |

# 5. 解析标注结果
python scripts/analyze_pronunciation_rules.py --parse data/pronunciation_templates/占_pronunciation.md

# 6. 查看生成的词典
cat data/pronunciation_templates/占_pronunciation.dict.json
```

### 批量处理多个多音字

```bash
# 1. 批量提取上下文（前20个高频多音字）
python scripts/extract_polyphone_contexts.py --batch "王,为,与,使,将,相,得,中,夫,行"

# 2. 批量生成标注模板
python scripts/analyze_pronunciation_rules.py --all

# 3. 人工标注（逐个处理）
# vim data/pronunciation_templates/王_pronunciation.md
# vim data/pronunciation_templates/为_pronunciation.md
# ...

# 4. 批量解析（使用脚本循环）
for file in data/pronunciation_templates/*_pronunciation.md; do
    python scripts/analyze_pronunciation_rules.py --parse "$file"
done
```

## 质量控制

### 标注规范

1. **拼音格式**：
   - 使用小写带声调的拼音：`zhān`, `zhàn`, `wáng`, `wàng`
   - 多音字只标注目标字的读音，不需要标注整个词组的拼音

2. **词组分类**：
   - **固定词组**（2-3字）：高频固定搭配，如"占病"、"王曰"
   - **成语短语**（4字）：四字成语或固定短语
   - **上下文模式**（≥5字）：需要上下文才能确定读音的模式

3. **说明补充**：
   - 在「说明」列补充词义或用法说明
   - 有助于后续维护和理解

### 验证检查

1. **覆盖率检查**：
   ```bash
   # 检查是否遗漏高频模式
   grep "出现次数.*[1-9][0-9]" data/polyphone_contexts/占_contexts.txt
   ```

2. **标注完整性检查**：
   ```bash
   # 检查是否有未标注的行（拼音列为空）
   grep "_____" data/pronunciation_templates/占_pronunciation.md
   ```

3. **拼音格式检查**：
   ```bash
   # 检查拼音是否符合规范
   grep -P "\| [^\|]+ \| [a-z]+[1-4] \|" data/pronunciation_templates/占_pronunciation.md
   ```
