## 2. 不同上下文不同读音是否正确分类？

### 2.1 初始分类（错误）

**问题1: 误将"为XX王"分类为动词wàng**

| 词组 | 初始标注 | 实际读音 | 错误原因 |
|------|---------|---------|---------|
| 为平王 | wàng | wáng | 误认为"称王、封王"动词 |
| 为梁王 | wàng | wáng | 同上 |
| 为魏王 | wàng | wáng | 同上 |
| 为韩王 | wàng | wáng | 同上 |

**根本原因**:
- 未查证原文上下文
- 误以为"为"是动词"成为、作为"
- 实际是"是为XX王"结构的一部分，"是"被省略或在上文

**验证方法**（应该做而未做）:
```bash
grep "是为平王\|为平王" archive/史记.简体.txt
```

结果显示：所有"为平王"都出现在"是为平王"结构中，表示"这就是平王"。

### 2.2 修正后分类（正确）

**wáng（名词）- 99.5%**:
1. **国王、诸侯王**: 齐王、秦王（指人）
2. **王位、王号**: 是为XX王、立为王、为XX王（指职位）
3. **王姓**: 王夫人、王太后

**wàng（动词）- 0.5%**:
1. **统治**: 王天下（11次）
2. **称王**: 称王（17次）
3. **想称王**: 欲王（11次）

**验证示例**:
```
wáng: "诸侯乃即申侯而共立故幽王太子宜臼，是为平王"
wàng: "今秦南面而王天下"（统治天下）
wàng: "秦惠王称王"（称王）
wàng: "沛公欲王关中"（想在关中称王）
```

**评估**: ✗ 初始分类错误 → ✓ 修正后正确

### 3.2 发现的管线问题

**问题1: 步骤3缺少验证机制**

当前问题：
- 人工标注时容易凭主观判断
- 没有强制要求查证原文
- 没有同行审核机制

**改进建议**:
```bash
# 标注前必须执行的验证
grep "为平王" archive/史记.简体.txt  # 查看实际用法
grep -B2 -A2 "为平王" archive/史记.简体.txt  # 查看上下文
```

**问题2: 步骤4缺少自动验证**

当前问题：
- 手动添加词条后未验证pinyin-pro处理
- 可能添加不必要的条目

**改进建议**:
```bash
# 添加词条前必须测试
node -e "
const { pinyin } = require('pinyin-pro');
console.log('为平王:', pinyin('为平王', { toneType: 'symbol', type: 'array' }));

```

如果pinyin-pro已正确处理，则不应添加到词表。

**问题3: 缺少"反思-修正"循环**

当前流程是单向的，发现错误后需要手动返回步骤3重做。

**改进建议**: 建立检查清单（见3.3）

3. **pinyin-pro能力评估**
   - 许多常见词组pinyin-pro已能正确处理

## 6. 改进后的标准操作流程（SOP）

### 6.1 人工标注阶段

**必须执行的4步验证**：

1. **查证原文**（每个词组）
   ```bash
   grep -B2 -A2 "词组" archive/史记.简体.txt | head -10
   ```

2. **分析语法结构**
   - 主谓宾关系
   - 动词/名词/形容词
   - 固定搭配vs自由组合

3. **区分所有读音**
   - 列出所有可能的读音
   - 确定每个读音的判断条件
   - 找出典型例句

4. **检查矛盾标注**
   - 同样结构是否标注一致
   - 相似词组是否遵循相同规则

### 6.2 词表更新阶段

**必须执行的3步测试**：

1. **测试pinyin-pro**（每个词条）
   ```bash
   node -e "const { pinyin } = require('pinyin-pro');
   console.log('词组:', pinyin('词组', { toneType: 'symbol', type: 'array' }));"
   ```

2. **判断是否必需**
   - pinyin-pro正确 → 不添加
   - pinyin-pro错误 → 添加，并记录错误内容

3. **运行Lint验证**
   ```bash
   node scripts/lint_pronunciation_dict.js
   ```

### 6.3 反思报告阶段

**必须回答的4个问题**：

1. 上下文是否完整收集？
2. 不同上下文不同读音是否正确分类？
3. 整个正音管线是否清晰？
4. 每个字完整反思后是否更新了词表？

**必须记录的内容**：
- 错误原因分析
- 经验教训
- 改进建议
- 检查清单
