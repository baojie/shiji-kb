# GitHub Discussions 创建操作指南

## 第一步：启用 Discussions（如果尚未启用）

1. 进入您的 GitHub 仓库主页
2. 点击 **Settings**（设置）
3. 在左侧菜单找到 **General**
4. 向下滚动到 **Features** 部分
5. 勾选 **✓ Discussions**
6. 点击保存

## 第二步：创建第一个 Discussion（身毒读音）

### 2.1 进入 Discussions 页面

1. 回到仓库主页
2. 点击顶部的 **Discussions** 标签

### 2.2 创建新讨论

1. 点击右上角的 **New discussion** 按钮
2. 选择分类：**Q&A**（问答）
3. 填写信息：
   - **Title**（标题）：`身毒的读音：shēn dú 还是 yuán dú？`
   - **Body**（正文）：打开 `01_身毒读音考证.md`，复制全部内容粘贴
4. 点击 **Start discussion**

### 2.3 记录 Discussion URL

创建成功后，浏览器地址栏会显示 URL，格式类似：
```
https://github.com/username/shiji-kb/discussions/1
```

**请复制并保存这个 URL！**

## 第三步：创建第二个 Discussion（月氏读音）

### 3.1 重复第二步的操作

1. 回到 Discussions 页面
2. 点击 **New discussion**
3. 选择分类：**Q&A**
4. 填写信息：
   - **Title**：`月氏的读音：yuè zhī 还是 ròu zhī？`
   - **Body**：打开 `02_月氏读音考证.md`，复制全部内容粘贴
5. 点击 **Start discussion**

### 3.2 记录 Discussion URL

URL 格式类似：
```
https://github.com/username/shiji-kb/discussions/2
```

**请复制并保存这个 URL！**

## 第四步：更新 HTML 文件中的链接

### 4.1 打开文件

编辑文件：`docs/pronunciation-debates.html`

### 4.2 更新身毒的链接（约第225行）

## 第五步：提交更改

```bash

git commit -m "更新读音争议页面的GitHub Discussion链接"
git push
```

## 第六步：测试链接

1. 在浏览器中打开任意章节页面
2. 点击右上角的 **⚙️** 图标打开设置面板
3. 点击"拼音注释"后面的 **[?]** 链接
4. 验证页面正确显示
5. 点击"查看详细考证和讨论"按钮
6. 确认能正确跳转到 GitHub Discussion

## 常见问题

### Q1: 如果我想添加更多读音争议词汇怎么办？

参考现有的两个文件，创建新的 `.md` 文件，然后：
1. 在 `pronunciation-debates.html` 中添加新的 `debate-item` 块
2. 在 GitHub 创建新的 Discussion
3. 更新链接

### Q2: 可以让用户在 Discussion 中投票或评论吗？

可以！GitHub Discussions 支持：
- 👍👎 点赞/反对
- 💬 评论和回复
- ✅ 标记最佳答案
- 📌 置顶重要讨论

### Q3: 如果有人提出不同的读音观点怎么办？

在 Discussion 中进行学术讨论，如果有充分证据，可以：
1. 更新 Discussion 内容
2. 在 `pronunciation-debates.html` 中添加新的争议读音
3. 保持透明和学术严谨
