## 二、术语vs工具分类优化

### 2.1 术语标注优化

**〖#标准知识单元〗优化：**

修改前（第113-115行）：
```

这里叫〖#Standard Knowledge Unit〗,叫〖#标准知识单元〗
```

修改后：
```
你还得有我们后来自己⟦◆定义⟧的一个词叫〖#标准知识单元〗,这是从电商里学来的。
这里叫〖#标准知识单元〗(Standard Knowledge Unit, SKU)
```

- SKU是缩写，不应单独标注为实体
- Standard Knowledge Unit是英文全称，应作为括号补充说明
- 〖#标准知识单元〗是核心概念，应该作为主要标注

## 六、标注规范总结

### 6.1 职业实体〖※〗使用原则

**应该使用〖※职业〗的情况：**
- 现代职业身份（程序员、工程师、业务经理）
- 专业技术人员（妇科医生）
- 学位作为职业身份（博士）

**应该使用〖;官职〗的情况：**
- 古代历史官职（大将、丞相、太尉等）
- 史记中的官职体系

### 6.2 术语vs工具的区分标准

**术语〖#〗：**
- 学术概念（知识图谱、语义结构）
- 理论术语（标准知识单元、三元组）
- 技术概念（语法高亮、人工智能）

**工具〖^〗：**
- 软件系统（Claude, IDE, GitHub）
- 编辑器（Protégé, Word, PPT）
- 应用程序（APP, 利器系统）

**方法论〖★〗：**
- 理论框架（四层语义结构）
- 方法体系（元技能、Bootstrap）
- 技术方案（PDF to Skill）
