# 语义标注阅读系统

这是一个将语义标注文章转换为交互式HTML的渲染系统，演示了文章《未来阅读不只是文字》中提出的理念。

## 📁 文件说明

### 核心文件
- **未来阅读不只是文字.md** - 原始文章（无标注）
- **未来阅读不只是文字.tagged.md** - 语义标注版本
- **未来阅读不只是文字.html** - 生成的交互式HTML

### 渲染系统
- **render_semantic_article.py** - Python渲染脚本（tagged.md → HTML）
- **semantic-reading.css** - 语义标注样式表
- **semantic-reading.js** - 交互增强脚本

## 🎨 功能特性

### 1. 语义可视化
- **15种实体类型**，每种类型有独特的颜色编码
- 实体类型包括：人名、书名、事件、时间、技术、概念、工具、职业、国家、头衔、可视化、网络、话题、项目、神话
- 悬停显示实体类型标签

### 2. 三种标注模式
- **完整标注** (full) - 显示颜色高亮和背景
- **最小标注** (minimal) - 仅显示下划线
- **无标注** (none) - 纯文本模式

### 3. 交互功能
- **点击实体** - 显示详细信息（类型、规范名、出现次数）
- **双击实体** - 高亮所有同名实体
- **类型过滤** - 显示/隐藏特定类型的实体
- **侧边栏** - 展示实体详情和关联信息

### 4. 统计分析
- 总标注数：154个
- 实体类型：15种
- 实体类型图例和统计面板

## 🚀 使用方法

### 生成HTML
```bash
cd resources/publications/draft
python3 render_semantic_article.py
```

### 查看效果
在浏览器中打开 `未来阅读不只是文字.html`

### 自定义标注
1. 编辑 `.tagged.md` 文件
2. 使用格式：`〖TYPE 文本〗` 或 `〖TYPE 显示名|规范名〗`
3. 重新运行渲染脚本

## 🏷️ 标注语法

### 基本语法
```
〖PERSON 项羽〗
〖BOOK 史记〗
〖EVENT 鸿门宴〗
```

### 消歧语法
```
〖TECH Stock Keeping Unit|标准知识单元〗
显示：Stock Keeping Unit
规范名：标准知识单元
```

## 🎯 设计理念

这个系统实践了文章中提出的**三层递进架构**：

### 第一层：语义标注
- Python脚本解析 `.tagged.md` 文件
- 识别并标记15种实体类型
- 生成结构化的HTML标记

### 第二层：语义排版
- CSS为不同实体类型定义视觉编码
- 使用颜色、背景、边框区分语义
- 悬停提示显示实体类型

### 第三层：超文本阅读
- JavaScript提供交互功能
- 点击查看详情，双击高亮关联
- 类型过滤、搜索、统计分析

## 🔧 技术栈

- **Python 3** - Markdown解析和HTML生成
- **纯CSS** - 无需任何CSS框架
- **纯JavaScript** - 无需任何JS库
- **语义HTML5** - 结构化标记

## 🎨 自定义样式

### 修改颜色方案
编辑 `semantic-reading.css` 的 `:root` 变量：
```css
:root {
  --color-person: #2563eb;  /* 修改人名颜色 */
  --color-book: #7c3aed;    /* 修改书名颜色 */
  /* ... */
}
```

### 添加新实体类型
1. 在 `render_semantic_article.py` 中添加类型标签
2. 在 `semantic-reading.css` 中定义颜色
3. 在文章中使用新类型标注
