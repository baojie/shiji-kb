# 功能借鉴与灵感

从参考项目中提炼的可借鉴功能，按优先级排列。

---

## 来源：dutongjian.com（读通鉴）

核心理念：**围绕一本书，集成所有辅助材料，在正确的时候提供正确的功能。**

### 高优先级

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 三家注嵌入 | 裴骃集解、司马贞索隐、张守节正义嵌入对应段落，点击展开/折叠 | 已有段落编号系统（Purple Numbers），注释数据公开可获取 |
| 地图联动 | 点击地名显示历史地图，古今对照 | 已有1,820个地名实体，缺可视化；可集成CHGIS数据 |

### 中优先级

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 人物上下文聚合 | 点击人物，显示介绍+全书所有出现上下文片段 | 已有实体索引（4,142人名），当前只有章节列表，缺上下文预览 |
| 史料对勘 | 逐段对照相关史料（如史记↔汉书重叠部分） | 扩展计划已含二十六史 |
| 多维度解构 | 按主题分类浏览（军事、外交、制度等），苏轼"八面受敌"读法 | 已有11类实体标注+11类事件类型，可生成"按主题浏览"视图 |

### 低优先级（需较多新开发）

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 决策卡 | 提取关键决策点，展示背景、选项、结果 | 3,092个事件中可筛选决策类 |
| AI问答 | 基于原文+知识图谱的RAG问答 | 有完整标注文本和结构化数据 |
| 划线笔记 | 用户个人阅读笔记 | 需前端+后端开发 |
| 注音 | 古文生僻字注音 | 可批量生成 |

## 来源：texta.studio（高僧传智能释读）

核心理念：**为每个文本单元（人物传记）提供多层次的结构化解读工具，从大纲到辞典到校勘到AI，层层深入。**

### 高优先级

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 结构化大纲与标签 | 每个人物/章节配有结构化大纲，点击定位段落 | 已有小节划分（sections_data.json）和段落编号，可进一步细化为大纲导航 |
| 辞典释义 | 点击词汇弹出释义窗口 | 已有2,504个官职、661个制度等专业词汇；可集成汉语大词典或自建释义库 |
| 校勘异文 | 展示不同版本文字差异（类似CBETA异文） | 史记有中华书局本、百衲本、日本古钞本等；三家注本身含大量校勘 |

### 中优先级

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 文献来源对勘 | 逐句对勘，展示原文与来源文献的对应关系 | 史记与《左传》《国语》《战国策》有大量重叠；汉书几乎照抄史记多篇 |
| 相似故事/段落匹配 | 向量化检索相似叙事模式（如"弑君""篡位""谋反"类似事件） | 已有3,092个事件和事件类型标签，可做事件级相似匹配 |
| 外典匹配 | 匹配传世古籍中引用/化用史记的段落 | 史记被后世大量引用（《文选》《太平御览》等），可反向检索 |
| AI解读 | 基于学术资料的AI问答，解答疑难段落 | 有完整标注文本+知识图谱，可做RAG；类似dutongjian的AI问答 |

### 低优先级

| 功能 | 说明 | 我们的基础 |
|------|------|-----------|
| 相似句型类聚 | 发现史记中反复出现的叙事套语和句型模式 | 可用embedding对段落聚类；史记有大量套语如"太史公曰""其后X年" |
| 版本研究 | 系统化的版本对比和传抄谱系 | 学术价值高但工程量大 |
| 佛典/经典匹配 | 对应到我们场景：匹配儒家经典（五经）中的引文 | 史记大量引用《尚书》《诗经》《易经》《春秋》，可做标注 |

---

## 综合启发

1. **阅读体验层** — 我们偏重知识提取和结构化（"拆书"），阅读体验层面（"读书"）还有很大提升空间
2. **集成思路** — 不只是处理原文，还要把围绕这本书的所有辅助材料（注释、解读、地图、相关史料）整合到一个界面
3. **按需呈现** — 功能不是越多越好，而是在用户需要时恰好出现（dutongjian的"按钮就在手边"原则）
